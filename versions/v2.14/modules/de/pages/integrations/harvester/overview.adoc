Eingeführt in Rancher v2.6.1, https://documentation.suse.com/cloudnative/virtualization/{harvester-docs-version}/en/introduction/overview.html[{harvester-product-name}] ist eine Open-Source-Software für hyperkonvergente Infrastrukturen (HCI), die auf Kubernetes basiert. Harvester wird auf Bare-Metal-Servern installiert und bietet integrierte Virtualisierungs- und verteilte Speicherfunktionen. Obwohl Harvester mit Kubernetes arbeitet, müssen die Benutzer keine Kubernetes-Konzepte kennen, was es zu einer benutzerfreundlicheren Anwendung macht.

[#_feature_flag]
== Funktionsflag

Das Harvester-Funktionsflag wird verwendet, um den Zugriff auf die Seite für das Virtualisierungsmanagement (VM) in Rancher zu verwalten, wo Benutzer direkt zu Harvester-Clustern navigieren und auf die Harvester-Benutzeroberfläche zugreifen können. Das Harvester-Funktionsflag ist standardmäßig aktiviert. Klicken Sie {xref-experimental-features}[hier], um weitere Informationen zu Funktionsflags in Rancher zu erhalten.

Um zum Harvester-Cluster zu navigieren, klicken Sie *☰ > Virtualisierungsmanagement*. Auf der Seite der Harvester-Cluster klicken Sie auf einen der aufgelisteten Cluster, um zur Ansicht des einzelnen Harvester-Clusters zu gelangen.

* Wenn das Harvester-Funktionsflag aktiviert ist, werden Harvester-Cluster von allen Seiten oder Apps (wie Continuous Delivery mit Fleet) herausgefiltert, die Kubernetes-Cluster auflisten.
* Wenn das Harvester-Funktionsflag deaktiviert ist und ein Harvester-Cluster importiert wird, wird der Harvester-Cluster in der Clusterliste von Rancher auf der Seite Clusterverwaltung angezeigt. Harvester-Cluster werden nur in der Clusterliste angezeigt, wenn das Funktionsflag deaktiviert ist.
* Mit der Harvester-Integration können Harvester-Cluster jetzt als Cluster-Typ `Harvester` in Rancher importiert werden.
* Benutzer können einen Harvester-Cluster nur auf der Seite für das Virtualisierungsmanagement importieren. Das Importieren eines Clusters auf der Seite Clusterverwaltung wird nicht unterstützt, und eine Warnung wird Sie auffordern, zur VM-Seite zurückzukehren, um dies zu tun.

[#__harvester_product_name_node_driver]
== {harvester-product-name} Node-Treiber

Der https://documentation.suse.com/cloudnative/virtualization/{harvester-docs-version}/en/integrations/rancher/node-driver/node-driver.html[{harvester-product-name} Node-Treiber] ist allgemein für K3s- und RKE2-Optionen in Rancher verfügbar. Der Node-Treiber ist verfügbar, unabhängig davon, ob das Harvester-Funktionsflag aktiviert ist oder nicht. Beachten Sie, dass der Node-Treiber standardmäßig deaktiviert ist. Benutzer können K3s- oder RKE2-Cluster auf Harvester nur von der Seite zur Clusterverwaltung aus erstellen.

Harvester ermöglicht das Hochladen und Anzeigen von `.ISO` Images über die Harvester-Benutzeroberfläche, jedoch wird dies in der Rancher-Benutzeroberfläche nicht unterstützt. Das liegt daran, dass `.ISO` Images in der Regel ein zusätzliches Setup erfordern, das eine saubere Bereitstellung (ohne Benutzerintervention) beeinträchtigt, und in Cloud-Umgebungen normalerweise nicht verwendet werden.

Siehe {xref-provisioning-drivers}#_node_drivers[Provisioning Drivers] für weitere Informationen zu Node-Treibern in Rancher.

[#_port_requirements]
== Portanforderungen

Die Portanforderungen für den Harvester-Cluster finden Sie https://documentation.suse.com/cloudnative/virtualization/{harvester-docs-version}/en/installation-setup/requirements.html#_network_requirements[hier].

Darüber hinaus gelten folgende weitere Netzwerküberlegungen:

* Stellen Sie sicher, dass die VLAN-Trunk-Ports des physischen Switches für VM-VLAN-Netzwerke aktiviert sind.
* Befolgen Sie die Anweisungen zum Netzwerk-Setup https://documentation.suse.com/cloudnative/virtualization/{harvester-docs-version}/en/networking/cluster-network.html[hier].

Für andere Portanforderungen für andere Gastcluster, wie K3s und RKE2, finden Sie https://documentation.suse.com/cloudnative/virtualization/{harvester-docs-version}/en/installation-setup/requirements.html#_port_requirements_for_k3s_or_rkerke2_clusters[diese Dokumente].
