Dieser Abschnitt bietet einen Überblick über die Architekturoptionen zur Installation von Rancher und beschreibt die Vorteile jeder Option.

[#_terminology]
== Terminologie

In diesem Abschnitt,

* *Der Rancher-Server* verwaltet und provisioniert Kubernetes-Cluster. Sie können über die Benutzeroberfläche des Rancher-Servers mit Downstream-Kubernetes-Clustern interagieren. Der Rancher-Management-Server kann auf jedem Kubernetes-Cluster installiert werden, einschließlich gehosteter Cluster wie Amazon EKS-Cluster.
* *RKE (Rancher Kubernetes Engine)* ist eine zertifizierte Kubernetes-Distribution sowie eine Kommandozeilenschnittstelle/Bibliothek, die ein Kubernetes-Cluster erstellt und verwaltet.
* *K3s (Lightweight Kubernetes)* ist ebenfalls eine vollständig konforme Kubernetes-Distribution. Es ist neuer als RKE, einfacher zu bedienen und leichter, mit einer Binärgröße von weniger als 100 MB.
* *RKE2* ist eine vollständig konforme Kubernetes-Distribution, die sich auf Sicherheit und Compliance innerhalb der USA konzentriert. US-Bundesbehörden liegt.

[#_overview_of_installation_options]
== Überblick über die Installationsoptionen

Rancher kann auf diesen Hauptarchitekturen installiert werden:

[#_high_availability_kubernetes_install_with_the_helm_cli]
=== Hochverfügbarkeits-Kubernetes-Installation mit der Helm-CLI

Wir empfehlen die Verwendung von Helm, einem Kubernetes-Paketmanager, um Rancher auf mehreren Knoten in einem dedizierten Kubernetes-Cluster zu installieren. Für RKE-Cluster sind drei Knoten erforderlich, um einen hochverfügbaren Cluster zu erreichen. Für K3s-Cluster sind nur zwei Knoten erforderlich.

[#_rancher_on_eks_install_with_the_aws_marketplace]
=== Rancher auf EKS-Installation mit dem AWS Marketplace

Rancher kann auf dem Amazon Elastic Kubernetes Service (EKS) {xref-qs-aws-marketplace}[über den AWS Marketplace] installiert werden. Der bereitgestellte EKS-Cluster ist produktionsbereit und folgt den besten Praktiken von AWS.

[#_single_node_kubernetes_install]
=== Einzelknoten-Kubernetes-Installation

Rancher kann auf einem Einzelknoten-Kubernetes-Cluster installiert werden. In diesem Fall hat der Rancher-Server keine hohe Verfügbarkeit, was für den Betrieb von Rancher in der Produktion wichtig ist.

Diese Option ist jedoch nützlich, wenn Sie kurzfristig Ressourcen sparen möchten, während Sie einen Migrationspfad zur hohen Verfügbarkeit beibehalten. In Zukunft können Sie dem Cluster Knoten hinzufügen, um einen hochverfügbaren Rancher-Server zu erhalten.

[#_docker_install]
=== Docker-Installation

Für Test- und Demonstrationszwecke kann Rancher mit Docker auf einem einzelnen Knoten installiert werden. Ein lokaler Kubernetes-Cluster wird im einzelnen Docker-Container installiert, und Rancher wird auf dem lokalen Cluster installiert.

Der Rancher-Sicherungs-Operator kann verwendet werden, um Rancher von der Einzel-Docker-Container-Installation auf eine Installation in einem Kubernetes-Cluster mit hoher Verfügbarkeit zu migrieren. Für Details siehe die Dokumentation zu {xref-migrate-cluster}[der Migration von Rancher zu einem neuen Cluster.]

=== Sonstige Optionen

| Zugangslevel zum Internet | Kubernetes-Installation - Dringend empfohlen | Docker-Installation

| Mit direktem Zugang zum Internet
| {xref-install-rancher}[Dokumente]
| xref:[Dokumente]

| Hinter einem HTTP-Proxy
| {xref-http-proxy}[Dokumente]
| Diese xref:[Dokumente,] plus diese xref:[Konfiguration]

| In einer Air Gap-Umgebung
| {xref-air-gap}[Dokumente]
| {xref-air-gap}[Dokumente]
|===

Wir empfehlen, Rancher auf einem Kubernetes-Cluster zu installieren, da der Rancher-Management-Server in einem Mehrknoten-Cluster hohe Verfügbarkeit erreicht. Diese hochverfügbare Konfiguration hilft, den konsistenten Zugang zu den Downstream-Kubernetes-Clustern aufrechtzuerhalten, die Rancher verwalten wird.

Aus diesem Grund empfehlen wir, für eine produktionsgerechte Architektur einen Kubernetes-Cluster mit hoher Verfügbarkeit einzurichten und dann Rancher darauf zu installieren. Nachdem Rancher installiert ist, können Sie Rancher verwenden, um Kubernetes-Cluster bereitzustellen und zu verwalten.

Für Test- oder Demonstrationszwecke können Sie Rancher in einem einzelnen Docker-Container installieren. In dieser Docker-Installation können Sie Rancher verwenden, um Kubernetes-Cluster sofort einzurichten. Die Docker-Installation ermöglicht es Ihnen, die Funktionalität des Rancher-Servers zu erkunden, ist jedoch nur für Entwicklungs- und Testzwecke gedacht.

Unsere {xref-install-rancher}[Anleitungen zur Installation von Rancher auf Kubernetes] beschreiben, wie Sie zunächst K3s oder RKE verwenden, um ein Kubernetes-Cluster zu erstellen und zu verwalten, und dann Rancher auf diesem Cluster installieren.

Wenn die Knoten in Ihrem Kubernetes-Cluster laufen und die {xref-install-reqs}[Knotenanforderungen] erfüllen, verwenden Sie Helm, um Rancher auf Kubernetes bereitzustellen. Helm verwendet das Helm-Chart von Rancher, um eine Kopie von Rancher auf jedem Knoten im Kubernetes-Cluster zu installieren. Wir empfehlen, einen Lastenausgleich zu verwenden, um den Datenverkehr an jede Kopie von Rancher im Cluster zu leiten.

Für eine ausführlichere Diskussion über die Architektur von Rancher verweisen Sie auf die {xref-rancher-architecture}[Architekturübersicht], {xref-arch-rec}[Empfehlungen für produktionsgerechte Architektur] oder unseren {xref-tips-for-running-rancher}[Leitfaden zu Best Practices].

[#_prerequisites]
== Voraussetzungen

Stellen Sie vor der Installation von Rancher sicher, dass Ihre Knoten alle {xref-install-reqs}[Installationsanforderungen] erfüllen.

[#_architecture_tip]
== Architektur-Tipp

Für die beste Leistung und höhere Sicherheit empfehlen wir einen separaten, dedizierten Kubernetes-Cluster für den Rancher-Management-Server. Es wird nicht empfohlen, Benutzerlasten in diesem Cluster auszuführen. Nach der Bereitstellung von Rancher können Sie {xref-cluster-deployment}[Cluster erstellen oder importieren], um Ihre Workloads auszuführen.

Für weitere Architektur-Empfehlungen verweisen Sie auf {xref-arch-rec}[diese Seite].

[#_more_options_for_installations_on_a_kubernetes_cluster]
=== Weitere Optionen für Installationen auf einem Kubernetes-Cluster

Verweisen Sie auf die {xref-helm-chart-options}[Helm-Chart-Optionen] für Details zur Installation von Rancher auf einem Kubernetes-Cluster mit anderen Konfigurationen, einschließlich:

* Mit {xref-helm-chart-options}#_api_audit_log[API-Auditing zur Aufzeichnung aller Transaktionen]
* Mit {xref-helm-chart-options}#_external_tls_termination[TLS-Terminierung auf einem Lastenausgleich]
* Mit einem {xref-helm-chart-options}#_customizing_your_ingress[benutzerdefinierten Ingress]

In den Installationsanweisungen für Rancher empfehlen wir, K3s oder RKE zu verwenden, um ein Kubernetes-Cluster einzurichten, bevor Sie Rancher auf dem Cluster installieren. Sowohl K3s als auch RKE bieten viele Konfigurationsoptionen, um den Kubernetes-Cluster an Ihre spezifische Umgebung anzupassen. Für die vollständige Liste ihrer Funktionen verweisen Sie auf deren Dokumentation:

* https://rancher.com/docs/rke/latest/en/config-options/[RKE-Konfigurationsoptionen]
* https://rancher.com/docs/k3s/latest/en/installation/install-options/[K3s-Konfigurationsoptionen]
