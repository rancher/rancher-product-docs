Dieser Abschnitt behandelt die Verwendung der Helm CLI zur Installation des Rancher-Servers in einer Air-Gapped-Umgebung. Eine Air-Gapped-Umgebung ist eine Umgebung, in der der Rancher-Server offline, hinter einer Firewall oder hinter einem Proxy installiert wird.

Die Installationsschritte unterscheiden sich je nachdem, ob Rancher auf einem K3s Kubernetes-Cluster oder in einem einzelnen Docker-Container installiert wird.

Für weitere Informationen zu jeder Installationsoption siehe {xref-install-upgrade}[diese Seite].

In den Installationsanweisungen wird es _Tabs_ für jede Installationsoption geben.

[NOTE]
.Wichtig:
====

Wenn Sie Rancher gemäß der Docker-Installationsanleitung installieren, gibt es keinen Upgrade-Pfad, um Ihre Docker-Installation auf eine Kubernetes-Installation zu übertragen.
====
