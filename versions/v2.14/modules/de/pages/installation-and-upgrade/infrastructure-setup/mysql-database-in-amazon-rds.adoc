Dieses Tutorial beschreibt, wie man eine MySQL-Datenbank im Relational Database Service (RDS) von Amazon einrichtet.

Diese Datenbank kann später als externer Datenspeicher für einen K3s Kubernetes-Cluster mit hoher Verfügbarkeit verwendet werden.

. Melden Sie sich bei der https://console.aws.amazon.com/rds/[Amazon AWS RDS-Konsole] an, um zu beginnen. Stellen Sie sicher, dass Sie die *Region* auswählen, in der Ihre EC2-Instanzen (Linux-Knoten) erstellt werden.
. Klicken Sie im linken Bereich auf *Datenbanken*.
. Klicken Sie auf *Datenbank erstellen*.
. Klicken Sie im Abschnitt *Engine-Typ* auf *MySQL*.
. Wählen Sie im Abschnitt *Version* *MySQL 5.7.22* aus.
. Geben Sie im Abschnitt *Einstellungen* unter *Benutzerdaten,* ein Master-Passwort für den *admin* Master-Benutzernamen ein. Bestätigen Sie das Passwort.
. Klappen Sie den Abschnitt *Zusätzliche Konfiguration* auf. Geben Sie im Feld *Initialer Datenbankname* einen Namen ein. Der Name darf nur Buchstaben, Zahlen und Unterstriche enthalten. Dieser Name wird verwendet, um eine Verbindung zur Datenbank herzustellen.
. Klicken Sie auf *Datenbank erstellen*.

Sie müssen die folgenden Informationen über die neue Datenbank erfassen, damit der K3s Kubernetes-Cluster eine Verbindung herstellen kann.

Um diese Informationen in der Amazon RDS-Konsole zu sehen, klicken Sie auf *Datenbanken,* und klicken Sie auf den Namen der Datenbank, die Sie erstellt haben.

* *Benutzername:* Verwenden Sie den Admin-Benutzernamen.
* *Passwort:* Verwenden Sie das Admin-Passwort.
* *Hostname:* Verwenden Sie den *Endpunkt* als Hostnamen. Der Endpunkt ist im Abschnitt *Konnektivität und Sicherheit* verfügbar.
* *Port:* Der Standardport sollte 3306 sein. Sie können dies im Abschnitt *Konnektivität und Sicherheit* bestätigen.
* *Datenbankname:* Bestätigen Sie den Namen, indem Sie auf den Tab *Konfiguration* wechseln. Der Name ist unter *Datenbankname* aufgeführt.

Diese Informationen werden verwendet, um sich im folgenden Format mit der Datenbank zu verbinden:

----
mysql://username:password@tcp(hostname:3306)/database-name
----

Für weitere Informationen zur Konfiguration des Datenspeichers für K3s, siehe die https://rancher.com/docs/k3s/latest/en/installation/datastore/[K3s-Dokumentation.]
