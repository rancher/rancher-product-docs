Nach der Installation des CSP-Adapters haben Sie die Möglichkeit, ein Supportconfig-Bundle zu erstellen. Dieses Bundle ist eine TAR-Datei, die verwendet werden kann, um schnell Informationen an den Support bereitzustellen.

Diese Bundles können über Rancher oder durch direkten Zugriff auf den Cluster, auf dem Rancher installiert ist, erstellt werden. Bitte beachten Sie, dass der Zugriff über Rancher bevorzugt wird.

NOTE: Nur Administratoren können Supportconfig-Bundles generieren oder herunterladen, unabhängig von der Methode.

[#_accessing_through_rancher]
== Zugriff über Rancher

Zuerst klicken Sie auf das Hamburger-Menü. Dann klicken Sie auf die Schaltfläche `Get Support`.

image::support-help.png[Support anfordern]

Auf der nächsten Seite klicken Sie auf die Schaltfläche `Generate Support Config`.

NOTE: Wenn der Adapter nicht installiert ist, wird die Option zum Generieren des Supportconfig-Bundles nicht vorhanden sein. Sie müssen den CSP-Adapter installieren, um ein Supportconfig-Bundle zu generieren.

image::generate-support-config.png[Support anfordern]

[#_accessing_without_rancher]
== Zugriff ohne Rancher

Zuerst generieren Sie eine Kubeconfig für den Cluster, auf dem Rancher installiert ist.

NOTE: Wenn Rancher nicht verfügbar ist, können Sie kein von Rancher generiertes Kubeconfig-Token verwenden, um auf den Cluster zuzugreifen.

Konfigurieren Sie Ihre Shell-Umgebung, um dieses Kubeconfig-Token zu verwenden:

[,bash]
----
export KUBECONFIG=$MY_KUBECONFIG_PATH
----

Es wird empfohlen, ein temporäres Arbeitsverzeichnis zu erstellen, während Sie diesen Befehl ausführen, wie unten gezeigt:

[,bash]
----
mkdir temp && cd temp
----

Dann holen Sie sich das Supportconfig-Bundle:

[,bash]
----
mkdir rancher && kubectl get configmap csp-config -n cattle-csp-adapter-system -o=jsonpath='{.data.data}' >> rancher/config.json && tar -c -f supportconfig_rancher.tar rancher && rm -rf rancher
----

Dies wird eine `supportconfig_rancher.tar`-Datei in Ihrem aktuellen Verzeichnis erstellen.

Benutzer, die diese Befehle auf einem Mac ausführen, können aufgrund von Inkompatibilitäten zwischen gnu-tar und bsd-tar auf Probleme stoßen. Wenn der Support Probleme hat, ein von Ihnen erzeugtes Supportconfig-Bundle zu lesen, versuchen Sie, den folgenden Befehl zu verwenden, nachdem Sie gnu-tar als `gtar` in Ihrem Pfad zugänglich gemacht haben:

[,bash]
----
mkdir rancher && kubectl get configmap csp-config -n cattle-csp-adapter-system -o=jsonpath='{.data.data}' >> rancher/config.json && gtar -c -f supportconfig_rancher.tar rancher && rm -rf rancher
----
