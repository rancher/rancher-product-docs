Dieses Feature ermöglicht eine UI, mit der Sie virtuelle Dienste und Zielregeln erstellen, lesen, aktualisieren und löschen können, die Verkehrsmanagement-Funktionen von Istio sind.

____
*Voraussetzung:* Das Aktivieren dieses Features aktiviert nicht Istio. Ein Cluster-Administrator muss {xref-istio-guides}[Istio für den Cluster aktivieren], um das Feature nutzen zu können.
____

Um dieses Feature zu aktivieren oder zu deaktivieren, beachten Sie die Anweisungen auf {xref-experimental-features}[der Hauptseite über das Aktivieren experimenteller Funktionen.]

|===
| Umgebungsvariablen-Schlüssel | Standardwert | Status | Verfügbar seit

| `istio-virtual-service-ui`
| `false`
| Experimentell
| v2.3.0

| `istio-virtual-service-ui`
| `true`
| GA
| v2.3.2
|===

[#_about_this_feature]
== Über dieses Feature

Ein zentraler Vorteil der Verkehrsmanagement-Funktionen von Istio ist, dass sie dynamisches Routing von Anfragen ermöglichen, was nützlich für Canary-Deployments, Blue/Green-Deployments oder A/B-Tests ist.

Wenn aktiviert, schaltet dieses Feature eine Seite frei, die es Ihnen ermöglicht, einige Verkehrsmanagement-Funktionen von Istio über die Rancher UI zu konfigurieren. Ohne dieses Feature müssen Sie `kubectl` verwenden, um den Verkehr mit Istio zu verwalten.

Das Feature aktiviert zwei UI-Tabs: einen Tab für *Virtuelle Dienste* und einen anderen für *Zielregeln*.

* *Virtuelle Dienste* fangen den Verkehr ab und leiten ihn an Ihre Kubernetes-Dienste weiter, sodass Sie Prozentsätze des Verkehrs von einer Anfrage auf verschiedene Dienste aufteilen können. Sie können diese verwenden, um eine Reihe von Routingregeln zu definieren, die angewendet werden, wenn ein Host adressiert wird. Für Details siehe die https://istio.io/docs/reference/config/networking/v1alpha3/virtual-service/[Istio-Dokumentation.]
* *Zielregeln* dienen als die einzige Quelle der Wahrheit darüber, welche Dienstversionen verfügbar sind, um Verkehr von virtuellen Diensten zu empfangen. Sie können diese Ressourcen verwenden, um Richtlinien zu definieren, die auf den Verkehr angewendet werden, der für einen Dienst bestimmt ist, nachdem das Routing erfolgt ist. Für Details siehe die https://istio.io/docs/reference/config/networking/v1alpha3/destination-rule[Istio-Dokumentation.]

Um diese Tabs zu sehen,

. Klicken Sie auf *☰ > Clusterverwaltung*.
. Gehen Sie zu dem Cluster, in dem Istio installiert ist, und klicken Sie auf *Erkunden*.
. Klicken Sie in der linken Navigationsleiste auf *Istio*.
. werden Sie Tabs für *Kiali* und *Jaeger* sehen. Im linken Navigationsbereich können Sie *Virtuelle Dienste* und *Zielregeln* anzeigen und konfigurieren.
