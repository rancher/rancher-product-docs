{xref-fleet}[Continuous Delivery mit Fleet] ist vorinstalliert in Rancher und kann nicht vollständig deaktiviert werden. Die Fleet-Funktion für GitOps Continuous Delivery kann jedoch mit dem `continuous-delivery` Feature-Flag deaktiviert werden.

Um dieses Feature zu aktivieren oder zu deaktivieren, beachten Sie die Anweisungen auf {xref-experimental-features}[der Hauptseite über das Aktivieren experimenteller Funktionen.]

|===
| Umgebungsvariablen-Schlüssel | Standardwert | Beschreibung

| `continuous-delivery`
| `true`
| Dieses Flag deaktiviert die GitOps Continuous Delivery-Funktion von Fleet.
|===

Wenn Fleet in Rancher v2.5.x deaktiviert war, wird es aktiviert, wenn Rancher auf v2.6.x geupgradet wird. Nur der Teil der Continuous Delivery von Fleet kann deaktiviert werden. Wenn `continuous-delivery` deaktiviert ist, wird die `gitjob` Implementierung nicht mehr im lokalen Cluster des Rancher-Servers bereitgestellt, und `continuous-delivery` wird nicht in der Rancher-Benutzeroberfläche angezeigt.
