Dieses Feature ermöglicht die Verwendung von Typen für Speicheranbieter und Bereitsteller, die standardmäßig nicht aktiviert sind.

Um dieses Feature zu aktivieren oder zu deaktivieren, beachten Sie die Anweisungen auf {xref-experimental-features}[der Hauptseite zum Aktivieren experimenteller Funktionen.]

|===
| Umgebungsvariablen-Schlüssel | Standardwert | Beschreibung

| `unsupported-storage-drivers`
| `false`
| Dieses Feature ermöglicht die Verwendung von Typen für Speicheranbieter und Bereitsteller, die standardmäßig nicht aktiviert sind.
|===

[#_types_for_persistent_volume_plugins_that_are_enabled_by_default]
== Typen für persistente Volume-Plugins, die standardmäßig aktiviert sind

Nachfolgend finden Sie eine Liste von Speichertypen für persistente Volume-Plugins, die standardmäßig aktiviert sind. Beim Aktivieren dieses Feature-Flags werden alle persistenten Volume-Plugins, die nicht auf dieser Liste stehen, als experimentell und nicht unterstützt betrachtet:

|===
| Name | Plugin

| Amazon EBS Disk
| `aws-ebs`

| AzureFile
| `azure-file`

| AzureDisk
| `azure-disk`

| Google Persistent Disk
| `gce-pd`

| VMware vSphere Volume
| `vsphere-volume`

| Lokal
| `local`

| Network File System
| `nfs`

| hostPath
| `host-path`
|===
