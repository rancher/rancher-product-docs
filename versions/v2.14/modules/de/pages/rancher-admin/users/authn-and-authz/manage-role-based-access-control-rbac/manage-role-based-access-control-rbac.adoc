Innerhalb von Rancher authentifiziert sich jede Person als _Benutzer_, was ein Login ist, das Ihnen Zugriff auf Rancher gewährt. Wie in {xref-authn-authz}[Authentifizierung] erwähnt, können Benutzer entweder lokal oder extern sein.

Nachdem Sie die externe Authentifizierung konfiguriert haben, ändert sich die Anzeige der Benutzer auf der *Benutzer*-Seite.

* Wenn Sie als lokaler Benutzer angemeldet sind, werden nur lokale Benutzer angezeigt.
* Wenn Sie als externer Benutzer angemeldet sind, werden sowohl externe als auch lokale Benutzer angezeigt.

[#_users_and_roles]
== Benutzer und Funktionen

Sobald der Benutzer sich bei Rancher anmeldet, wird seine _Autorisierung_, oder seine Zugriffsrechte im System, durch _globale Berechtigungen_ und _Cluster- und Projektrollen_ bestimmt.

* {xref-global-permissions}[Globale Berechtigungen]:
+
Definieren Sie die Benutzerautorisierung außerhalb des Rahmens eines bestimmten Clusters.

* {xref-cluster-project-roles}[Cluster- und Projektrollen]:
+
Definieren Sie die Benutzerautorisierung innerhalb des spezifischen Clusters oder Projekts, in dem ihnen die Rolle zugewiesen wird.

Sowohl globale Berechtigungen als auch Cluster- und Projektrollen werden auf https://kubernetes.io/docs/reference/access-authn-authz/rbac/[Kubernetes RBAC] implementiert. Daher erfolgt die Durchsetzung von Berechtigungen und Rollen durch Kubernetes.
