Sie können Rollen auf den Status `locked` setzen. Das Sperren von Rollen verhindert, dass sie in Zukunft Benutzern zugewiesen werden.

Gesperrte Rollen:

* Können nicht Benutzern zugewiesen werden, die sie nicht bereits zugewiesen bekommen haben.
* Werden nicht im *Mitglieder-Rollen*-Dropdown-Menü angezeigt, wenn Sie einen Benutzer zu einem Cluster oder Projekt hinzufügen.
* Beeinflussen nicht die Benutzer, die die Rolle erhalten haben, bevor Sie die Rolle sperren. Diese Benutzer behalten den Zugriff, den die Rolle bietet.
+
*Beispiel:* Angenommen, Ihre Organisation erstellt eine interne Richtlinie, die besagt, dass Benutzern, die einem Cluster zugewiesen sind, das Erstellen neuer Projekte untersagt ist. Es ist Ihre Aufgabe, diese Richtlinie durchzusetzen.
+
Um dies durchzusetzen, sollten Sie, bevor Sie neue Benutzer zum Cluster hinzufügen, die folgenden Rollen sperren: `Cluster Owner`, `Cluster Member` und `Create Projects`. Dann könnten Sie eine neue benutzerdefinierte Rolle erstellen, die die gleichen Berechtigungen wie ein *Clustermitglied* enthält, mit Ausnahme der Möglichkeit, Projekte zu erstellen. Dann verwenden Sie diese neue benutzerdefinierte Rolle, wenn Sie Benutzer zu einem Cluster hinzufügen.

Rollen können von den folgenden Benutzern gesperrt werden:

* Jeder Benutzer, der die `Administrator` globale Berechtigung zugewiesen bekommen hat.
* Jeder Benutzer, der die `Custom Users` Berechtigung sowie die `Manage Roles` Rolle zugewiesen bekommen hat.

[#_locking_unlocking_roles]
== Rollen sperren/entsperren

Wenn Sie verhindern möchten, dass eine Rolle Benutzern zugewiesen wird, können Sie sie auf den Status `locked` setzen.

Sie können Rollen in zwei Kontexten sperren:

* Wenn Sie {xref-custom-roles}[eine benutzerdefinierte Rolle hinzufügen].
* Wenn Sie eine vorhandene Rolle bearbeiten (siehe unten).

Cluster-Rollen und Projekt-/Namespace-Rollen können gesperrt werden, aber globale Rollen können nicht gesperrt werden.

. Klicken Sie in der oberen linken Ecke auf *☰ > Benutzer & Authentifizierung*.
. Klicken Sie in der linken Navigationsleiste auf *Rollenvorlagen*.
. Gehen Sie zum *Cluster*-Tab oder zum *Projekt/Namensräume*-Tab.
. Wählen Sie aus der Rolle, die Sie sperren (oder entsperren) möchten, *⋮ > Konfiguration bearbeiten* aus.
. Wählen Sie aus der Option *Gesperrt* die Radiobuttons *Ja* oder *Nein* aus. Klicken Sie dann auf *Speichern*.
