Globale Ressourcen sind nicht-namenspezifische Ressourcen, die von Rancher bereitgestellt werden. Benutzer, die Cluster-Besitzer oder Projekt-Besitzer sind, haben standardmäßig keinen Zugriff auf mehrere dieser Ressourcen. Zum Beispiel auf Principals und RoleTemplates.

Die Ressourcen sind hier dokumentiert, um Administratoren bei der Erstellung und/oder Modifizierung benutzerdefinierter Berechtigungssets zu unterstützen, indem sie das minimale Set an Berechtigungen finden, das für eine bestimmte Aufgabe im Dashboard erforderlich ist.

Als Beispiel erfordert die Nutzung der Seiten für Cluster-/Projektberechtigungen im Dashboard Berechtigungen für Principals, um einen lesbaren Namen für Benutzer zu suchen/anzuzeigen, und für RoleTemplates, um eine Liste verwendbarer RoleTemplates zu sehen sowie einen lesbaren Namen für die Rolle anzuzeigen.

Bitte sehen Sie sich die folgende Liste an, um zu bestimmen, welche Berechtigungen Sie möglicherweise benötigen, wenn Sie einen Benutzer mit minimalen Rechten erstellen.
