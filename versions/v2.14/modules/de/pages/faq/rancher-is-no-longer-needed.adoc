Diese Seite soll Fragen beantworten, was passiert, wenn Sie Rancher nicht mehr verwenden, wenn Sie nicht möchten, dass ein Cluster weiterhin von Rancher verwaltet wird, oder wenn der Rancher-Server gelöscht wird.

[#_if_the_rancher_server_is_deleted_what_happens_to_the_workloads_in_my_downstream_clusters]
== Was passiert mit den Workloads in meinen Downstream-Clustern, wenn der Rancher-Server gelöscht wird?

Wenn Rancher jemals gelöscht oder nicht wiederherstellbar ist, werden alle Workloads in den Downstream-Kubernetes-Clustern, die von Rancher verwaltet werden, weiterhin normal funktionieren.

[#_if_the_rancher_server_is_deleted_how_do_i_access_my_downstream_clusters]
== Wenn der Rancher-Server gelöscht wird, wie greife ich auf meine Downstream-Cluster zu?

Die Möglichkeit, auf einen Downstream-Cluster ohne Rancher zuzugreifen, hängt von der Art des Clusters und der Art und Weise ab, wie der Cluster erstellt wurde. Zusammenfassung:

* *Registrierte/Importierte Cluster:* Der Cluster bleibt unbeeinflusst und Sie können auf den Cluster mit denselben Methoden zugreifen, die Sie verwendet haben, bevor der Cluster in Rancher registriert wurde.
ifeval::["{build-type}" != "srfa"] 
* *Gehostete Kubernetes-Cluster:* Wenn Sie den Cluster bei einem cloud-gehosteten Kubernetes-Anbieter wie EKS, GKE oder AKS erstellt haben, können Sie den Cluster weiterhin mit den Cloud-Anmeldeinformationen Ihres Anbieters verwalten.
endif::[]
ifeval::["{build-type}" == "srfa"]
* *Gehostete Kubernetes-Cluster:* Wenn Sie den Cluster bei einem cloud-gehosteten Kubernetes-Anbieter wie EKS erstellt haben, können Sie den Cluster weiterhin mit den Cloud-Anmeldeinformationen Ihres Anbieters verwalten.
endif::[]
* *Von Rancher bereitgestellte Cluster:* Um auf einen {xref-launch-k8s-with-rancher}[RKE2/K3s-Cluster] zuzugreifen, muss der Cluster den {xref-arch-com-ds-cluster}#_4_autorisierte_cluster_endpunkt[autorisierte Cluster-Endpunkt] aktiviert haben, und Sie müssen die kubeconfig-Datei des Clusters bereits über die Rancher-Benutzeroberfläche heruntergeladen haben. (Der autorisierte Cluster-Endpunkt ist standardmäßig für RKE-Cluster aktiviert.) Mit diesem Endpunkt können Sie direkt mit kubectl auf Ihren Cluster zugreifen, anstatt über den {xref-arch-com-ds-cluster}#_1_the_authentication_proxy[Authentication Proxy] des Rancher-Servers zu kommunizieren. Für Anweisungen zur Konfiguration von kubectl zur Verwendung des autorisierten Cluster-Endpunkts verweisen Sie auf den Abschnitt über den direkten Zugriff auf Cluster mit {xref-access-cluster-kubectl}#_authenticating_directly_with_a_downstream_cluster[kubectl und der kubeconfig-Datei.] Diese Cluster verwenden einen Snapshot der Authentifizierung, wie sie konfiguriert war, als Rancher entfernt wurde.

[#_what_if_i_dont_want_rancher_anymore]
== Was ist, wenn ich Rancher nicht mehr verwenden möchte?

[NOTE]
====

Das zuvor empfohlene xref:[System Tools] wurde seit Juni 2022 ausgelaufen.
====


Wenn Sie {xref-install-rancher}[Rancher auf einem Kubernetes-Cluster installiert haben,] entfernen Sie Rancher mit dem https://github.com/rancher/rancher-cleanup[Rancher Cleanup] Tool.

Die Deinstallation von Rancher im Hochverfügbarkeitsmodus (HA) entfernt auch alle `helm-operation-*` Pods und die folgenden Apps:

* fleet
* fleet-agent
* rancher-operator
* rancher-webhook

Benutzerdefinierte Ressourcen (CRDs) und benutzerdefinierte Namespaces müssen weiterhin manuell entfernt werden.

Wenn Sie Rancher mit Docker installiert haben, können Sie Rancher deinstallieren, indem Sie den einzelnen Docker-Container entfernen, in dem es läuft.

Importierte Cluster werden nicht von der Deinstallation von Rancher betroffen sein. Für andere Arten von Clustern verweisen Sie auf den Abschnitt über <<_if_the_rancher_server_is_deleted_how_do_i_access_my_downstream_clusters,den Zugriff auf Downstream-Cluster, wenn Rancher entfernt wird.>>

[#_what_if_i_dont_want_my_registered_cluster_managed_by_rancher]
== Was ist, wenn ich nicht mehr möchte, dass mein registrierter Cluster von Rancher verwaltet wird?

Wenn ein registrierter Cluster aus der Rancher-Benutzeroberfläche gelöscht wird, wird der Cluster von Rancher getrennt, bleibt jedoch intakt und ist mit denselben Methoden zugänglich, die verwendet wurden, um darauf zuzugreifen, bevor er in Rancher registriert wurde.

Um den Cluster zu trennen,

. Klicken Sie in der oberen linken Ecke auf *☰ > Clusterverwaltung*.
. Gehen Sie zu dem registrierten Cluster, der von Rancher getrennt werden soll, und klicken Sie auf *⋮ > Löschen*.
. Klicken Sie auf *Löschen*.

*Ergebnis:* Der registrierte Cluster wird von Rancher getrennt und funktioniert normal außerhalb von Rancher.

[#_what_if_i_dont_want_my_hosted_kubernetes_cluster_managed_by_rancher]
== Was ist, wenn ich nicht möchte, dass mein gehosteter Kubernetes-Cluster von Rancher verwaltet wird?

Derzeit gibt es keine Funktionalität, um diese Cluster von Rancher zu trennen. In diesem Kontext wird "trennen" als die Fähigkeit definiert, Rancher-Komponenten vom Cluster zu entfernen und den Zugriff auf den Cluster unabhängig von Rancher zu verwalten.

Die Fähigkeit, diese Cluster ohne Rancher zu verwalten, wird in diesem https://github.com/rancher/rancher/issues/25234[Issue.] verfolgt.

Für Informationen darüber, wie man auf Cluster zugreift, wenn der Rancher-Server gelöscht wird, verweisen Sie auf <<_if_the_rancher_server_is_deleted_how_do_i_access_my_downstream_clusters,diesen Abschnitt.>>
