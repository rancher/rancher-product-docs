Rancher erweitert Kubernetes mit zusätzlichen APIs, indem es einen Erweiterung-API-Server registriert, der die https://kubernetes.io/docs/concepts/extend-kubernetes/api-extension/apiserver-aggregation/[Kubernetes API Aggregationschicht] nutzt.

[#_aggregation_layer_is_required]
== Aggregation-Schicht ist erforderlich

Die API-Aggregationsschicht muss im lokalen Kubernetes-Cluster konfiguriert werden, damit die `v1.ext.cattle.io` `APIService` korrekt funktioniert. Wenn der `APIService` nach dem Start des Rancher-Servers keine Registrierungsanfrage erhält, stürzt der Pod ab und es wird ein Protokolleintrag angezeigt, der den Fehler angibt. Wenn Ihre Pods trotz korrekt konfiguriertem Cluster ständig Schwierigkeiten haben, die Registrierung zu erkennen, können Sie das Timeout erhöhen, indem Sie die `.Values.aggregationRegistrationTimeout` in Helm festlegen.

In dieser Version von Rancher sind die von RKE2/K3s unterstützten Kubernetes-Versionen standardmäßig mit der Aggregationsschicht konfiguriert und aktiviert. Wenn Sie jedoch vermuten, dass Ihre Clusterkonfiguration fehlerhaft ist, konsultieren Sie die https://kubernetes.io/docs/tasks/extend-kubernetes/configure-aggregation-layer/[Kubernetes Aggregationschicht-Dokumentation] für Informationen zur Konfiguration der Aggregationschicht.

[NOTE]
====
Wenn die zugrunde liegende Kubernetes-Distribution die Aggregationsschicht nicht unterstützt, müssen Sie vor dem Upgrade zu einer Kubernetes-Distribution migrieren, die dies tut.
====
