Stellen Sie eine Arbeitslast bereit, um eine Anwendung in einem oder mehreren Containern auszuführen.

. Klicken Sie in der oberen linken Ecke auf *☰ > Clusterverwaltung*.
. Gehen Sie zu dem Cluster, in dem Sie eine Arbeitslast upgraden möchten, und klicken Sie auf *Explore*.
. Klicken Sie in der linken Navigationsleiste auf *Workload*.
. Klicken Sie auf *Erstellen*.
. Wählen Sie den Typ der Arbeitslast aus.
. Wählen Sie den Namespace aus, in dem die Arbeitslast bereitgestellt wird.
. Geben Sie einen *Name* für die Arbeitslast ein.
. Geben Sie im Feld *Container Image* den Namen des Docker-Images ein, das Sie im Projekt bereitstellen möchten, und fügen Sie optional den Host der Registry hinzu (z. B. `quay.io`, `registry.gitlab.com` usw.). Während der Bereitstellung zieht Rancher dieses Image aus der angegebenen öffentlichen oder privaten Registry. Wenn kein Registry-Host angegeben ist, zieht Rancher das Image von https://hub.docker.com/explore/[Docker Hub]. Geben Sie den Namen genau so ein, wie er im Registry-Server erscheint, einschließlich des erforderlichen Pfads, und fügen Sie optional das gewünschte Tag hinzu (z. B. `registry.gitlab.com/user/path/image:tag`). Wenn kein Tag angegeben ist, wird automatisch das `latest` Tag verwendet.
. Wählen Sie entweder einen vorhandenen Namespace aus oder klicken Sie auf *Add to a new namespace* und geben Sie einen neuen Namespace ein.
. Klicken Sie auf *Add Port*, um eine Portzuordnung einzugeben, die den Zugriff auf die Anwendung innerhalb und außerhalb des Clusters ermöglicht. Für weitere Informationen siehe {xref-workloads-and-pods}#_services[Services].
. Konfigurieren Sie die verbleibenden Optionen:
 ** *Umgebungsvariablen*
+
Verwenden Sie diesen Abschnitt, um entweder Umgebungsvariablen für Ihre Arbeitslast zur Laufzeit anzugeben oder sie aus einer anderen Quelle, wie einem Geheimnis oder {xref-configmaps}[ConfigMap], abzurufen.

 ** *Knotenplanung*
 ** *Zustandsprüfung*
 ** *Volumes*
+
Verwenden Sie diesen Abschnitt, um Speicher für Ihre Arbeitslast hinzuzufügen. Sie können das Volumen, das Sie hinzufügen möchten, manuell angeben, einen persistenten Volumenanspruch verwenden, um ein Volumen für die Arbeitslast dynamisch zu erstellen, oder Daten für ein Volumen aus einer Datei wie einer {xref-configmaps}[ConfigMap] lesen.
+
Wenn Sie ein Stateful Set bereitstellen, sollten Sie eine Volumenanspruchsvorlage verwenden, wenn Sie persistente Volumes nutzen. Dies stellt sicher, dass persistente Volumes dynamisch erstellt werden, wenn Sie Ihr Stateful Set skalieren.

 ** *Skalierungs-/Upgrade-Richtlinie*

+

[NOTE]
.Amazon-Hinweis zu Volumes:
====
Um ein Amazon EBS-Volume zu mounten:
 ** In https://aws.amazon.com/[Amazon AWS] müssen die Knoten in derselben Verfügbarkeitszone sein und IAM-Berechtigungen besitzen, um Volumes anzuhängen/abzuhängen.
 ** Der Cluster muss die Option https://github.com/kubernetes/website/blob/release-1.18/content/en/docs/concepts/cluster-administration/cloud-providers.md#aws[AWS-Cloudanbieter] verwenden. Für weitere Informationen zur Aktivierung dieser Option siehe {xref-infra-aws}[Erstellen eines Amazon EC2-Clusters] oder {xref-custom-clusters}[Erstellen eines benutzerdefinierten Clusters].
====

. Klicken Sie auf *Erweiterte Optionen anzeigen* und konfigurieren Sie:
 ** *Kommando*
 ** *Netzwerke*
 ** *Labels & Anmerkungen*
 ** *Sicherheits- und Hostkonfiguration*
. Klicken Sie auf *Starten*.

*Ergebnis:* Die Arbeitslast wird im gewählten Namespace bereitgestellt. Sie können den Status der Arbeitslast im *Arbeitslasten*-Bereich des Projekts einsehen.
