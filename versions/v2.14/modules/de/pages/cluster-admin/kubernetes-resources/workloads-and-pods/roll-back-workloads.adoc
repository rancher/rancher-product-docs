Manchmal besteht die Notwendigkeit, einen Rollback auf die vorherige Version der Anwendung durchzuführen, entweder zu Debugging-Zwecken oder weil ein Upgrade nicht wie geplant verlief.

. Klicken Sie in der oberen linken Ecke auf *☰ > Clusterverwaltung*.
. Gehen Sie zu dem Cluster, in dem Sie eine Arbeitslast upgraden möchten, und klicken Sie auf *Explore*.
. Klicken Sie in der linken Navigationsleiste auf *Arbeitslast*.
. Suchen Sie die Arbeitslast, bei der Sie einen Rollback durchführen möchten, und wählen Sie *⋮ > Rollback* aus.
. Wählen Sie die Revision aus, auf die Sie einen Rollback durchführen möchten. Klicken Sie auf *Rollback*.

*Ergebnis:* Ihre Arbeitslast wird auf die vorherige Version zurückgesetzt, die Sie ausgewählt haben. Warten Sie einige Minuten, bis die Aktion abgeschlossen ist.
