Wenn eine neue Version eines Anwendungsimages auf Docker Hub veröffentlicht wird, können Sie alle Workloads, die eine vorherige Version der Anwendung ausführen, auf die neue Version upgraden.

. Klicken Sie in der oberen linken Ecke auf *☰ > Clusterverwaltung*.
. Gehen Sie zu dem Cluster, in dem Sie eine Arbeitslast upgraden möchten, und klicken Sie auf *Explore*.
. Klicken Sie in der linken Navigationsleiste auf *Workload*.
. Suchen Sie den Workload, den Sie upgraden möchten, und wählen Sie *⋮ > Konfiguration bearbeiten* aus.
. Aktualisieren Sie das *Container-Image* und alle Optionen, die Sie ändern möchten.
. Überprüfen und bearbeiten Sie die *Skalierungs- und Upgrade-Richtlinie* des Workloads.
+
Diese Optionen steuern, wie das Upgrade auf die derzeit laufenden Container ausgerollt wird. Zum Beispiel können Sie bei skalierbaren Bereitstellungen wählen, ob Sie alte Pods beenden möchten, bevor Sie neue bereitstellen, oder umgekehrt, sowie die Upgrade-Batch-Größe.

. Klicken Sie auf *Speichern*.

*Ergebnis:* Der Workload beginnt damit, seine Container zu upgraden, gemäß Ihren Vorgaben. Beachten Sie, dass das Hochskalieren der Bereitstellung oder das Aktualisieren der Upgrade-/Skalierungsrichtlinie nicht zur Neuerstellung der Pods führt.
