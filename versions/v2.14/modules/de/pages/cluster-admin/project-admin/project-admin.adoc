_Projekte_ sind Objekte, die in Rancher eingeführt wurden und dabei helfen, Namespaces in Ihrem Kubernetes-Cluster zu organisieren. Sie können Projekte verwenden, um Multi-Tenant-Cluster zu erstellen, die es einer Gruppe von Benutzern ermöglichen, die gleichen zugrunde liegenden Ressourcen zu teilen, ohne mit den Anwendungen der anderen zu interagieren.

In Bezug auf die Hierarchie:

* Cluster enthalten Projekte
* Projekte enthalten Namespaces

Innerhalb von Rancher ermöglichen Projekte die Verwaltung mehrerer Namespaces als eine einzige Einheit. In nativem Kubernetes, das keine Projekte umfasst, werden Funktionen wie rollenbasierte Zugriffsrechte oder Clusterressourcen einzelnen Namespaces zugewiesen. In Clustern, in denen mehrere Namespaces die gleichen Zugriffsrechte benötigen, kann es mühsam werden, diese Rechte jedem einzelnen Namespace zuzuweisen. Selbst wenn alle Namespaces die gleichen Rechte benötigen, gibt es keine Möglichkeit, diese Rechte in einer einzigen Aktion auf alle Ihre Namespaces anzuwenden. Sie müssten diese Rechte wiederholt jedem Namespace zuweisen!

Rancher-Projekte lösen dieses Problem, indem sie es Ihnen ermöglichen, Ressourcen und Zugriffsrechte auf Projektebene anzuwenden. Jeder Namespace im Projekt erbt dann diese Ressourcen und Richtlinien, sodass Sie sie nur einmal dem Projekt zuweisen müssen, anstatt sie jedem einzelnen Namespace zuzuweisen.

Sie können Projekte verwenden, um Aktionen wie Folgendes auszuführen:

* {xref-add-users-to-projects}[][Benutzern Zugriff auf eine Gruppe von Namespaces zuweisen]
* Benutzern {xref-cluster-project-roles}#_project_roles[spezifische Rollen in einem Projekt] zuweisen. Eine Rolle kann Projektbesitzer, Mitglied, Nur-Lesezugriff oder {xref-custom-roles}[benutzerdefiniert] sein.
* {xref-project-resource-quotas}[Ressourcenkontingente festlegen]
* {xref-namespaces}[Namespaces verwalten]
* {xref-rancher-project-tools}[Werkzeuge konfigurieren]

[#_authorization]
== Autorisierung

Nicht-administrative Benutzer sind nur nach einem {xref-global-permissions}[Administrator], {xref-cluster-project-roles}#_cluster_roles[Clusterbesitzer oder Mitglied], oder {xref-cluster-project-roles}#_project_roles[Projektbesitzer] autorisiert, auf das Projekt zuzugreifen, wenn sie zum Tab *Mitglieder* des Projekts hinzugefügt werden.

Wer das Projekt erstellt, wird automatisch {xref-cluster-project-roles}#_project_roles[Projektbesitzer].

[#_switching_between_projects]
== Wechseln zwischen Projekten

Um zwischen Projekten zu wechseln, verwenden Sie das Dropdown-Menü in der Navigationsleiste. Alternativ können Sie direkt in der Navigationsleiste zwischen Projekten wechseln.

. Klicken Sie in der oberen linken Ecke auf *☰ > Clusterverwaltung*.
. Gehen Sie auf der Seite *Clusters* zu dem Cluster, in dem Sie die Projekte wechseln möchten, und klicken Sie auf *Erkunden*.
. Wählen Sie in der oberen Navigationsleiste das Projekt aus, das Sie öffnen möchten.
