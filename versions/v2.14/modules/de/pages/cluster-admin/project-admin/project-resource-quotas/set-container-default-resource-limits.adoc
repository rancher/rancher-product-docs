Beim Festlegen von Ressourcenquoten sollten Sie, wenn Sie auf einem Projekt oder Namespace Einstellungen bezüglich CPU oder Speicher (d.h. Limits oder Reservierungen) vornehmen, sicherstellen, dass bei der Erstellung jedes Containers das entsprechende CPU- oder Speicherfeld gesetzt wird. Siehe die https://kubernetes.io/docs/concepts/policy/resource-quotas/#requests-vs-limits[Kubernetes-Dokumentation] für weitere Details, warum dies erforderlich ist.

Um zu vermeiden, dass diese Limits bei jeder Containererstellung festgelegt werden, kann ein Standard-Ressourcengrenzwert für Container auf dem Namespace festgelegt werden.

[#_editing_the_container_default_resource_limit]
== Bearbeiten der Standardressourcengrenze für Container

Bearbeiten Sie die Standardressourcengrenze für Container, wenn:

* Sie eine CPU- oder Speicherressourcenquote für ein Projekt festgelegt haben und die entsprechenden Standardwerte für einen Container bereitstellen möchten.
* Sie die Standardressourcengrenze für Container bearbeiten möchten.

. Klicken Sie in der oberen linken Ecke auf *☰ > Clusterverwaltung*.
. Gehen Sie auf der Seite *Cluster* zu dem Cluster, in dem Sie die Standardressourcengrenze bearbeiten möchten, und klicken Sie auf *Erkunden*.
. Klicken Sie auf menu:Cluster[Projekte/Namespaces].
. Finden Sie das Projekt, dessen Standardressourcengrenze für Container Sie bearbeiten möchten. Wählen Sie aus diesem Projekt *⋮ > Konfiguration bearbeiten*.
. Erweitern Sie *Standardressourcengrenze für Container* und bearbeiten Sie die Werte.

[#_resource_limit_propagation]
== Propagation der Ressourcengrenzen

Wenn die Standardressourcengrenze für Container auf Projektebene festgelegt ist, wird der Parameter auf jeden Namespace propagiert, der nach Festlegung des Limits im Projekt erstellt wird. Für jeden bestehenden Namespace in einem Projekt wird dieses Limit nicht automatisch propagiert. Sie müssen die Standardressourcengrenze für Container manuell für alle bestehenden Namespaces im Projekt festlegen, damit sie bei der Erstellung von Containern verwendet werden kann.

Sie können eine Standardressourcengrenze für Container auf einem Projekt festlegen und beliebige Kataloganwendungen starten.

Sobald eine Standardressourcengrenze für Container in einem Namespace konfiguriert ist, wird der Standardwert für alle in diesem Namespace erstellten Container vorausgefüllt. Diese Limits/Reservierungen können während der Erstellung von Arbeitslasten jederzeit überschrieben werden.

[#_container_resource_quota_types]
== Arten von Ressourcenquoten für Container

Die folgenden Ressourcenlimits können konfiguriert werden:

|===
| Ressourcentyp | Beschreibung

| CPU Limit
| Die maximale Menge an CPU (in https://kubernetes.io/docs/concepts/configuration/manage-compute-resources-container/#meaning-of-cpu[Millicores]), die dem Container zugewiesen ist.

| CPU-Reservierung
| Die minimale Menge an CPU (in Millicores), die dem Container garantiert wird.

| Speicherlimit
| Die maximale Menge an Speicher (in Bytes), die dem Container zugewiesen ist.

| Speicherreservierung
| Die minimale Menge an Speicher (in Bytes), die dem Container garantiert wird.

| NVIDIA GPU-Limit/Reservierung
| Die Anzahl der GPUs, die dem Container zugewiesen sind. Das Limit und die Reservierung sind für GPUs immer gleich.
|===
