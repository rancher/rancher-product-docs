Rancher unterstützt die Verwaltung von Clustern, die aufgrund von Netzwerkunterbrechungen, fehlender Verfügbarkeit der Steuerungsebene oder ausgefallener Clusterknoten möglicherweise nicht immer online sind. Derzeit gibt es in der neuesten veröffentlichten Rancher-Version keine bekannten Probleme mit getrennten Clustern.

Während ein verwalteter Cluster von Rancher getrennt ist, sind Verwaltungsoperationen nicht verfügbar, und die Rancher-Benutzeroberfläche erlaubt keine Navigation zum Cluster. Sobald die Verbindung jedoch wiederhergestellt ist, wird die Funktionalität vollständig wiederhergestellt.

[#_best_practices_for_managing_disconnected_clusters]
== Best Practices für die Verwaltung von getrennten Clustern

* *Clusterverfügbarkeit während Rancher-Upgrades*: Es wird empfohlen, dass alle oder zumindest die meisten verwalteten Cluster während eines Rancher-Upgrades online sind. Der Grund dafür ist, dass ein Upgrade von Rancher automatisch die Rancher-Agentensoftware aktualisiert, die auf verwalteten Clustern läuft. Die Angleichung der Agenten- und Rancher-Versionen gewährleistet eine konsistente Funktionalität. Alle Cluster, die während des Upgrades getrennt sind, werden ihre Agenten aktualisiert, sobald sie sich wieder verbinden.

* *Bereinigung von getrennten Clustern*: Entfernen Sie regelmäßig Cluster, die sich nicht mehr mit Rancher verbinden werden (z. B. Cluster, die stillgelegt oder zerstört wurden). Das Beibehalten solcher Cluster im Rancher-Managementsystem verbraucht unnötige Ressourcen, was die Leistung von Rancher im Laufe der Zeit beeinträchtigen könnte.

* *Überlegungen zur Zertifikatsrotation*: Bei der Gestaltung von Prozessen, die regelmäßig das Herunterfahren von Clustern beinhalten, unabhängig davon, ob sie mit Rancher verbunden sind oder nicht, sollten die Richtlinien zur Zertifikatsrotation berücksichtigt werden. Zum Beispiel können RKE2/K3s-Cluster beim Start Zertifikate rotieren, wenn sie ihre Lebensdauer überschritten haben.
