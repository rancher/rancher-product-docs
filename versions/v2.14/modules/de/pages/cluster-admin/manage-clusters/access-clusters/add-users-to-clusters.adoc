Um einem Benutzer Zugriff auf bestimmte clusterbezogene Ressourcen zu gewähren und neue Projekte zu erstellen, weisen Sie dem Benutzer eine **Cluster-Mitgliedschaft** zu. Cluster-Mitglieder können Projekte erstellen und Ressourcen in diesen Projekten verwalten. Allerdings sind nicht alle Ressourcen, Namespaces und Workloads in einem Cluster für Cluster-Mitglieder zugänglich.

[TIP]
====
Möchten Sie einem Benutzer Zugriff auf ein _spezifisches_ Projekt innerhalb eines Clusters gewähren? Siehe stattdessen {xref-add-users-to-projects}[Projektmitglieder hinzufügen].
====

Es gibt zwei Kontexte, in denen Sie Cluster-Mitglieder hinzufügen können:

* Mitglieder zu einem neuen Cluster hinzufügen
+
Sie können Mitglieder zu einem Cluster hinzufügen, während Sie es erstellen (empfohlen, wenn möglich).

* <<_editing_cluster_membership,Mitglieder zu einem bestehenden Cluster hinzufügen>>
+
Sie können immer Mitglieder zu einem Cluster hinzufügen, nachdem es bereitgestellt wurde.

[#_editing_cluster_membership]
== Cluster-Mitgliedschaft bearbeiten

Cluster-Administratoren können die Mitgliedschaft eines Clusters bearbeiten und steuern, welche Rancher-Benutzer auf den Cluster zugreifen können und welche Funktionen sie nutzen können.

. Klicken Sie auf *☰ > Clusterverwaltung*.
. Gehen Sie zu dem Cluster, dem Sie Mitglieder hinzufügen möchten, und klicken Sie auf *⋮ > Konfiguration bearbeiten*.
. Klicken Sie im *Mitgliederrollen*-Tab auf *Mitglied hinzufügen*.
. Suchen Sie nach dem Benutzer oder der Gruppe, die Sie zum Cluster hinzufügen möchten.
+
[NOTE]
.Anmerkungen
====
* Mindestens 2 Zeichen müssen in das Suchfeld eingegeben werden, damit Ergebnisse angezeigt werden.
* Benutzer können anhand ihres Benutzernamens oder Anzeigenamens gesucht werden.
* Die Suche basiert auf Präfixen (z. B. wird ein Benutzer mit dem Namen `Stan Dard` angezeigt, wenn nach `Sta` gesucht wird, jedoch nicht, wenn nach `Dar` gesucht wird) und ist groß-/kleinschreibungssensitiv.
====
+
Wenn die externe Authentifizierung konfiguriert ist:


 ** Rancher gibt Benutzer aus Ihrer {xref-authn-authz}[externen Authentifizierungs]-Quelle zurück, während Sie tippen.
+

[NOTE]
.Verwenden Sie AD, aber finden Sie Ihre Benutzer nicht?
====
Es könnte ein Problem mit Ihrer Konfiguration des Suchattributs geben. Siehe {xref-configure-ad}[Konfigurieren der Active Directory-Authentifizierung: Schritt 5].
====


 ** Ein Dropdown-Menü ermöglicht es Ihnen, Gruppen anstelle von einzelnen Benutzern hinzuzufügen. Das Dropdown-Menü listet nur Gruppen auf, zu denen Sie, der angemeldete Benutzer, gehören.
+

[NOTE]
====
Wenn Sie als lokaler Benutzer angemeldet sind, werden externe Benutzer nicht in Ihren Suchergebnissen angezeigt. Für weitere Informationen siehe {xref-authn-authz}#_externe_authentifizierungskonfiguration_und_hauptbenutzer[Externe Authentifizierungskonfiguration und Hauptbenutzer].
====

. Weisen Sie dem Benutzer oder der Gruppe *Cluster* Rollen zu.
+
{xref-cluster-project-roles}[Was sind Clusterrollen?]
+

[TIP]
====
Für benutzerdefinierte Rollen können Sie die Liste der verfügbaren einzelnen Rollen zur Zuweisung ändern.

* Um Rollen zur Liste hinzuzufügen, {xref-custom-roles}[Fügen Sie eine benutzerdefinierte Rolle hinzu].
* Um Rollen aus der Liste zu entfernen, {xref-locked-roles}[Lock/Unlock Roles].
====

*Ergebnis:* Die ausgewählten Benutzer werden dem Cluster hinzugefügt.

* Um die Cluster-Mitgliedschaft zu widerrufen, wählen Sie den Benutzer aus und klicken Sie auf *Löschen*. Diese Aktion löscht die Mitgliedschaft, nicht den Benutzer.
* Um die Rollen eines Benutzers im Cluster zu ändern, löschen Sie ihn aus dem Cluster und fügen Sie ihn dann mit geänderten Rollen wieder hinzu.
