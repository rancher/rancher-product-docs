Rancher unterstützt persistenten Speicher mit einer Vielzahl von Volume-Plugins. Bevor Sie jedoch eines dieser Plugins verwenden, um persistenten Speicher an Ihre Workloads zu binden, müssen Sie den Speicher selbst konfigurieren, egal ob es sich um eine cloudbasierte Lösung von einem Dienstanbieter oder um eine lokale Lösung handelt, die Sie selbst verwalten.

Zu Ihrer Bequemlichkeit bietet Rancher die Dokumentation zur Konfiguration einiger der gängigen Speichermethoden an:

* {xref-nfs-storage}[NFS]
ifeval::["{build-type}" != "srfa"]
* {xref-vsphere-storage-example}[vSphere]
endif::[]
* {xref-amazon-ebs-storage}[EBS]
