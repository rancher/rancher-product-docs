ServiceMonitors und PodMonitors sind beide Pseudo-CRDs, die die Scrape-Konfiguration der Prometheus-Custom-Ressource abbilden.

Diese Konfigurationsobjekte geben deklarativ die Endpunkte an, von denen Prometheus Metriken abruft.

ServiceMonitors werden häufiger verwendet als PodMonitors, und wir empfehlen sie für die meisten Anwendungsfälle.

[NOTE]
====

Dieser Abschnitt setzt voraus, dass Sie mit der Funktionsweise der Überwachungskomponenten vertraut sind. Für weitere Informationen über Alertmanager siehe {xref-how-monitoring-works}[diesen Abschnitt.]
====


[#_servicemonitors]
== ServiceMonitors

Dieses Pseudo-CRD entspricht einem Abschnitt der Konfiguration der Prometheus-Custom-Ressource. Es gibt deklarativ an, wie Gruppen von Kubernetes-Diensten überwacht werden sollten.

Wenn ein ServiceMonitor erstellt wird, aktualisiert der Prometheus-Operator die Scrape-Konfiguration von Prometheus, um die Konfiguration des ServiceMonitors einzuschließen. Dann beginnt Prometheus, Metriken vom im ServiceMonitor definierten Endpunkt abzurufen.

Alle Dienste in Ihrem Cluster, die mit den Labels im ServiceMonitor `selector` übereinstimmen, werden basierend auf dem `endpoints` überwacht, das im ServiceMonitor angegeben ist. Für weitere Informationen darüber, welche Felder angegeben werden können, siehe bitte das https://github.com/prometheus-operator/prometheus-operator/blob/main/Documentation/api-reference/api.md#servicemonitor[spec], das vom Prometheus-Operator bereitgestellt wird.

Für weitere Informationen darüber, wie ServiceMonitors funktionieren, siehe die https://github.com/prometheus-operator/prometheus-operator/blob/main/Documentation/user-guides/running-exporters.md[Dokumentation des Prometheus-Operators.]

[#_podmonitors]
== PodMonitors

Dieses Pseudo-CRD entspricht einem Abschnitt der Konfiguration der Prometheus-Custom-Ressource. Es gibt deklarativ an, wie eine Gruppe von Pods überwacht werden sollte.

Wenn ein PodMonitor erstellt wird, aktualisiert der Prometheus-Operator die Scrape-Konfiguration von Prometheus, um die Konfiguration des PodMonitors einzuschließen. Dann beginnt Prometheus, Metriken vom im PodMonitor definierten Endpunkt abzurufen.

Alle Pods in Ihrem Cluster, die mit den Labels im PodMonitor `selector` übereinstimmen, werden basierend auf dem `podMetricsEndpoints` überwacht, das im PodMonitor angegeben ist. Für weitere Informationen darüber, welche Felder angegeben werden können, siehe bitte das https://github.com/prometheus-operator/prometheus-operator/blob/main/Documentation/api-reference/api.md#podmonitor[spec], das vom Prometheus-Operator bereitgestellt wird.
