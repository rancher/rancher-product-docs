Diese Seite enthält einige der wichtigsten Optionen zur Konfiguration von Monitoring V2 in der Rancher UI.

Für Informationen zur Konfiguration benutzerdefinierter Scrape-Ziele und Regeln für Prometheus konsultieren Sie bitte die Upstream-Dokumentation für den https://github.com/prometheus-operator/prometheus-operator[Prometheus Operator.] Einige der wichtigsten benutzerdefinierten Ressourcen werden in der Design-Dokumentation des Prometheus Operators https://github.com/prometheus-operator/prometheus-operator/blob/main/Documentation/getting-started/design.md[erklärt.] Die Dokumentation des Prometheus Operators kann Ihnen auch helfen, RBAC, Thanos oder benutzerdefinierte Konfiguration einzurichten.

[#_setting_resource_limits_and_requests]
== Einstellung von Ressourcengrenzen und -anforderungen

Die Ressourcenanforderungen und -grenzen für die Monitoring-Anwendung können bei der Installation von `rancher-monitoring` konfiguriert werden. Für weitere Informationen zu den Standardgrenzen siehe {xref-helm-chart-options}#_configuring_resource_limits_and_requests[diese Seite.]

[TIP]
====

In einem leeren Cluster kann Monitoring eine hohe CPU-Auslastung aufweisen. Um die Leistung zu verbessern, schalten Sie den Prometheus-Adapter aus.
====


[#_prometheus_configuration]
== Prometheus-Konfiguration

Es ist normalerweise nicht erforderlich, die benutzerdefinierte Ressource von Prometheus direkt zu bearbeiten.

Stattdessen genügt es, einen neuen ServiceMonitor oder PodMonitor zu erstellen, damit Prometheus zusätzliche Metriken erfasst.

[#_alertmanager_configuration]
== Alertmanager-Konfiguration

Die benutzerdefinierte Ressource von Alertmanager muss normalerweise nicht direkt bearbeitet werden. Für die häufigsten Anwendungsfälle können Sie Alarme verwalten, indem Sie Routen und Empfänger aktualisieren.

Routen und Empfänger sind Teil der Konfiguration der benutzerdefinierten Ressource von Alertmanager. In der Rancher UI sind Routen und Empfänger keine echten benutzerdefinierten Ressourcen, sondern pseudo-benutzerdefinierte Ressourcen, die der Prometheus Operator verwendet, um Ihre Konfiguration mit der benutzerdefinierten Ressource von Alertmanager zu synchronisieren. Wenn Routen und Empfänger aktualisiert werden, wird die Monitoring-Anwendung Alertmanager automatisch aktualisieren, um diese Änderungen widerzuspiegeln.

Für einige fortgeschrittene Anwendungsfälle möchten Sie möglicherweise den Alertmanager direkt konfigurieren. Für weitere Informationen siehe {xref-configure-alertmanager}[diese Seite.]

[#_receivers]
=== Empfänger

Empfänger werden verwendet, um Benachrichtigungen einzurichten. Für Details zur Konfiguration von Empfängern siehe {xref-receivers}[diese Seite.]

[#_routes]
=== Routen

Routen filtern Benachrichtigungen, bevor sie die Empfänger erreichen. Jede Route muss auf einen Empfänger verweisen, der bereits konfiguriert wurde. Für Details zur Konfiguration von Routen siehe {xref-monitoring-routes}[diese Seite.]
