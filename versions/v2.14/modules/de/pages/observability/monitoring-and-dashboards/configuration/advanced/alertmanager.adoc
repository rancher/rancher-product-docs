Es ist normalerweise nicht erforderlich, die benutzerdefinierte Ressource des Alertmanagers direkt zu bearbeiten. Für die meisten Anwendungsfälle müssen Sie nur die Empfänger und Routen bearbeiten, um Benachrichtigungen zu konfigurieren.

Wenn Empfänger und Routen aktualisiert werden, wird die benutzerdefinierte Ressource des Alertmanagers automatisch aktualisiert, um mit diesen Änderungen übereinzustimmen.

[NOTE]
====

Dieser Abschnitt setzt voraus, dass Sie mit der Funktionsweise der Überwachungskomponenten vertraut sind. Für weitere Informationen über den Alertmanager siehe {xref-how-monitoring-works}#_3_how_alertmanager_works[diesen Abschnitt.]

[#_about_the_alertmanager_custom_resource]
== Über die benutzerdefinierte Ressource des Alertmanagers

Standardmäßig stellt Rancher Monitoring einen einzelnen Alertmanager in einem Cluster bereit, der ein Standard-Alertmanager-Konfigurations-Secret verwendet.

Sie möchten möglicherweise die benutzerdefinierte Ressource des Alertmanagers bearbeiten, wenn Sie von erweiterten Optionen profitieren möchten, die in den Formularen der Rancher-Benutzeroberfläche nicht verfügbar sind, wie die Möglichkeit, eine Routingbaumstruktur zu erstellen, die mehr als zwei Ebenen tief ist.

Es ist auch möglich, mehr als einen Alertmanager in einem Cluster zu erstellen, was nützlich sein kann, wenn Sie eine auf Namespace-Ebene beschränkte Überwachung implementieren möchten. In diesem Fall sollten Sie die benutzerdefinierten Ressourcen des Alertmanagers mit demselben zugrunde liegenden Alertmanager-Konfigurations-Secret verwalten.

[#_deeply_nested_routes]
=== Tief verschachtelte Routen

Während die Rancher-Benutzeroberfläche nur einen Routingbaum unterstützt, der zwei Ebenen tief ist, können Sie tiefer verschachtelte Routingstrukturen konfigurieren, indem Sie die Alertmanager-YAML bearbeiten.

[#_multiple_alertmanager_replicas]
=== Mehrere Alertmanager-Replikate

Im Rahmen der Implementierungsoptionen des Charts können Sie sich entscheiden, die Anzahl der Replikate des Alertmanagers, die in Ihrem Cluster bereitgestellt werden, zu erhöhen. Die Replikate können alle mit demselben zugrunde liegenden Alertmanager-Konfigurations-Secret verwaltet werden.

Dieses Secret sollte aktualisiert oder geändert werden, wann immer Sie:

* Neue Benachrichtiger oder Empfänger hinzufügen
* Die Alarme ändern, die an bestimmte Benachrichtiger oder Empfänger gesendet werden sollen
* Die Gruppe von Alarmen ändern, die versendet werden

Standardmäßig können Sie entweder wählen, ein vorhandenes Alertmanager Config Secret bereitzustellen (d.h. ein beliebiges Secret im `cattle-monitoring-system` Namespace) oder Rancher Monitoring zu erlauben, ein Standard-Alertmanager Config Secret in Ihrem Cluster bereitzustellen.

Standardmäßig wird das von Rancher erstellte Alertmanager Config Secret bei einem Upgrade oder einer Deinstallation des `rancher-monitoring` Charts niemals geändert oder gelöscht. Diese Einschränkung verhindert, dass Benutzer ihre Alarmkonfiguration verlieren oder überschreiben, wenn sie Operationen am Chart ausführen.

Für weitere Informationen darüber, welche Felder im Alertmanager Config Secret angegeben werden können, schauen Sie bitte in die https://prometheus.io/docs/alerting/latest/alertmanager/[Dokumentation zum Prometheus Alertmanager.]

Die vollständige Spezifikation für die Alertmanager-Konfigurationsdatei und was sie benötigt, finden Sie https://prometheus.io/docs/alerting/latest/configuration/#configuration-file[hier.]
