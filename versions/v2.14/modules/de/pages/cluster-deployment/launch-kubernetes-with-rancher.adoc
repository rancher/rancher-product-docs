Rancher ermöglicht es Ihnen, ein Kubernetes-Cluster auf verschiedenen Arten von Knoten zu starten, einschließlich Bare-Metal-Servern, lokalen virtuellen Maschinen und virtuellen Maschinen von einem Infrastruktur-Anbieter. Beim Bereitstellen von Kubernetes auf diesen Knoten gibt Rancher Ihnen die Möglichkeit, {url-rke2-introduction}[RKE2] oder {url-k3s}[K3s] zu verwenden.

Rancher kann Kubernetes auf vorhandenen Knoten installieren oder es kann dynamisch Knoten bei einem Infrastruktur-Anbieter bereitstellen und Kubernetes auf ihnen installieren.

Rancher kann auch Pools von Maschinen erstellen. Ein Vorteil der Installation von Kubernetes auf Maschinenpools, die von einem Infrastruktur-Anbieter gehostet werden, besteht darin, dass Rancher automatisch einen weiteren Knoten erstellen kann, um dem Cluster beizutreten, falls ein Knoten die Verbindung zum Cluster verliert, um sicherzustellen, dass die Anzahl des Maschinenpools wie erwartet ist.

[#__rke2_product_name]
== {rke2-product-name}

Rancher v2.6 führte die Bereitstellung von {url-rke2-introduction}[RKE2] Clustern direkt über die Rancher-GUI ein. RKE2, auch bekannt als RKE Government, ist eine vollständig konforme Kubernetes-Distribution, deren Schwerpunkt auf Sicherheit und Compliance innerhalb der USA liegt. US-Bundesbehörden liegt. In Rancher v.2.6.4 und früher war die RKE2-Bereitstellung in der technischen Vorschau.

Beachten Sie, dass in Rancher v2.6.5 die RKE2-Bereitstellung GA wurde.

[#_requirements]
=== Anforderungen

Wenn Sie RKE2 verwenden, um ein Cluster einzurichten, müssen Ihre Knoten die {url-rke2-requirements}[Anforderungen] für Knoten in Downstream-Benutzerclustern erfüllen.

[#_launching_kubernetes_on_new_nodes_in_an_infrastructure_provider]
=== Starten von Kubernetes auf neuen Knoten bei einem Infrastrukturprovider

Die RKE2-Bereitstellung basiert auf einem neuen Bereitstellungsrahmen, der das upstream https://github.com/kubernetes-sigs/cluster-api[Cluster API (CAPI)] Projekt nutzt. Mit diesem neuen Bereitstellungsrahmen können Sie:

* RKE2-Cluster auf jedem Anbieter bereitstellen, für den Rancher einen Knoten-Treiber hat.
* RKE2-Cluster innerhalb von Rancher vollständig konfigurieren.
* CNI-Optionen wie Calico, Cilium und Multus zusätzlich zu Canal auswählen.

Wenn Sie Änderungen an Ihrer Cluster-Konfiguration in RKE2 vornehmen, kann dies dazu führen, dass Knoten neu bereitgestellt werden. Dies wird von CAPI-Controllern und nicht von Rancher selbst gesteuert. Beachten Sie, dass für etcd-Knoten dasselbe Verhalten nicht zutrifft.

Ein Beispiel für eine Konfigurationsänderung, die das beschriebene Verhalten verursachen kann, ist das Bearbeiten eines Clusters und das Aktivieren von Drain vor dem Löschen. In diesem Szenario werden die vorhandenen Control-Plane-Knoten und die Worker-Knoten gelöscht und neue Knoten erstellt.

Die Bereitstellung von RKE2 umfasst auch die Installation von RKE2 auf Clustern mit Windows-Knoten.

Windows-Funktionen für RKE2 umfassen:

* Windows unterstützt den vSphere-Knotentreiber
* Calico und Flannel CNI für benutzerdefinierte Windows-RKE2-Cluster
* Projekt-Netzwerkisolierung (PNI) für Calico
* Windows-Container mit RKE2, betrieben von containerd
* Bereitstellung von Windows-RKE2-Clustern über Terraform
* Bereitstellung von benutzerdefinierten Windows-RKE2-Clustern direkt über die Rancher-Benutzeroberfläche

Die Unterstützung von Windows für benutzerdefinierte RKE2-Cluster erfordert die Wahl von Calico als CNI.

[#_launching_kubernetes_on_existing_custom_nodes]
=== Kubernetes auf vorhandenen benutzerdefinierten Knoten starten

Die Bereitstellung von RKE2 ermöglicht es Ihnen auch, benutzerdefinierte Cluster auf zuvor bereitgestellten VMs oder Bare-Metal-Knoten zu installieren.

Wenn Sie einen Knoten aus einem vorherigen benutzerdefinierten Cluster wiederverwenden möchten, {xref-clean-cluster-nodes}#_cleaning_up_nodes[reinigen Sie den Knoten], bevor Sie ihn erneut in einem Cluster verwenden. Wenn Sie einen Knoten wiederverwenden, der nicht gereinigt wurde, kann die Clusterbereitstellung fehlschlagen.

=== Programmgesteuertes Erstellen von {rke2-product-name} Clustern

Der gebräuchlichste Weg, RKE2-Cluster programmgesteuert über Rancher bereitzustellen, ist die Verwendung des https://registry.terraform.io/providers/rancher/rancher2/latest/docs/resources/cluster_v2[Rancher2 Terraform-Anbieters].
