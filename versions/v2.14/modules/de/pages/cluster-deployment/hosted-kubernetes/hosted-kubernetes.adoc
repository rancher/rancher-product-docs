ifeval::["{build-type}" != "srfa"]
In diesem Szenario provisioniert Rancher kein Kubernetes, da es von Anbietern wie Google Kubernetes Engine (GKE), Amazon Elastic Container Service für Kubernetes oder Azure Kubernetes Service installiert wird.

Wenn Sie einen Kubernetes-Anbieter wie Google GKE verwenden, integriert sich Rancher mit dessen Cloud-APIs, sodass Sie die rollenbasierte Zugriffskontrolle für den gehosteten Cluster über die Rancher-Benutzeroberfläche erstellen und verwalten können.

In diesem Anwendungsfall sendet Rancher eine Anfrage an einen gehosteten Anbieter über die API des Anbieters. Der Anbieter provisioniert und hostet dann den Cluster für Sie. Wenn der Cluster fertiggestellt ist, können Sie ihn über die Rancher-Benutzeroberfläche verwalten, zusammen mit Clustern, die Sie provisioniert haben und die lokal oder bei einem Infrastruktur-Anbieter gehostet werden.

Rancher unterstützt die folgenden Kubernetes-Anbieter:

* https://cloud.google.com/kubernetes-engine/[Google GKE (Google Kubernetes Engine)]
* https://aws.amazon.com/eks/[Amazon EKS (Amazon Elastic Container Service for Kubernetes)]
* https://azure.microsoft.com/en-us/services/kubernetes-service/[Microsoft AKS (Azure Kubernetes Service)]

endif::[]
