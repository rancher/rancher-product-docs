[#_basic_options]
== Grundlegende Optionen

[#_region]
=== Region

Die geografische https://docs.aws.amazon.com/AWSEC2/latest/UserGuide/using-regions-availability-zones.html[Region], in der Sie Ihren Cluster erstellen möchten.

[#_zone]
=== Zone

Die https://docs.aws.amazon.com/AWSEC2/latest/UserGuide/using-regions-availability-zones.html#concepts-availability-zones[Zone], ein isolierter Standort innerhalb einer Region, um Ihren Cluster zu erstellen.

[#_instance_type]
=== Instanztyp

Der https://docs.aws.amazon.com/AWSEC2/latest/UserGuide/instance-types.html[Instanztyp], der die Hardwaremerkmale bestimmt, die zur Bereitstellung Ihres Clusters verwendet werden.

[#_root_disk_size]
=== Root-Disk-Größe

Konfigurieren Sie die Größe (in GB) für Ihr https://docs.aws.amazon.com/AWSEC2/latest/UserGuide/RootDeviceStorage.html[Root-Gerät].

[#_vpc_subnet]
=== VPC/Subnet

Die https://docs.aws.amazon.com/vpc/latest/userguide/configure-your-vpc.html[VPC] oder das spezifische https://docs.aws.amazon.com/vpc/latest/userguide/configure-subnets.html[Subnetz], ein IP-Bereich in Ihrer VPC, um Ihre Ressourcen hinzuzufügen.

[#_iam_instance_profile_name]
=== IAM Instanzprofilname

Der Name des https://docs.aws.amazon.com/IAM/latest/UserGuide/id_roles_use_switch-role-ec2_instance-profiles.html[Instanzprofils], das verwendet wird, um eine IAM-Rolle an eine EC2-Instanz zu übergeben.

[#_advanced_options]
== Erweiterte Optionen

[#_ami_id]
=== AMI ID

Das https://docs.aws.amazon.com/AWSEC2/latest/UserGuide/AMIs.html[Amazon Machine Image], das für die Knoten in Ihrem Cluster verwendet wird.

[#_ssh_username_for_ami]
=== SSH-Benutzername für AMI

Der Benutzername für die Verbindung zu Ihren gestarteten Instanzen. Siehe https://docs.aws.amazon.com/AWSEC2/latest/UserGuide/connection-prereqs.html[hier] für die Standardbenutzernamen zu den ausgewählten AMIs. Für nicht aufgelistete AMIs, wenden Sie sich an den AMI-Anbieter.

[#_security_group]
=== Sicherheitsgruppe

Wählen Sie die Standard-Sicherheitsgruppe oder konfigurieren Sie eine Sicherheitsgruppe.

Bitte beziehen Sie sich auf {xref-port-reqs}#_rancher_aws_ec2_security_group[Amazon EC2-Sicherheitsgruppe bei Verwendung des Node Drivers], um zu sehen, welche Regeln in der `rancher-nodes` Sicherheitsgruppe erstellt werden.

[#_ebs_root_volume_type]
=== EBS Root-Volume-Typ

Der https://docs.aws.amazon.com/AWSEC2/latest/UserGuide/ebs-volume-types.html[EBS-Volumentyp], der für das Root-Gerät verwendet werden soll.

[#_encrypt_ebs_volume]
=== EBS-Volume verschlüsseln

Aktivieren Sie https://docs.aws.amazon.com/AWSEC2/latest/UserGuide/EBSEncryption.html[die Amazon EBS-Verschlüsselung].

[#_request_spot_instance]
=== Spot-Instanz anfordern

Aktivieren Sie die Option, https://docs.aws.amazon.com/AWSEC2/latest/UserGuide/spot-requests.html[Spot-Instanzen anzufordern] und geben Sie den maximalen Preis pro Stunde an, den Sie bereit sind zu zahlen.

[#_use_only_private_address]
=== Nur private Adresse verwenden

Aktivieren Sie die Option, nur https://docs.aws.amazon.com/AWSEC2/latest/UserGuide/using-instance-addressing.html[private Adressen] zu verwenden.

[#_ebs_optimized_instance]
=== EBS-optimierte Instanz

Verwenden Sie eine https://docs.aws.amazon.com/AWSEC2/latest/UserGuide/ebs-optimized.html[EBS-optimierte Instanz].

[#_allow_access_to_ec2_metadata]
=== Zugriff auf EC2-Metadaten erlauben

Zugriff auf https://docs.aws.amazon.com/AWSEC2/latest/UserGuide/ec2-instance-metadata.html[EC2-Metadaten] aktivieren.

[#_use_tokens_for_metadata]
=== Tokens für Metadaten verwenden

Verwenden Sie https://docs.aws.amazon.com/AWSEC2/latest/UserGuide/configuring-instance-metadata-service.html[Instance Metadata Service Version 2 (IMDSv2)], eine tokenbasierte Methode zum Zugriff auf Metadaten.

[#_add_tag]
=== Tag hinzufügen

Fügen Sie Metadaten mit https://docs.aws.amazon.com/AWSEC2/latest/UserGuide/Using_Tags.html[Tags] hinzu, um Ressourcen zu kategorisieren.

[#_ipv6_address_count]
=== IPv6-Adressenanzahl

Geben Sie an, wie viele IPv6-Adressen der Netzwerkschnittstelle der Instanz zugewiesen werden sollen.

[#_ipv6_address_only]
=== Nur IPv6-Adresse

Aktivieren Sie diese Option, wenn die Instanz ausschließlich IPv6 verwenden soll. IPv6-only VPCs oder Subnetze erfordern dies. Wenn aktiviert, hat die Instanz IPv6 als einzige Adresse, und die Anzahl der IPv6-Adressen muss größer als null sein.

[#_http_protocol_ipv6]
=== HTTP-Protokoll IPv6

Aktivieren oder Deaktivieren von IPv6-Endpunkten für den Instanz-Metadatenservice.

[#_enable_primary_ipv6]
=== Primäres IPv6 aktivieren

Aktivieren Sie diese Option, um die erste zugewiesene IPv6-Adresse als primäre Adresse festzulegen. Dies gewährleistet eine konsistente, unveränderliche IPv6-Adresse für die Instanz. Es steuert nicht, ob IPv6-Adressen zugewiesen werden.
