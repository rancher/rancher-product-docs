Für weitere Informationen zu Azure, siehe die offizielle https://docs.microsoft.com/en-us/azure/?product=featured[Azure-Dokumentation.]

Die Kontozugangsinformationen werden als Cloud-Anmeldeinformationen gespeichert. Cloud-Anmeldeinformationen werden als Kubernetes-Geheimnisse gespeichert. Mehrere Node-Vorlagen können dieselben Cloud-Anmeldeinformationen verwenden. Sie können vorhandene Cloud-Anmeldeinformationen verwenden oder neue erstellen.

* *Platzierung* legt die geografische Region fest, in der Ihr Cluster gehostet wird, sowie andere Standortmetadaten.
* *Netzwerk* konfiguriert das Netzwerk, das in Ihrem Cluster verwendet wird.
* *Instanz* passt Ihre VM-Konfiguration an.
+

[NOTE]
====
Wenn Sie ein VNet in einer anderen Ressourcengruppe als die VMs verwenden, sollte der VNet-Name mit dem Namen der Ressourcengruppe vorangestellt werden. Beispiel: `<resource group>:<vnet>`. 
====


Wenn Sie Docker verwenden, umfassen die https://docs.docker.com/engine/docker-overview/#the-docker-daemon[Docker-Daemon] Konfigurationsoptionen:

* *Labels:* Für Informationen zu Labels siehe die https://docs.docker.com/config/labels-custom-metadata/[Dokumentation zu Docker-Objekt-Labels.].
* *Docker Engine Installations-URL:* Bestimmt, welche Docker-Version auf der Instanz installiert wird.
+

[NOTE]
====
Wenn Sie Red Hat Enterprise Linux (RHEL) oder CentOS-Knoten bereitstellen, lassen Sie das Feld *Docker Install URL* auf dem Standardwert oder wählen Sie *keine* aus. Dies umgeht eine Überprüfung der Docker-Installation, da Docker bereits auf diesen Knotentypen installiert ist.

Wenn Sie *Docker Install URL* auf einen anderen Wert als den Standard oder *keine* setzen, könnte eine Fehlermeldung wie die folgende angezeigt werden: `Error creating machine: RHEL ssh command error: command: sudo -E yum install -y curl err: exit status 1 output: Updating Subscription Management repositories.`
====


* *Registry-Spiegel:* Docker-Registry-Mirror, der vom Docker-Daemon verwendet werden soll.
* *Weitere erweiterte Optionen:* Siehe die https://docs.docker.com/engine/reference/commandline/dockerd/[Referenz zu Docker-Daemon-Optionen].
