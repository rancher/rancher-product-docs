Wenn Sie Azure-VMs für Ihre Knoten verwenden, können Sie https://docs.microsoft.com/en-us/azure/aks/azure-files-dynamic-pv[Azure-Dateien] als StorageClass für den Cluster verwenden.

Um die erforderlichen Speicherressourcen auf der Azure-Plattform zu erstellen, befolgen Sie diese Schritte:

. {xref-azure-cloud-provider}[Konfigurieren Sie den Azure-Cloud-Anbieter.]
. Konfigurieren Sie `kubectl`, um eine Verbindung zu Ihrem Cluster herzustellen.
. Kopieren Sie das `ClusterRole` und das `ClusterRoleBinding` Manifest für das Dienstkonto:
  ```yml
  --
  apiVersion: rbac.authorization.k8s.io/v1
  kind: ClusterRole
  metadata:
    name: system:azure-cloud-provider
  rules:
 ** apiGroups: ['']
resources: ['secrets']
verbs:     ['get','create']
  --
  apiVersion: rbac.authorization.k8s.io/v1
  kind: ClusterRoleBinding
  metadata:
name: system:azure-cloud-provider
  roleRef:
kind: ClusterRole
apiGroup: rbac.authorization.k8s.io
name: system:azure-cloud-provider
  subjects:
 ** kind: ServiceAccount
name: persistent-volume-binder
namespace: kube-system
  ```
. Erstellen Sie diese in Ihrem Cluster mit einem der folgenden Befehle.
+
----
# kubectl create -f <MANIFEST>
----
