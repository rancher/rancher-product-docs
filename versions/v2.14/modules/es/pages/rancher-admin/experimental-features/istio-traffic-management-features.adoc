Esta característica habilita una interfaz de usuario que te permite crear, leer, actualizar y eliminar servicios virtuales y reglas de destino, que son características de gestión de tráfico de Istio.

____
*Requisito previo:* Activar esta característica no habilita Istio. Un administrador de clúster necesita {xref-istio-guides}[habilitar Istio para el clúster] para poder usar la característica.
____

Para habilitar o deshabilitar esta característica, consulta las instrucciones en {xref-experimental-features}[la página principal sobre habilitar características experimentales.]

[#_about_this_feature]
== Acerca de esta característica

Una ventaja central de las características de gestión de tráfico de Istio es que permiten el enrutamiento dinámico de solicitudes, lo cual es útil para despliegues canarios, despliegues azul/verde o pruebas A/B.

Cuando está habilitada, esta característica activa una página que te permite configurar algunas características de gestión de tráfico de Istio utilizando la interfaz de usuario de Rancher. Sin esta característica, necesitas usar `kubectl` para gestionar el tráfico con Istio.

La característica habilita dos pestañas en la interfaz de usuario: una pestaña para *Servicios Virtuales* y otra para *Reglas de Destino*.

* *Los servicios virtuales* interceptan y dirigen el tráfico a tus servicios de Kubernetes, permitiéndote dirigir porcentajes de tráfico de una solicitud a diferentes servicios. Puedes utilizarlos para definir un conjunto de reglas de enrutamiento que se aplican cuando se dirige a un host. Para más detalles, consulta la https://istio.io/docs/reference/config/networking/v1alpha3/virtual-service/[documentación de Istio.]
* *Las reglas de destino* sirven como la única fuente de verdad sobre qué versiones de servicio están disponibles para recibir tráfico de los servicios virtuales. Puedes usar estos recursos para definir políticas que se aplican al tráfico destinado a un servicio después de que se ha realizado el enrutamiento. Para más detalles, consulta la https://istio.io/docs/reference/config/networking/v1alpha3/destination-rule[documentación de Istio.]

Para ver estas pestañas,

. Haz clic en *☰ > Gestión de Clústeres*.
. Ve al clúster donde está instalado Istio y haz clic en *Explorar*.
. En la barra de navegación izquierda, haz clic en *Istio*.
. Verás pestañas para *Kiali* y *Jaeger*. Desde la barra de navegación izquierda, puedes ver y configurar *Servicios Virtuales* y *Reglas de Destino*.
