Esta característica permite utilizar tipos para proveedores de almacenamiento y aprovisionadores que no están habilitados por defecto.

Para habilitar o deshabilitar esta característica, consulta las instrucciones en {xref-experimental-features}[la página principal sobre habilitar características experimentales.]

|===
| Clave de variable de entorno | Valor por defecto | Descripción

| `unsupported-storage-drivers`
| `false`
| Esta característica habilita tipos para proveedores de almacenamiento y aprovisionadores que no están habilitados por defecto.
|===

[#_types_for_persistent_volume_plugins_that_are_enabled_by_default]
== Tipos de complementos de volumen persistente que están habilitados por defecto

A continuación se muestra una lista de tipos de almacenamiento para complementos de volumen persistente que están habilitados por defecto. Al habilitar este flag de característica, cualquier complemento de volumen persistente que no esté en esta lista se considera experimental y no compatible:

|===
| Nombre | Plugin

| Amazon EBS Disk
| `aws-ebs`

| AzureFile
| `azure-file`

| AzureDisk
| `azure-disk`

| Google Persistent Disk
| `gce-pd`

| VMware vSphere Volume
| `vsphere-volume`

| Local
| `local`

| NFS (Network File System)
| `nfs`

| hostPath
| `host-path`
|===
