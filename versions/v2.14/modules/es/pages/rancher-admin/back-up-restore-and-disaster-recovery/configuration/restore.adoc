La página Crear Restauración te permite proporcionar detalles de la copia de seguridad desde la que restaurar

image::backup_restore/restore/restore.png[]

[#_backup_source]
== Fuente de copia de seguridad

Proporciona detalles del archivo de copia de seguridad y su ubicación de almacenamiento, que el operador utilizará para realizar la restauración. Selecciona entre las siguientes opciones para proporcionar estos detalles

[#_an_existing_backup_config]
=== Una configuración de copia de seguridad existente

Seleccionar esta opción llenará el desplegable *Copia de seguridad de destino* con las copias de seguridad disponibles en este clúster. Selecciona la copia de seguridad del desplegable, y eso completará el campo *Nombre de archivo de copia de seguridad* para ti, y también pasará la información de la fuente de copia de seguridad de la copia de seguridad seleccionada al operador.

image::backup_restore/restore/existing.png[]

Si el recurso personalizado de copia de seguridad no existe en el clúster, necesitas obtener el nombre de archivo exacto y proporcionar los detalles de la fuente de copia de seguridad con el objetivo de almacenamiento predeterminado o un almacén de objetos compatible con S3.

[#_the_default_storage_target]
=== El objetivo de almacenamiento predeterminado

Selecciona esta opción si estás restaurando desde un archivo de copia de seguridad que existe en la ubicación de almacenamiento predeterminada configurada a nivel de operador. La configuración a nivel de operador es la ubicación de almacenamiento que se configuró cuando se instaló o se actualizó la versión del `rancher-backup` operador. Proporciona el nombre de archivo exacto en el campo *Nombre de archivo de copia de seguridad*.

image::backup_restore/restore/default.png[]

[#_an_s3_compatible_object_store]
=== Un almacén de objetos compatible con S3

Selecciona esta opción si no se ha configurado ninguna ubicación de almacenamiento predeterminada a nivel de operador, O si el archivo de copia de seguridad existe en un cubo S3 diferente al que se configuró como la ubicación de almacenamiento predeterminada. Proporciona el nombre de archivo exacto en el campo *Nombre de archivo de copia de seguridad*. Consulta <<_getting_the_backup_filename_from_s3,esta sección>> para los pasos exactos sobre cómo obtener el nombre de archivo de copia de seguridad de S3. Rellena todos los detalles para el almacén de objetos compatible con S3. Sus campos son exactamente los mismos que los de la configuración `backup.StorageLocation` en {xref-rancher-backup-config}#_storage_location[Recurso personalizado de copia de seguridad.]

image::backup_restore/restore/s3store.png[]

[#_encryption]
== Cifrado

Si la copia de seguridad se creó con la encriptación habilitada, su archivo tendrá el sufijo `.enc`. Al elegir tal copia de seguridad, o al proporcionar un nombre de archivo de copia de seguridad con el sufijo `.enc`, se mostrará un menú desplegable adicional llamado *Secreto de configuración de encriptación*.

image::backup_restore/restore/encryption.png[]

El secreto seleccionado de este menú desplegable debe tener el mismo contenido que el utilizado para el recurso personalizado de copia de seguridad al realizar la copia de seguridad. Si la configuración de encriptación no coincide, la restauración fallará.

El menú desplegable `Encryption Config Secret` filtrará y listará solo aquellos Secretos que tengan esta clave exacta.

|===
| Nombre de Directiva YAML | Descripción

| `encryptionConfigSecretName`
| Proporciona el nombre del Secreto del espacio de nombres `cattle-resources-system`, que contiene el archivo de configuración de encriptación.
|===

[NOTE]
.Importante:
====

Este campo solo debe establecerse si la copia de seguridad se creó con la encriptación habilitada. Proporcionar la configuración de encriptación incorrecta hará que la restauración falle.
====


[#_prune_during_restore]
== Podar durante la restauración

* *Podar*:  Para restaurar completamente Rancher desde una copia de seguridad, y volver al estado exacto en el que se encontraba cuando se realizó la copia de seguridad, necesitamos eliminar cualquier recurso adicional que haya sido creado por Rancher después de que se tomó la copia de seguridad. El operador lo hace si la bandera *Podar* está habilitada. Podar está habilitado por defecto y se recomienda mantenerlo habilitado.
* *Tiempo de espera para eliminar*: Este es el tiempo que el operador esperará mientras elimina un recurso antes de editar el recurso para eliminar los finalizadores y volver a intentar la eliminación.

|===
| Nombre de Directiva YAML | Descripción

| `prune`
| Eliminar los recursos gestionados por Rancher que no están presentes en la copia de seguridad (Recomendado).

| `deleteTimeoutSeconds`
| Cantidad de tiempo que el operador esperará mientras elimina un recurso antes de editar el recurso para eliminar los finalizadores y volver a intentar la eliminación.
|===

[#_getting_the_backup_filename_from_s3]
== Obteniendo el nombre de archivo de copia de seguridad de S3

Este es el nombre del archivo de copia de seguridad que el operador `rancher-backup` utilizará para realizar la restauración.

Para obtener este nombre de archivo de S3, ve a tu bucket de S3 (y a la carpeta si se especificó al realizar la copia de seguridad).

Copia el nombre de archivo y guárdalo en tu recurso personalizado de restauración. Así que suponiendo que el nombre de tu archivo de copia de seguridad es `backupfile`,

* Si el nombre de tu bucket es `s3bucket` y no se especificó ninguna carpeta, entonces el `backupFilename` a utilizar será `backupfile`.
* Si el nombre de tu bucket es `s3bucket` y la carpeta base es ``s3folder``, el `backupFilename` a utilizar es solo `backupfile`.
* Si hay una subcarpeta dentro de `s3Folder` llamada `s3sub`, y esa contiene tu archivo de copia de seguridad, entonces el `backupFilename` a utilizar es `s3sub/backupfile`.

|===
| Nombre de Directiva YAML | Descripción

| `backupFilename`
| Este es el nombre del archivo de copia de seguridad que el operador `rancher-backup` utilizará para realizar la restauración.
|===
