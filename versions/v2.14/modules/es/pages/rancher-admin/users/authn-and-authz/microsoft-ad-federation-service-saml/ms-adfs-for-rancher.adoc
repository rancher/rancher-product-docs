Antes de configurar Rancher para soportar el Servicio de Federación de Active Directory (AD FS), debes añadir Rancher como una https://docs.microsoft.com/en-us/windows-server/identity/ad-fs/technical-reference/understanding-key-ad-fs-concepts[confianza de parte dependiente] en AD FS.

. Inicia sesión en tu servidor AD como un usuario administrativo.
. Abre la consola de *Gestión de AD FS*. Selecciona *Añadir Confianza de Parte Dependiente..* en el menú de *Acciones* y haz clic en *Iniciar*.
+
image::adfs/adfs-overview.png[]

. Selecciona *Introducir datos sobre la parte dependiente manualmente* como la opción para obtener datos sobre la parte dependiente.
+
image::adfs/adfs-add-rpt-2.png[]

. Introduce el *Nombre para mostrar* deseado para tu Confianza de Parte Dependiente. Por ejemplo, `Rancher`.
+
image::adfs/adfs-add-rpt-3.png[]

. Selecciona *perfil de AD FS* como el perfil de configuración para tu confianza de parte dependiente.
+
image::adfs/adfs-add-rpt-4.png[]

. Deja el *certificado de cifrado de token opcional* vacío, ya que Rancher AD FS no utilizará uno.
+
image::adfs/adfs-add-rpt-5.png[]

. Selecciona *Habilitar soporte para el protocolo SAML 2.0 WebSSO* y introduce `\https://<rancher-server>/v1-saml/adfs/saml/acs` para la URL del servicio.
+
image::adfs/adfs-add-rpt-6.png[]

. Añade `\https://<rancher-server>/v1-saml/adfs/saml/metadata` como el *identificador de confianza de parte dependiente*.
+
image::adfs/adfs-add-rpt-7.png[]

. Este tutorial no cubrirá la autenticación multifactor; por favor consulta la https://docs.microsoft.com/en-us/windows-server/identity/ad-fs/operations/configure-additional-authentication-methods-for-ad-fs[documentación de Microsoft] si deseas configurar la autenticación multifactor.
+
image::adfs/adfs-add-rpt-8.png[]

. Desde *Elegir Reglas de Autorización de Emisión*, puedes seleccionar cualquiera de las opciones disponibles según el caso de uso. Sin embargo, para los propósitos de esta guía, selecciona *Permitir a todos los usuarios acceder a esta confianza de parte dependiente*.
+
image::adfs/adfs-add-rpt-9.png[]

. Después de revisar tu configuración, selecciona *Siguiente* para añadir la confianza de parte dependiente.
+
image::adfs/adfs-add-rpt-10.png[]

. Selecciona *Abrir las Reglas de Reclamación de Edición..* y haz clic en *Cancelar*.
+
image::adfs/adfs-add-rpt-11.png[]

. En la pestaña de *Reglas de Transformación de Emisión*, haz clic en *Añadir Regla..*.
+
image::adfs/adfs-edit-cr.png[]

. Selecciona *Enviar atributos LDAP como reclamaciones* como la *plantilla de regla de reclamación*.
+
image::adfs/adfs-add-tcr-1.png[]

. Establece el *nombre de la regla de reclamación* a tu nombre deseado (por ejemplo, `Rancher Attributes`) y selecciona *Active Directory* como el *almacén de atributos*. Crea el siguiente mapeo para reflejar la tabla a continuación:
+
|===
| Atributo LDAP
| Tipo de reclamación saliente

| Nombre de pila
| Nombre

| Nombre principal de usuario
| UPN

| Grupos de Token - Calificado por nombre de dominio largo
| Grupo

| Nombre de cuenta SAM
| Nombre
|===
+
image::adfs/adfs-add-tcr-2.png[]

. Descarga el `federationmetadata.xml` de tu servidor AD en:
+
----
https://<AD_SERVER>/federationmetadata/2007-06/federationmetadata.xml
----

*Resultado:* Has añadido Rancher como una confianza de parte dependiente. Ahora puedes configurar Rancher para aprovechar AD.
