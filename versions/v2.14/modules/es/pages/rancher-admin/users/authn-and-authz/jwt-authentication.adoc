Muchas integraciones de terceros disponibles para Kubernetes, como GitLab y HashiCorp Vault, implican dar acceso a un proceso externo a la API de Kubernetes utilizando un token de cuenta de servicio nativo de Kubernetes para la autenticación.

En Rancher v2.9.0 y versiones posteriores, las cuentas de servicio en clústeres en sentido descendente ahora pueden autenticarse a través de un JSON Web Token (JWT) utilizando el proxy de autenticación de Rancher. En versiones de Rancher anteriores a v2.9.0, solo se admitían tokens emitidos por Rancher.

Para habilitar esta función, sigue estos pasos:

. En la esquina superior izquierda, haz clic en *☰ > Gestión del Clúster*.
. Haz clic en *Avanzado* para abrir el menú desplegable.
. Selecciona *Autenticación JWT*.
. Marca la casilla para el clúster para el que deseas habilitar la autenticación JWT y haz clic en *Habilitar*. Alternativamente, puedes hacer clic en *⋮* > *Habilitar*.
