El Rancher CLI (interfaz de línea de comandos) es una herramienta unificada que puedes utilizar para interactuar con Rancher. Con esta herramienta, puedes operar Rancher utilizando la interfaz de línea de comandos en lugar de la GUI.

[#_download_rancher_cli]
== Descargar Rancher CLI

El binario se puede descargar directamente desde la interfaz de usuario.

. En la esquina superior izquierda, haz clic en *☰*.
. En la parte inferior del menú de la barra lateral de navegación, haz clic en *Acerca de*.
. Bajo la *sección de Descargas de CLI*, hay enlaces para descargar los binarios para Windows, Mac y Linux. También puedes consultar la https://github.com/rancher/cli/releases[página de lanzamientos para nuestro CLI] para descargas directas del binario.

[#_requirements]
== Requisitos

Después de descargar el Rancher CLI, necesitas realizar algunas configuraciones. Rancher CLI requiere:

* La URL de tu servidor Rancher, que se utiliza para conectarse al servidor Rancher.
* Un token de portador de API, que se utiliza para autenticarte con Rancher. Para más información sobre cómo obtener un token de portador, consulta {xref-user-api-keys}[Creando una clave de API].

[#_cli_authentication]
== Autenticación de CLI

Antes de que puedas utilizar Rancher CLI para controlar tu servidor Rancher, debes autenticarte utilizando un token de portador de API. Inicia sesión utilizando el siguiente comando (reemplaza `<BEARER_TOKEN>` y `<SERVER_URL>` con tu información):

[,bash]
----
$ ./rancher login https://<SERVER_URL> --token <BEARER_TOKEN>
----

Si el servidor Rancher utiliza un certificado autofirmado, Rancher CLI te pedirá que continúes con la conexión.

[#_project_selection]
== Selección de Proyecto

Antes de que puedas realizar cualquier comando, debes seleccionar un proyecto de Rancher para ejecutar esos comandos. Para seleccionar un {xref-projects-and-namespaces}[proyecto] en el que trabajar, utiliza el comando `./rancher context switch`. Cuando ingreses este comando, se mostrará una lista de proyectos disponibles. Introduce un número para elegir tu proyecto.

*Ejemplo: `./rancher context switch` Salida*

----
User:rancher-cli-directory user$ ./rancher context switch
NUMBER    CLUSTER NAME   PROJECT ID              PROJECT NAME
1         cluster-2      c-7q96s:p-h4tmb         project-2
2         cluster-2      c-7q96s:project-j6z6d   Default
3         cluster-1      c-lchzv:p-xbpdt         project-1
4         cluster-1      c-lchzv:project-s2mch   Default
Select a Project:
----

Después de que ingreses un número, la consola mostrará un mensaje indicando que has cambiado de proyecto.

----
INFO[0005] Setting new context to project project-1
INFO[0005] Saving config to /Users/markbishop/.ranchcli2.json
----

Asegúrate de que puedes ejecutar `rancher kubectl get pods` con éxito.

[#_commands]
== Comandos

Los siguientes comandos están disponibles para su uso en Rancher CLI.

|===
| Comando | Resultado

| `clusters, [cluster]`
| Realiza operaciones en tus {xref-cluster-deployment}[clusters].

| `context`
| Cambia entre Rancher {xref-projects-and-namespaces}[proyectos]. Para un ejemplo, consulta <<_project_selection,Selección de Proyecto>>.

| `inspect [OPTIONS] [RESOURCEID RESOURCENAME]`
| Muestra detalles sobre https://kubernetes.io/docs/reference/kubectl/cheatsheet/#resource-types[recursos de Kubernetes] o recursos de Rancher (es decir: {xref-projects-and-namespaces}[proyectos] y {xref-workloads-and-pods}[cargas de trabajo]). Especifica recursos por nombre o ID.

| `kubectl`
| Ejecuta https://kubernetes.io/docs/reference/kubectl/overview/#operations[comandos de kubectl].

| `login, [l]`
| Inicia sesión en un Servidor Rancher. Para un ejemplo, consulta <<_cli_authentication,Autenticación de CLI>>.

| `machines, [machine]`
| Realiza operaciones en máquinas.

| `namespaces, [namespace]`
| Realiza operaciones en {xref-namespaces}[espacios de nombres].

| `nodes, [node]`
| Realiza operaciones en {xref-nodes-machinepools}[nodos].

| `projects, [project]`
| Realiza operaciones en {xref-projects-and-namespaces}[proyectos].

| `ps`
| Muestra {xref-workloads-and-pods}[cargas de trabajo] en un proyecto.

| `server`
| Realiza operaciones para el servidor.

| `settings, [setting]`
| Muestra la configuración actual de tu servidor Rancher.

| `ssh`
| Se conecta a uno de los nodos de tu clúster utilizando el protocolo SSH.

| `up`
| Aplica la configuración de composición.

| `wait`
| Espera a los recursos del clúster, app, proyecto, multiClusterApp.

| `token`
| Se autentica y genera un nuevo token kubeconfig.

| `help, [h]`
| Muestra una lista de comandos o ayuda para un comando.
|===

[#_rancher_cli_help]
== Ayuda de Rancher CLI

Una vez que hayas iniciado sesión en el servidor Rancher utilizando la CLI, introduce `./rancher --help` para obtener una lista de comandos.

Todos los comandos aceptan la `--help` bandera, que documenta el uso de cada comando.

[#_limitations]
== limitaciones

La CLI de Rancher *no puede* ser utilizada para instalar {xref-helm-charts-in-rancher}[aplicaciones de panel o gráficos de características de Rancher].
