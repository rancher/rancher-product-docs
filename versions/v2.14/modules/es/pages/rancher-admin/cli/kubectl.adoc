[#_kubectl]
== kubectl

Interactúa con Rancher utilizando kubectl.

[#_kubectl_utility]
=== Utilidad kubectl

Instala la utilidad `kubectl`. Consulta https://kubernetes.io/docs/tasks/tools/install-kubectl/[instalar kubectl].

Configura kubectl visitando tu clúster en la interfaz web de Rancher, haciendo clic en `Kubeconfig`, copiando el contenido y colocándolo en tu archivo `~/.kube/config`.

Ejecuta `kubectl cluster-info` o `kubectl get pods` con éxito.

[#_authentication_with_kubectl_and_kubeconfig_tokens_with_ttl]
=== Autenticación con kubectl y tokens kubeconfig con TTL

_Requisitos_

Si los administradores tienen xref:api/api-tokens.adoc#_disable_tokens_in_generated_kubeconfigs[desactivada la generación de tokens kubeconfig], el archivo kubeconfig requiere que el {xref-rancher-cli}[CLI de Rancher] esté presente en tu PATH cuando ejecutes `kubectl`. De lo contrario, verás un error como:
`Unable to connect to the server: getting credentials: exec: exec: "rancher": executable file not found in $PATH`.

Esta función permite que kubectl se autentique con el servidor de Rancher y obtenga un nuevo token kubeconfig cuando sea necesario. Los siguientes proveedores de autenticación son actualmente compatibles:

. Local
. Active Directory (solo LDAP)
. FreeIPA
. OpenLDAP
. Proveedores SAML: Ping, Okta, ADFS, Keycloak, Shibboleth
. Azure AD

Cuando ejecutes kubectl por primera vez, por ejemplo, `kubectl get pods`, se te pedirá que elijas un proveedor de autenticación e inicies sesión con el servidor de Rancher. El token kubeconfig se almacena en caché en el directorio desde el que ejecutas kubectl bajo `./.cache/token`. Este token es válido hasta que xref:api/api-tokens.adoc#_disable_tokens_in_generated_kubeconfigs[expire], o xref:api/api-tokens.adoc#_deleting_tokens[se elimine del servidor de Rancher]. Al expirar, debes iniciar sesión con el servidor de Rancher nuevamente para ejecutar el comando `kubectl get pods`.
