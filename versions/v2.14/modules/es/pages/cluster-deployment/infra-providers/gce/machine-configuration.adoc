Para más información sobre Google Cloud Platform (GCP) y Google Compute Engine (GCE), consulta la https://cloud.google.com/docs[documentación oficial de GCP].

[#_basic_options]
== Opciones básicas

[#_zone]
=== Zona

La región y zona de GCP a la que se desplegará la VM. Por ejemplo, `us-east1-b`.

[#_machine_image_project]
=== Proyecto de imagen de máquina

El proyecto de imagen al que pertenecen las familias de imágenes deseadas.

[#_machine_image_family]
=== Familia de imagen de máquina

La familia de imágenes a la que pertenece el sistema operativo de la máquina deseada.

[#_machine_image]
=== Imagen de máquina

El sistema operativo que se instalará en la VM.

[#_disk_type]
=== Tipo de disco

El tipo de disco adjunto a la VM. Los tipos disponibles pueden diferir entre regiones.

[#_disk_size]
=== Tamaño del disco

El tamaño del disco adjunto a la VM, en Gigabytes.

[#_machine_type]
=== Tipo de máquina

El tipo de VM que se desplegará. Los tipos de máquina determinan el número de recursos (vCPU, RAM, etc.) asignados a cada nodo.

[#_network]
=== Red

La red VPC en la que se creará la VM. Este valor no se puede cambiar una vez que se ha aprovisionado el grupo de máquinas.

[#_subnet]
=== Subred

La subred VPC en la que se creará la VM. Este valor no se puede cambiar una vez que se ha aprovisionado el grupo de máquinas.

[#_username]
=== Nombre de usuario

Un nombre de usuario personalizado establecido como el usuario predeterminado de la VM GCE. 

[#_external_address]
=== Dirección externa

La dirección IP externa deseada para la VM GCE.

[#_scopes]
=== Ámbitos

Una lista de ámbitos de OAuth2 que permiten a la VM acceder a otras API de GCP.

[#_allow_internal_communication]
=== Permitir comunicación interna

Por defecto, se crea automáticamente una regla de firewall VPC para exponer un conjunto fijo de puertos dentro de la VPC y facilitar la comunicación entre los nodos del clúster. Este comportamiento se puede desactivar por grupo de máquinas, al hacer clic en la opción `Show Advanced` y desmarcando la casilla `Allow Internal Communication`. 

[#_expose_external_ports]
=== Exponer puertos externos

Una lista de puertos que se abrirán _externamente_ a la internet más amplia. Los puertos abiertos se definen a nivel de grupo de máquinas. Habilitar esta opción resultará en la creación automática de una regla de firewall VPC. Esta regla se eliminará automáticamente cuando se elimine el clúster o el grupo de máquinas.

[#_network_tags]
=== Etiquetas de red

Las etiquetas son una lista de _etiquetas de red_, que se pueden usar para asociar reglas de firewall preexistentes con todas las VMs dentro de un grupo de máquinas.

[#_labels]
=== Etiquetas

Una lista de etiquetas personalizadas separadas por comas que se adjuntarán a todas las VMs dentro de un grupo de máquinas determinado. A diferencia de las etiquetas, las etiquetas personalizadas no influyen en el comportamiento de la red y solo sirven para organizar los recursos en la nube.

[#_advanced_options]
== Opciones avanzadas

Al crear clústeres a través de la interfaz de usuario de Rancher, algunas opciones se configuran automáticamente para ti. Sin embargo, al crear objetos de configuración de máquinas manualmente, debes asegurarte de configurar correctamente los campos a continuación.

[#_external_firewall_rule_prefix]
=== external-firewall-rule-prefix

Un prefijo que se utilizará al crear la regla de firewall para exponer puertos públicamente. Idealmente, esto debería ser una concatenación del nombre del grupo de máquinas y el nombre del clúster. Este campo debe establecerse si el grupo de máquinas está configurado para exponer puertos públicamente, de lo contrario, se puede omitir.

[#_internal_firewall_rule_prefix]
=== internal-firewall-rule-prefix

Un prefijo que se utilizará al crear la regla de firewall interna que permite la comunicación entre nodos dentro del clúster. Si se omite este campo, no se creará ninguna regla de firewall interna.
