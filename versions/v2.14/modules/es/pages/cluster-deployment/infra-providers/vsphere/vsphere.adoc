Al utilizar Rancher con VMware vSphere, puedes llevar las operaciones en la nube in situ.

Rancher puede aprovisionar nodos en vSphere e instalar Kubernetes en ellos. Al crear un clúster de Kubernetes en vSphere, Rancher primero aprovisiona el número especificado de máquinas virtuales comunicándose con la API de vCenter. Luego instala Kubernetes sobre ellas.

Un clúster de vSphere puede consistir en múltiples grupos de máquinas virtuales con propiedades distintas, como la cantidad de memoria o el número de vCPUs. Este agrupamiento permite un control detallado sobre el tamaño de los nodos para cada rol de Kubernetes.

En {xref-provision-cluster-in-vsphere}[esta sección,] aprenderás a utilizar Rancher para instalar un {url-rke2-landing-page}[RKE2] o {url-k3s-landing-page}[K3s] clúster de Kubernetes en vSphere.

[#_provisioning_storage]
== Aprovisionamiento de Almacenamiento

Para un ejemplo de cómo aprovisionar almacenamiento en vSphere utilizando Rancher, consulta {xref-vsphere-storage-example}[esta sección]. Para aprovisionar almacenamiento de forma dinámica en vSphere, el proveedor de vSphere debe estar habilitado. Consulta {xref-cloud-provider-vsphere}[la configuración integrada de vSphere] y {xref-cloud-provider-vsphere-out-of-tree}[la configuración externa de vSphere].

[#_enabling_the_vmware_vsphere_cloud_provider]
== Habilitando el proveedor de nube de VMware vSphere

Cuando se configura un proveedor de nube en Rancher, el servidor de Rancher puede aprovisionar automáticamente nueva infraestructura para el clúster, incluyendo nuevos nodos o dispositivos de almacenamiento persistente.

Para obtener detalles sobre cómo habilitar el proveedor de nube vSphere, consulte {xref-cloud-provider-vsphere}[configuración vSphere en árbol] y {xref-cloud-provider-vsphere-out-of-tree}[configuración vSphere fuera de árbol].
