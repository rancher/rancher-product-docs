Para más información sobre Azure, consulta la https://docs.microsoft.com/en-us/azure/?product=featured[documentación oficial de Azure.]

La información de acceso a la cuenta se almacena como una credencial en la nube. Las credenciales en la nube se almacenan como secretos de Kubernetes. Múltiples plantillas de nodo pueden utilizar la misma credencial en la nube. Puedes utilizar una credencial en la nube existente o crear una nueva.

* *Ubicación* establece la región geográfica donde se aloja tu clúster y otros metadatos de ubicación.
* *Red* configura la red utilizada en tu clúster.
* *Instancia* personaliza la configuración de tu VM.
+

[NOTE]
====
Si utilizas una VNet en un Grupo de Recursos diferente al de las VMs, el nombre de la VNet debe ir precedido por el nombre del Grupo de Recursos. Por ejemplo, `<resource group>:<vnet>`.
====


Si utilizas Docker, las opciones de configuración del https://docs.docker.com/engine/docker-overview/#the-docker-daemon[Docker daemon] incluyen:

* *Etiquetas:* Para información sobre etiquetas, consulta la https://docs.docker.com/config/labels-custom-metadata/[documentación de etiquetas de objetos de Docker.].
* *URL de instalación de Docker Engine:* Determina qué versión de Docker se instalará en la instancia.
+

[NOTE]
====
Si estás aprovisionando nodos de Red Hat Enterprise Linux (RHEL) o CentOS, deja el campo *URL de instalación de Docker* con el valor predeterminado, o selecciona *ninguno*. Esto eludirá una comprobación de la instalación de Docker, ya que Docker ya está instalado en estos tipos de nodos.

Si estableces *URL de instalación de Docker* a un valor diferente del predeterminado o *ninguno*, podrías ver un mensaje de error como el siguiente: `Error creating machine: RHEL ssh command error: command: sudo -E yum install -y curl err: exit status 1 output: Updating Subscription Management repositories.`
====


* *Espejos de registro:* Espejo de registro de Docker que será utilizado por el daemon de Docker.
* *Otras opciones avanzadas:* Consulta la https://docs.docker.com/engine/reference/commandline/dockerd/[referencia de opciones del Docker daemon].
