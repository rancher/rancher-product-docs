En esta sección, aprenderás a desplegar un {url-rke2-landing-page}[RKE2] o {url-k3s-landing-page}[K3s] clúster de Kubernetes en Azure a través de Rancher.

Primero, configuras tus credenciales de nube de Azure en Rancher.

Luego, creas un clúster de Azure en Rancher, y al configurar el nuevo clúster, defines grupos de máquinas para él. Cada grupo de máquinas tiene un rol de Kubernetes de etcd, controlplane o worker. Rancher instala Kubernetes en los nuevos nodos y configura cada nodo con el rol de Kubernetes definido por el grupo de máquinas.

[CAUTION]
====

Cuando el clúster RKE2/K3s de Rancher está en funcionamiento en Azure y tiene un balanceador de carga de Azure delante, el flujo saliente falla. La solución alternativa para este problema es la siguiente:

* Termina el SSL/TLS en el balanceador de carga interno
* Utiliza el balanceador de carga L7

Para más información, consulta la documentación sobre https://docs.microsoft.com/en-us/azure/load-balancer/components#_limitations[limitaciones del balanceador de carga de Azure].
====


Para más información sobre la configuración del clúster de Kubernetes que Rancher instala en los nodos de Azure, consulta las referencias de configuración del clúster {xref-rke2-config}[RKE2] y {xref-k3s-config}[K3s].


Para más información sobre la configuración de máquinas de Azure, consulta la {xref-azure-machine-configuration}[referencia de configuración de máquinas de Azure].

[#_preparation_in_azure]
== Preparación en Azure

Antes de que se pueda desplegar un clúster, debemos configurar nuestra suscripción de Azure para permitir la manipulación de sus recursos por un tercero, como Rancher.

Para ello, primero crearemos un nuevo *principal de servicio (SP)* en Azure *Active Directory (AD)*, que, en Azure, es un usuario de aplicación que tiene permiso para gestionar recursos de Azure.

El siguiente es un guion de plantilla `az cli` que debes ejecutar para crear un principal de servicio, donde debes introducir el nombre de tu SP, rol y ámbito:

----
az ad sp create-for-rbac \
  --name="<Rancher ServicePrincipal name>" \
  --role="Contributor" \
  --scopes="/subscriptions/<subscription Id>"
----

La creación de este principal de servicio devuelve el **ID de aplicación**, también llamado **ID de cliente**, y el **secreto de cliente**. Esta información se utiliza cuando creas tus credenciales de nube.

[#_creating_an_azure_cluster]
== Creando un clúster de Azure

[#_create_your_cloud_credentials]
=== Crea tus credenciales de nube

Si ya tienes un conjunto de credenciales de nube para usar, omite esta sección.

. Haz clic en *☰ > Gestión de Clústeres*.
. Haz clic en *Credenciales de Nube*.
. Haga clic en *Crear*.
. Haz clic en *Azure*.
. Introduce tus credenciales de Azure.
. Haga clic en *Crear*.

*Resultado:* Has creado las credenciales de nube que se utilizan para aprovisionar nodos en tu clúster.

[#_create_your_cluster]
=== Crea tu clúster

Utiliza Rancher para crear un clúster de Kubernetes en Azure.

. Haz clic en *☰ > Gestión de Clústeres*.
. En la página *Clusters*, haz clic en *Crear*.
. Cambia el interruptor a *RKE2/K3s*.
. Haz clic en *Azure*.
. Selecciona una *Credencial de nube*, si existe más de una. De lo contrario, está preseleccionada.
. Introduce un *Nombre del clúster*.
. Crea un grupo de máquinas para cada rol de Kubernetes. Consulta las {xref-infra-providers}#_roles_de_nodo[mejores prácticas] para recomendaciones sobre asignaciones y conteos de roles.
 .. Para cada grupo de máquinas, define la configuración de la máquina. Consulta la {xref-azure-machine-configuration}[referencia de configuración de máquinas de Azure] para información sobre las opciones de configuración.
. Utiliza la *Configuración del Clúster* para elegir la versión de Kubernetes que deseas instalar, qué proveedor de red utilizar y si deseas habilitar el aislamiento de red del proyecto.  Para ayuda configurando el clúster, consulta las {xref-rke2-config}[referencias de configuración de clúster RKE2] y {xref-k3s-config}[K3s].
. Utiliza *Roles de Miembro* para configurar la autorización de usuarios para el clúster. Haz clic en *Añadir Miembro* para añadir usuarios que puedan acceder al clúster. Utiliza el menú desplegable *Rol* para establecer permisos para cada usuario.
. Haga clic en *Crear*.

*Resultado:*

Tu clúster ha sido creado y se le ha asignado un estado de *Aprovisionamiento*. Rancher está configurando tu clúster.

Puedes acceder a tu clúster después de que su estado se actualice a *Activo*.

Los clústeres *Activos* tienen asignados dos proyectos:

* `Default`, que contiene el espacio de nombres `default`
* `System`, que contiene los espacios de nombres `cattle-system`, `traefik`, `kube-public` y `kube-system`

[#_optional_next_steps]
=== Pasos siguientes opcionales

Después de crear tu clúster, puedes acceder a él a través de la interfaz de usuario de Rancher. Como mejor práctica, recomendamos configurar estas formas alternativas de acceder a tu clúster:

* *Accede a tu clúster con la CLI de kubectl:* Sigue {xref-access-cluster-kubectl}#_accediendo_a_clústeres_con_kubectl_desde_tu_estación_de_trabajo[estos pasos] para acceder a clústeres con kubectl en tu estación de trabajo. En este caso, serás autenticado a través del proxy de autenticación del servidor de Rancher, luego Rancher te conectará al clúster descendente. Este método te permite gestionar el clúster sin la interfaz de usuario de Rancher.
* *Accede a tu clúster con la CLI de kubectl, utilizando el punto final autorizado del clúster:* Sigue {xref-access-cluster-kubectl}#_autenticando_directamente_con_un_clúster_descendente[estos pasos] para acceder a tu clúster con kubectl directamente, sin autenticarte a través de Rancher. Recomendamos configurar este método alternativo para acceder a tu clúster, de modo que en caso de que no puedas conectarte a Rancher, aún puedas acceder al clúster.
