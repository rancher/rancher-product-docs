[#_region]
== Región

En el campo *Región*, selecciona la misma región que utilizaste al crear tus credenciales de nube.

[#_cloud_credentials]
== Credenciales de Nube

La información de acceso de tu cuenta de AWS, almacenada en una {xref-cloud-credentials}[credencial de nube.]

Consulta https://docs.aws.amazon.com/IAM/latest/UserGuide/id_credentials_access-keys.html#Using_CreateAccessKey[Documentación de Amazon: Creación de Claves de Acceso] cómo crear una Clave de Acceso y una Clave Secreta.

Consulta https://docs.aws.amazon.com/IAM/latest/UserGuide/access_policies_create.html#access_policies_create-start[Documentación de Amazon: Creación de Políticas IAM (Consola)] cómo crear una política IAM.

Consulta https://docs.aws.amazon.com/IAM/latest/UserGuide/id_users_change-permissions.html#users_change_permissions-add-console[Documentación de Amazon: Añadiendo Permisos a un Usuario (Consola)] cómo adjuntar una política IAM

Consulta nuestros tres ejemplos de políticas JSON:

* {xref-infra-aws}#_example_iam_policy[Example IAM Policy]
* {xref-infra-aws}#_ejemplo_politica_iam_con_passrole[Ejemplo de Política IAM con PassRole] (necesario si deseas utilizar {xref-set-up-cloud-providers}[Proveedor de Nube de Kubernetes] o quieres pasar un Perfil IAM a una instancia)
* {xref-infra-aws}#_ejemplo_politica_iam_para_permitir_volumenes_ebs_encriptados[Ejemplo de Política IAM para permitir volúmenes EBS cifrados] a un usuario.

[#_authenticate_configure_nodes]
== Autenticar y configurar nodos

Elige una zona de disponibilidad y la configuración de red para tu clúster.

[#_security_group]
== Grupo de seguridad

Elige el grupo de seguridad predeterminado o configura un grupo de seguridad.

Por favor, consulta {xref-port-reqs}#_rancher_aws_ec2_security_group[grupo de seguridad de Amazon EC2 al usar el controlador de nodos] para ver qué reglas se crean en el `rancher-nodes` grupo de seguridad.

Si proporcionas tu propio grupo de seguridad para una instancia EC2, ten en cuenta que Rancher no lo modificará. Por lo tanto, serás responsable de asegurarte de que tu grupo de seguridad esté configurado para permitir los {xref-port-reqs}#_ports_for_rancher_server_nodes_on_rke2[puertos necesarios para que Rancher aprovisione la instancia]. Para más información sobre cómo controlar el tráfico entrante y saliente a las instancias EC2 con grupos de seguridad, consulta https://docs.aws.amazon.com/vpc/latest/userguide/VPC_SecurityGroups.html#WorkingWithSecurityGroups[aquí].

[#_instance_options]
== Opciones de instancia

Configura las instancias que se crearán. Asegúrate de configurar el *Usuario SSH* correcto para la AMI configurada. Es posible que una región seleccionada no soporte el tipo de instancia predeterminado. En este escenario, debes seleccionar un tipo de instancia que sí exista, de lo contrario, ocurrirá un error indicando que la configuración solicitada no es compatible.

Si necesitas pasar un *Nombre del perfil de instancia IAM* (no ARN), por ejemplo, cuando quieras usar un {xref-set-up-cloud-providers}[Proveedor de Nube Kubernetes], necesitarás un permiso adicional en tu política. Consulta {xref-infra-aws}#_example_iam_policy_with_passrole[Ejemplo de política IAM con PassRole] para un ejemplo de política.

[#_engine_options]
== Opciones del motor

En la sección *Opciones del motor* de la plantilla del nodo, puedes configurar el daemon de contenedor. Puede que desees especificar la versión del contenedor o un espejo del registro de imágenes del contenedor.
