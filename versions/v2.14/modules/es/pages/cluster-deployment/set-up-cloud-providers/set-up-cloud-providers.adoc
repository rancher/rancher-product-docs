Un _proveedor de nube_ es un módulo en Kubernetes que proporciona una interfaz para gestionar nodos, balanceadores de carga y rutas de red.

Cuando se configura un proveedor de nube en Rancher, el servidor de Rancher puede aprovisionar automáticamente nuevos nodos, balanceadores de carga o dispositivos de almacenamiento persistente al lanzar definiciones de Kubernetes, si el proveedor de nube que está utilizando admite dicha automatización.

Su clúster no se aprovisionará correctamente si configura un clúster de nodos de proveedor de nube que no cumpla con los requisitos previos.

Por defecto, la opción *proveedor de nube* está configurada en `None`.

Los siguientes proveedores de nube pueden ser habilitados:

* Amazon
* Azure
* GCE (Google Compute Engine)
* vSphere

[#_setting_up_a_custom_cloud_provider]
== Configuración de un proveedor de nube personalizado

El proveedor de nube `Custom` está disponible si deseas configurar cualquier proveedor de nube de Kubernetes.

Para la opción de proveedor de nube personalizado, puedes consultar la https://rancher.com/docs/rke/latest/en/config-options/cloud-providers/[documentación de RKE] sobre cómo editar el archivo yaml para tu proveedor de nube específico. Existen proveedores de nube específicos que tienen una configuración más detallada:

* https://rke.docs.rancher.com/config-options/cloud-providers/vsphere[vSphere]
* https://rancher.com/docs/rke/latest/en/config-options/cloud-providers/openstack/[OpenStack]
