Si estáis utilizando máquinas virtuales de Azure para vuestros nodos, podéis usar https://docs.microsoft.com/en-us/azure/aks/azure-files-dynamic-pv[Azure files] como una StorageClass para el clúster.

Para que la plataforma de Azure cree los recursos de almacenamiento requeridos, seguid estos pasos:

. {xref-azure-cloud-provider}[Configurar el proveedor de nube de Azure.]
. Configurar `kubectl` para conectar con vuestro clúster.
. Copiad el manifiesto de `ClusterRole` y `ClusterRoleBinding` para la cuenta de servicio:
  ```yml
  --
  apiVersion: rbac.authorization.k8s.io/v1
  kind: ClusterRole
  metadata:
    name: system:azure-cloud-provider
  rules:
 ** apiGroups: ['']
resources: ['secrets']
verbs:     ['get','create']
  --
  apiVersion: rbac.authorization.k8s.io/v1
  kind: ClusterRoleBinding
  metadata:
name: system:azure-cloud-provider
  roleRef:
kind: ClusterRole
apiGroup: rbac.authorization.k8s.io
name: system:azure-cloud-provider
  subjects:
 ** kind: ServiceAccount
name: persistent-volume-binder
namespace: kube-system
  ```
. Cread estos en vuestro clúster utilizando uno de los siguientes comandos.
+
----
# kubectl create -f <MANIFEST>
----
