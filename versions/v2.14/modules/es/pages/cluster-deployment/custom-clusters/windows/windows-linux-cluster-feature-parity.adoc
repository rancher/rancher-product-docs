Los clústeres de Windows no comparten el mismo soporte de características que los clústeres de Linux.

El siguiente gráfico describe la paridad de características entre Windows y Linux en Rancher:

|===
| *Componente* | *Linux* | *Windows*

| *Distribuciones*
|
|

| RKE
| Admitido
| Admitido

| RKE2
| Admitido
| Admitido

| K3S
| Admitido
| No compatible

| EKS
| Admitido
| No compatible

| GKE
| Admitido
| No compatible

| AKS
| Admitido
| No compatible

| *Componentes de Rancher*
|
|

| Servidor
| Admitido
| No compatible

| oficina postal
| Admitido
| Admitido

| Fleet
| Admitido
| Admitido

| Operador EKS
| Admitido
| No compatible

| Operador AKS
| No compatible
| No compatible

| Operador GKE
| No compatible
| No compatible

| Monitoreo/Alertas v2
| Admitido
| Admitido

| Registro v2
| Admitido
| Admitido

| Istio
| Admitido
| No compatible

| Catálogo v2
| Admitido
| No compatible

| Admitido
| No compatible

| Admitido

| Admitido

|

| Flannel

| Traefik

| NGINX Ingress
| Admitido
| No compatible
|===

Para información actualizada sobre el soporte de características, puede visitar https://github.com/rancher/windows[rancher/windows] en GitHub.
