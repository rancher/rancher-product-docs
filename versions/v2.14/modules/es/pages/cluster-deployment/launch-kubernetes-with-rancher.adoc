Rancher permite lanzar un clúster de Kubernetes en diferentes tipos de nodos, incluidos servidores de equipo sin sistema operativo, máquinas virtuales locales y máquinas virtuales de un proveedor de infraestructura. Al desplegar Kubernetes en estos nodos, Rancher te da la opción de usar {url-rke2-introduction}[RKE2] o {url-k3s}[K3s].

Rancher puede instalar Kubernetes en nodos existentes, o puede aprovisionar dinámicamente nodos en un proveedor de infraestructura e instalar Kubernetes en ellos.

Rancher también puede crear grupos de máquinas. Un beneficio de instalar Kubernetes en grupos de máquinas alojados por un proveedor de infraestructura es que si un nodo pierde conectividad con el clúster, Rancher puede crear automáticamente otro nodo para unirse al clúster y asegurar que el número del grupo de máquinas sea el esperado.

[#__rke2_product_name]
== {rke2-product-name}

Rancher v2.6 introdujo el aprovisionamiento para clústeres {url-rke2-introduction}[RKE2] directamente desde la interfaz de usuario de Rancher. RKE2, también conocido como RKE Government, es una distribución de Kubernetes totalmente conforme que se centra en la seguridad y el cumplimiento de las normativas en EE. UU. Gobierno federal de Estados Unidos. En Rancher v.2.6.4 y versiones anteriores, el aprovisionamiento de RKE2 estaba en vista previa técnica.

Ten en cuenta que en Rancher v2.6.5, el aprovisionamiento de RKE2 se convirtió en GA.

[#_requirements]
=== Requisitos

Si utilizas RKE2 para configurar un clúster, tus nodos deben cumplir con los {url-rke2-requirements}[requisitos] para nodos en clústeres de usuarios en sentido descendente.

[#_launching_kubernetes_on_new_nodes_in_an_infrastructure_provider]
=== Lanzando Kubernetes en Nuevos Nodos en un Proveedor de Infraestructura

El aprovisionamiento de RKE2 se basa en un nuevo marco de aprovisionamiento que aprovecha el proyecto en sentido ascendente https://github.com/kubernetes-sigs/cluster-api[Cluster API (CAPI)]. Con este nuevo marco de aprovisionamiento, puedes:

* Aprovisionar clústeres RKE2 en cualquier proveedor para el cual Rancher tenga un controlador de nodos.
* Configurar completamente los clústeres RKE2 dentro de Rancher.
* Elegir opciones de CNI como Calico, Cilium y Multus además de Canal.

Cuando realices cambios en la configuración de tu clúster en RKE2, esto puede resultar en la reprovisión de nodos. Esto es controlado por los controladores de CAPI y no por Rancher en sí. Ten en cuenta que para los nodos etcd, el mismo comportamiento no se aplica.

Un cambio de configuración de ejemplo que puede causar el comportamiento descrito es al editar un clúster y habilitar el drenaje antes de eliminarlo. En este escenario, se eliminan los nodos del plano de control existentes y el nodo de trabajo, y se crean nuevos nodos.

El aprovisionamiento de RKE2 también incluye la instalación de RKE2 en clústeres con nodos de Windows.

Las características de Windows para RKE2 incluyen:

* Windows soporta el controlador de nodo vSphere
* Calico y Flannel CNI para clústeres personalizados de Windows RKE2
* Aislamiento de red del proyecto (PNI) para Calico
* Contenedores de Windows con RKE2 impulsados por containerd
* Aprovisionamiento de clústeres de Windows RKE2 a través de Terraform
* Aprovisionamiento de clústeres personalizados de Windows RKE2 directamente desde la interfaz de usuario de Rancher

El soporte de Windows para clústeres personalizados de RKE2 requiere elegir Calico como el CNI.

[#_launching_kubernetes_on_existing_custom_nodes]
=== Lanzando Kubernetes en Nodos Personalizados Existentes

El aprovisionamiento de RKE2 también permite instalar clústeres personalizados en máquinas virtuales o en nodos de equipo sin sistema operativo previamente aprovisionados.

Si deseas reutilizar un nodo de un clúster personalizado anterior, {xref-clean-cluster-nodes}#_cleaning_up_nodes[limpia el nodo] antes de usarlo de nuevo en un clúster. Si reutilizáis un nodo que no ha sido limpiado, la provisión del clúster puede fallar.

[#_programmatically_creating_rke2_product_name_clusters]
=== Creación programática de {rke2-product-name} clústeres

La forma más común de desplegar programáticamente clústeres de RKE2 a través de Rancher es utilizando el https://registry.terraform.io/providers/rancher/rancher2/latest/docs/resources/cluster_v2[proveedor de Terraform Rancher2].
