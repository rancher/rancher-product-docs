Este tutorial describe cómo configurar una base de datos MySQL en el Servicio de Base de Datos Relacional (RDS) de Amazon.

Esta base de datos puede ser utilizada más tarde como un almacén de datos externo para un clúster de Kubernetes K3s de alta disponibilidad.

. Inicie sesión en la https://console.aws.amazon.com/rds/[Consola de Amazon AWS RDS] para comenzar. Asegúrese de seleccionar la *Región* donde se crean sus instancias EC2 (nodos Linux).
. En el panel izquierdo, haga clic en *Bases de datos*.
. Haga clic en *Crear base de datos*.
. En la sección *Tipo de motor*, haga clic en *MySQL*.
. En la sección *Versión*, elija *MySQL 5.7.22*.
. En la sección *Configuración*, bajo *Configuración de credenciales,* introduzca una contraseña maestra para el nombre de usuario maestro *admin*. Confirme la contraseña.
. Expanda la sección *Configuración adicional*. En el campo *Nombre de la base de datos inicial*, introduzca un nombre. El nombre solo puede contener letras, números y guiones bajos. Este nombre se utilizará para conectarse a la base de datos.
. Haga clic en *Crear base de datos*.

Necesitará capturar la siguiente información sobre la nueva base de datos para que el clúster de Kubernetes K3s pueda conectarse a ella.

Para ver esta información en la consola de Amazon RDS, haga clic en *Bases de datos,* y haga clic en el nombre de la base de datos que creó.

* *Nombre de usuario:* Utilice el nombre de usuario administrador.
* *Contraseña:* Utilice la contraseña del administrador.
* *Nombre de host:* Utilice el *Punto de enlace* como el nombre de host. El punto de enlace está disponible en la sección *Conectividad y seguridad*.
* *Puerto:* El puerto debe ser 3306 por defecto. Puede confirmarlo en la sección *Conectividad y seguridad*.
* *Nombre de la base de datos:* Confirme el nombre yendo a la pestaña *Configuración*. El nombre se encuentra listado bajo *Nombre de la base de datos*.

Esta información se utilizará para conectarse a la base de datos en el siguiente formato:

----
mysql://username:password@tcp(hostname:3306)/database-name
----

Para más información sobre cómo configurar el almacén de datos para K3s, consulte la https://rancher.com/docs/k3s/latest/en/installation/datastore/[documentación de K3s.]
