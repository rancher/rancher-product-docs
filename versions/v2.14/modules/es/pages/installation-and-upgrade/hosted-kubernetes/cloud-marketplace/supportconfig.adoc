Después de instalar el adaptador CSP, tendrás la capacidad de generar un paquete de soporteconfig. Este paquete es un archivo tar que se puede utilizar para proporcionar rápidamente información al soporte.

Estos paquetes se pueden crear a través de Rancher o mediante acceso directo al clúster en el que está instalado Rancher. Ten en cuenta que se prefiere acceder a través de Rancher.

NOTE: Solo los usuarios administradores pueden generar/descargar paquetes de soporteconfig, independientemente del método.

[#_accessing_through_rancher]
== Accediendo a través de Rancher

Primero, haz clic en el menú hamburguesa. Luego haz clic en el botón `Get Support`.

image::support-help.png[Obtener soporte]

En la siguiente página, haz clic en el botón `Generate Support Config`.

NOTE: Si el adaptador no está instalado, la opción para generar el paquete de soporteconfig no estará presente. Debes instalar el adaptador CSP para generar un paquete de soporteconfig.

image::generate-support-config.png[Obtener soporte]

[#_accessing_without_rancher]
== Accediendo sin Rancher

Primero, genera un kubeconfig para el clúster en el que está instalado Rancher.

NOTE: Si Rancher está caído, no podrás usar un token kubeconfig generado por Rancher para acceder al clúster.

Configura tu entorno de shell para usar este token kubeconfig:

[,bash]
----
export KUBECONFIG=$MY_KUBECONFIG_PATH
----

Se recomienda crear un directorio de trabajo temporal mientras ejecutas este comando, como se muestra a continuación:

[,bash]
----
mkdir temp && cd temp
----

Luego, recupera el paquete de soporteconfig:

[,bash]
----
mkdir rancher && kubectl get configmap csp-config -n cattle-csp-adapter-system -o=jsonpath='{.data.data}' >> rancher/config.json && tar -c -f supportconfig_rancher.tar rancher && rm -rf rancher
----

Esto creará un archivo `supportconfig_rancher.tar` en tu directorio actual.

Los usuarios que ejecutan estos comandos en Mac pueden experimentar problemas debido a incompatibilidades entre gnu-tar y bsd-tar. Si el soporte tiene problemas para leer un supportconfig que tú produces, intenta usar el siguiente comando después de hacer gnu-tar accesible como `gtar` en tu vía:

[,bash]
----
mkdir rancher && kubectl get configmap csp-config -n cattle-csp-adapter-system -o=jsonpath='{.data.data}' >> rancher/config.json && gtar -c -f supportconfig_rancher.tar rancher && rm -rf rancher
----
