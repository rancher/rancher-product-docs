[#_air_gapped_installations]
== Instalaciones en entorno aislado

Sigue {xref-air-gap}[estos pasos] para instalar el servidor Rancher en un entorno aislado.

Un entorno aislado podría ser donde se instalará el servidor Rancher sin conexión, detrás de un firewall o detrás de un proxy.

[#_docker_installations]
== Instalaciones de Docker

La xref:[instalación de Docker de un solo nodo] es para usuarios de Rancher que desean probar Rancher. En lugar de ejecutarse en un clúster de Kubernetes utilizando Helm, instalas el componente del servidor Rancher en un solo nodo utilizando un `docker run` comando.

La instalación de Docker es solo para entornos de desarrollo y pruebas.

Dado que solo hay un nodo y un único contenedor de Docker, si el nodo falla, no hay copia de los datos de etcd disponible en otros nodos y perderás todos los datos de tu servidor Rancher.

El operador de copia de seguridad de Rancher se puede utilizar para migrar Rancher de la instalación de un solo contenedor de Docker a una instalación en un clúster de Kubernetes de alta disponibilidad. Para más detalles, consulta la documentación sobre {xref-migrate-cluster}[migrar Rancher a un nuevo clúster.]
