Esta sección trata sobre el uso del CLI de Helm para instalar el servidor Rancher en un entorno aislado. Un entorno aislado podría ser donde se instalará el servidor Rancher sin conexión, detrás de un firewall o detrás de un proxy.

Los pasos de instalación difieren dependiendo de si Rancher se instala en un clúster de Kubernetes K3s o en un único contenedor Docker.

Para más información sobre cada opción de instalación, consulta {xref-install-upgrade}[esta página.]

A lo largo de las instrucciones de instalación, habrá _pestañas_ para cada opción de instalación.

[NOTE]
.Importante:
====

Si instalas Rancher siguiendo la guía de instalación de Docker, no hay una vía de actualización para transitar tu instalación de Docker a una instalación de Kubernetes.
====
