[#_specify_a_hostname_to_use]
== Especifica un nombre de host a utilizar

Si utilizas esta opción, Ingress dirige las solicitudes para un nombre de host al servicio o carga de trabajo que especifiques.

. Especifica una vía de tipo `Prefix` y especifica una vía como `/`.
. Añade un *Servicio Objetivo*.
. *Opcional:* Si deseas especificar una carga de trabajo o servicio cuando se envía una solicitud a una vía de nombre de host particular, añade una *Vía* para el objetivo. Por ejemplo, si deseas que las solicitudes para `www.mysite.com/contact-us` se envíen a un servicio diferente al `www.mysite.com`, introduce `/contact-us` en el campo *Vía*. Típicamente, la primera regla que creas no incluye una vía.
. Introduce el número de *Puerto* en el que opera cada objetivo.

[#_certificates]
== Certificados

[NOTE]
====

Debes tener un certificado SSL que Ingress pueda utilizar para encriptar y desencriptar las comunicaciones. Para más información, consulta {xref-http-encrypt}[Añadiendo Certificados SSL].
====


. Para crear un controlador de Ingress, haz clic en la pestaña *Certificados*.
. Haz clic en *Añadir Certificado*.
. Selecciona un *Nombre del Certificado - Secreto* de la lista desplegable.
. Introduce el host utilizando comunicación encriptada.
. Para añadir más hosts que utilicen el mismo certificado, haz clic en *Añadir Hosts*.
