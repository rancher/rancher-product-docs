Cette section contient des informations pour vous aider à résoudre les problèmes lors de l'utilisation de Rancher.

* {xref-k8s-component-troubleshooting}[Kubernetes components]
+
Si vous avez besoin d'aide pour dépanner les composants du noyau du cluster Kubernetes tels que :

 ** `etcd`
 ** `kube-apiserver`
 ** `kube-controller-manager`
 ** `kube-scheduler`
 ** `kubelet`
 ** `kube-proxy`
 ** `nginx-proxy`

* xref:troubleshooting/other-troubleshooting-tips/kubernetes-resources.adoc[Ressources Kubernetes]
+
Les options pour dépanner les ressources Kubernetes telles que les nœuds, le contrôleur d'Ingress et les agents Rancher sont décrites dans cette section.

* xref:troubleshooting/other-troubleshooting-tips/networking.adoc[Réseautique]
+
Les étapes pour dépanner les problèmes de réseau peuvent être trouvées ici.

* xref:troubleshooting/other-troubleshooting-tips/dns.adoc[DNS]
+
Lorsque vous rencontrez des problèmes de résolution de noms dans votre cluster.

* {xref-rancher-ha-troubleshooting}[Dépannage de Rancher installé sur Kubernetes]
+
Si vous rencontrez des problèmes avec votre {xref-install-rancher}[serveur Rancher installé sur Kubernetes]

* {xref-logging-troubleshooting}[Journalisation]
+
Lisez-en plus sur les niveaux de journalisation qui peuvent être configurés et comment configurer un niveau de journalisation.

* xref:troubleshooting/other-troubleshooting-tips/user-id-tracking-in-audit-logs.adoc[Suivi des identifiants d'utilisateur dans les journaux d'audit]
+
Lisez-en plus sur la façon dont un administrateur Rancher peut tracer un événement à partir des journaux d'audit de Rancher et dans les journaux d'audit Kubernetes en utilisant le nom d'utilisateur du fournisseur d'identité externe.

* {xref-expired-webhook-cert-rotation}[Certificats de webhook expirés]
+
Lisez-en plus sur la façon de renouveler un secret de certificat de webhook Rancher après son expiration sur une base annuelle.
