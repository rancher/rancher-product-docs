Rancher étend Kubernetes avec des API supplémentaires en enregistrant un serveur API d'extension en utilisant le https://kubernetes.io/docs/concepts/extend-kubernetes/api-extension/apiserver-aggregation/[couche d'agrégation de l'API Kubernetes].

[#_aggregation_layer_is_required]
== La couche d'agrégation est requise

La couche d'agrégation de l'API doit être configurée sur le cluster Kubernetes local pour que le `v1.ext.cattle.io` `APIService` fonctionne correctement. Si le `APIService` ne reçoit pas de demande d'enregistrement après le démarrage du serveur Rancher, le pod plante avec une entrée de journal indiquant l'erreur. Si vos pods échouent systématiquement à détecter l'enregistrement malgré une configuration correcte du cluster, vous pouvez augmenter le délai d'attente en définissant le `.Values.aggregationRegistrationTimeout` dans Helm.

Dans cette version de Rancher, les versions de Kubernetes prises en charge par RKE2/K3s ont la couche d'agrégation configurée et activée par défaut. Cependant, si vous soupçonnez que la configuration de votre cluster est incorrecte, consultez la https://kubernetes.io/docs/tasks/extend-kubernetes/configure-aggregation-layer/[documentation de la couche d'agrégation Kubernetes] pour des informations sur la configuration de la couche d'agrégation.

[NOTE]
====
Si la distribution Kubernetes sous-jacente ne prend pas en charge la couche d'agrégation, vous devez migrer vers une distribution Kubernetes qui le fait avant de procéder à la mise à niveau.
====
