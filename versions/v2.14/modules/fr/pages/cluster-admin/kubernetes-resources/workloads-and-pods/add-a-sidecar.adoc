Un _sidecar_ est un conteneur qui étend ou améliore le conteneur principal dans un pod. Le conteneur principal et le sidecar partagent un pod, et partagent donc le même espace réseau et de stockage. Vous pouvez ajouter des sidecars aux charges de travail existantes en utilisant l'option *Ajouter un Sidecar*.

. Dans le coin supérieur gauche, cliquez sur *☰ > Gestion des clusters*.
. Allez dans le cluster où vous souhaitez ajouter un sidecar et cliquez sur *Explorer*.
. Dans la barre de navigation à gauche, cliquez sur *Charge de travail*.
. Trouvez la charge de travail que vous souhaitez étendre. Sélectionnez *⋮ > + Ajouter un Sidecar*.
. Entrez un *Nom* pour le sidecar.
. Dans la section *Général*, sélectionnez un type de sidecar. Cette option détermine si le conteneur sidecar est déployé avant ou après le déploiement du conteneur principal.
 ** *Conteneur Standard :*
+
Le conteneur sidecar est déployé après le conteneur principal.

 ** *Conteneur Init :*
+
Le conteneur sidecar est déployé avant le conteneur principal.
. Dans le champ *Image du Conteneur*, entrez le nom de l'image du conteneur que vous souhaitez déployer en soutien au conteneur principal. Lors du déploiement, Rancher récupère cette image depuis https://hub.docker.com/explore/[Docker Hub]. Entrez le nom exactement comme il apparaît sur Docker Hub.
. Définissez les options restantes. Vous pouvez en lire plus à ce sujet dans {xref-deploy-workloads}[Déploiement des charges de travail].
. Cliquez sur *Lancer*.

*Résultat :* Le sidecar est déployé selon vos paramètres. Après son déploiement, vous pouvez visualiser le sidecar en sélectionnant *⋮ icône (...) > Modifier* pour le déploiement principal.
