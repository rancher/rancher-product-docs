La configuration des pods est gérée par les Déploiements, StatefulSets et Daemonsets, tandis que les services dirigent le trafic vers les pods à l'aide de sélecteurs.

Pour chaque charge de travail (avec au moins un port configuré) créée, une entrée de découverte de service complémentaire est créée. Cette entrée de découverte de service permet la résolution DNS pour les pods de la charge de travail en utilisant la convention de nommage suivante{nbsp}:
`<workload>.<namespace>.svc.cluster.local`.

Vous pouvez créer des services supplémentaires afin qu'un espace de noms donné se résolve avec une ou plusieurs adresses IP externes, un nom d'hôte externe, un alias vers un autre enregistrement DNS, d'autres charges de travail, ou un ensemble de pods qui correspondent à un sélecteur que vous créez.

. Dans le coin supérieur gauche, cliquez sur *☰ > Gestion des clusters*.
. Allez dans le cluster où vous souhaitez ajouter un service et cliquez sur *Explorer*.
. Cliquez sur menu:Découverte de service[Services].
. Cliquez sur *Create*.
. Choisissez le type de service que vous souhaitez créer.
. Sélectionnez un *Espace de noms* dans la liste déroulante.
. Entrez un *Nom* pour le service. Ce nom est utilisé pour la résolution DNS.
. Remplissez le reste du formulaire. Pour obtenir de l'aide, consultez la documentation Kubernetes en amont concernant https://kubernetes.io/docs/concepts/services-networking/service/[les services.]
. Cliquez sur *Create*.

*Résultat :* Un nouveau service est créé.

* Vous pouvez voir l'enregistrement depuis l'onglet *Découverte de service* du projet.
* Lorsque vous visitez le nouveau nom DNS pour le nouvel enregistrement que vous avez créé (`<recordname>.<namespace>.svc.cluster.local`), il résout l'espace de noms choisi.
