:product-path: cluster-admin/project-admin/project-resource-quotas/resource-quota-types.adoc

Lorsque vous créez un quota de ressources, vous configurez le pool de ressources disponibles pour le projet. Rancher prend en charge l'utilisation de références de ressources arbitraires et de leurs quotas. Cela vous permet d'utiliser tous les types en amont https://kubernetes.io/docs/concepts/policy/resource-quotas/#types-of-resource-quota[Kubernetes `ResourceQuota`] lors de la gestion des quotas de ressources du projet.

Vous pouvez définir des limites de ressources pour les types de ressources prédéfinis suivants, où le type `Custom` permet de spécifier des ressources arbitraires et leurs quotas.

[NOTE]
====
Le support des références de ressources arbitraires utilisant le type `Custom` ne couvre pas les ressources dans le groupe d'API `ext.cattle.io`.
====

|===
| Type de ressource | Description

| Limite d'UC*
| La quantité maximale d'UC (en https://kubernetes.io/docs/concepts/configuration/manage-compute-resources-container/#meaning-of-cpu[millicores]) allouée au projet/espace de noms.^1^

| Réservation d'UC*
| La quantité minimale d'UC (en millicores) garantie au projet/espace de noms.^1^

| Limite de mémoire*
| La quantité maximale de mémoire (en https://kubernetes.io/docs/concepts/configuration/manage-resources-containers/#meaning-of-memory[octets]) allouée au projet/espace de noms.^1^

| Réservation de mémoire*
| La quantité minimale de mémoire (en octets) garantie au projet/espace de noms.^1^

| Réservation de stockage
| La quantité minimale de stockage (en gigaoctets) garantie au projet/espace de noms.

| Services d'équilibrage de charge
| Le nombre maximum de services d'équilibrage de charge pouvant exister dans le projet/espace de noms.

| Services de port de nœud
| Le nombre maximum de services de port de nœud pouvant exister dans le projet/espace de noms.

| Pods
| Le nombre maximum de pods pouvant exister dans le projet/espace de noms dans un état non terminal (c'est-à-dire, des pods avec un état de `.status.phase in (Failed, Succeeded)` égal à vrai).

| Services
| Le nombre maximum de services pouvant exister dans le projet/espace de noms.

| ConfigMaps
| Le nombre maximum de ConfigMaps pouvant exister dans le projet/espace de noms.

| Demandes de volumes persistants
| Le nombre maximum de demandes de volumes persistants pouvant exister dans le projet/espace de noms.

| Contrôleurs de réplication
| Le nombre maximum de contrôleurs de réplication pouvant exister dans le projet/espace de noms.

| Sécrets
| Le nombre maximum de secrets pouvant exister dans le projet/espace de noms.

| Personnalisé**
| La spécification de ressources arbitraires et de leurs quotas, au-delà des types de ressources intégrés dans les projets, comme indiqué ci-dessus.
|===

[NOTE]
.*^*^*
====
Lors de la définition des quotas de ressources, si vous définissez quoi que ce soit lié à l'UC ou à la mémoire (c'est-à-dire des limites ou des réservations) sur un projet ou un espace de noms, tous les conteneurs devront avoir un champ d'UC ou de mémoire respectif défini lors de leur création. Une limite de ressources par défaut pour les conteneurs peut être définie en même temps pour éviter la nécessité de définir explicitement ces limites pour chaque charge de travail. Voir la https://kubernetes.io/docs/concepts/policy/resource-quotas/#requests-vs-limits[documentation Kubernetes] pour plus de détails sur les raisons pour lesquelles cela est nécessaire.
====

[NOTE]
.*^*\^**^*^*
====
Par exemple :

* `requests.nvidia.com/gpu: 4`
* `gold.storageclass.storage.k8s.io/requests.storage: 500Gi`
* `count/podtemplates: 10`

Voir la https://kubernetes.io/docs/concepts/policy/resource-quotas/#quota-for-extended-resources[documentation Kubernetes] pour de nombreux autres exemples.
====
