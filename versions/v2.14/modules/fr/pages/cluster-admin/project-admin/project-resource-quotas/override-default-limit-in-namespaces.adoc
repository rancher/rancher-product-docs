Bien que la *limite par défaut de l'espace de noms* se propage du projet à chaque espace de noms lors de sa création, dans certains cas, vous devrez peut-être augmenter (ou diminuer) les quotas pour un espace de noms spécifique. Dans cette situation, vous pouvez remplacer les limites par défaut en modifiant l'espace de noms.

Dans le diagramme ci-dessous, l'administrateur Rancher a un quota de ressources en vigueur pour son projet. Cependant, l'administrateur souhaite remplacer les limites de l'UC pour `Namespace 3` afin qu'il dispose de plus de ressources disponibles. Par conséquent, l'administrateur {xref-projects-and-namespaces}[augmente les limites de l'UC] pour `Namespace 3` afin que l'espace de noms puisse accéder à plus de ressources.

== Modifier les Quotas de Ressources de l'Espace de Noms

S'il existe un quota de ressources configuré pour un projet, vous pouvez remplacer la limite par défaut de l'espace de noms pour fournir à un espace de noms spécifique un accès à plus (ou moins) de ressources du projet.

. Dans le coin supérieur gauche, cliquez sur *☰ > Gestion des clusters*.
. Sur la page *Clusters*, allez au cluster où vous souhaitez modifier un quota de ressources d'espace de noms et cliquez sur *Explorer*.
. Cliquez sur menu:Cluster[Projets/Espaces de noms].
. Trouvez l'espace de noms pour lequel vous souhaitez modifier le quota de ressources. Cliquez sur *⋮ > Modifier la configuration*.
. Modifiez les limites de ressources.  Ces limites déterminent les ressources disponibles pour l'espace de noms. Les limites doivent être définies dans les limites de projet configurées.
+
Pour plus d'informations sur chaque *Type de ressource*, consultez {xref-resource-quota-types}[la référence de type].
+

[NOTE]
====

 ** Si un quota de ressources n'est pas configuré pour le projet, ces options ne seront pas disponibles.
 ** Si vous entrez des limites qui dépassent les limites de projet configurées, Rancher ne vous permettra pas d'enregistrer vos modifications.

+
====


*Résultat :* Votre substitution est appliquée au quota de ressources de l'espace de noms.
