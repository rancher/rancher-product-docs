Cette section concerne les outils pouvant être utilisés pour accéder aux clusters gérés par Rancher.

Pour des informations sur la façon de donner aux utilisateurs la permission d'accéder à un cluster, voir la section sur {xref-add-users-to-clusters}[ajouter des utilisateurs aux clusters.]

Pour plus d'informations sur le contrôle d'accès en fonction du rôle, voir {xref-manage-rbac}[cette section.]

Pour des informations sur la façon de configurer un système d'authentification, voir {xref-authn-authz}[cette section.]

[#_clusters_in_rancher_ui]
== Clusters dans l'interface utilisateur de Rancher

Il existe plusieurs chemins pour visualiser et gérer les clusters via l'interface utilisateur de Rancher.

[#_clusters_page]
=== Page des clusters

Vous pouvez accéder à la page *Clusters* depuis le menu *☰*{nbsp}:

. Click *☰*.
. Sélectionnez *Cluster Management*.

Vous pouvez également accéder à la page *Clusters* en cliquant sur le bouton *Gérer* au-dessus du tableau des clusters sur la page d'accueil de l'interface utilisateur de Rancher *Accueil*.

Sur la page *Clusters*, sélectionnez *⁝* à la fin de chaque ligne pour afficher un sous-menu avec les options suivantes{nbsp}:

* {xref-access-cluster-kubectl}[Kubectl Shell]
* Télécharger KubeConfig
* Copier KubeConfig dans le presse-papiers
* Modifier la configuration
* Voir YAML
* Télécharger YAML

[#_cluster_dashboard]
=== Tableau de bord du cluster

Sur la page *Clusters*, sélectionnez le bouton *Explorer* à la fin de chaque ligne pour voir le *Tableau de bord du cluster*. Vous pouvez également voir le tableau de bord en cliquant sur le nom d'un cluster dans le tableau, puis en cliquant sur le bouton *Explorer* sur la page *Cluster*.

Le *Tableau de bord du cluster* est également accessible depuis la page d'accueil de l'interface utilisateur de Rancher *Accueil*, en cliquant sur le nom d'un cluster.

Vous pouvez également accéder au *Tableau de bord du cluster* depuis le *☰* dans la barre de navigation supérieure{nbsp}:

. Click *☰*.
. Sélectionnez le nom d'un cluster dans l'option de menu *Explorer le cluster*.

Le *Tableau de bord du cluster* liste des informations sur un cluster spécifique, telles que le nombre de nœuds, l'utilisation de la mémoire, les événements et les ressources.

[#_kubectl]
== kubectl

Vous pouvez utiliser l'outil de ligne de commande Kubernetes, https://kubernetes.io/docs/reference/kubectl/overview/[kubectl], pour gérer vos clusters. Vous avez deux options pour utiliser kubectl{nbsp}:

* *Shell kubectl Rancher :* Interagissez avec vos clusters en lançant un shell kubectl disponible dans l'interface utilisateur de Rancher. Cette option ne nécessite aucune action de configuration de votre part. Pour plus d'informations, voir {xref-access-cluster-kubectl}[Accéder aux clusters avec le shell kubectl].
* *Connexion terminale distante :* Vous pouvez également interagir avec vos clusters en installant https://kubernetes.io/docs/tasks/tools/install-kubectl/[kubectl] sur votre bureau local, puis en copiant le fichier kubeconfig du cluster dans votre répertoire local `~/.kube/config`. Pour plus d'informations, voir {xref-access-cluster-kubectl}[Accéder aux clusters avec kubectl et un fichier kubeconfig].

[#_rancher_cli]
== Rancher CLI

Vous pouvez contrôler vos clusters en téléchargeant l'interface de ligne de commande propre à Rancher, {xref-rancher-cli}[Rancher CLI]. Cet outil CLI peut interagir directement avec différents clusters et projets ou leur passer des `kubectl` commandes.

[#_rancher_api]
== API Rancher

Enfin, vous pouvez interagir avec vos clusters via l'API Rancher. Avant d'utiliser l'API, vous devez obtenir une {xref-user-api-keys}[clé API]. Pour voir les différents champs de ressources et actions pour un objet API, ouvrez l'interface API, accessible en cliquant sur *Voir dans l'API* pour tout objet de l'interface utilisateur de Rancher.
