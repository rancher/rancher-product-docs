Pour plus de détails sur les nœuds EC2, consultez la documentation officielle pour la https://aws.amazon.com/ec2[Console de gestion EC2].

[#_basic_options]
== Options de base

[#_region]
=== Région

La https://docs.aws.amazon.com/AWSEC2/latest/UserGuide/using-regions-availability-zones.html[région] géographique dans laquelle construire votre cluster.

[#_zone]
=== Zone

La https://docs.aws.amazon.com/AWSEC2/latest/UserGuide/using-regions-availability-zones.html#concepts-availability-zones[zone], un emplacement isolé au sein d'une région pour construire votre cluster

[#_instance_type]
=== Type d'instance

Le https://docs.aws.amazon.com/AWSEC2/latest/UserGuide/instance-types.html[type d'instance], qui détermine les caractéristiques matérielles, utilisé pour provisionner votre cluster.

[#_root_disk_size]
=== Taille du disque racine

Configurez la taille (en Go) de votre https://docs.aws.amazon.com/AWSEC2/latest/UserGuide/RootDeviceStorage.html[dispositif racine].

[#_vpc_subnet]
=== VPC/Subnet

Le https://docs.aws.amazon.com/vpc/latest/userguide/configure-your-vpc.html[VPC] ou le https://docs.aws.amazon.com/vpc/latest/userguide/configure-subnets.html[sous-réseau] spécifique, une plage IP dans votre VPC, pour ajouter vos ressources.

[#_iam_instance_profile_name]
=== Nom du profil d'instance IAM

Le nom du https://docs.aws.amazon.com/IAM/latest/UserGuide/id_roles_use_switch-role-ec2_instance-profiles.html[profil d'instance] utilisé pour passer un rôle IAM à une instance EC2.

[#_advanced_options]
== Options avancées

[#_ami_id]
=== ID AMI

L'https://docs.aws.amazon.com/AWSEC2/latest/UserGuide/AMIs.html[image machine Amazon] utilisée pour les nœuds de votre cluster.

[#_ssh_username_for_ami]
=== Nom d'utilisateur SSH pour AMI

Le nom d'utilisateur pour se connecter à vos instances lancées. Référez-vous à https://docs.aws.amazon.com/AWSEC2/latest/UserGuide/connection-prereqs.html[ici] pour les noms d'utilisateur par défaut des AMIs sélectionnées. Pour les AMIs non listées, vérifiez auprès du fournisseur d'AMI.

[#_security_group]
=== Groupe de sécurité

Choisissez le groupe de sécurité par défaut ou configurez un groupe de sécurité.

Veuillez vous référer à {xref-port-reqs}#_rancher_aws_ec2_security_group[groupe de sécurité Amazon EC2 lors de l'utilisation du pilote Node] pour voir quelles règles sont créées dans le `rancher-nodes` groupe de sécurité.

[#_ebs_root_volume_type]
=== Type de volume racine EBS

Le type de volume https://docs.aws.amazon.com/AWSEC2/latest/UserGuide/ebs-volume-types.html[EBS] à utiliser pour le dispositif racine.

[#_encrypt_ebs_volume]
=== Chiffrer le volume EBS

Activer https://docs.aws.amazon.com/AWSEC2/latest/UserGuide/EBSEncryption.html[le chiffrement Amazon EBS].

[#_request_spot_instance]
=== Demander une instance Spot

Activer l'option pour https://docs.aws.amazon.com/AWSEC2/latest/UserGuide/spot-requests.html[demander des instances Spot] et spécifier le prix maximum par heure que vous êtes prêt à payer.

[#_use_only_private_address]
=== Utiliser uniquement l'adresse privée

Activer l'option d'utiliser uniquement https://docs.aws.amazon.com/AWSEC2/latest/UserGuide/using-instance-addressing.html[des adresses privées].

[#_ebs_optimized_instance]
=== Instance optimisée pour EBS

Utiliser une https://docs.aws.amazon.com/AWSEC2/latest/UserGuide/ebs-optimized.html[instance optimisée pour EBS].

[#_allow_access_to_ec2_metadata]
=== Autoriser l'accès aux métadonnées EC2

Activer l'accès aux https://docs.aws.amazon.com/AWSEC2/latest/UserGuide/ec2-instance-metadata.html[métadonnées EC2].

[#_use_tokens_for_metadata]
=== Utiliser des jetons pour les métadonnées

Utiliser https://docs.aws.amazon.com/AWSEC2/latest/UserGuide/configuring-instance-metadata-service.html[la version 2 du service de métadonnées d'instance (IMDSv2)], une méthode basée sur des jetons pour accéder aux métadonnées.

[#_add_tag]
=== Ajouter une étiquette

Ajouter des métadonnées en utilisant https://docs.aws.amazon.com/AWSEC2/latest/UserGuide/Using_Tags.html[des étiquettes] pour catégoriser les ressources.

[#_ipv6_address_count]
=== Nombre d'adresses IPv6

Spécifiez combien d'adresses IPv6 attribuer à l'interface réseau de l'instance.

[#_ipv6_address_only]
=== Adresse IPv6 uniquement

Activez cette option si l'instance doit utiliser exclusivement IPv6. Les VPC ou sous-réseaux uniquement IPv6 nécessitent cela. Lorsque cette option est activée, l'instance a l'IPv6 comme seule adresse, et le nombre d'adresses IPv6 doit être supérieur à zéro.

[#_http_protocol_ipv6]
=== HTTP Protocol IPv6

Activer ou désactiver les points de terminaison IPv6 pour le service de métadonnées de l'instance.

[#_enable_primary_ipv6]
=== Activer l'IPv6 principal

Activez cette option pour désigner la première adresse IPv6 assignée comme adresse principale. Cela garantit une adresse IPv6 cohérente et invariable pour l'instance. Cela ne contrôle pas si des adresses IPv6 sont assignées.
