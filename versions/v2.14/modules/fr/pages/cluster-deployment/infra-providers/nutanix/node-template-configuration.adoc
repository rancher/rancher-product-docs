[#_instance_options]
== Options d'instance

Dans la section *Options d'instance*, configurez le nombre de vCPUs, la mémoire et la taille du disque pour les VMs créées par ce modèle.

[cols="<,^,<,<"]
|===
| Paramètre | Requis | Description | Par défaut

| Nombre de processeurs
|
| Nombre de vCPUs alloués à la VM (cœurs)
| 2

| Mémoire
|
| Quantité de RAM allouée à la VM (Mo)
| 2 Go

| Image de modèle
| ✓
| Nom du modèle d'image de disque à cloner comme disque principal de la VM (doit prendre en charge cloud-init)
|

| Taille du disque de la VM
|
| Nouvelle taille du disque principal de la VM (en GiB)
|

| Taille de disque supplémentaire
|
| Taille d'un disque supplémentaire à ajouter à la VM (en GiB)
|

| Conteneur de stockage
|
| Conteneur de stockage _UUID_ dans lequel provisionner un disque supplémentaire
|

| Cloud Config YAML
|
| Cloud-init à fournir à la VM (sera patché avec l'utilisateur root de Rancher)
|

| Réseau
| ✓
| Nom(s) des réseau(x) à attacher à la VM
|

| Catégories de VM
|
| Nom(s) des catégories à appliquer à la VM
|
|===

La VM peut utiliser tout système d'exploitation Linux moderne configuré pour prendre en charge https://cloudinit.readthedocs.io/en/latest/[cloud-init] en utilisant la https://cloudinit.readthedocs.io/en/latest/reference/datasources/configdrive.html[Config Drive v2 datasource].

[#_networks]
== Réseaux

Le modèle de nœud permet de provisionner une VM avec plusieurs réseaux. Dans le champ *Network*, vous pouvez cliquer sur *Add* pour ajouter les réseaux disponibles dans AOS.

[#_vm_categories]
== Catégories de VM

Une catégorie est un regroupement d'entités en une paire clé-valeur. En général, les VM sont assignées à une catégorie en fonction de certains critères. Des politiques peuvent ensuite être liées à ces entités qui sont assignées (regroupées par) une valeur de catégorie spécifique.

[#_cloud_init]
== cloud-init

https://cloudinit.readthedocs.io/en/latest/[Cloud-init] vous permet d'initialiser vos nœuds en appliquant une configuration au premier démarrage. Cela peut impliquer des choses telles que la création d'utilisateurs ou l'autorisation de clés SSH.

Pour utiliser l'initialisation cloud-init, collez une configuration cloud en utilisant une syntaxe YAML valide dans le champ *Cloud Config YAML*. Référez-vous à la https://cloudinit.readthedocs.io/en/latest/topics/examples.html[documentation cloud-init] pour un ensemble d'exemples commentés de directives de configuration cloud prises en charge.

Notez que la configuration réseau basée sur cloud-init n'est pas recommandée et n'est prise en charge que via les données utilisateur `runcmd` plutôt que par NoCloud ou d'autres sources de données de configuration réseau.

La gestion des adresses IP Nutanix (IPAM) ou un autre service DHCP est recommandée.

[#_engine_options]
== Options du moteur

Dans la section *Options du moteur* du modèle de nœud, vous pouvez configurer le daemon de conteneur. Vous souhaiterez peut-être spécifier la version du conteneur ou un miroir de registre d'images de conteneur.

[NOTE]
====
Si vous provisionnez des nœuds Red Hat Enterprise Linux (RHEL) ou CentOS, laissez le champ *URL d'installation de Docker* à la valeur par défaut, ou sélectionnez *aucun*. Cela contournera une vérification de l'installation de Docker, car Docker est déjà installé sur ces types de nœuds.

Si vous définissez *URL d'installation de Docker* à une valeur autre que la valeur par défaut ou *aucun*, vous pourriez voir un message d'erreur tel que le suivant{nbsp}: `Error creating machine: RHEL ssh command error: command: sudo -E yum install -y curl err: exit status 1 output: Updating Subscription Management repositories.`
====
