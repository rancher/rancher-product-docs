Pour plus d'informations sur Azure, consultez la https://docs.microsoft.com/en-us/azure/?product=featured[documentation officielle d'Azure.]

Les informations d'accès au compte sont stockées en tant qu'identifiant cloud. Les identifiants cloud sont stockés en tant que secrets Kubernetes. Plusieurs modèles de nœuds peuvent utiliser le même identifiant cloud. Vous pouvez utiliser un identifiant cloud existant ou en créer un nouveau.

* *Placement* définit la région géographique où votre cluster est hébergé et d'autres métadonnées de localisation.
* *Réseau* configure le réseau utilisé dans votre cluster.
* *Instance* personnalise la configuration de votre VM.
+

[NOTE]
====
Si vous utilisez un VNet dans un groupe de ressources différent de celui des VM, le nom du VNet doit être préfixé par le nom du groupe de ressources. Par exemple : `<resource group>:<vnet>`.
====


Si vous utilisez Docker, les options de configuration du https://docs.docker.com/engine/docker-overview/#the-docker-daemon[daemon Docker] incluent{nbsp}:

* *Libellés :* Pour des informations sur les libellés, référez-vous à la https://docs.docker.com/config/labels-custom-metadata/[documentation des libellés d'objet Docker.].
* *URL d'installation de Docker Engine :* Détermine quelle version de Docker sera installée sur l'instance.
+

[NOTE]
====
Si vous provisionnez des nœuds Red Hat Enterprise Linux (RHEL) ou CentOS, laissez le champ *URL d'installation de Docker* à la valeur par défaut, ou sélectionnez *aucun*. Cela contournera une vérification de l'installation de Docker, car Docker est déjà installé sur ces types de nœuds.

Si vous définissez *URL d'installation de Docker* à une valeur autre que la valeur par défaut ou *aucun*, vous pourriez voir un message d'erreur tel que le suivant{nbsp}: `Error creating machine: RHEL ssh command error: command: sudo -E yum install -y curl err: exit status 1 output: Updating Subscription Management repositories.`
====


* *Miroirs de registre :* Miroir de registre Docker à utiliser par le daemon Docker.
* *Autres options avancées :* Référez-vous à la https://docs.docker.com/engine/reference/commandline/dockerd/[référence des options du daemon Docker].
