[#_basic_options]
== Options de base

[#_environment]

[#_location]
=== Emplacement

Configurez le cluster et https://docs.microsoft.com/en-us/azure/virtual-machines/regions[l'emplacement] du nœud.

[#_resource_group]
=== Groupe de ressources

Un groupe de ressources est un conteneur qui contient des ressources liées pour une solution Azure. Le groupe de ressources peut inclure toutes les ressources de la solution, ou seulement celles que vous souhaitez gérer en tant que groupe. Vous décidez comment vous souhaitez allouer des ressources aux groupes de ressources en fonction de ce qui a le plus de sens pour votre organisation. En général, ajoutez des ressources qui partagent le même cycle de vie au même groupe de ressources afin de pouvoir les déployer, les mettre à jour et les supprimer facilement en tant que groupe.

Utilisez un groupe de ressources existant ou saisissez un nom de groupe de ressources et un groupe sera créé pour vous.

Pour des informations sur la gestion des groupes de ressources, consultez la https://docs.microsoft.com/en-us/azure/azure-resource-manager/management/manage-resource-groups-portal[documentation Azure.]

[#_availability_set_unmanaged]
=== Groupe de disponibilité (non géré)

Nom ou ID d'un https://docs.microsoft.com/en-us/azure/virtual-machines/availability-set-overview[groupe de disponibilité] existant auquel ajouter la machine virtuelle.

[#_image]
=== Image

Le nom de l'image du système d'exploitation fourni en tant qu'identifiant de ressource ARM. Nécessite l'utilisation d'un disque géré.

[#_vm_size]
=== Taille de la VM :

Choisissez une taille pour chaque VM dans le pool de nœuds. Pour des détails sur chaque taille de VM, consultez https://azure.microsoft.com/en-us/pricing/details/virtual-machines/linux/[cette page.]

[#_advanced_options]
== Options avancées

[#_fault_domain_count]
=== Nombre de domaines de pannes

Les domaines de pannes définissent le groupe de machines virtuelles qui partagent une source d'alimentation et un commutateur réseau communs. Si le groupe de disponibilité a déjà été créé, le nombre de domaines de pannes sera ignoré.

Pour plus d'informations sur les domaines de pannes, https://docs.microsoft.com/en-us/azure/virtual-machines/availability-set-overview#how-do-availability-sets-work[voir ici].

[#_update_domain_count]
=== Nombre de domaines de mise à jour

Les domaines de mise à jour indiquent des groupes de machines virtuelles et de matériel physique sous-jacent qui peuvent être redémarrés en même temps. Si le groupe de disponibilité a déjà été créé, le nombre de domaines de mise à jour sera ignoré.

Pour plus d'informations sur les domaines de mise à jour, https://docs.microsoft.com/en-us/azure/virtual-machines/availability-set-overview#how-do-availability-sets-work[voir ici].

[#_purchase_plan]
=== Plan d'achat

Certaines images de machines virtuelles dans le Marketplace Azure nécessitent un plan. Le cas échéant, sélectionnez un plan d'achat, formaté comme `publisher:product:plan`, à utiliser avec l'image choisie.

[#_subnet]
=== Sous-réseau

Le nom du sous-réseau à utiliser lors de la création d'un nouveau VNet ou en faisant référence à un existant.

Par défaut{nbsp}: `docker-machine`

[#_subnet_prefix]
=== Préfixe de sous-réseau

Le préfixe d'adresse IP de sous-réseau à utiliser lors de la création d'un nouveau VNet au format CIDR.

Par défaut{nbsp}: `192.168.0.0/16`

[#_virtual_network]
=== Réseau virtuel

Le https://docs.microsoft.com/en-us/azure/virtual-network/virtual-networks-overview[réseau virtuel] à utiliser ou à créer si aucun n'existe. Formaté comme `[resourcegroup:]name`.

[#_public_ip_options]
=== Options d'adresse IP publique

[#_no_public_ip]
==== Pas d'adresse IP publique

Ne pas allouer d'adresse IP publique.

[#_static_public_ip]
==== Adresse IP publique statique

Allouer une adresse IP publique statique.

[#_use_private_ip]
=== Utiliser l'adresse IP privée

Utiliser une adresse IP privée statique.

[#_private_ip_address]
=== Adresse IP privée

Configurer une adresse IP privée statique à utiliser.

[#_network_security_group]
=== Groupe de sécurité réseau

Le https://docs.microsoft.com/en-us/azure/virtual-network/network-security-groups-overview[groupe de sécurité réseau] à utiliser. Tous les nœuds utilisant ce modèle utiliseront le groupe de sécurité réseau fourni. Si aucun groupe de sécurité réseau n'est fourni, un nouveau sera créé pour chaque nœud.

[#_dns_label]
=== Étiquette DNS

Une étiquette de nom DNS unique pour l'adresse IP publique.

[#_storage_type]
=== Type de stockage

Le type de https://docs.microsoft.com/en-us/azure/storage/common/storage-account-overview[compte de stockage] à utiliser avec vos machines virtuelles. Les options incluent Standard LRS, Standard ZRS, Standard GRS, Standard RAGRS et Premium LRS.

[#_use_managed_disks]
=== Utiliser des disques gérés

https://docs.microsoft.com/en-us/azure/virtual-machines/managed-disks-overview[Les disques gérés par Azure] sont des volumes de stockage au niveau des blocs qui sont gérés par Azure et utilisés avec les machines virtuelles Azure. Les disques gérés sont conçus pour une disponibilité de 99,999%. Les disques gérés atteignent cela en vous fournissant trois répliques de vos données, permettant une grande durabilité.

[#_managed_disk_size]
=== Taille du disque géré

La taille en Go pour le disque de chaque nœud.

[#_ssh_username]
=== Nom d'utilisateur SSH

Le nom d'utilisateur utilisé pour créer une connexion SSH à vos nœuds.

[#_open_port]
=== Port ouvert

Ouvre le trafic entrant sur les ports spécifiés. Lors de l'utilisation d'un groupe de sécurité réseau existant, les ports ouverts sont ignorés.

Par défaut{nbsp}: `2379/tcp, 2380/tcp, 6443/tcp, 9796/tcp, 10250/tcp, 10251/tcp, 10252/tcp, 10256/tcp` et `8472/udp, 4789/udp`
