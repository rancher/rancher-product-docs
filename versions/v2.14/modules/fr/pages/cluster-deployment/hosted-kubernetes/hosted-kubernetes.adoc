ifeval::["{build-type}" != "srfa"]
Dans ce scénario, Rancher ne provisionne pas Kubernetes car il est installé par des fournisseurs tels que Google Kubernetes Engine (GKE), Amazon Elastic Container Service for Kubernetes ou Azure Kubernetes Service.

Si vous utilisez un fournisseur Kubernetes tel que Google GKE, Rancher s'intègre à ses API cloud, vous permettant de créer et de gérer le contrôle d'accès en fonction du rôle pour le cluster hébergé depuis l'interface utilisateur de Rancher.

Dans ce cas d'utilisation, Rancher envoie une demande à un fournisseur hébergé en utilisant l'API du fournisseur. Le fournisseur provisionne ensuite et héberge le cluster pour vous. Lorsque le cluster a terminé sa construction, vous pouvez le gérer depuis l'interface utilisateur de Rancher, ainsi que les clusters que vous avez provisionnés et qui sont hébergés sur site ou chez un fournisseur d'infrastructure.

Rancher prend en charge les fournisseurs Kubernetes suivants{nbsp}:

* https://cloud.google.com/kubernetes-engine/[Google GKE (Google Kubernetes Engine)]
* https://aws.amazon.com/eks/[Amazon EKS (Amazon Elastic Container Service for Kubernetes)]
* https://azure.microsoft.com/en-us/services/kubernetes-service/[Microsoft AKS (Azure Kubernetes Service)]

endif::[]

[#_hosted_kubernetes_provider_authentication]
== Authentification du fournisseur Kubernetes hébergé

ifeval::["{build-type}" != "srfa"]
Lorsque vous utilisez Rancher pour créer un cluster hébergé par un fournisseur, vous êtes invité à fournir des informations d'authentification. Ces informations sont nécessaires pour accéder à l'API du fournisseur. Pour plus d'informations sur la façon d'obtenir ces informations, consultez les procédures suivantes :

* {xref-deploy-gke}[Création d'un cluster GKE]
* {xref-deploy-eks}[Création d'un cluster EKS]
* {xref-deploy-aks}[Création d'un cluster AKS]
endif::[]
