Si vous utilisez des machines virtuelles Azure pour vos nœuds, vous pouvez utiliser https://docs.microsoft.com/en-us/azure/aks/azure-files-dynamic-pv[Fichiers Azure] comme classe de stockage pour le cluster.

Pour que la plateforme Azure crée les ressources de stockage requises, suivez ces étapes{nbsp}:

. {xref-azure-cloud-provider}[Configurer le fournisseur cloud Azure.]
. Configurer `kubectl` pour se connecter à votre cluster.
. Copiez le manifeste `ClusterRole` et `ClusterRoleBinding` pour le compte de service{nbsp}: +
  ```yml +
  -- +
  apiVersion: rbac.authorization.k8s.io/v1 +
  kind: ClusterRole +
  metadata: +
    name: system:azure-cloud-provider +
  rules:
 ** apiGroups: [''] +
resources: ['secrets'] +
verbs:     ['get','create'] +
  -- +
  apiVersion: rbac.authorization.k8s.io/v1 +
  kind: ClusterRoleBinding +
  metadata: +
name: system:azure-cloud-provider +
  roleRef: +
kind: ClusterRole +
apiGroup: rbac.authorization.k8s.io +
name: system:azure-cloud-provider +
  subjects:
 ** kind: ServiceAccount +
name: persistent-volume-binder +
namespace: kube-system +
  ```
. Créez-les dans votre cluster en utilisant l'une des commandes suivantes.
+
----
# kubectl create -f <MANIFEST>
----
