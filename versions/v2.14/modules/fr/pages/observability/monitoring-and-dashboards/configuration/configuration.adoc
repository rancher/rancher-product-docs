Cette page présente certaines des options les plus importantes pour configurer Monitoring V2 dans la GUI de Rancher.

Pour des informations sur la configuration des cibles de récupération et des règles personnalisées pour Prometheus, veuillez vous référer à la documentation en amont pour le https://github.com/prometheus-operator/prometheus-operator[Prometheus Operator.] Certaines des ressources personnalisées les plus importantes sont expliquées dans la https://github.com/prometheus-operator/prometheus-operator/blob/main/Documentation/getting-started/design.md[documentation de conception du Prometheus Operator.] La documentation du Prometheus Operator peut également vous aider à configurer RBAC, Thanos ou une configuration personnalisée.

[#_setting_resource_limits_and_requests]
== Configuration des limites et des demandes de ressources

Les demandes et limites de ressources pour l'application Monitoring peuvent être configurées lors de l'installation de `rancher-monitoring`. Pour plus d'informations sur les limites par défaut, voir {xref-helm-chart-options}#_configuring_resource_limits_and_requests[cette page.]

[TIP]
====

Sur un cluster inactif, Monitoring peut avoir une utilisation élevée de l'UC. Pour améliorer les performances, désactivez l'adaptateur Prometheus.
====


[#_prometheus_configuration]
== Configuration de Prometheus

Il n'est généralement pas nécessaire de modifier directement la ressource personnalisée de Prometheus.

Au lieu de cela, pour que Prometheus récupère des métriques personnalisées, il vous suffira de créer un nouveau ServiceMonitor ou PodMonitor pour récupérer des métriques supplémentaires.

[#_alertmanager_configuration]
== Configuration d'Alertmanager

La ressource personnalisée d'Alertmanager n'a généralement pas besoin d'être modifiée directement. Pour les cas d'utilisation les plus courants, vous pouvez gérer les alertes en mettant à jour les Routes et les Récepteurs.

Les Routes et les Récepteurs font partie de la configuration de la ressource personnalisée d'Alertmanager. Dans l'interface utilisateur de Rancher, les Routes et les Récepteurs ne sont pas de vraies ressources personnalisées, mais des pseudo-ressources personnalisées que le Prometheus Operator utilise pour synchroniser votre configuration avec la ressource personnalisée d'Alertmanager. Lorsque les Routes et les Récepteurs sont mis à jour, l'application Monitoring mettra automatiquement à jour Alertmanager pour refléter ces changements.

Pour certains cas d'utilisation avancés, vous souhaiterez peut-être configurer Alertmanager directement. Pour plus d'informations, reportez-vous à {xref-configure-alertmanager}[cette page.]

[#_receivers]
=== Récepteurs

Les Récepteurs sont utilisés pour configurer des notifications. Pour des détails sur la façon de configurer les Récepteurs, consultez {xref-receivers}[cette page.]

[#_routes]
=== Routes

Les Routes filtrent les notifications avant qu'elles n'atteignent les Récepteurs. Chaque Route doit faire référence à un Récepteur qui a déjà été configuré. Pour des détails sur la façon de configurer les Routes, consultez {xref-monitoring-routes}[cette page.]

=== Avancé

Pour plus d'informations sur l'édition directe de la ressource personnalisée d'Alertmanager, qui peut être utile dans des cas d'utilisation avancés, consultez {xref-configure-alertmanager}[cette page.]
