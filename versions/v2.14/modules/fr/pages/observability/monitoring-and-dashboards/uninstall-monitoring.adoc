= Désinstaller le monitoring
ifeval::["{build-type}" != "srfa"]
:page-languages: [en, zh]
endif::[]
:revdate: 2025-08-05
:page-revdate: {revdate}
:community-path: how-to-guides/advanced-user-guides/monitoring-alerting-guides/uninstall-monitoring.adoc 
:product-path: observability/monitoring-and-dashboards/uninstall-monitoring.adoc

. Cliquez sur *☰ > Gestion des clusters*.
. Allez au cluster que vous avez créé et cliquez sur *Explorer*.
. Dans la barre de navigation à gauche, cliquez sur *Applis*.
. Cliquez sur *Applis installées*.
. Allez dans l'espace de noms `cattle-monitoring-system` et cochez les cases pour `rancher-monitoring-crd` et `rancher-monitoring`.
. Cliquez sur *Supprimer*.
. Confirmez *Supprimer*.

*Résultat\&nbsp;:* `rancher-monitoring` est désinstallé.

[NOTE]
.Tableaux de bord Grafana persistants :
====

Pour les utilisateurs qui utilisent Monitoring V2 v9.4.203 ou une version inférieure, la désinstallation du graphique de surveillance supprimera l'espace de noms cattle-dashboards, ce qui supprimera tous les tableaux de bord persistants, à moins que l'espace de noms ne soit marqué avec l'annotation `helm.sh/resource-policy: "keep"`. Cette annotation est ajoutée par défaut dans Monitoring V2 v14.5.100+ mais peut être appliquée manuellement sur l'espace de noms cattle-dashboards avant une désinstallation si une version antérieure du graphique de surveillance est actuellement installée sur votre cluster.
====
