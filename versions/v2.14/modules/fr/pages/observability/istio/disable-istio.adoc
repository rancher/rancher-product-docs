[WARNING]
====
https://github.com/rancher/charts/tree/release-v2.11/charts/rancher-istio[Rancher-Istio] a cessé la prise en charge depuis Rancher v2.12.0{nbsp}; tournez-vous vers l'https://apps.rancher.io[SUSE Application Collection] d'Istio pour une sécurité renforcée (incluse dans les abonnements SUSE Rancher Prime).
Des informations détaillées peuvent être trouvées dans https://forums.suse.com/t/deprecation-of-rancher-istio/45043[cette annonce].
====

Cette section décrit comment désinstaller Istio dans un cluster ou désactiver un espace de noms ou une charge de travail.

[#_uninstall_istio_in_a_cluster]
== Désinstaller Istio dans un cluster

Pour désinstaller Istio,

. Cliquez sur *☰ > Gestion des clusters*.
. Allez au cluster que vous avez créé et cliquez sur *Explorer*.
. Dans la barre de navigation à gauche, cliquez sur menu:Applis[Applis installées].
. Dans l'espace de noms `istio-system`, allez à `rancher-istio` et cliquez sur *⋮ > Supprimer*.
. Après que `rancher-istio` soit supprimé, vous pouvez ensuite sélectionner toutes les applications restantes dans l'espace de noms `istio-system` et cliquer sur *Supprimer*.

*Résultat :* L'appli `rancher-istio` dans le cluster est supprimée. Le sidecar Istio ne peut pas être déployé sur des charges de travail dans le cluster.

[NOTE]
====

Vous ne pouvez plus désactiver et réactiver votre installation d'Istio. Si vous souhaitez enregistrer vos paramètres pour une installation future, visualisez et enregistrez les fichiers YAML individuels pour vous y référer / les réutiliser pour de futures installations.
====


*Dépannage de la désinstallation :* Si vous n'avez pas suivi les étapes de désinstallation, vous pourriez rencontrer un avertissement lors de la désinstallation{nbsp}:

`Error: uninstallation completed with 1 error(s): unable to build kubernetes objects for delete: unable to recognize "": no matches for kind "MonitoringDashboard" in version "monitoring.kiali.io/v1alpha1"`

Cela pourrait signifier plusieurs choses. Vous avez soit sélectionné toutes les applications dans l'espace de noms `istio-system` et les avez supprimées en même temps, soit vous avez supprimé les dépendances de chartes `rancher-istio` avant de supprimer la charte `rancher-istio`. Puisque la désinstallation ne s'est pas terminée correctement, vous aurez des ressources restantes dans l'espace de noms `istio-system` que vous devrez nettoyer manuellement. Une autre option pour éviter le nettoyage manuel est de réinstaller `rancher-istio`, puis de le désinstaller dans le bon ordre.

[#_disable_istio_in_a_namespace]
== Désactiver Istio dans un espace de noms

. Cliquez sur *☰ > Gestion des clusters*.
. Allez dans le cluster que vous avez créé et cliquez sur *Explorer*.
. Cliquez sur menu:Cluster[Projets/Espaces de noms].
. Allez dans l'espace de noms où vous souhaitez activer Istio et cliquez sur *⋮  > Activer l'injection automatique d'Istio*. Alternativement, cliquez sur l'espace de noms, puis sur la page de détails de l'espace de noms, cliquez sur *⋮  > Activer l'injection automatique d'Istio*.

*Résultat :* Lorsque les charges de travail sont déployées dans cet espace de noms, elles n'auront pas le sidecar Istio.

[#_remove_the_istio_sidecar_from_a_workload]
== Supprimez le sidecar Istio d'une charge de travail.

Désactivez Istio dans l'espace de noms, puis redéployez les charges de travail qui s'y trouvent. Elles seront déployées sans le sidecar Istio.
