[WARNING]
====
https://github.com/rancher/charts/tree/release-v2.11/charts/rancher-istio[Rancher-Istio] a cessé la prise en charge depuis Rancher v2.12.0{nbsp}; tournez-vous vers l'https://apps.rancher.io[SUSE Application Collection] d'Istio pour une sécurité renforcée (incluse dans les abonnements SUSE Rancher Prime).
Des informations détaillées peuvent être trouvées dans https://forums.suse.com/t/deprecation-of-rancher-istio/45043[cette annonce].
====

Lors de l'installation ou de la mise à niveau du chart Helm d'Istio via *Applis,*

. Si vous installez le chart, cliquez sur *Personnaliser les options Helm avant l'installation* et cliquez sur *Suivant*.
. Vous verrez des options pour configurer le chart Helm d'Istio. Dans l'onglet *Composants*, cochez la case à côté de *CNI activé*.
. Ajoutez un fichier de superposition personnalisé spécifiant `cniBinDir` et `cniConfDir`. Pour plus d'informations sur ces options, consultez la https://istio.io/latest/docs/setup/additional-setup/cni/#helm-chart-parameters[documentation d'Istio.] Un exemple est ci-dessous{nbsp}:

[tabs]
======
RKE2::
+
--
[,yaml]
----
apiVersion: install.istio.io/v1alpha1
kind: IstioOperator
spec:
  components:
    cni:
      enabled: true
      k8s:
        overlays:
        - apiVersion: "apps/v1"
          kind: "DaemonSet"
          name: "istio-cni-node"
          patches:
          - path: spec.template.spec.containers.[name:install-cni].securityContext.privileged
            value: true
  values:
    cni:
      cniBinDir: /opt/cni/bin
      cniConfDir: /etc/cni/net.d
----
--

K3s::
+
--
[,yaml]
----
apiVersion: install.istio.io/v1alpha1
kind: IstioOperator
spec:
  components:
    cni:
      enabled: true
      k8s:
        overlays:
        - apiVersion: "apps/v1"
          kind: "DaemonSet"
          name: "istio-cni-node"
          patches:
          - path: spec.template.spec.containers.[name:install-cni].securityContext.privileged
            value: true
  values:
    cni:
      cniBinDir: /var/lib/rancher/k3s/data/current/bin
      cniConfDir: /var/lib/rancher/k3s/agent/etc/cni/net.d
----
--
======

*Résultat :* Vous devriez maintenant être en mesure d'utiliser Istio comme souhaité, y compris l'injection de sidecar et la surveillance via Kiali.
