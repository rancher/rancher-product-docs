De nombreuses intégrations tierces disponibles pour Kubernetes, telles que GitLab et HashiCorp Vault, impliquent de donner à un processus externe l'accès à l'API Kubernetes en utilisant un token de compte de service Kubernetes natif pour l'authentification.

Dans Rancher v2.9.0 et versions ultérieures, les comptes de service sur les clusters en aval peuvent désormais s'authentifier via JSON Web Token (JWT) en utilisant le proxy d'authentification Rancher. Dans les versions de Rancher antérieures à v2.9.0, seuls les jetons émis par Rancher étaient pris en charge.

Pour activer cette fonctionnalité, suivez ces étapes{nbsp}:

. Dans le coin supérieur gauche, cliquez sur *☰ > Gestion des clusters*.
. Cliquez sur *Avancé* pour ouvrir le menu déroulant.
. Sélectionnez *Authentification JWT*.
. Cochez la case pour le cluster pour lequel vous souhaitez activer l'authentification JWT, puis cliquez sur *Activer*. Alternativement, vous pouvez cliquer sur *⋮* > *Activer*.
