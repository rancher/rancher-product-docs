[CAUTION]
====
L'agrégation de modèles de rôle est une fonctionnalité expérimentale dans la version 2.13 qui modifie l'architecture RBAC utilisée pour les modèles de rôle, les liaisons de modèles de rôle de cluster et les liaisons de modèles de rôle de projet. **Elle n'est pas prise en charge pour les environnements de production**. Des changements majeurs peuvent survenir entre la version 2.13 et la version 2.14.
====

L'agrégation de modèles de rôle implémente des modèles de rôle, des liaisons de modèles de rôle de cluster et des liaisons de modèles de rôle de projet en utilisant la fonctionnalité Kubernetes https://kubernetes.io/docs/reference/access-authn-authz/rbac/#aggregated-clusterroles[Rôles de cluster agrégés]. La nouvelle architecture entraîne une réduction nette des objets RBAC (Rôles, Liaisons de rôles, Rôles de cluster et Liaisons de rôles de cluster) tant dans le cluster Rancher que dans les clusters en aval.

Pour plus d'informations sur la façon dont cette fonctionnalité peut améliorer l'évolutivité et les performances, veuillez consulter le https://www.suse.com/c/rancher_blog/fewer-bindings-more-power-ranchers-rbac-boost-for-enhanced-performance-and-scalability/[post du blog Rancher].

|===
| Clé de variable d'environnement | Valeur par défaut | Description

| `aggregated-roletemplates`
| `false`
| [Bêta] Faire en sorte que les modèles de rôle utilisent l'agrégation pour les rôles RBAC générés.
|===

La valeur de ce drapeau de fonctionnalité est verrouillée lors de l'installation, ce qui apparaît dans l'interface utilisateur sous la forme d'un symbole de verrou à côté du drapeau de fonctionnalité. Cela signifie que la fonctionnalité ne peut être définie que lors de la toute première installation de Rancher. Après cela, toutes les tentatives de modification de la valeur sont refusées.
