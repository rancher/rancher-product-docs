[CAUTION]
====
Fonctionnement sur une plateforme d'architecture mixte ARM64 est actuellement une fonctionnalité expérimentale et n'est pas encore officiellement prise en charge dans Rancher. Par conséquent, nous ne recommandons pas d'utiliser des nœuds basés sur une architecture mixte ARM64 dans un environnement de production.
====

Les options suivantes sont disponibles lors de l'utilisation d'une plateforme ARM64{nbsp}:

* Créer un cluster personnalisé et ajouter des nœuds basés sur ARM64
 ** La version du cluster Kubernetes doit être 1.12 ou supérieure
* Importer des clusters contenant des nœuds basés sur ARM64
 ** La version du cluster Kubernetes doit être 1.12 ou supérieure

Selon votre provisionnement de cluster, référez-vous à {xref-rke2-config}[options de configuration du cluster RKE2] ou {xref-k3s-config}[options de configuration du cluster K3s] pour plus d'informations.

Les fonctionnalités suivantes ne sont pas testées{nbsp}:

* Surveillance, alertes, notificateurs, pipelines et journalisation
* Lancement d'applications depuis le catalogue
