Cette fonctionnalité vous permet d'utiliser des types pour les fournisseurs de stockage et les provisionneurs qui ne sont pas activés par défaut.

Pour activer ou désactiver cette fonctionnalité, reportez-vous aux instructions sur {xref-experimental-features}[la page principale concernant l'activation des fonctionnalités expérimentales.]

|===
| Clé de variable d'environnement | Valeur par défaut | Description

| `unsupported-storage-drivers`
| `false`
| Cette fonctionnalité active des types pour les fournisseurs de stockage et les provisionneurs qui ne sont pas activés par défaut.
|===

[#_types_for_persistent_volume_plugins_that_are_enabled_by_default]
== Types pour les plugins de volume persistant qui sont activés par défaut

Voici une liste des types de stockage pour les plugins de volume persistant qui sont activés par défaut. Lors de l'activation de ce drapeau de fonctionnalité, tous les plugins de volume persistant qui ne figurent pas sur cette liste sont considérés comme expérimentaux et non pris en charge{nbsp}:

|===
| Nom | Plugin

| Disque Amazon EBS
| `aws-ebs`

| AzureFile
| `azure-file`

| AzureDisk
| `azure-disk`

| Disque persistant Google
| `gce-pd`

| Volume VMware vSphere
| `vsphere-volume`

| Local
| `local`

| NFS (Network File System)
| `nfs`

| hostPath
| `host-path`
|===
