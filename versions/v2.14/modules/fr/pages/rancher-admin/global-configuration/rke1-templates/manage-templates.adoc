Cette section décrit comment gérer les modèles RKE et les révisions. Vous pouvez créer, partager, mettre à jour et supprimer des modèles depuis la vue *Gestion des clusters* sous menu:Configuration RKE1[Modèles RKE].

Les mises à jour des modèles sont gérées par un système de révision. Lorsque les propriétaires de modèles souhaitent modifier ou mettre à jour un modèle, ils créent une nouvelle révision du modèle. Les révisions individuelles ne peuvent pas être modifiées. Cependant, si vous souhaitez empêcher une révision d'être utilisée pour créer un nouveau cluster, vous pouvez la désactiver.

Les révisions de modèles peuvent être utilisées de deux manières{nbsp}: pour créer un nouveau cluster ou pour mettre à niveau un cluster qui a été créé avec une version antérieure du modèle. Le créateur du modèle peut choisir une révision par défaut, mais lorsque les utilisateurs finaux créent un cluster, ils peuvent choisir n'importe quel modèle et n'importe quelle révision de modèle qui leur est disponible. Après la création du cluster à partir d'une révision spécifique, il ne peut pas changer de modèle, mais le cluster peut être mis à niveau vers une révision plus récente disponible du même modèle.

Le propriétaire du modèle a un contrôle total sur les révisions de modèles, et peut créer de nouvelles révisions pour mettre à jour le modèle, supprimer ou désactiver des révisions qui ne devraient pas être utilisées pour créer des clusters, et choisir quelle révision de modèle est par défaut.

[#_prerequisites]
== Conditions préalables

Vous pouvez créer des modèles RKE si vous avez la permission *Créer des modèles RKE*, qui peut être {xref-rke1-creator-permissions}[donnée par un administrateur.]

Vous pouvez réviser, partager et supprimer un modèle si vous en êtes le propriétaire. Pour des détails sur la façon de devenir propriétaire d'un modèle, référez-vous à xref:./access-or-share-templates.adoc#_sharing_ownership_of_templates[la documentation sur le partage de la propriété des modèles.]

[#_creating_a_template]
== Création d'un modèle

. Dans le coin supérieur gauche, cliquez sur *☰ > Gestion des clusters*.
. Cliquez sur menu:Configuration RKE1[Modèles de nœuds].
. Cliquez sur *Ajouter un modèle*.
. Indiquez le nom du modèle. Un nom auto-généré est déjà fourni pour la première version du modèle, qui est créé avec ce modèle.
. Facultatives : Partagez le modèle avec d'autres utilisateurs ou groupes en xref:./access-or-share-templates.adoc#_sharing_templates_with_specific_users_or_groups[les ajoutant en tant que membres.] Vous pouvez également rendre le modèle public pour le partager avec tout le monde dans la configuration de Rancher.
. Ensuite, suivez le formulaire à l'écran pour enregistrer les paramètres de configuration du cluster dans la révision du modèle. La révision peut être marquée comme par défaut pour ce modèle.

*Résultat :* Un modèle RKE avec une révision est configuré. Vous pouvez utiliser cette révision de modèle RKE plus tard lorsque vous {xref-launch-k8s-with-rancher}[provisionnez un cluster lancé par Rancher]. Après qu'un cluster soit géré par un modèle RKE, il ne peut pas être déconnecté et l'option de décocher *Utiliser un modèle RKE et une révision existants* sera indisponible.

[#_updating_a_template]
== Mise à jour d'un modèle

Lorsque vous mettez à jour un modèle RKE, vous créez une révision du modèle existant. Les clusters qui ont été créés avec une version antérieure du modèle peuvent être mis à jour pour correspondre à la nouvelle révision.

Vous ne pouvez pas modifier des révisions individuelles. Puisque vous ne pouvez pas modifier des révisions individuelles d'un modèle, pour empêcher une révision d'être utilisée, vous pouvez <<_disabling_a_template_revision,la désactiver.>>

Lorsque de nouvelles révisions de modèle sont créées, les clusters utilisant une révision plus ancienne du modèle ne sont pas affectés.

. Dans le coin supérieur gauche, cliquez sur *☰ > Gestion des clusters*.
. Dans le menu de navigation à gauche, cliquez sur menu:Configuration RKE1[Modèles RKE].
. Allez au modèle que vous souhaitez modifier et cliquez sur *⋮ > Modifier*.
. Modifiez les informations requises et cliquez sur *Enregistrer*.
. Facultatives : Vous pouvez changer la révision par défaut de ce modèle et également changer avec qui il est partagé.

*Résultat :* Le modèle est mis à jour. Pour l'appliquer à un cluster utilisant une version plus ancienne du modèle, référez-vous à la section sur <<_upgrading_a_cluster_to_use_a_new_template_revision,la mise à niveau d'un cluster pour utiliser une nouvelle révision d'un modèle.>>

[#_deleting_a_template]
== Suppression d'un modèle

Lorsque vous n'utilisez plus un modèle RKE pour aucun de vos clusters, vous pouvez le supprimer.

. Dans le coin supérieur gauche, cliquez sur *☰ > Gestion des clusters*.
. Cliquez sur menu:Configuration RKE1[Modèles RKE].
. Allez au modèle RKE que vous souhaitez supprimer et cliquez sur *⋮ > Supprimer*.
. Confirmez la suppression.

*Résultat :* Le modèle est supprimé.

[#_creating_a_revision_based_on_the_default_revision]
== Création d'une révision basée sur la révision par défaut.

Vous pouvez cloner la révision du modèle par défaut et mettre à jour rapidement ses paramètres plutôt que de créer une nouvelle révision à partir de zéro. Cloner des modèles vous évite le tracas de ressaisir les clés d'accès et d'autres paramètres nécessaires à la création de clusters.

. Dans le coin supérieur gauche, cliquez sur *☰ > Gestion des clusters*.
. Dans le menu de navigation à gauche, cliquez sur menu:Configuration RKE1[Modèles RKE].
. Allez au modèle RKE que vous souhaitez cloner et cliquez sur *⋮ > Nouvelle révision à partir de la révision par défaut*.
. Complétez le reste du formulaire pour créer une nouvelle révision.

*Résultat :* La révision du modèle RKE est clonée et configurée.

[#_creating_a_revision_based_on_a_cloned_revision]
== Création d'une révision basée sur une révision clonée.

Lors de la création de nouvelles révisions de modèles RKE à partir de vos paramètres utilisateur, vous pouvez cloner une révision existante et mettre à jour rapidement ses paramètres plutôt que de créer une nouvelle à partir de zéro. Cloner des révisions de modèles vous évite le tracas de ressaisir les paramètres du cluster.

. Dans le coin supérieur gauche, cliquez sur *☰ > Gestion des clusters*.
. Sous *Configuration RKE1*, cliquez sur *Modèles RKE*.
. Allez à la révision du modèle que vous souhaitez cloner. Puis sélectionnez *⋮ > Cloner la révision*.
. Complétez le reste du formulaire.

*Résultat :* La révision du modèle RKE est clonée et configurée. Vous pouvez utiliser la révision du modèle RKE plus tard lorsque vous provisionnez un cluster. Tout cluster existant utilisant ce modèle RKE peut être mis à niveau vers cette nouvelle révision.

[#_disabling_a_template_revision]
== Désactivation d'une révision de modèle

Lorsque vous ne souhaitez plus qu'une révision de modèle RKE soit utilisée pour créer de nouveaux clusters, vous pouvez la désactiver. Une révision désactivée peut être réactivée.

Vous pouvez désactiver la révision si elle n'est utilisée par aucun cluster.

. Dans le coin supérieur gauche, cliquez sur *☰ > Gestion des clusters*.
. Dans le menu de navigation à gauche, cliquez sur menu:Configuration RKE1[Modèles RKE].
. Allez à la révision du modèle que vous souhaitez désactiver. Puis sélectionnez *⋮ > Désactiver*.

*Résultat :* La révision du modèle RKE ne peut pas être utilisée pour créer un nouveau cluster.

[#_re_enabling_a_disabled_template_revision]
== Réactivation d'une révision de modèle désactivée.

Si vous décidez qu'une révision de modèle RKE désactivée doit être utilisée pour créer de nouveaux clusters, vous pouvez la réactiver.

. Dans le coin supérieur gauche, cliquez sur *☰ > Gestion des clusters*.
. Sous *Configuration RKE1*, cliquez sur *Modèles RKE*.
. Allez à la révision du modèle que vous souhaitez réactiver. Puis sélectionnez *⋮ > Activer*.

*Résultat :* La révision du modèle RKE peut être utilisée pour créer un nouveau cluster.

[#_setting_a_template_revision_as_default]
== Définir une révision de modèle par défaut

Lorsque les utilisateurs finaux créent un cluster en utilisant un modèle RKE, ils peuvent choisir quelle révision utiliser pour créer le cluster. Vous pouvez configurer quelle révision est utilisée par défaut.

Pour définir une révision de modèle RKE comme par défaut,

. Dans le coin supérieur gauche, cliquez sur *☰ > Gestion des clusters*.
. Dans le menu de navigation à gauche, cliquez sur menu:Configuration RKE1[Modèles RKE].
. Allez à la révision du modèle RKE qui doit être par défaut et cliquez sur *⋮ > Définir comme par défaut*.

*Résultat :* La révision du modèle RKE sera utilisée comme option par défaut lors de la création de clusters avec le modèle.

[#_deleting_a_template_revision]
== Suppression d'une révision de modèle

Vous pouvez supprimer toutes les révisions d'un modèle sauf la révision par défaut.

Pour supprimer définitivement une révision,

. Dans le coin supérieur gauche, cliquez sur *☰ > Gestion des clusters*.
. Dans le menu de navigation à gauche, cliquez sur menu:Configuration RKE1[Modèles RKE].
. Allez à la révision du modèle RKE qui doit être supprimée et cliquez sur *⋮ > Supprimer*.

*Résultat :* La révision du modèle RKE est supprimée.

[#_upgrading_a_cluster_to_use_a_new_template_revision]
== Mise à niveau d'un cluster pour utiliser une nouvelle révision de modèle

[NOTE]
====

Cette section suppose que vous avez déjà un cluster qui {xref-rke1-template-apply}[a un modèle RKE appliqué.]

Cette section suppose également que vous avez <<_updating_a_template,mis à jour le modèle que le cluster utilise>> afin qu'une nouvelle révision de modèle soit disponible.
====


Pour mettre à niveau un cluster afin d'utiliser une nouvelle révision de modèle,

. Dans le coin supérieur gauche, cliquez sur *☰ > Gestion des clusters*.
. Allez au cluster que vous souhaitez mettre à niveau et cliquez sur *⋮ > Modifier la configuration*.
. Dans la section *Options du cluster*, cliquez sur le menu déroulant pour la révision du modèle, puis sélectionnez la nouvelle révision de modèle.
. Cliquez sur *Enregistrer*.

*Résultat :* Le cluster est mis à niveau pour utiliser les paramètres définis dans la nouvelle révision de modèle.

[#_exporting_a_running_cluster_to_a_new_rke_template_and_revision]
== Exporter un cluster en cours d'exécution vers un nouveau modèle et une nouvelle révision RKE

Vous pouvez enregistrer les paramètres d'un cluster existant en tant que modèle RKE.

Cela exporte les paramètres du cluster en tant que nouveau modèle RKE et lie également le cluster à ce modèle. Le résultat est que le cluster ne peut être modifié que si le <<_updating_a_template,modèle est mis à jour,>> et le cluster est mis à niveau pour <<_upgrading_a_cluster_to_use_a_new_template_revision,utiliser une version plus récente du modèle>>.

Pour convertir un cluster existant pour utiliser un modèle RKE,

. Dans le coin supérieur gauche, cliquez sur *☰ > Gestion des clusters*.
. Allez au cluster qui sera converti pour utiliser un modèle RKE et *⋮ > Enregistrer en tant que modèle RKE*.
. Entrez un nom pour le modèle RKE dans le formulaire qui apparaît, puis cliquez sur *Créer*.

*Résultats :*

* Un nouveau modèle RKE est créé.
* Le cluster est converti pour utiliser le nouveau modèle.
* De nouveaux clusters peuvent être xref:./apply-templates.adoc#_creating_a_cluster_from_an_rke_template[créés à partir du nouveau modèle et de la nouvelle révision.]
