La page de création de restauration vous permet de fournir des détails sur la sauvegarde à restaurer

image::backup_restore/restore/restore.png[]

[#_backup_source]
== Source de sauvegarde

Fournissez les détails du fichier de sauvegarde et de son emplacement de stockage, que l'opérateur utilisera ensuite pour effectuer la restauration. Sélectionnez parmi les options suivantes pour fournir ces détails

[#_an_existing_backup_config]
=== Une configuration de sauvegarde existante

Sélectionner cette option remplira le menu déroulant *Sauvegarde cible* avec les sauvegardes disponibles dans ce cluster. Sélectionnez la sauvegarde dans le menu déroulant, cela remplira le champ *Nom du fichier de sauvegarde* pour vous, et transmettra également les informations de source de sauvegarde de la sauvegarde sélectionnée à l'opérateur.

image::backup_restore/restore/existing.png[]

Si la ressource personnalisée de sauvegarde n'existe pas dans le cluster, vous devez obtenir le nom de fichier exact et fournir les détails de la source de sauvegarde avec la cible de stockage par défaut ou un stockage d'objets compatible S3.

[#_the_default_storage_target]
=== La cible de stockage par défaut

Sélectionnez cette option si vous restaurez à partir d'un fichier de sauvegarde qui existe dans l'emplacement de stockage par défaut configuré au niveau de l'opérateur. La configuration au niveau de l'opérateur est l'emplacement de stockage qui a été configuré lors de l'installation ou de la mise à niveau de `rancher-backup`l'opérateur. Fournissez le nom de fichier exact dans le champ *Nom du fichier de sauvegarde*.

image::backup_restore/restore/default.png[]

[#_an_s3_compatible_object_store]
=== Un stockage d'objets compatible S3

Sélectionnez cette option si aucun emplacement de stockage par défaut n'est configuré au niveau de l'opérateur, OU si le fichier de sauvegarde existe dans un autre compartiment S3 que celui configuré comme emplacement de stockage par défaut. Fournissez le nom de fichier exact dans le champ *Nom du fichier de sauvegarde*. Reportez-vous à <<_getting_the_backup_filename_from_s3,cette section>> pour les étapes exactes sur l'obtention du nom de fichier de sauvegarde depuis S3. Remplissez tous les détails pour le stockage d'objets compatible S3. Ses champs sont exactement les mêmes que ceux de la `backup.StorageLocation` configuration dans la {xref-rancher-backup-config}#_storage_location[ressource personnalisée de sauvegarde.]

image::backup_restore/restore/s3store.png[]

[#_encryption]
== Chiffrement

Si la sauvegarde a été créée avec le chiffrement activé, son fichier aura le suffixe `.enc`. Choisir une telle sauvegarde, ou fournir un nom de fichier de sauvegarde avec le suffixe `.enc`, affichera un autre menu déroulant nommé *Secret de configuration de chiffrement*.

image::backup_restore/restore/encryption.png[]

Le Secret sélectionné dans ce menu déroulant doit avoir le même contenu que celui utilisé pour la ressource personnalisée de sauvegarde lors de la réalisation de la sauvegarde. Si la configuration de chiffrement ne correspond pas, la restauration échouera.

Le menu déroulant `Encryption Config Secret` filtrera et listera uniquement les Secrets qui ont cette clé exacte.

|===
| Nom de la Directive YAML | Description

| `encryptionConfigSecretName`
| Fournissez le nom du Secret dans l'espace de noms `cattle-resources-system`, qui contient le fichier de configuration de chiffrement.
|===

[NOTE]
.Important :
====

Ce champ ne doit être défini que si la sauvegarde a été créée avec le chiffrement activé. Fournir une configuration de chiffrement incorrecte entraînera l'échec de la restauration.
====


[#_prune_during_restore]
== Nettoyer lors de la restauration.

* *Nettoyer*:  Pour restaurer complètement Rancher à partir d'une sauvegarde, et revenir à l'état exact dans lequel il se trouvait lors de la réalisation de la sauvegarde, nous devons supprimer toutes les ressources supplémentaires créées par Rancher après la prise de la sauvegarde. L'opérateur le fait si le drapeau *Prune* est activé. Prune est activé par défaut et il est recommandé de le laisser activé.
* *Délai de suppression*: C'est la durée pendant laquelle l'opérateur attendra lors de la suppression d'une ressource avant de modifier la ressource pour supprimer les finalizers et tenter à nouveau la suppression.

|===
| Nom de la Directive YAML | Description

| `prune`
| Supprimer les ressources gérées par Rancher qui ne sont pas présentes dans la sauvegarde (recommandé).

| `deleteTimeoutSeconds`
| Durée pendant laquelle l'opérateur attendra lors de la suppression d'une ressource avant de modifier la ressource pour supprimer les finalizers et tenter à nouveau la suppression.
|===

[#_getting_the_backup_filename_from_s3]
== Obtenir le nom de fichier de sauvegarde depuis S3.

C'est le nom du fichier de sauvegarde que `rancher-backup`l'opérateur utilisera pour effectuer la restauration.

Pour obtenir ce nom de fichier depuis S3, allez dans votre bucket S3 (et dossier si cela a été spécifié lors de la réalisation de la sauvegarde).

Copiez le nom de fichier et conservez-le dans votre ressource personnalisée de restauration. En supposant que le nom de votre fichier de sauvegarde soit `backupfile`,

* Si le nom de votre bucket est `s3bucket` et qu'aucun dossier n'a été spécifié, alors le `backupFilename` à utiliser sera `backupfile`.
* Si le nom de votre bucket est `s3bucket` et que le dossier de base est ``s3folder``, le `backupFilename` à utiliser est seulement `backupfile` .
* S'il y a un sous-dossier à l'intérieur de `s3Folder` appelé `s3sub`, et qu'il contient votre fichier de sauvegarde, alors le `backupFilename` à utiliser est `s3sub/backupfile`.

|===
| Nom de la Directive YAML | Description

| `backupFilename`
| C'est le nom du fichier de sauvegarde que `rancher-backup`l'opérateur utilisera pour effectuer la restauration.
|===
