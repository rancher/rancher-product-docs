Cette section traite de l'utilisation du Helm CLI pour installer le serveur Rancher dans un environnement isolé physiquement. Un environnement isolé physiquement pourrait être un endroit où le serveur Rancher sera installé hors ligne, derrière une passerelle de périmètre de sécurité ou un proxy.

Les étapes d'installation diffèrent selon que Rancher est installé sur un cluster Kubernetes K3s ou un conteneur Docker unique.

Pour plus d'informations sur chaque option d'installation, consultez {xref-install-upgrade}[cette page.]

Tout au long des instructions d'installation, il y aura _onglets_ pour chaque option d'installation.

[NOTE]
.Important :
====

Si vous installez Rancher en suivant le guide d'installation Docker, il n'y a pas de chemin de mise à niveau pour passer de votre installation Docker à une installation Kubernetes.
====

== Aperçu de l'installation

. {xref-air-gap-publish-images}[Collectez et publiez des images dans votre registre privé]
. {xref-air-gap-install-k8s}[Configurez un cluster Kubernetes (Ignorez cette étape pour les installations Docker)]
. {xref-air-gap-rancher-ha}[Installez Rancher]
