Ce tutoriel décrit comment configurer une base de données MySQL dans le service de base de données relationnelle (RDS) d'Amazon.

Cette base de données peut ensuite être utilisée comme un magasin de données externe pour un cluster Kubernetes K3s à haute disponibilité.

. Connectez-vous à la https://console.aws.amazon.com/rds/[Console Amazon AWS RDS] pour commencer. Assurez-vous de sélectionner la *Région* où vos instances EC2 (nœuds Linux) sont créées.
. Dans le panneau de gauche, cliquez sur *Bases de données*.
. Cliquez sur *Créer une base de données*.
. Dans la section *Type de moteur*, cliquez sur *MySQL*.
. Dans la section *Version*, choisissez *MySQL 5.7.22*.
. Dans la section *Paramètres*, sous *Paramètres d'identification,* entrez un mot de passe principal pour le nom d'utilisateur principal *admin*. Confirmez-le.
. Développez la section *Configuration supplémentaire*. Dans le champ *Nom de la base de données initiale*, entrez un nom. Le nom ne peut contenir que des lettres, des chiffres et des traits de soulignement. Ce nom sera utilisé pour se connecter à la base de données.
. Cliquez sur *Créer une base de données*.

Vous devrez capturer les informations suivantes sur la nouvelle base de données afin que le cluster Kubernetes K3s puisse s'y connecter.

Pour voir ces informations dans la console Amazon RDS, cliquez sur *Bases de données,* et cliquez sur le nom de la base de données que vous avez créée.

* *Nom d'utilisateur :* Utilisez le nom d'utilisateur admin.
* *Mot de passe :* Utilisez le mot de passe admin.
* *Nom d'hôte :* Utilisez le *Point de terminaison* comme nom d'hôte. Le point de terminaison est disponible dans la section *Connectivité et sécurité*.
* *Port :* Le port doit être 3306 par défaut. Vous pouvez le confirmer dans la section *Connectivité et sécurité*.
* *Nom de la base de données :* Confirmez le nom en allant dans l'onglet *Configuration*. Le nom est indiqué sous *Nom de la base de données*.

Cette information sera utilisée pour se connecter à la base de données dans le format suivant{nbsp}:

----
mysql://username:password@tcp(hostname:3306)/database-name
----

Pour plus d'informations sur la configuration du magasin de données pour K3s, consultez la https://rancher.com/docs/k3s/latest/en/installation/datastore/[documentation K3s.]
