Lorsque vous installez Rancher, vous pouvez définir un mot de passe de démarrage pour le premier compte administrateur.

Si vous choisissez de ne pas définir de mot de passe de démarrage, Rancher génère aléatoirement un mot de passe de démarrage pour le premier compte administrateur.

Pour des détails sur la façon de définir le mot de passe de démarrage, voir ci-dessous.

[#_password_requirements]
== Exigences de mot de passe

Le mot de passe de démarrage peut avoir n'importe quelle longueur.

Lorsque vous réinitialisez le mot de passe du premier compte administrateur après la première connexion, le nouveau mot de passe doit comporter au moins 12 caractères.

Vous pouvez {xref-users-and-groups}#_minimum_password_length[personnaliser la longueur minimale du mot de passe] pour les comptes utilisateurs, dans certaines limites.

La longueur minimale du mot de passe peut être n'importe quelle valeur entière positive entre 2 et 256. Les valeurs décimales et les zéros non significatifs ne sont pas autorisés.

[#_specifying_the_bootstrap_password]
== Spécification du mot de passe de démarrage

Lors de {xref-install-rancher}[l'installation de Rancher], définissez `bootstrapPassword` aux côtés de tous les autres indicateurs pour le chart Helm de Rancher. Par exemple :

[,bash]
----
helm install rancher rancher-<chart-repo>/rancher \
  --set bootstrapPassword=<password>
----

## Récupération du mot de passe de démarrage

Le mot de passe de démarrage est stocké dans les journaux du conteneur Docker. Après l'installation de Rancher, l'interface utilisateur affiche des instructions sur la façon de récupérer le mot de passe en fonction de votre méthode d'installation. 

[,bash]
----
kubectl get secret --namespace cattle-system bootstrap-secret -o go-template='{{ .data.bootstrapPassword|base64decode}}{{ "\n" }}'
----
