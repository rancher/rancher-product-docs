Après avoir installé l'adaptateur CSP, vous aurez la possibilité de générer un bundle de supportconfig. Ce bundle est un fichier tar qui peut être utilisé pour fournir rapidement des informations au support.

Ces bundles peuvent être créés via Rancher ou par un accès direct au cluster sur lequel Rancher est installé. Notez qu'il est préférable d'accéder via Rancher.

NOTE: Seuls les utilisateurs administrateurs peuvent générer/télécharger des bundles de supportconfig, quelle que soit la méthode.

[#_accessing_through_rancher]
== Accès via Rancher

Tout d'abord, cliquez sur le menu hamburger. Ensuite, cliquez sur le bouton `Get Support`.

image::support-help.png[Obtenir de l'assistance]

Sur la page suivante, cliquez sur le bouton `Generate Support Config`.

NOTE: Si l'adaptateur n'est pas installé, l'option de générer le bundle de supportconfig ne sera pas présente. Vous devez installer l'adaptateur CSP pour générer un bundle de supportconfig.

image::generate-support-config.png[Obtenir de l'assistance]

[#_accessing_without_rancher]
== Accès sans Rancher

Tout d'abord, générez un kubeconfig pour le cluster sur lequel Rancher est installé.

NOTE: Si Rancher est hors service, vous ne pourrez pas utiliser un jeton kubeconfig généré par Rancher pour accéder au cluster.

Configurez votre environnement shell pour utiliser ce token kubeconfig{nbsp}:

[,bash]
----
export KUBECONFIG=$MY_KUBECONFIG_PATH
----

Il est recommandé de créer un répertoire de travail temporaire lors de l'exécution de cette commande, comme ci-dessous{nbsp}:

[,bash]
----
mkdir temp && cd temp
----

Ensuite, récupérez le bundle de supportconfig{nbsp}:

[,bash]
----
mkdir rancher && kubectl get configmap csp-config -n cattle-csp-adapter-system -o=jsonpath='{.data.data}' >> rancher/config.json && tar -c -f supportconfig_rancher.tar rancher && rm -rf rancher
----

Cela créera un fichier `supportconfig_rancher.tar` dans votre répertoire actuel.

Les utilisateurs qui exécutent ces commandes sur Mac peuvent rencontrer des problèmes en raison d'incompatibilités entre gnu-tar et bsd-tar. Si le support a des problèmes pour lire un supportconfig que vous produisez, essayez d'utiliser la commande ci-dessous après avoir rendu gnu-tar accessible en tant que `gtar` dans votre chemin{nbsp}:

[,bash]
----
mkdir rancher && kubectl get configmap csp-config -n cattle-csp-adapter-system -o=jsonpath='{.data.data}' >> rancher/config.json && gtar -c -f supportconfig_rancher.tar rancher && rm -rf rancher
----
