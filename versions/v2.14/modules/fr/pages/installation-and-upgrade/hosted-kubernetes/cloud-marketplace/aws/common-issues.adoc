*Après avoir installé l'adaptateur, un message en bannière apparaît dans Rancher indiquant "Adaptateur AWS Marketplace{nbsp}: Impossible d'exécuter l'adaptateur, veuillez vérifier les journaux de l'adaptateur"*

Cette erreur indique que, bien que l'adaptateur ait été installé dans le cluster, une erreur s'est produite qui l'empêche d'enregistrer ou de désenregistrer correctement les licences.

Cela se produit souvent parce que le rôle IAM n'a pas été configuré correctement. Vérifiez les {xref-aws-cloud-marketplace-adapter-reqs}[prérequis] et assurez-vous que{nbsp}:

* Un fournisseur OIDC a été créé/associé au cluster sur lequel Rancher fonctionne.
* Le rôle IAM a été configuré pour faire confiance à ce fournisseur OIDC.
* Le rôle IAM dispose d'au moins les autorisations décrites dans la stratégie.

Si tout ce qui précède a été configuré correctement, contactez le support pour obtenir de l'aide.

*Je vois un message en bannière indiquant, "Adaptateur AWS Marketplace{nbsp}: Vous avez dépassé votre nombre de nœuds licenciés. Au moins x licence(s) supplémentaire(s) sont requises dans AWS pour être conforme"*

Ce message indique que vous n'avez pas suffisamment de droits pour le nombre de nœuds que Rancher gère actuellement.

Veuillez garder à l'esprit les limitations suivantes{nbsp}:

* Chaque droit est valide pour un certain nombre de nœuds.
* Chaque nœud actuellement géré par Rancher compte dans votre total d'utilisation (à l'exception des nœuds dans le cluster sur lequel Rancher est installé).
* Chaque droit peut être utilisé par au maximum une instance Rancher. Par exemple, si vous avez deux instances Rancher en cours d'exécution dans votre compte (chacune installée sur un cluster EKS séparé), vous aurez besoin d'au moins deux droits.

Vous avez peut-être également récemment désinstallé/réinstallé l'adaptateur. Si l'adaptateur perd la trace des licences qu'il gère actuellement, cela peut prendre jusqu'à une heure pour résoudre l'état réel des licences.
