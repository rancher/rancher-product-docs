[#_overview]
== Présentation

Rancher propose une intégration avec AWS Marketplace qui permet aux utilisateurs d'acheter un contrat de support avec SUSE. Cette intégration vous permet d'ajuster facilement vos besoins en support à mesure que vous commencez à prendre en charge davantage de clusters.

[#_limitations]
== limites

* Vous devez exécuter Rancher v2.6.7 ou une version supérieure
* Rancher doit être déployé avec des métriques supplémentaires activées.
* Rancher doit être installé sur un cluster EKS.
* Vous devez acheter au moins un droit à l'assistance Rancher via AWS Marketplace.
* Vous pourriez avoir besoin d'une configuration supplémentaire pour prendre en charge les cas d'utilisation de proxy/airgap. Voir les {xref-aws-cloud-marketplace-adapter-reqs}[prérequis] pour plus d'informations.

[#_how_to_use]
== Comment utiliser

. Complétez les {xref-aws-cloud-marketplace-adapter-reqs}[étapes prérequises].
. {xref-aws-cloud-marketplace-install-adapter}[Installez l'adaptateur CSP].

[#_faq]
== FAQ

*Puis-je acheter une assistance pour plus de nœuds plus tard\&nbsp;?*

Oui. Il vous suffit de vous rendre sur l'entrée AWS Marketplace que vous avez utilisée pour acheter initialement l'assistance et d'augmenter le nombre de droits.

*Puis-je utiliser plusieurs instances de Rancher dans le même compte AWS\&nbsp;?*

Oui. Cependant, chaque cluster dans lequel Rancher est installé devra respecter les prérequis.

De plus, gardez à l'esprit qu'un droit donné ne peut être utilisé que par un seul serveur de gestion Rancher à la fois.
