La majorité des logiciels Rancher 2.x s'exécutent sur le Serveur Rancher. Le Serveur Rancher comprend tous les composants logiciels utilisés pour gérer l'ensemble du déploiement Rancher.

Le diagramme ci-dessous illustre l'architecture de haut niveau de Rancher 2.x. La figure représente une installation de Serveur Rancher qui gère deux clusters Kubernetes en aval{nbsp}: l'un créé par RKE2 et l'autre créé par Amazon EKS (Elastic Kubernetes Service).

Pour la meilleure performance et sécurité, nous recommandons un cluster Kubernetes dédié pour le serveur de gestion Rancher. Il n'est pas conseillé d'exécuter des charges de travail utilisateur sur ce cluster. Après avoir déployé Rancher, vous pouvez {xref-cluster-deployment}[créer ou importer des clusters] pour exécuter vos charges de travail.

Le diagramme montre comment les utilisateurs peuvent manipuler à la fois les clusters Kubernetes {xref-launch-k8s-with-rancher}[lancés par Rancher] et les clusters Kubernetes {xref-hosted-k8s-clusters}[hébergés] via le proxy d'authentification de Rancher{nbsp}:

Vous pouvez installer Rancher sur un seul nœud, ou sur un cluster Kubernetes à haute disponibilité.

Une installation Kubernetes à haute disponibilité est recommandée pour la production.

Une installation de Rancher via Docker est recommandée uniquement à des fins de développement et de test. La capacité de migrer Rancher vers un cluster à haute disponibilité dépend de la version de Rancher.

Pour voir les ressources déployées, exécutez la commande suivante{nbsp}:

[,bash]
----
kubectl get all -n <namespace>
----

Si vous avez des privilèges d'administrateur, vous pouvez également voir ces ressources répertoriées dans l'interface utilisateur de Rancher{nbsp}:

. Cliquez *☰* et sélectionnez un cluster.
. Sélectionnez *Plus de ressources* dans le menu de navigation latéral pour afficher les ressources déployées par type.
. Sélectionnez *Cluster* > *Projets/Espaces de noms* dans le menu de navigation latéral, puis sélectionnez un espace de noms pour voir les ressources déployées par espace de noms.
L'opérateur de sauvegarde de Rancher peut être utilisé pour migrer Rancher de l'installation unique de conteneur Docker vers une installation sur un cluster Kubernetes à haute disponibilité. Pour plus de détails, consultez la documentation sur {xref-migrate-cluster}[la migration de Rancher vers un nouveau cluster].

Le serveur Rancher, quelle que soit la méthode d'installation, doit toujours fonctionner sur des nœuds séparés des clusters utilisateurs en aval qu'il gère. Si Rancher est installé sur un cluster Kubernetes à haute disponibilité, il doit fonctionner sur un cluster séparé de celui ou ceux qu'il gère.
