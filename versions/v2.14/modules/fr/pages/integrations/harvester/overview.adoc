Introduit dans Rancher v2.6.1, https://documentation.suse.com/cloudnative/virtualization/{harvester-docs-version}/en/introduction/overview.html[{harvester-product-name}] est un logiciel d'infrastructure hyper-convergée (HCI) Open Source construit sur Kubernetes. Harvester s'installe sur des serveurs à matériel sans système d'exploitation et fournit des fonctionnalités intégrées de virtualisation et de stockage distribué. Bien que Harvester fonctionne avec Kubernetes, il n'est pas nécessaire que les utilisateurs connaissent les concepts de Kubernetes, ce qui en fait une application plus conviviale.

[#_feature_flag]
== Drapeau de fonctionnalité

Le drapeau de fonctionnalité Harvester est utilisé pour gérer l'accès à la page de gestion de la virtualisation (VM) dans Rancher où les utilisateurs peuvent naviguer directement vers les clusters Harvester et accéder à l'interface utilisateur de Harvester. Le drapeau de fonctionnalité Harvester est activé par défaut. Cliquez {xref-experimental-features}[ici] pour plus d'informations sur les drapeaux de fonctionnalité dans Rancher.

Pour naviguer vers le cluster Harvester, cliquez sur *☰ > Gestion de la virtualisation*. Depuis la page des Clusters Harvester, cliquez sur l'un des clusters listés pour accéder à la vue d'un seul cluster Harvester.

* Si le drapeau de fonctionnalité Harvester est activé, les clusters Harvester seront filtrés de toutes les pages ou applications (comme la Livraison Continue avec Fleet) qui listent les clusters Kubernetes.
* Si le drapeau de fonctionnalité Harvester est désactivé et qu'un cluster Harvester est importé, le cluster Harvester sera affiché dans la liste des clusters Rancher sur la page de Gestion des Clusters. Les clusters Harvester ne seront affichés dans la liste des clusters que lorsque le drapeau de fonctionnalité est désactivé.
* Avec l'intégration de Harvester, les clusters Harvester peuvent désormais être importés dans Rancher en tant que type de cluster `Harvester`.
* Les utilisateurs peuvent importer un cluster Harvester uniquement sur la page de Gestion de la virtualisation. L'importation d'un cluster sur la page de Gestion des Clusters n'est pas prise en charge, et un avertissement vous conseillera de revenir à la page VM pour le faire.

[#__harvester_product_name_node_driver]
== {harvester-product-name} Pilote de nœud

Le https://documentation.suse.com/cloudnative/virtualization/{harvester-docs-version}/en/integrations/rancher/node-driver/node-driver.html[{harvester-product-name} pilote de nœud] est généralement disponible pour les options K3s et RKE2 dans Rancher. Le pilote de nœud est disponible que le drapeau de fonctionnalité Harvester soit activé ou non. Notez que le pilote de nœud est désactivé par défaut. Les utilisateurs peuvent créer des clusters K3s ou RKE2 sur Harvester uniquement à partir de la page de gestion des clusters.

Harvester permet de télécharger et d'afficher des images `.ISO` via l'interface utilisateur de Harvester, mais cela n'est pas pris en charge dans l'interface utilisateur de Rancher. Cela est dû au fait que les images `.ISO` nécessitent généralement une configuration supplémentaire qui interfère avec un déploiement propre (sans nécessiter d'intervention de l'utilisateur), et elles ne sont pas typiquement utilisées dans des environnements cloud.

Voir {xref-provisioning-drivers}#_node_drivers[Pilotes de provisionnement] pour plus d'informations sur les pilotes de nœud dans Rancher.

[#_port_requirements]
== Exigences de port

Les exigences de port pour le cluster Harvester peuvent être trouvées https://documentation.suse.com/cloudnative/virtualization/{harvester-docs-version}/en/installation-setup/requirements.html#_network_requirements[ici].

De plus, d'autres considérations réseau sont les suivantes:

* Assurez-vous d'activer les ports de tronc VLAN du commutateur physique pour les réseaux VLAN de VM.
* Suivez les conseils de configuration réseau https://documentation.suse.com/cloudnative/virtualization/{harvester-docs-version}/en/networking/cluster-network.html[ici].

Pour d'autres exigences de port pour d'autres clusters invités, tels que K3s et RKE2, veuillez consulter https://documentation.suse.com/cloudnative/virtualization/{harvester-docs-version}/en/installation-setup/requirements.html#_port_requirements_for_k3s_or_rkerke2_clusters[ces documents].
