Lorsqu'une ressource personnalisée ClusterScan est créée, elle lance une nouvelle analyse de conformité sur le cluster pour le ClusterScanProfile choisi.

[NOTE]
====
Il y a actuellement une limitation à l'exécution d'une seule analyse de conformité à la fois pour un cluster. Si vous créez plusieurs ressources personnalisées ClusterScan, elles seront exécutées les unes après les autres par l'opérateur, et tant qu'une analyse n'est pas terminée, le reste des ressources personnalisées ClusterScan sera dans l'état "En attente".
====


Pour exécuter une analyse,

. Dans le coin supérieur gauche, cliquez sur *☰ > Gestion des clusters*.
. Sur la page *Clusters*, allez au cluster où vous souhaitez exécuter une analyse de conformité et cliquez sur *Explorer*.
. Cliquez sur *Conformité > Analyse*.
. Cliquez sur *Create*.
. Choisissez un profil d'analyse de conformité. Le profil détermine quelle version du CIS Benchmark sera utilisée et quels tests seront effectués. Si vous choisissez le profil par défaut, alors l'opérateur de conformité choisira un profil applicable au type de cluster Kubernetes sur lequel il est installé.
. Cliquez sur *Create*.

*Résultat :* Un rapport est généré avec les résultats de l'analyse. Pour voir les résultats, cliquez sur le nom de l'analyse qui apparaît.
