Ingresses podem ser adicionados para cargas de trabalho para fornecer balanceamento de carga, terminação SSL e roteamento baseado em host/caminho. Ao usar ingresses em um projeto, você pode programar o nome do host do ingress para um DNS externo configurando uma entrada de DNS Global.

. No canto superior esquerdo, clique em *☰ > Gerenciamento de Cluster*.
. Vá para o cluster ao qual você deseja adicionar um ingress e clique em *Explorar*.
. Clique em menu:Descoberta de Serviços[Ingresses].
. Clique em *Criar*.
. Selecione um *Namespace* existente na lista suspensa.
. Insira um *Nome* para o ingress.
. {xref-ingress-configuration}[Criar] encaminhamento de ingress *Regras*. Se alguma das suas regras de ingress lidar com solicitações para portas criptografadas, adicione um certificado para criptografar/descriptografar comunicações.
. *Opcional:* clique em *Adicionar Regra* para criar regras de ingress adicionais. Por exemplo, após criar regras de ingress para direcionar solicitações para seu nome de host, você provavelmente desejará criar um backend padrão para lidar com 404s.
. Clique em *Criar* no canto inferior direito.

*Resultado:* Seu ingress é adicionado ao projeto. O ingress começa a aplicar suas regras de ingress.
