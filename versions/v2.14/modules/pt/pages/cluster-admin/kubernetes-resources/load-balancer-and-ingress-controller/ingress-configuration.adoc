[#_specify_a_hostname_to_use]
== Especifique um nome de host a ser usado

Se você usar esta opção, o Ingress roteia solicitações para um nome de host para o serviço ou carga de trabalho que você especificar.

. Especifique um caminho do tipo `Prefix` e especifique um caminho como `/`.
. Adicione um *Serviço de Destino*.
. *Opcional:* Se você quiser especificar uma carga de trabalho ou serviço quando uma solicitação for enviada para um caminho de nome de host específico, adicione um *Caminho* para o destino. Por exemplo, se você quiser que as solicitações para `www.mysite.com/contact-us` sejam enviadas para um serviço diferente de `www.mysite.com`, insira `/contact-us` no campo *Caminho*. Normalmente, a primeira regra que você cria não inclui um caminho.
. Insira o número da *Porta* em que cada destino opera.

[#_certificates]
== Certificados

[NOTE]
====

Você deve ter um certificado SSL que o Ingress possa usar para criptografar e descriptografar comunicações. Para mais informações, veja {xref-http-encrypt}[Adicionando Certificados SSL].
====


. Para criar um controlador de Ingress, clique na aba *Certificados*.
. Clique em *Adicionar Certificado*.
. Selecione um *Nome do Certificado - Segredo* na lista suspensa.
. Insira o host usando comunicação criptografada.
. Para adicionar mais hosts que utilizam o mesmo certificado, clique *Adicionar Hosts*.
