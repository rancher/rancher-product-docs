= Dimensionador Automático de Cluster
ifeval::["{build-type}" != "srfa"]
:page-languages: [en, zh]
endif::[]
:revdate: 2025-08-05
:page-revdate: {revdate}

Nesta seção, você aprenderá como instalar e usar o https://github.com/kubernetes/autoscaler/blob/master/cluster-autoscaler/[Kubernetes cluster-autoscaler] em clusters personalizados do Rancher usando Grupos de Auto Scaling do AWS EC2.

O Dimensionador Automático de Cluster é uma ferramenta que ajusta automaticamente o tamanho do cluster Kubernetes quando uma das seguintes condições é verdadeira:

* Existem pods que falharam ao rodar no cluster devido a recursos insuficientes.
* Existem nós no cluster que estão subutilizados por um período prolongado e seus pods podem ser alocados em outros nós existentes.

Para evitar que seu pod seja desalojado, defina uma `priorityClassName: system-cluster-critical` propriedade na especificação do seu pod.

O Dimensionador Automático de Cluster foi projetado para ser executado em nós mestres do Kubernetes. Ele pode ser executado no namespace `kube-system`. O Dimensionador Automático de Cluster não reduz o número de nós com pods `kube-system` não espelhados em execução neles.

É possível executar uma implantação personalizada do Dimensionador Automático de Cluster em nós de trabalho, mas é necessário ter cuidado extra para garantir que o Dimensionador Automático de Cluster permaneça em funcionamento.
