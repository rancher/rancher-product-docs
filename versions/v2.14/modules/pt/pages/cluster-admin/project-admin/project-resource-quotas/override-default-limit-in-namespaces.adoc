Embora o *Limite Padrão do namespace* se propague do projeto para cada namespace quando criado, em alguns casos, pode ser necessário aumentar (ou diminuir) as cotas para um namespace específico. Nessa situação, você pode substituir os limites padrão editando o namespace.

No diagrama abaixo, o administrador Rancher tem uma cota de recursos em vigor para seu projeto. No entanto, o administrador deseja substituir os limites de CPU para `Namespace 3` para que tenha mais recursos disponíveis. Portanto, o administrador {xref-projects-and-namespaces}[aumenta os limites de CPU] para `Namespace 3` para que o namespace possa acessar mais recursos.

== Editando Cotas de Recursos do namespace

Se houver uma cota de recursos configurada para um projeto, você pode substituir o limite padrão do namespace para fornecer a um namespace específico acesso a mais (ou menos) recursos do projeto.

. No canto superior esquerdo, clique em *☰ > Gerenciamento de Cluster*.
. Na página *Clusters*, vá para o cluster onde deseja editar uma cota de recursos do namespace e clique em *Explorar*.
. Clique em menu:Cluster[Projetos/Namespaces].
. Encontre o namespace para o qual deseja editar a cota de recursos. Clique em *⋮ > Editar Configuração*.
. Edite os limites de recursos.  Esses limites determinam os recursos disponíveis para o namespace. Os limites devem ser definidos dentro dos limites do projeto configurados.
+
Para mais informações sobre cada *Tipo de Recurso*, consulte {xref-resource-quota-types}[a referência do tipo].
+

[NOTE]
====

 ** Se uma cota de recursos não estiver configurada para o projeto, essas opções não estarão disponíveis.
 ** Se você inserir limites que excedam os limites do projeto configurados, o Rancher não permitirá que você salve suas edições.

+
====


*Resultado:* Sua substituição é aplicada à cota de recursos do namespace.
