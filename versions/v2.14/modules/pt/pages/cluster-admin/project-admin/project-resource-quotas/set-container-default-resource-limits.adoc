Ao definir cotas de recursos, se você definir qualquer coisa relacionada a CPU ou Memória (ou seja, limites ou reservas) em um projeto / namespace, todos os contêineres precisarão de um campo de CPU ou Memória correspondente definido durante a criação. Consulte a https://kubernetes.io/docs/concepts/policy/resource-quotas/#requests-vs-limits[documentação do Kubernetes] para mais detalhes sobre por que isso é necessário.

Para evitar definir esses limites em cada contêiner durante a criação da carga de trabalho, um limite de recursos padrão para contêineres pode ser especificado no namespace.

[#_editing_the_container_default_resource_limit]
== Editando o Limite de Recursos Padrão do Contêiner

Edite o limite de recursos padrão do contêiner quando:

* Você tem uma cota de recursos de CPU ou Memória definida em um projeto e deseja fornecer os valores padrão correspondentes para um contêiner.
* Você deseja editar o limite de recursos padrão do contêiner.

. No canto superior esquerdo, clique em *☰ > Gerenciamento de Cluster*.
. Na página *Clusters*, vá para o cluster onde deseja editar o limite de recursos padrão e clique em *Explorar*.
. Clique em menu:Cluster[Projetos/Namespaces].
. Encontre o projeto que você deseja editar o limite de recursos padrão do contêiner. Desse projeto, selecione *⋮ > Editar Config*.
. Expanda *Limite de Recursos Padrão do Contêiner* e edite os valores.

[#_resource_limit_propagation]
== Propagação do Limite de Recursos

Quando o limite de recursos padrão do contêiner é definido em um nível de projeto, o parâmetro será propagado para qualquer namespace criado no projeto após o limite ter sido definido. Para qualquer namespace existente em um projeto, esse limite não será automaticamente propagado. Você precisará definir manualmente o limite de recursos padrão do contêiner para quaisquer namespaces existentes no projeto para que ele seja utilizado ao criar quaisquer contêineres.

Você pode definir um limite de recursos padrão do contêiner em um projeto e lançar quaisquer aplicativos do catálogo.

Uma vez que um limite de recursos padrão do contêiner é configurado em um namespace, o padrão será preenchido automaticamente para quaisquer contêineres criados nesse namespace. Esses limites/reservas podem sempre ser substituídos durante a criação de cargas de trabalho.

[#_container_resource_quota_types]
== Tipos de Cota de Recursos do Contêiner

Os seguintes limites de recursos podem ser configurados:

|===
| Tipo de Recurso | Descrição

| CPU Limit
| A quantidade máxima de CPU (em https://kubernetes.io/docs/concepts/configuration/manage-compute-resources-container/#meaning-of-cpu[milicores]) alocada para o contêiner.

| Reserva de CPU
| A quantidade mínima de CPU (em milicores) garantida para o contêiner.

| Limite de Memória
| A quantidade máxima de memória (em bytes) alocada para o contêiner.

| Reserva de Memória
| A quantidade mínima de memória (em bytes) garantida para o contêiner.

| Limite/Reserva de NVIDIA GPU
| A quantidade de GPUs alocadas para o contêiner. O limite e a reserva são sempre os mesmos para GPUs.
|===
