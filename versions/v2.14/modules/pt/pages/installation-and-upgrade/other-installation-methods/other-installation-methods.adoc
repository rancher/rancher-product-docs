[#_air_gapped_installations]
== Instalações air-gapped

Siga {xref-air-gap}[estas etapas] para instalar o servidor Rancher em um ambiente air-gapped.

Um ambiente air-gapped pode ser onde o servidor Rancher será instalado offline, atrás de um firewall ou de um proxy.

[#_docker_installations]
== Instalações do Docker

A xref:[instalação Docker de nó único] é para usuários do Rancher que desejam testar o Rancher. Em vez de rodar em um cluster Kubernetes usando Helm, você instala o componente do servidor Rancher em um único nó usando um comando `docker run`.

A instalação do Docker é apenas para ambientes de desenvolvimento e teste.

Como há apenas um nó e um único contêiner Docker, se o nó falhar, não há cópia dos dados do etcd disponível em outros nós e você perderá todos os dados do seu servidor Rancher.

O operador de backup do Rancher pode ser usado para migrar o Rancher da instalação de contêiner Docker único para uma instalação em um cluster Kubernetes de alta disponibilidade. Para mais detalhes, consulte a documentação sobre {xref-migrate-cluster}[migrando o Rancher para um novo cluster.]
