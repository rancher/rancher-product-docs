O repositório https://github.com/rancher/charts[charts] contém todos os itens do catálogo Helm necessários para recursos como monitoramento, registro, alerta e Istio.

Em uma instalação air-gapped do Rancher, você precisará configurar o Rancher para usar uma cópia local dos charts do sistema. Esta seção descreve como usar charts do sistema local utilizando um sinalizador da CLI.

[#_using_local_system_charts]
== Usando charts do sistema local

Uma cópia local de `system-charts` foi empacotada no contêiner `rancher/rancher`. Para poder usar esses recursos em uma instalação air-gapped, você precisará executar o comando de instalação do Rancher com uma variável de ambiente extra, `CATTLE_SYSTEM_CATALOG=bundled`, que informa ao Rancher para usar a cópia local dos charts em vez de tentar buscá-los no GitHub.

Exemplos de comandos para uma instalação do Rancher com um `system-charts` empacotado estão incluídos nas instruções de {xref-air-gap}[instalação air-gapped] para instalações Docker e Helm.
