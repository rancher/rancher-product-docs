Esta seção trata do uso do Helm CLI para instalar o servidor Rancher em um ambiente air-gapped. Um ambiente air-gapped pode ser onde o servidor Rancher será instalado offline, atrás de um firewall ou de um proxy.

Os passos de instalação diferem dependendo se o Rancher é instalado em um cluster Kubernetes K3s ou em um único contêiner Docker.

Para mais informações sobre cada opção de instalação, consulte {xref-install-upgrade}[esta página.]

Ao longo das instruções de instalação, haverá _abas_ para cada opção de instalação.

[NOTE]
.Importante:
====

Se você instalar o Rancher seguindo o guia de instalação do Docker, não há um caminho para fazer upgrade da sua instalação Docker para uma instalação Kubernetes.
====
