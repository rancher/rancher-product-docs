Este tutorial descreve como configurar um banco de dados MySQL no Serviço de Banco de Dados Relacional (RDS) da Amazon.

Este banco de dados pode ser usado posteriormente como um armazenamento externo para um cluster Kubernetes K3s de alta disponibilidade.

. Faça login no https://console.aws.amazon.com/rds/[Console do Amazon AWS RDS] para começar. Certifique-se de selecionar a *Região* onde suas instâncias EC2 (nós Linux) foram criadas.
. No painel esquerdo, clique em *Banco de Dados*.
. Clique em *Criar banco de dados*.
. Na seção *Tipo de mecanismo*, clique em *MySQL*.
. Na seção *Versão*, escolha *MySQL 5.7.22*.
. Na seção *Configurações*, sob *Configurações de Credenciais,* insira uma senha mestre para o nome de usuário mestre *admin*. Confirme a senha.
. Expanda a seção *Configuração adicional*. No campo *Nome do banco de dados inicial*, insira um nome. O nome pode conter apenas letras, números e sublinhados. Este nome será usado para conectar ao banco de dados.
. Clique em *Criar banco de dados*.

Você precisará capturar as seguintes informações sobre o novo banco de dados para que o cluster Kubernetes K3s possa se conectar a ele.

Para ver essas informações no console do Amazon RDS, clique em *Banco de Dados,* e clique no nome do banco de dados que você criou.

* *Nome do usuário:* Use o nome de usuário admin.
* *Senha:* Use a senha do admin.
* *Nome do host:* Use o *Endpoint* como o nome do host. O endpoint está disponível na seção *Conectividade e segurança*.
* *Porta:* A porta deve ser 3306 por padrão. Você pode confirmar isso na seção *Conectividade e segurança*.
* *Nome do banco de dados:* Confirme o nome acessando a guia *Configuração*. O nome está listado sob *Nome do banco de dados*.

Essas informações serão usadas para conectar ao banco de dados no seguinte formato:

----
mysql://username:password@tcp(hostname:3306)/database-name
----

Para mais informações sobre como configurar o datastore para K3s, consulte a https://rancher.com/docs/k3s/latest/en/installation/datastore/[documentação do K3s.]
