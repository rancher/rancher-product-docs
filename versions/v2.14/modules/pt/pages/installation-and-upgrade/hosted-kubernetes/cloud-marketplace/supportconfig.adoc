Após instalar o adaptador CSP, você terá a capacidade de gerar um pacote supportconfig. Este pacote é um arquivo tar que pode ser usado para fornecer rapidamente informações ao suporte.

Esses pacotes supportconfig podem ser criados através do Rancher ou por meio de acesso direto ao cluster onde o Rancher está instalado. Observe que o acesso através do Rancher é preferido.

NOTE: Apenas usuários administradores podem gerar/baixar pacotes supportconfig, independentemente do método.

[#_accessing_through_rancher]
== Acessando Através do Rancher

Primeiro, clique no menu hambúrguer. Em seguida, clique no `Get Support` botão.

image::support-help.png[Obter Suporte]

Na próxima página, clique no `Generate Support Config` botão.

NOTE: Se o adaptador não estiver instalado, a opção de gerar o pacote supportconfig não estará presente. Você deve instalar o adaptador CSP para gerar um pacote supportconfig.

image::generate-support-config.png[Obter Suporte]

[#_accessing_without_rancher]
== Acessando Sem o Rancher

Primeiro, gere um kubeconfig para o cluster onde o Rancher está instalado.

NOTE: Se o Rancher estiver fora do ar, você não poderá usar um token kubeconfig gerado pelo Rancher para acessar o cluster.

Configure seu ambiente de shell para usar este token kubeconfig:

[,bash]
----
export KUBECONFIG=$MY_KUBECONFIG_PATH
----

É recomendado criar um diretório de trabalho temporário ao executar este comando, como abaixo:

[,bash]
----
mkdir temp && cd temp
----

Em seguida, recupere o pacote supportconfig:

[,bash]
----
mkdir rancher && kubectl get configmap csp-config -n cattle-csp-adapter-system -o=jsonpath='{.data.data}' >> rancher/config.json && tar -c -f supportconfig_rancher.tar rancher && rm -rf rancher
----

Isso criará um `supportconfig_rancher.tar` arquivo no seu diretório atual.

Usuários que executam esses comandos no Mac podem enfrentar problemas devido a incompatibilidades entre gnu-tar e bsd-tar. Se o suporte tiver problemas para ler um supportconfig que você produzir, tente usar o comando abaixo após tornar gnu-tar acessível como `gtar` no seu caminho:

[,bash]
----
mkdir rancher && kubectl get configmap csp-config -n cattle-csp-adapter-system -o=jsonpath='{.data.data}' >> rancher/config.json && gtar -c -f supportconfig_rancher.tar rancher && rm -rf rancher
----
