*Após instalar o adaptador, uma mensagem de banner aparece no Rancher que diz: "AWS Marketplace Adapter: Não foi possível executar o adaptador, por favor, verifique os logs do adaptador"*

Esse erro indica que, embora o adaptador tenha sido instalado no cluster, ocorreu um erro que impede que ele realize corretamente o check-in e o check-out das licenças.

Isso geralmente ocorre porque a função IAM não foi configurada corretamente. Revise os {xref-aws-cloud-marketplace-adapter-reqs}[pré-requisitos] e verifique se:

* Um provedor OIDC foi criado/associado ao cluster em que o Rancher está sendo executado.
* A função IAM foi configurada para confiar neste provedor OIDC.
* A função IAM possui pelo menos as permissões descritas na política.

Se tudo acima foi configurado corretamente, entre em contato com o suporte para assistência.

*Eu vejo uma mensagem de banner que afirma: "AWS Marketplace Adapter: Você excedeu sua contagem de nós licenciados. Pelo menos x licença(s) a mais são necessárias na AWS para se tornar compatível"*

Essa mensagem indica que você não possui direitos suficientes para a quantidade de nós que o Rancher está gerenciando atualmente.

Lembre-se das seguintes limitações:

* Cada direito é válido para um certo número de nós.
* Cada nó atualmente gerenciado pelo Rancher conta para o seu total de uso (com exceção dos nós no cluster onde o Rancher está instalado).
* Cada direito pode ser usado por no máximo uma instância do Rancher. Por exemplo, se você tiver duas instâncias do Rancher em execução em sua conta (cada uma instalada em um cluster EKS separado), então você precisará de pelo menos dois direitos.

Você também pode ter desinstalado/reinstalado recentemente o adaptador. Se o adaptador perder o controle das licenças que está gerenciando atualmente, pode levar até uma hora para resolver o estado real das licenças.
