A segurança está no coração de todos os recursos do Rancher. Desde a integração com todas as ferramentas e serviços de autenticação populares até uma {xref-manage-rbac}[capacidade RBAC] de nível empresarial, o Rancher torna seus clusters Kubernetes ainda mais seguros.

Nesta página, fornecemos documentação relacionada à segurança, juntamente com recursos para ajudá-lo a proteger sua instalação do Rancher e seus clusters Kubernetes downstream.

[#__neuvector_product_name_integration_with_rancher]
== {neuvector-product-name} Integração com o Rancher

NeuVector é um aplicativo de segurança focado em contêineres e de código aberto que agora está integrado ao Rancher. NeuVector fornece segurança em produção, proteção contra vulnerabilidades DevOps e um firewall de contêiner, entre outros. Consulte os {xref-neuvector}[documentos do Rancher] e os https://documentation.suse.com/cloudnative/security/{neuvector-docs-version}/en/overview.html[{neuvector-product-name} documentos] para mais informações.

[#_running_a_compliance_security_scan_on_a_kubernetes_cluster]
== Executando uma Verificação de Segurança de Conformidade em um Cluster Kubernetes

O Rancher utiliza https://github.com/aquasecurity/kube-bench[kube-bench] para executar uma verificação de segurança para verificar se o Kubernetes está implantado de acordo com as melhores práticas de segurança.

Quando o Rancher executa uma verificação de conformidade em um cluster, ele gera um relatório mostrando os resultados de cada teste, incluindo um resumo com o número de testes aprovados, ignorados e reprovados. O relatório também inclui etapas de remediação para quaisquer testes reprovados.

Para detalhes, consulte a seção sobre {xref-compliance-scans-howto}[verificações de segurança].

[#_selinux_rpm]
== SELinux RPM

Fornecemos três RPMs (RPM Package Manager) que permitem que os produtos Rancher funcionem corretamente em hosts que aplicam SELinux: `rancher-selinux`, `rke2-selinux` e `k3s-selinux`.  Para detalhes, veja {xref-selinux-rpm}[esta página].

[#_rancher_hardening_guide]
== Guia de Proteção do Rancher

O Guia de Proteção do Rancher é baseado em controles e melhores práticas encontrados no https://www.cisecurity.org/benchmark/kubernetes/[CIS Kubernetes Benchmark] do Center for Internet Security.

Os guias de proteção fornecem orientações prescritivas para proteger uma instalação de produção do Rancher. Consulte os guias do Rancher para <<_the_cis_benchmark_and_self_assessment,Autoavaliação do CIS Kubernetes Benchmark>> para a lista completa de controles de segurança.

____
Os guias de proteção descrevem como proteger os nós em seu cluster, e é recomendado seguir um guia de proteção antes de instalar o Kubernetes.
____

Cada versão do guia de proteção destina-se a ser usada com versões específicas do CIS Kubernetes Benchmark, Kubernetes e Rancher.

[#_the_cis_benchmark_and_self_assessment]
== O Benchmark CIS e a Autoavaliação

A autoavaliação do benchmark é um complemento ao guia de proteção de segurança do Rancher. Enquanto o guia de proteção mostra como proteger o cluster, o guia de benchmark é destinado a ajudá-lo a avaliar o nível de segurança do cluster protegido.

Este guia percorre os vários controles e fornece comandos de exemplo atualizados para auditar a conformidade em clusters criados pelo Rancher. Os documentos originais do benchmark podem ser baixados do https://www.cisecurity.org/benchmark/kubernetes/[site do CIS].

Cada versão do guia de autoavaliação do Rancher corresponde a versões específicas do guia de proteção, Rancher, Kubernetes e do CIS Benchmark.

Os ambientes em teste seguem os guias de proteção fornecidos pelo Rancher no momento dos testes. Relatórios de testes de penetração anteriores estão disponíveis abaixo.

Resultados:

* https://releases.rancher.com/documents/security/pen-tests/2019/RAN-01-cure53-report.final.pdf[Teste de Penetração Cure53 - Julho de 2019]
* https://releases.rancher.com/documents/security/pen-tests/2019/UntamedTheory-Rancher_SecurityAssessment-20190712_v5.pdf[Teste de Penetração Untamed Theory - Março de 2019]

Por favor, note que novos relatórios não são mais compartilhados ou disponibilizados publicamente.

[#_rancher_kubernetes_distributions_k3s_rke2_self_assessment_and_hardening_guides]
== Guias de Autoavaliação e Proteção das Distribuições Kubernetes do Rancher (K3s/RKE2)

Rancher utiliza as seguintes distribuições Kubernetes:

* https://documentation.suse.com/en-us/cloudnative/rke2/[*RKE2*] é uma distribuição Kubernetes totalmente compatível que se concentra na segurança e conformidade nos EUA. Governo Federal dos EUA.
* https://documentation.suse.com/en-us/cloudnative/k3s/[*K3s*] é uma distribuição Kubernetes leve e totalmente compatível. É fácil de instalar, com metade da necessidade de memória do Kubernetes upstream, tudo em um binário de menos de 100 MB.

Para proteger um cluster Kubernetes que está executando uma distribuição diferente das listadas, consulte a documentação do seu provedor de Kubernetes.

[#_hardening_guides_and_benchmark_versions]
=== Guias de Proteção e Versões de Benchmark

Cada guia de autoavaliação é acompanhado por um guia de proteção. Esses guias foram testados juntamente com as versões do Rancher listadas. Cada guia de autoavaliação foi testado em uma versão específica do Kubernetes e versão de benchmark CIS. Se um benchmark CIS não foi validado para a sua versão do Kubernetes, você pode usar os guias existentes até que um guia para a sua versão seja adicionado.
