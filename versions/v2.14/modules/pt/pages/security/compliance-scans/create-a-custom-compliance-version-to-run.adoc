Pode haver algumas configurações de cluster Kubernetes que exigem configurações personalizadas dos testes de conformidade. Por exemplo, o caminho para os arquivos de configuração do Kubernetes ou certificados pode ser diferente do local padrão onde o upstream de conformidade os procura.

Agora é possível criar uma versão de conformidade personalizada para executar uma varredura de cluster usando o aplicativo `rancher-compliance`.

Para obter detalhes, consulte {xref-compliance-scan-custom-benchmark}[esta página.]
