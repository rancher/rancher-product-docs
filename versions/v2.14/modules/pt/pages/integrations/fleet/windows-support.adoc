Antes do Rancher v2.5.6, o `agent` não tinha manifests nativos do Windows em clusters downstream com nós Windows. Isso resultaria em um `agent` pod com falha no cluster.

Se você estiver atualizando de uma versão mais antiga do Rancher para v2.5.6+, pode implantar um `agent` funcional com o seguinte fluxo de trabalho _no cluster downstream_:

. Cordon todos os nós Windows.
. Aplique a tolerância abaixo ao `agent` workload.
. Descordon todos os nós Windows.
. Exclua todos os `agent` pods. Novos pods devem ser criados com a nova tolerância.
. Uma vez que os `agent` pods estejam em execução, e a atualização automática esteja habilitada para o Fleet, eles devem ser atualizados para uma `agent` versão compatível com Windows.

[,yaml]
----
tolerations:
- effect: NoSchedule
  key: cattle.io/os
  operator: Equal
  value: linux
----
