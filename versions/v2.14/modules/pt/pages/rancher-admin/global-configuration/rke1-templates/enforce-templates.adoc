Esta seção descreve como os administradores de modelos podem aplicar modelos no Rancher, restringindo a capacidade dos usuários de criar clusters sem um modelo.

Por padrão, qualquer usuário padrão no Rancher pode criar clusters. Mas quando a aplicação de modelos RKE está ativada,

* Apenas um administrador tem a capacidade de criar clusters sem um modelo.
* Todos os usuários padrão devem usar um modelo RKE para criar um novo cluster.
* Usuários padrão não podem criar um cluster sem usar um modelo.

Os usuários só podem criar novos modelos se o administrador xref:./creator-permissions.adoc#_allowing_a_user_to_create_templates[der permissão a eles.]

Após um cluster ser criado com um modelo RKE, o criador do cluster não pode editar as configurações que estão definidas no modelo. A única maneira de mudar essas configurações após a criação do cluster é xref:./apply-templates.adoc#_updating_a_cluster_created_with_an_rke_template[fazer upgrade do cluster para uma nova revisão] do mesmo modelo. Se os criadores de clusters quiserem mudar as configurações definidas pelo modelo, precisarão contatar o proprietário do modelo para obter uma nova revisão do modelo. Para detalhes sobre como as revisões de modelos funcionam, consulte a {xref-manage-rke1-templates}#_atualizando_um_modelo[documentação sobre a revisão de modelos.]

[#_requiring_new_clusters_to_use_an_rke_template]
== Exigindo que Novos Clusters Usem um Modelo RKE

Você pode querer exigir que novos clusters usem um modelo para garantir que qualquer cluster lançado por um {xref-global-permissions}[usuário padrão] usará as configurações do Kubernetes e/ou Rancher que foram aprovadas pelos administradores.

Para exigir que novos clusters usem um modelo RKE, os administradores podem ativar a aplicação de modelos RKE com os seguintes passos:

. Clique em *☰ > Configurações Globais*.
. Vá até a configuração `cluster-template-enforcement`. Clique em *⋮ > Editar configuração*.
. Defina o valor como *Verdadeiro* e clique em *Salvar*.
+

[NOTE]
.Importante:
====
Quando o administrador define o `cluster-template-enforcement` como *Verdadeiro*, ele também precisa compartilhar o `clusterTemplates` com os usuários para que eles possam selecionar um desses modelos para criar o cluster.
====


*Resultado:* Todos os clusters provisionados pelo Rancher devem usar um modelo, a menos que o criador seja um administrador.

[#_disabling_rke_template_enforcement]
== Desabilitando a Aplicação do Modelo RKE

Para permitir que novos clusters sejam criados sem um modelo RKE, os administradores podem desativar a aplicação do modelo RKE com os seguintes passos:

. Clique em *☰ > Configurações Globais*.
. Vá até a configuração `cluster-template-enforcement`. Clique em *⋮ > Editar configuração*.
. Defina o valor como *Falso* e clique em *Salvar*.

*Resultado:* Quando os clusters são provisionados pelo Rancher, eles não precisam usar um modelo.
