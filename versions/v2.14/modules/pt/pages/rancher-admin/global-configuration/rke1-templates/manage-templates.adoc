Esta seção descreve como gerenciar modelos RKE e revisões. Você pode criar, compartilhar, atualizar e excluir modelos a partir da visualização *Gerenciamento de Cluster* sob menu:Configuração RKE1[Modelos RKE].

As atualizações de modelo são gerenciadas por meio de um sistema de revisões. Quando os proprietários de modelos desejam alterar ou atualizar um modelo, eles criam uma nova revisão do modelo. Revisões individuais não podem ser editadas. No entanto, se você quiser impedir que uma revisão seja usada para criar um novo cluster, pode desativá-la.

As revisões de modelo podem ser usadas de duas maneiras: para criar um novo cluster ou para fazer upgrade de um cluster que foi criado com uma versão anterior do modelo. O criador do modelo pode escolher uma revisão padrão, mas quando os usuários finais criam um cluster, eles podem escolher qualquer modelo e qualquer revisão de modelo que esteja disponível para eles. Depois que o cluster é criado a partir de uma revisão específica, ele não pode mudar para outro modelo, mas o cluster pode fazer upgrade para uma revisão mais nova disponível do mesmo modelo.

O proprietário do modelo tem controle total sobre as revisões do modelo e pode criar novas revisões para atualizar o modelo, excluir ou desativar revisões que não devem ser usadas para criar clusters e escolher qual revisão de modelo é a padrão.

[#_prerequisites]
== Pré-requisitos

Você pode criar modelos RKE se tiver a permissão *Criar Modelos RKE*, que pode ser {xref-rke1-creator-permissions}[dada por um administrador.]

Você pode revisar, compartilhar e excluir um modelo se for o proprietário do modelo. Para detalhes sobre como se tornar um proprietário de um modelo, consulte xref:./access-or-share-templates.adoc#_sharing_ownership_of_templates[a documentação sobre compartilhamento de propriedade de modelo.]

[#_creating_a_template]
== Criando um modelo

. No canto superior esquerdo, clique em *☰ > Gerenciamento de Cluster*.
. Clique em menu:Configuração RKE1[Modelos de Nó].
. Clique em *Adicionar Modelo*.
. Forneça um nome para o modelo. Um nome gerado automaticamente já é fornecido para a primeira versão do modelo, que é criada junto com este modelo.
. Opcional: Compartilhe o modelo com outros usuários ou grupos xref:./access-or-share-templates.adoc#_sharing_templates_with_specific_users_or_groups[adicionando-os como membros.] Você também pode tornar o modelo público para compartilhar com todos na configuração do Rancher.
. Em seguida, siga o formulário na tela para salvar os parâmetros de configuração do cluster como parte da revisão do modelo. A revisão pode ser marcada como padrão para este modelo.

*Resultado:* Um modelo RKE com uma revisão está configurado. Você pode usar esta revisão do gabarito RKE mais tarde quando {xref-launch-k8s-with-rancher}[provisionar um cluster iniciado pelo Rancher]. Depois que um cluster é gerenciado por um gabarito RKE, ele não pode ser desconectado e a opção para desmarcar *Usar um Gabarito e Revisão RKE existentes* ficará indisponível.

[#_updating_a_template]
== Atualizando um Modelo

Quando você atualiza um modelo RKE, está criando uma revisão do modelo existente. Clusters que foram criados com uma versão mais antiga do modelo podem ser atualizados para corresponder à nova revisão.

Você não pode editar revisões individuais. Como você não pode editar revisões individuais de um modelo, para evitar que uma revisão seja utilizada, você pode <<_disabling_a_template_revision,desativá-la.>>

Quando novas revisões de modelos são criadas, clusters que usam uma revisão mais antiga do modelo não são afetados.

. No canto superior esquerdo, clique em *☰ > Gerenciamento de Cluster*.
. No menu de navegação à esquerda, clique em menu:Configuração RKE1[Modelos RKE].
. Vá para o modelo que você deseja editar e clique em *⋮ > Editar*.
. Edite as informações necessárias e clique em *Salvar*.
. Opcional: Você pode mudar a revisão padrão deste gabarito e também alterar com quem ele é compartilhado.

*Resultado:* O modelo é atualizado. Para aplicá-lo a um cluster usando uma versão mais antiga do modelo, consulte a seção sobre <<_upgrading_a_cluster_to_use_a_new_template_revision,atualizando um cluster para usar uma nova revisão de um modelo.>>

[#_deleting_a_template]
== Apagando um Modelo

Quando você não usar mais um modelo RKE para nenhum de seus clusters, você pode apagá-lo.

. No canto superior esquerdo, clique em *☰ > Gerenciamento de Cluster*.
. Clique em menu:Configuração RKE1[Modelos RKE].
. Vá até o modelo RKE que você deseja apagar e clique em *⋮ > Apagar*.
. Confirme a exclusão.

*Resultado:* O modelo é apagado.

[#_creating_a_revision_based_on_the_default_revision]
== Criando uma Revisão Baseada na Revisão Padrão

Você pode clonar a revisão do modelo padrão e atualizar rapidamente suas configurações em vez de criar uma nova revisão do zero. Clonar modelos economiza o trabalho de reinserir as chaves de acesso e outros parâmetros necessários para a criação do cluster.

. No canto superior esquerdo, clique em *☰ > Gerenciamento de Cluster*.
. No menu de navegação à esquerda, clique em menu:Configuração RKE1[Modelos RKE].
. Vá até o modelo RKE que você deseja clonar e clique em *⋮ > Nova Revisão a partir do Padrão*.
. Complete o restante do formulário para criar uma nova revisão.

*Resultado:* A revisão do modelo RKE é clonada e configurada.

[#_creating_a_revision_based_on_a_cloned_revision]
== Criando uma Revisão Baseada em uma Revisão Clonada

Ao criar novas revisões de modelos RKE a partir das configurações do seu usuário, você pode clonar uma revisão existente e atualizar rapidamente suas configurações em vez de criar uma nova do zero. Clonar revisões de modelos economiza o trabalho de reinserir os parâmetros do cluster.

. No canto superior esquerdo, clique em *☰ > Gerenciamento de Cluster*.
. Em *Configuração RKE1*, clique em *Modelos RKE*.
. Vá até a revisão do modelo que você deseja clonar. Em seguida, selecione *⋮ > Clonar Revisão*.
. Complete o restante do formulário.

*Resultado:* A revisão do modelo RKE é clonada e configurada. Você pode usar a revisão do modelo RKE mais tarde, quando provisionar um cluster. Qualquer cluster existente que utilize este modelo RKE pode ser atualizado para esta nova revisão.

[#_disabling_a_template_revision]
== Desabilitando uma Revisão de Modelo

Quando você não quiser mais que uma revisão do modelo RKE seja usada para criar novos clusters, você pode desabilitá-la. Uma revisão desabilitada pode ser reabilitada.

Você pode desabilitar a revisão se ela não estiver sendo usada por nenhum cluster.

. No canto superior esquerdo, clique em *☰ > Gerenciamento de Cluster*.
. No menu de navegação à esquerda, clique em menu:Configuração RKE1[Modelos RKE].
. Vá para a revisão do modelo RKE que você deseja desabilitar. Então selecione *⋮ > Desabilitar*.

*Resultado:* A revisão do modelo RKE não pode ser usada para criar um novo cluster.

[#_re_enabling_a_disabled_template_revision]
== Reabilitando uma Revisão de Modelo Desabilitada

Se você decidir que uma revisão do modelo RKE desabilitada deve ser usada para criar novos clusters, você pode reabilitá-la.

. No canto superior esquerdo, clique em *☰ > Gerenciamento de Cluster*.
. Em *Configuração RKE1*, clique em *Modelos RKE*.
. Vá para a revisão do modelo RKE que você deseja reabilitar. Então selecione *⋮ > Habilitar*.

*Resultado:* A revisão do modelo RKE pode ser usada para criar um novo cluster.

[#_setting_a_template_revision_as_default]
== Definindo uma Revisão de Modelo como Padrão

Quando os usuários finais criam um cluster usando um modelo RKE, eles podem escolher qual revisão usar para criar o cluster. Você pode configurar qual revisão é usada por padrão.

Para definir uma revisão do modelo RKE como padrão,

. No canto superior esquerdo, clique em *☰ > Gerenciamento de Cluster*.
. No menu de navegação à esquerda, clique em menu:Configuração RKE1[Modelos RKE].
. Vá para a revisão do modelo RKE que deve ser padrão e clique em *⋮ > Definir como Padrão*.

*Resultado:* A revisão do modelo RKE será usada como a opção padrão quando os clusters forem criados com o modelo RKE.

[#_deleting_a_template_revision]
== Apagando uma Revisão de Modelo

Você pode excluir todas as revisões de um modelo, exceto pela revisão padrão.

Para excluir uma revisão permanentemente,

. No canto superior esquerdo, clique em *☰ > Gerenciamento de Cluster*.
. No menu de navegação à esquerda, clique em menu:Configuração RKE1[Modelos RKE].
. Vá para a revisão do modelo RKE que deve ser excluída e clique em *⋮ > Excluir*.

*Resultado:* A revisão do modelo RKE é excluída.

[#_upgrading_a_cluster_to_use_a_new_template_revision]
== Atualizando um Cluster para Usar uma Nova Revisão de Modelo

[NOTE]
====

Esta seção assume que você já possui um cluster que {xref-rke1-template-apply}[tem um modelo RKE aplicado.]

Esta seção também assume que você <<_updating_a_template,atualizou o modelo que o cluster está usando>> para que uma nova revisão de modelo esteja disponível.
====


Para atualizar um cluster para usar uma nova revisão de modelo,

. No canto superior esquerdo, clique em *☰ > Gerenciamento de Cluster*.
. Vá para o cluster que você deseja fazer upgrade e clique em *⋮ > Editar Configuração*.
. Na seção *Opções do Cluster*, clique no menu suspenso para a revisão do modelo e selecione a nova revisão do modelo.
. Clique em *Salvar*.

*Resultado:* O cluster é atualizado para usar as configurações definidas na nova revisão do modelo.

[#_exporting_a_running_cluster_to_a_new_rke_template_and_revision]
== Exportando um cluster em execução para um novo modelo RKE e revisão.

Você pode salvar as configurações de um cluster existente como um modelo RKE.

Isso exporta as configurações do cluster como um novo modelo RKE e também vincula o cluster a esse modelo. O resultado é que o cluster só pode ser alterado se o <<_updating_a_template,modelo for atualizado,>> e o cluster fizer upgrade para <<_upgrading_a_cluster_to_use_a_new_template_revision,usar uma versão mais recente do modelo>>.

Para converter um cluster existente para usar um modelo RKE,

. No canto superior esquerdo, clique em *☰ > Gerenciamento de Cluster*.
. Vá para o cluster que será convertido para usar um modelo RKE e *⋮ > Salvar como Modelo RKE*.
. Digite um nome para o modelo RKE no formulário que aparece e clique em *Criar*.

*Resultados:*

* Um novo modelo RKE é criado.
* O cluster é convertido para usar o novo modelo.
* Novos clusters podem ser xref:./apply-templates.adoc#_creating_a_cluster_from_an_rke_template[criados a partir do novo modelo e revisão.]
