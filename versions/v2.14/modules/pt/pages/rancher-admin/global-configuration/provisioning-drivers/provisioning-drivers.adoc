Os drivers no Rancher permitem que você gerencie quais provedores podem ser usados para implantar {xref-hosted-k8s-clusters}[clusters Kubernetes hospedados] ou {xref-infra-providers}[nós em um provedor de infraestrutura] para permitir que o Rancher implemente e gerencie o Kubernetes.

[#_rancher_drivers]
== Drivers do Rancher

Com os drivers do Rancher, você pode ativar/desativar drivers incorporados existentes que estão empacotados no Rancher. Alternativamente, você pode adicionar seu próprio driver se o Rancher ainda não o implementou.

Existem dois tipos de drivers dentro do Rancher:

* <<_cluster_drivers,Drivers de Cluster>>
* <<_node_drivers,Drivers de Nó>>

[#_cluster_drivers]
== Drivers de Cluster

Drivers de cluster são usados para provisionar {xref-hosted-k8s-clusters}[clusters Kubernetes hospedados], como GKE, EKS, AKS, etc. A disponibilidade de qual driver de cluster exibir ao criar um cluster é definida com base no status do driver de cluster. Apenas `active` drivers de cluster serão exibidos como opção para criar clusters de Kubernetes hospedados. Por padrão, o Rancher é empacotado com vários drivers de cluster existentes, mas você também pode criar drivers de cluster personalizados para adicionar ao Rancher.

Por padrão, o Rancher ativou vários provedores de nuvem Kubernetes hospedados, incluindo:

* {xref-deploy-eks}[Amazon EKS]
* {xref-deploy-gke}[Google GKE]
* {xref-deploy-aks}[Azure AKS]

[#_node_drivers]
== Drivers de Nó

Os drivers de nó são usados para provisionar hosts, que o Rancher utiliza para lançar e gerenciar clusters Kubernetes. Um driver de nó é o mesmo que um https://github.com/docker/docs/blob/vnext-engine/machine/drivers/index.md[driver do Docker Machine]. A disponibilidade de qual driver de nó exibir ao criar modelos de nó é definida com base no status do driver de nó. Apenas `active` drivers de nó serão exibidos como uma opção para criar modelos de nó. Por padrão, o Rancher é empacotado com muitos drivers de Docker Machine existentes, mas você também pode criar drivers de nó personalizados para adicionar ao Rancher.

Se houver drivers de nó específicos que você não deseja mostrar aos seus usuários, você precisaria desativar esses drivers de nó.

O Rancher suporta vários provedores de nuvem importantes, mas por padrão, esses drivers de nó estão ativos e disponíveis para implantação:

* {xref-infra-aws}[Amazon EC2]
* {xref-infra-azure}[Azure]
* {xref-infra-digitalocean}[Digital Ocean]
* {xref-infra-vsphere}[vSphere]

Existem vários outros drivers de nó que estão desativados por padrão, mas estão empacotados no Rancher:

* {xref-harvester-overview}#_harvester_node_driver/[{harvester-product-name}], disponível a partir do Rancher v2.6.1
* {xref-infra-gce}[Google GCE], disponível a partir do Rancher v2.12.0
