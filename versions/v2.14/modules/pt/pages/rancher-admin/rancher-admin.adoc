Após a instalação, o {xref-global-permissions}[administrador do sistema] deve configurar o Rancher para configurar autenticação, autorização, segurança, configurações padrão, políticas de segurança, drivers e entradas de DNS globais.

[#_first_log_in]
== Primeiro Acesso

Após você fazer login no Rancher pela primeira vez, o Rancher solicitará a *URL do Servidor Rancher*. Você deve definir a URL como o ponto de entrada principal para o Servidor Rancher. Quando um balanceador de carga está na frente de um cluster do Servidor Rancher, a URL deve resolver para o balanceador de carga. O sistema tentará automaticamente inferir a URL do Servidor Rancher a partir do endereço IP ou nome de host do host que está executando o Servidor Rancher. Isso está correto apenas se você estiver executando uma instalação do Servidor Rancher em um único nó. Na maioria dos casos, portanto, você precisa definir a URL do Servidor Rancher para o valor correto você mesmo.

[WARNING]
====

Após você definir a URL do Servidor Rancher, não suportamos atualizá-la. Defina a URL com extrema cautela.
====


[#_authentication]
== Autenticação

Uma das principais características que o Rancher adiciona ao Kubernetes é a autenticação de usuários centralizada. Esse recurso permite configurar usuários locais e/ou conectar-se a um provedor de autenticação externo. Ao se conectar a um provedor de autenticação externo, você pode aproveitar os usuários e grupos desse provedor.

Para mais informações sobre como a autenticação funciona e como configurar cada provedor, veja {xref-authn-authz}[Autenticação].

[#_authorization]
== Autorização

Dentro do Rancher, cada pessoa se autentica como um _usuário_, que é um login que concede acesso ao Rancher. Uma vez que o usuário faz login no Rancher, sua _autorização_, ou seus direitos de acesso dentro do sistema, é determinada pela função do usuário. O Rancher fornece funções incorporadas para permitir que você configure facilmente os direitos de acesso de um usuário aos recursos, mas o Rancher também oferece a capacidade de personalizar as funções para cada recurso do Kubernetes.

Para mais informações sobre como a autorização funciona e como personalizar funções, veja {xref-manage-rbac}[Controle de Acesso com Base em Função (RBAC)].

[#_user_retention]
== Retenção de Usuários

No Rancher v2.8.5 e versões posteriores, você pode habilitar a retenção de usuários. Esse recurso remove automaticamente usuários inativos após um período de tempo configurável.

O recurso de retenção de usuários está desativado por padrão.

Para mais informações, veja {xref-user-retention}[Habilitando a Retenção de Usuários].

[#_provisioning_drivers]
== Drivers de Provisionamento

Os drivers no Rancher permitem que você gerencie quais provedores podem ser usados para provisionar {xref-hosted-k8s-clusters}[clusters Kubernetes hospedados] ou {xref-infra-providers}[nós em um provedor de infraestrutura] para permitir que o Rancher implemente e gerencie o Kubernetes.

Para mais informações, veja {xref-provisioning-drivers}[Drivers de Provisionamento].

[#_adding_kubernetes_versions_into_rancher]
== Adicionando Versões do Kubernetes ao Rancher

Com esse recurso, você pode fazer upgrade para a versão mais recente do Kubernetes assim que for lançada, sem fazer upgrade do Rancher. Esse recurso permite que você faça upgrade facilmente das versões de patch do Kubernetes (ou seja, `v1.15.X`), mas não é destinado a fazer upgrade das versões menores do Kubernetes (ou seja, `v1.X.0`), pois o Kubernetes tende a descontinuar ou adicionar APIs entre versões menores.

As informações que o Rancher usa para provisionar {xref-launch-k8s-with-rancher}[clusters RKE] agora estão localizadas nos Metadados do Kubernetes do Rancher. Para detalhes sobre a configuração de metadados e como alterar a versão do Kubernetes usada para provisionar clusters RKE, veja {xref-backups-without-uprading-rancher}[Metadados do Kubernetes do Rancher].

Os Metadados do Kubernetes do Rancher contêm informações sobre a versão do Kubernetes que o Rancher usa para provisionar {xref-launch-k8s-with-rancher}[clusters RKE].

Para mais informações sobre como os metadados funcionam e como configurar a configuração de metadados, veja {xref-backups-without-uprading-rancher}[Metadados do Rancher Kubernetes].

[#_global_settings]
== Configurações Globais

Opções que controlam certas configurações globais do Rancher estão disponíveis na barra de navegação superior.

Clique *☰* no canto superior esquerdo, depois selecione *Configurações Globais* para visualizar e configurar as seguintes configurações:

* *Configurações*: Vários padrões do Rancher, como o comprimento mínimo para a senha de um usuário (`password-min-length`). Você deve ter cautela ao modificar essas configurações, pois valores inválidos podem quebrar sua instalação do Rancher.
* *Flags de Recursos*: Recursos do Rancher que podem ser ativados ou desativados. Algumas dessas flags são para <<_enabling_experimental_features,recursos experimentais>>.
* *Faixas*: Elementos que você pode adicionar a locais fixos no portal. Por exemplo, você pode usar essas opções para {xref-custom-branding}#_fixed_banners[definir uma faixa personalizada] para os usuários quando eles fizerem login no Rancher.
* *Branding*: Elementos de design da interface do usuário do Rancher que você pode {xref-custom-branding}[personalizar]. Você pode adicionar um logotipo ou favicon personalizado e modificar as cores da interface do usuário.
* *Desempenho*: Configurações de desempenho para a interface do usuário do Rancher, como carregamento incremental de recursos.
* *Links da Página Inicial*: Links exibidos na página *Página Inicial* da interface do usuário do Rancher. Você pode modificar a visibilidade dos links padrão ou adicionar seus próprios links.

[#_enabling_experimental_features]
=== Habilitando Recursos Experimentais

O Rancher inclui algumas funcionalidades que são experimentais e/ou desativadas por padrão. As flags de recurso permitem que você ative esses recursos. Para mais informações, consulte a seção sobre {xref-experimental-features}[flags de recurso].

[#_global_configuration]
=== Configuração Global

As opções de *Configuração Global* não estão visíveis a menos que você ative a *legado* {xref-experimental-features}[flag de recurso]. A flag *legado* está desativada por padrão em novas instalações do Rancher a partir da versão v2.6. Se você atualizar de uma versão anterior do Rancher, ou ativar a *legado* flag de recurso no Rancher v2.6 ou posterior, *Configuração Global* estará disponível no menu de navegação superior:

. Clique em *☰* no canto superior esquerdo.
. Selecione *Configuração Global* em *Apps Legados*.

Os seguintes recursos estão disponíveis em *Configuração Global*:

* *Catálogos*
* *Global DNS Entries*
* *Provedores DNS Globais*

Como estas são funcionalidades legadas, consulte a documentação do Rancher v2.0--v2.4 sobre xref:[catálogos], xref:/versioned_docs/version-2.0-2.4/how-to-guides/new-user-guides/helm-charts-in-rancher/globaldns.adoc#_adding_a_global_dns_entry[entradas DNS globais] e xref:/versioned_docs/version-2.0-2.4/how-to-guides/new-user-guides/helm-charts-in-rancher/globaldns.adoc#_editing_a_global_dns_provider[provedores DNS globais] para mais detalhes.
