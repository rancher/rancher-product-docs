Muitas integrações de terceiros disponíveis para Kubernetes, como GitLab e HashiCorp Vault, envolvem conceder a um processo externo acesso à API do Kubernetes usando um token de conta de serviço nativo do Kubernetes para autenticação.

No Rancher v2.9.0 e versões posteriores, contas de serviço em clusters downstream agora podem se autenticar através de um JSON Web Token (JWT) usando o proxy de autenticação do Rancher. Nas versões do Rancher anteriores à v2.9.0, apenas tokens emitidos pelo Rancher eram suportados.

Para habilitar esse recurso, siga estas etapas:

. No canto superior esquerdo, clique em *☰ > Gerenciamento de Cluster*.
. Clique em *Avançado* para abrir o menu suspenso.
. Selecione *Autenticação JWT*.
. Marque a caixa de seleção para o cluster que você deseja habilitar a autenticação JWT e clique em *Habilitar*. Alternativamente, você pode clicar em *⋮* > *Habilitar*.
