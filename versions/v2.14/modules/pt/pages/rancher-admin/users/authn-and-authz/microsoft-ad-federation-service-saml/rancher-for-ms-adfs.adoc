Após completar {xref-ms-adfs-for-rancher}[Configurando Microsoft AD FS para Rancher], insira suas informações do Active Directory Federation Service (AD FS) no Rancher para que os usuários do AD FS possam se autenticar no Rancher.

[NOTE]
.Notas Importantes Para Configurar Seu Servidor ADFS:
====

* A URL do Serviço do Protocolo SAML 2.0 WebSSO é: `\https://<RANCHER_SERVER>/v1-saml/adfs/saml/acs`
* A URL do identificador de Confiança da Parte Confiável é: `\https://<RANCHER_SERVER>/v1-saml/adfs/saml/metadata`
* Você deve exportar o arquivo `federationmetadata.xml` do seu servidor AD FS. Isso pode ser encontrado em: `\https://<AD_SERVER>/federationmetadata/2007-06/federationmetadata.xml`
====


. No canto superior esquerdo, clique em *☰ > Usuários e Autenticação*.
. No menu de navegação à esquerda, clique em *Provedor de Autenticação*.
. Clique em *ADFS*.
. Complete o formulário *Configurar Conta AD FS*. O Microsoft AD FS permite que você especifique um servidor Active Directory (AD) existente. A <<_configuration,seção de configuração abaixo>> descreve como você pode mapear atributos do AD para campos dentro do Rancher.
. Após completar o formulário *Configurar Conta AD FS*, clique em *Habilitar*.
+
O Rancher redireciona você para a página de login do AD FS. Insira credenciais que autentiquem com o Microsoft AD FS para validar sua configuração do Rancher AD FS.
+

[NOTE]
====
Você pode precisar desativar seu bloqueador de pop-ups para ver a página de login do AD FS.
====


*Resultado:* O Rancher está configurado para trabalhar com AD FS. Seus usuários agora podem fazer login no Rancher usando seus logins do AD FS.

[#_configuration]
== Configuração

|===
| Campo | Descrição

| Campo Nome de Exibição
| O atributo do AD que contém o nome de exibição dos usuários. +
 +
Exemplo: `http://schemas.xmlsoap.org/ws/2005/05/identity/claims/name`

| Campo Nome de Usuário
| O atributo do AD que contém o nome de usuário/nome dado. +
 +
Exemplo: `http://schemas.xmlsoap.org/ws/2005/05/identity/claims/givenname`

| Campo UID
| Um atributo do AD que é único para cada usuário. +
 +
Exemplo: `http://schemas.xmlsoap.org/ws/2005/05/identity/claims/upn`

| Campo Grupos
| Faça entradas para gerenciar associações a grupos. +
 +
Exemplo: `http://schemas.xmlsoap.org/claims/Group`

| Host da API do Rancher
| A URL do seu Servidor Rancher.

| Chave Privada / Certificado
| Este é um par chave-certificado para criar um shell seguro entre o Rancher e seu AD FS. Certifique-se de definir o Nome Comum (CN) para a URL do seu Servidor Rancher. +
 +
<<_example_certificate_creation_command,Comando de criação de certificado>>

| Metadata XML
| O arquivo `federationmetadata.xml` exportado do seu servidor AD FS. +
 +
Você pode encontrar este arquivo em `\https://<AD_SERVER>/federationmetadata/2007-06/federationmetadata.xml`.
|===

[#_example_certificate_creation_command]
=== Exemplo de Comando de Criação de Certificado

Você pode gerar um certificado usando um comando openssl. Por exemplo:

----
openssl req -x509 -newkey rsa:2048 -keyout myservice.key -out myservice.cert -days 365 -nodes -subj "/CN=myservice.example.com"
----
