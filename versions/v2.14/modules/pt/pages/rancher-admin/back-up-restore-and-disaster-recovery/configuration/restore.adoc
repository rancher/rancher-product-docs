A página Criar Restauração permite que você forneça detalhes do backup a ser restaurado

image::backup_restore/restore/restore.png[]

[#_backup_source]
== Fonte do Backup

Forneça detalhes do arquivo de backup e sua localização de armazenamento, que o operador usará para realizar a restauração. Selecione entre as seguintes opções para fornecer esses detalhes

[#_an_existing_backup_config]
=== Uma configuração de backup existente

Selecionar esta opção preencherá o menu suspenso *Backup de Destino* com os backups disponíveis neste cluster. Selecione o backup no menu suspenso, e isso preencherá o campo *Nome do Arquivo de Backup* para você, e também passará as informações da fonte do backup selecionado para o operador.

image::backup_restore/restore/existing.png[]

Se o recurso personalizado de Backup não existir no cluster, você precisa obter o nome exato do arquivo e fornecer os detalhes da fonte do backup com o alvo de armazenamento padrão ou um armazenamento de objetos compatível com S3.

[#_the_default_storage_target]
=== O alvo de armazenamento padrão

Selecione esta opção se você estiver restaurando de um arquivo de backup que existe na localização de armazenamento padrão configurada no nível do operador. A configuração no nível do operador é a localização de armazenamento que foi configurada quando o operador `rancher-backup` foi instalado ou fez upgrade. Forneça o nome exato do arquivo no campo *Nome do Arquivo de Backup*.

image::backup_restore/restore/default.png[]

[#_an_s3_compatible_object_store]
=== Um armazenamento de objetos compatível com S3

Selecione esta opção se nenhuma localização de armazenamento padrão estiver configurada no nível do operador, OU se o arquivo de backup existir em um bucket S3 diferente do que foi configurado como a localização de armazenamento padrão. Forneça o nome exato do arquivo no campo *Nome do Arquivo de Backup*. Consulte <<_getting_the_backup_filename_from_s3,esta seção>> para obter os passos exatos sobre como obter o nome do arquivo de backup do S3. Preencha todos os detalhes para o armazenamento de objetos compatível com S3. Seus campos são exatamente os mesmos que os da configuração `backup.StorageLocation` na {xref-rancher-backup-config}#_storage_location[Recurso personalizado de Backup.]

image::backup_restore/restore/s3store.png[]

[#_encryption]
== Criptografia

Se o backup foi criado com a criptografia habilitada, seu arquivo terá o sufixo `.enc`. Escolhendo um backup assim, ou fornecendo um nome de arquivo de backup com o sufixo `.enc`, será exibido outro menu suspenso chamado *Segredo da Configuração de Criptografia*.

image::backup_restore/restore/encryption.png[]

O segredo selecionado neste menu suspenso deve ter o mesmo conteúdo que o utilizado para o recurso personalizado de backup durante a realização do backup. Se a configuração de criptografia não corresponder, a restauração falhará.

O menu suspenso `Encryption Config Secret` filtrará e listará apenas aqueles segredos que possuem essa chave exata.

|===
| Nome da Diretiva YAML | Descrição

| `encryptionConfigSecretName`
| Forneça o nome do Secret do namespace `cattle-resources-system`, que contém o arquivo de configuração de criptografia.
|===

[NOTE]
.Importante:
====

Este campo deve ser definido apenas se o backup foi criado com a criptografia habilitada. Fornecer a configuração de criptografia incorreta fará com que a restauração falhe.
====


[#_prune_during_restore]
== Podar Durante a Restauração

* *Podar*:  Para restaurar completamente o Rancher a partir de um backup, e voltar ao estado exato em que estava quando o backup foi realizado, precisamos excluir quaisquer recursos adicionais que foram criados pelo Rancher após o backup ser feito. O operador faz isso se a flag *Podar* estiver habilitada. Podar está habilitado por padrão e é recomendado mantê-lo habilitado.
* *Tempo Limite de Exclusão*: Este é o tempo que o operador aguardará enquanto exclui um recurso antes de editar o recurso para remover finalizadores e tentar a exclusão novamente.

|===
| Nome da Diretiva YAML | Descrição

| `prune`
| Excluir os recursos gerenciados pelo Rancher que não estão presentes no backup (Recomendado).

| `deleteTimeoutSeconds`
| Quantidade de tempo que o operador aguardará enquanto exclui um recurso antes de editar o recurso para remover finalizadores e tentar a exclusão novamente.
|===

[#_getting_the_backup_filename_from_s3]
== Obtendo o nome de arquivo de backup do S3

Este é o nome do arquivo de backup que o operador `rancher-backup` usará para realizar a restauração.

Para obter este nome de arquivo do S3, vá para o seu bucket S3 (e pasta, se foi especificada durante a realização do backup).

Copie o nome de arquivo e armazene-o no seu recurso personalizado de Restauração. Então, assumindo que o nome do seu arquivo de backup é `backupfile`,

* Se o nome do seu bucket for `s3bucket` e nenhuma pasta foi especificada, então o `backupFilename` a ser usado será `backupfile`.
* Se o nome do seu bucket for `s3bucket` e a pasta base for ``s3folder``, o `backupFilename` a ser usado é apenas `backupfile`.
* Se houver uma subpasta dentro de `s3Folder` chamada `s3sub`, e essa contiver seu arquivo de backup, então o `backupFilename` a ser usado é `s3sub/backupfile`.

|===
| Nome da Diretiva YAML | Descrição

| `backupFilename`
| Este é o nome do arquivo de backup que o operador `rancher-backup` usará para realizar a restauração.
|===
