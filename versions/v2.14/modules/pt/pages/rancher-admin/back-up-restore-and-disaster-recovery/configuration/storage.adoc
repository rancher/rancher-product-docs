Configure um local de armazenamento onde todos os backups são salvos por padrão. Você terá a opção de substituir isso em cada backup, mas estará limitado a usar um armazenamento de objetos compatível com S3.

Apenas um local de armazenamento pode ser configurado no nível do operador.

[#_storage_location_configuration]
== Configuração do Local de Armazenamento

[#_no_default_storage_location]
=== Sem Local de Armazenamento Padrão

Você pode optar por não ter nenhum local de armazenamento configurado no nível do operador. Se você selecionar esta opção, deve configurar um armazenamento de objetos compatível com S3 como o local de armazenamento para cada backup individual.

[#_s3_compatible_object_store]
=== Armazenamento de Objetos Compatível com S3

|===
| Parâmetro | Descrição

| Segredo de Credenciais
| Escolha as credenciais para S3 a partir de seus segredos no Rancher. {xref-rancher-backup-config}#_example_credentialsecret[Exemplo].

| Nome do Bucket
| Digite o nome do https://docs.aws.amazon.com/AmazonS3/latest/dev/UsingBucket.html[bucket S3] onde os backups serão armazenados. Padrão: `rancherbackups`.

| Região
| A https://aws.amazon.com/about-aws/global-infrastructure/regions_az/[região da AWS] onde o bucket S3 está localizado.

| Pasta
| A https://docs.aws.amazon.com/AmazonS3/latest/user-guide/using-folders.html[pasta no bucket S3] onde os backups serão armazenados. Se este campo for deixado em branco, o comportamento padrão é armazenar os arquivos de backup na pasta raiz do bucket S3.

| Segurança
| O https://docs.aws.amazon.com/general/latest/gr/s3.html[endpoint S3], por exemplo, `s3.us-west-2.amazonaws.com`.

| Endpoint CA
| O certificado CA usado para o endpoint S3. Padrão: certificado CA codificado em base64

| insecureTLSSkipVerify
| Defina como verdadeiro se você não estiver usando TLS.
|===

[#_existing_storageclass]
=== StorageClass existente

Instalar o `rancher-backup` chart selecionando a opção StorageClass criará uma Persistent Volume Claim (PVC), e o Kubernetes, por sua vez, provisionará dinamicamente uma Persistent Volume (PV) onde todos os backups serão salvos por padrão.

Para informações sobre como criar classes de armazenamento, consulte {xref-dynamically-provision-new-storage}[esta seção.]

[NOTE]
.Importante:
====

É altamente recomendável usar uma StorageClass com uma política de recuperação de "Retain". Caso contrário, se a PVC criada pelo `rancher-backup` chart for excluída (seja durante o fazer upgrade do app ou acidentalmente), a PV também será excluída, o que significa que todos os backups salvos nela serão excluídos.
Se nenhuma StorageClass desse tipo estiver disponível, após a PV ser provisionada, certifique-se de editar sua política de recuperação e defini-la como "Retain" antes de armazenar backups nela.
====


[#_existing_persistent_volume]
=== Persistent Volume existente

Selecione um Persistent Volume (PV) existente que será usado para armazenar seus backups. Para informações sobre como criar PersistentVolumes no Rancher, consulte {xref-existing-storage-setup}#_2_add_a_persistentvolume_that_refers_to_the_persistent_storage[esta seção.]

[NOTE]
.Importante:
====

É altamente recomendável usar um Persistent Volume com uma política de recuperação de "Retain". Caso contrário, se a PVC criada pelo `rancher-backup` chart for excluída (seja durante o fazer upgrade do app ou acidentalmente), a PV também será excluída, o que significa que todos os backups salvos nela serão excluídos.
====
