Este recurso habilita uma interface do usuário que permite criar, ler, atualizar e excluir serviços virtuais e regras de destino, que são recursos de gerenciamento de tráfego do Istio.

____
*Pré-requisitos:* Ativar este recurso não habilita o Istio. Um administrador de cluster precisa {xref-istio-guides}[habilitar o Istio para o cluster] para usar o recurso.
____

Para habilitar ou desabilitar este recurso, consulte as instruções na {xref-experimental-features}[página principal sobre como habilitar recursos experimentais.]

[#_about_this_feature]
== Sobre este Recurso

Uma vantagem central dos recursos de gerenciamento de tráfego do Istio é que eles permitem o roteamento dinâmico de requisições, o que é útil para implantações canário, implantações blue/green ou testes A/B.

Quando habilitado, este recurso ativa uma página que permite configurar alguns recursos de gerenciamento de tráfego do Istio usando a interface do usuário do Rancher. Sem este recurso, você precisa usar `kubectl` para gerenciar o tráfego com o Istio.

O recurso habilita duas abas na interface: uma aba para *Serviços Virtuais* e outra para *Regras de Destino*.

* *Serviços virtuais* interceptam e direcionam o tráfego para seus serviços Kubernetes, permitindo que você direcione porcentagens de tráfego de uma requisição entre diferentes serviços. Você pode usá-los para definir um conjunto de regras de roteamento a serem aplicadas quando um host é endereçado. Para detalhes, consulte a https://istio.io/docs/reference/config/networking/v1alpha3/virtual-service/[documentação do Istio.]
* *Regras de destino* servem como a única fonte de verdade sobre quais versões de serviço estão disponíveis para receber tráfego dos serviços virtuais. Você pode usar esses recursos para definir políticas que se aplicam ao tráfego destinado a um serviço após o roteamento ter ocorrido. Para detalhes, consulte a https://istio.io/docs/reference/config/networking/v1alpha3/destination-rule[documentação do Istio.]

Para ver essas abas,

. Clique em *☰ > Gerenciamento de Cluster*.
. Vá para o cluster onde o Istio está instalado e clique em *Explorar*.
. Na barra de navegação à esquerda, clique em *Istio*.
. Você verá abas para *Kiali* e *Jaeger*. Na barra de navegação à esquerda, você pode visualizar e configurar *Serviços Virtuais* e *Regras de Destino*.
