Este recurso permite que você use tipos para provedores de armazenamento e provisionadores que não estão habilitados por padrão.

Para habilitar ou desabilitar este recurso, consulte as instruções na {xref-experimental-features}[página principal sobre como habilitar recursos experimentais.]

|===
| Chave da Variável de Ambiente | Valor Padrão | Descrição

| `unsupported-storage-drivers`
| `false`
| Este recurso habilita tipos para provedores de armazenamento e provisionadores que não estão habilitados por padrão.
|===

[#_types_for_persistent_volume_plugins_that_are_enabled_by_default]
== Tipos para Plugins de Volume Persistente que Estão Habilitados por Padrão

Abaixo está uma lista de tipos de armazenamento para plugins de volume persistente que estão habilitados por padrão. Ao habilitar este sinalizador de recurso, todos os plugins de volume persistente que não estiverem nesta lista serão considerados experimentais e não suportados:

|===
| Nome | Plugin

| Amazon EBS Disk
| `aws-ebs`

| AzureFile
| `azure-file`

| AzureDisk
| `azure-disk`

| Disco Persistente do Google
| `gce-pd`

| Volume VMware vSphere
| `vsphere-volume`

| Local
| `local`

| Network File System (Sistema de arquivos de rede)
| `nfs`

| hostPath
| `host-path`
|===
