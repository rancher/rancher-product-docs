As informações de acesso à conta são armazenadas como uma credencial em nuvem. As credenciais em nuvem são armazenadas como segredos do Kubernetes. Vários templates de nó podem usar a mesma credencial em nuvem. Você pode usar uma credencial em nuvem existente ou criar uma nova.

[#_droplet_options]
== Opções de Droplet

As *Opções de Droplet* provisionam a região geográfica e as especificações do seu cluster.

[#_docker_daemon]
== Daemon do Docker

Se você usar o Docker, as opções de configuração do https://docs.docker.com/engine/docker-overview/#the-docker-daemon[daemon do Docker] incluem:

* *Rótulos:* Para informações sobre rótulos, consulte a https://docs.docker.com/config/labels-custom-metadata/[documentação de rótulos de objetos do Docker.]
* *URL de Instalação do Docker Engine:* Determina qual versão do Docker será instalada na instância.
* *Espelhos de registro:* Espelho do registro do Docker a ser usado pelo daemon do Docker
* *Outras opções avançadas:* Consulte a https://docs.docker.com/engine/reference/commandline/dockerd/[referência de opções do daemon do Docker].
