[#_basic_options]
== opções básicas

[#_environment]
=== Ambiente

A Microsoft oferece múltiplas https://docs.microsoft.com/en-us/cli/azure/cloud?view=azure-cli-latest[nuvens] para conformidade com as leis regionais, que estão disponíveis para seu uso:

* AzurePublicCloud
* AzureGermanCloud
* AzureChinaCloud
* AzureUSGovernmentCloud

[#_location]
=== Localização

Configure a https://docs.microsoft.com/en-us/azure/virtual-machines/regions[localização] do cluster e do nó.

[#_resource_group]
=== Grupo de Recursos

Um grupo de recursos é um contêiner que mantém recursos relacionados para uma solução do Azure. O grupo de recursos pode incluir todos os recursos da solução ou apenas aqueles recursos que você deseja gerenciar como um grupo. Você decide como deseja alocar recursos para grupos de recursos com base no que faz mais sentido para sua organização. Geralmente, adicione recursos que compartilham o mesmo ciclo de vida ao mesmo grupo de recursos para que você possa implantá-los, atualizá-los e excluí-los facilmente como um grupo.

Use um grupo de recursos existente ou insira um nome de grupo de recursos e um será criado para você.

Para informações sobre como gerenciar grupos de recursos, consulte a https://docs.microsoft.com/en-us/azure/azure-resource-manager/management/manage-resource-groups-portal[documentação do Azure.]

[#_availability_set_unmanaged]
=== Conjunto de Disponibilidade (não gerenciado)

Nome ou ID de um https://docs.microsoft.com/en-us/azure/virtual-machines/availability-set-overview[conjunto de disponibilidade] existente para adicionar a VM.

[#_image]
=== Imagem

O nome da imagem do sistema operacional fornecida como um identificador de recurso da família de arquiteturas ARM. Requer o uso de disco gerenciado.

[#_vm_size]
=== Tamanho da VM

Escolha um tamanho para cada VM no pool de nós. Para detalhes sobre cada tamanho de VM, consulte https://azure.microsoft.com/en-us/pricing/details/virtual-machines/linux/[esta página.]

[#_advanced_options]
== Opções Avançadas

[#_fault_domain_count]
=== Contagem de Domínio de Falha

Domínios de falha definem o grupo de máquinas virtuais que compartilham uma fonte de energia e um switch de rede comuns. Se o conjunto de disponibilidade já foi criado, a contagem de domínios de falha será ignorada.

Para mais informações sobre domínios de falha, consulte https://docs.microsoft.com/en-us/azure/virtual-machines/availability-set-overview#how-do-availability-sets-work[aqui].

[#_update_domain_count]
=== Contagem de Domínios de Atualização

Domínios de atualização indicam grupos de máquinas virtuais e hardware físico subjacente que podem ser reiniciados ao mesmo tempo. Se o conjunto de disponibilidade já foi criado, a contagem de domínios de atualização será ignorada.

Para mais informações sobre domínios de atualização, consulte https://docs.microsoft.com/en-us/azure/virtual-machines/availability-set-overview#how-do-availability-sets-work[aqui].

[#_purchase_plan]
=== Plano de Compra

Algumas imagens de VM no Azure Marketplace requerem um plano. Se aplicável, selecione um plano de compra, formatado como `publisher:product:plan`, para usar com a imagem escolhida.

[#_subnet]
=== Sub-rede

O nome da sub-rede ao criar uma nova VNet ou referenciando uma existente.

Padrão: `docker-machine`

[#_subnet_prefix]
=== Prefixo da Sub-rede

O prefixo do endereço IP da sub-rede a ser usado ao criar uma nova VNet no formato CIDR.

Padrão: `192.168.0.0/16`

[#_virtual_network]
=== Rede virtual

A https://docs.microsoft.com/en-us/azure/virtual-network/virtual-networks-overview[rede virtual] a ser usada ou criada se uma não existir. Formatado como `[resourcegroup:]name`.

[#_public_ip_options]
=== Opções de IP Público

[#_no_public_ip]
==== Sem IP Público

Não alocar um endereço IP público.

[#_static_public_ip]
==== IP Público Estático

Aloque um endereço IP público estático.

[#_use_private_ip]
=== Use IP Privado

Use um endereço IP privado estático.

[#_private_ip_address]
=== Endereço IP privado

Configure um endereço IP privado estático para usar.

[#_network_security_group]
=== Grupo de Segurança de Rede

O https://docs.microsoft.com/en-us/azure/virtual-network/network-security-groups-overview[grupo de segurança de rede] a ser utilizado. Todos os nós que usam este modelo utilizarão o grupo de segurança de rede fornecido. Se nenhum grupo de segurança de rede for fornecido, um novo será criado para cada nó.

[#_dns_label]
=== DNS Label

Um rótulo de nome DNS exclusivo para o endereço IP público.

[#_storage_type]
=== Tipo de armazenamento

O https://docs.microsoft.com/en-us/azure/storage/common/storage-account-overview[tipo de conta de armazenamento] a ser utilizado com suas VMs. As opções incluem Standard LRS, Standard ZRS, Standard GRS, Standard RAGRS e Premium LRS.

[#_use_managed_disks]
=== Use Discos Gerenciados

https://docs.microsoft.com/en-us/azure/virtual-machines/managed-disks-overview[discos gerenciados do Azure] são volumes de armazenamento em nível de bloco que são gerenciados pelo Azure e usados com Máquinas Virtuais do Azure. Discos gerenciados são projetados para 99,999% de disponibilidade. Discos gerenciados alcançam isso fornecendo três réplicas de seus dados, permitindo alta durabilidade.

[#_managed_disk_size]
=== Tamanho do disco gerenciado

O tamanho em GB para o disco de cada nó.

[#_ssh_username]
=== Nome de usuário SSH

O nome de usuário utilizado para criar uma conexão SSH com seus nós.

[#_open_port]
=== Porta aberta

Abre o tráfego de entrada nas portas especificadas. Ao usar um Grupo de Segurança de Rede existente, as Portas Abertas são ignoradas.

Padrão: `2379/tcp, 2380/tcp, 6443/tcp, 9796/tcp, 10250/tcp, 10251/tcp, 10252/tcp, 10256/tcp` e `8472/udp, 4789/udp`
