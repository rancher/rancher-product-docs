Esta seção descreve como criar um nome de usuário e uma senha do VMware vSphere. Você deve fornecer essas credenciais do vSphere ao Rancher, o que permite que o Rancher aprovisione recursos no vSphere.

A tabela a seguir lista as permissões necessárias para a conta de usuário do vSphere:

|===
| Grupo de Privilégios | Operações

| Privilégios Cns
| Pesquisável

| Biblioteca de conteúdo
| Ler Armazenamento +
(Necessário apenas se implantar o Template a partir da Biblioteca de Conteúdo)

| Operações Criptográficas
| Acesso Direto

| Armazenamento de dados
| AllocateSpace +
Pesquisar +
Gerenciamento de Arquivos (Operações de arquivo de baixo nível) +
UpdateVirtualMachineFiles +
UpdateVirtualMachineMetadata

| Global
| Definir atributo personalizado

| Rede
| Designar

| Resource
| AssignVMToPool

| Máquina virtual
| Configuração (Todas) +
Operações do Convidado (Todas) +
Interagir (Todas) +
Inventário (Todas) +
Aprovisionamento (Todas)

| Marcação do vSphere
| Atribuir ou Remover Etiqueta do vSphere +
Atribuir ou Remover Etiqueta do vSphere em Objeto
|===

Os seguintes passos criam uma função com os privilégios necessários e, em seguida, a atribuem a um novo usuário no console do vSphere:

. No console *vSphere*, vá para a página *Administração*.
. Vá para a aba *Funções*.
. Crie uma nova função.  Dê um nome e selecione os privilégios listados na tabela de permissões acima.
+
image::rancherroles1.png[]

. Vá para a aba *Usuários e Grupos*.
. Crie um novo usuário. Preencha o formulário e clique em *OK*. Certifique-se de anotar o nome de usuário e a senha, pois você precisará deles ao criar credenciais de nuvem no Rancher.
+
image::rancheruser.png[]

. Vá para a aba *Permissões Globais*.
. Crie uma nova Permissão Global. Adicione o usuário que você criou anteriormente e atribua a ele a função que você criou anteriormente. Clique em *OK*.
+
image::globalpermissionuser.png[]
+
image::globalpermissionrole.png[]

*Resultado:* Agora você tem credenciais que o Rancher pode usar para manipular recursos do vSphere.
