Para mais detalhes sobre os nós do EC2, consulte a documentação oficial para o https://aws.amazon.com/ec2[Console de Gerenciamento do EC2].

[#_basic_options]
== opções básicas

[#_region]
=== Região

A https://docs.aws.amazon.com/AWSEC2/latest/UserGuide/using-regions-availability-zones.html[região] geográfica na qual construir seu cluster.

[#_zone]
=== Zona

A https://docs.aws.amazon.com/AWSEC2/latest/UserGuide/using-regions-availability-zones.html#concepts-availability-zones[zona], um local isolado dentro de uma região para construir seu cluster

[#_instance_type]
=== Tipo de Instância

O https://docs.aws.amazon.com/AWSEC2/latest/UserGuide/instance-types.html[tipo de instância], que determina as características de hardware, usado para provisionar seu cluster.

[#_root_disk_size]
=== Tamanho do Disco Raiz

Configure o tamanho (em GB) para seu https://docs.aws.amazon.com/AWSEC2/latest/UserGuide/RootDeviceStorage.html[dispositivo raiz].

[#_vpc_subnet]
=== VPC/Subnet

A https://docs.aws.amazon.com/vpc/latest/userguide/configure-your-vpc.html[VPC] ou https://docs.aws.amazon.com/vpc/latest/userguide/configure-subnets.html[sub-rede] específica, um intervalo de IP na sua VPC, para adicionar seus recursos.

[#_iam_instance_profile_name]
=== Nome do Perfil de Instância IAM

O nome do https://docs.aws.amazon.com/IAM/latest/UserGuide/id_roles_use_switch-role-ec2_instance-profiles.html[perfil de instância] usado para passar uma função IAM para uma instância EC2.

[#_advanced_options]
== Opções Avançadas

[#_ami_id]
=== ID DA AMI

A https://docs.aws.amazon.com/AWSEC2/latest/UserGuide/AMIs.html[Imagem de Máquina da Amazon] usada para os nós em seu cluster.

[#_ssh_username_for_ami]
=== Nome de Usuário SSH para AMI

O nome de usuário para conectar-se às suas instâncias lançadas. Consulte https://docs.aws.amazon.com/AWSEC2/latest/UserGuide/connection-prereqs.html[aqui] para os nomes de usuário padrão das AMIs selecionadas. Para AMIs não listadas, verifique com o provedor da AMI.

[#_security_group]
=== Grupo de Segurança

Escolha o grupo de segurança padrão ou configure um grupo de segurança.

Por favor, consulte {xref-port-reqs}#_rancher_aws_ec2_security_group[grupo de segurança do Amazon EC2 ao usar o Driver de Nó] para ver quais regras são criadas no `rancher-nodes` Grupo de Segurança.

[#_ebs_root_volume_type]
=== Tipo de Volume Raiz EBS

O https://docs.aws.amazon.com/AWSEC2/latest/UserGuide/ebs-volume-types.html[tipo de volume EBS] a ser usado para o dispositivo raiz.

[#_encrypt_ebs_volume]
=== Criptografar Volume EBS

Ativar https://docs.aws.amazon.com/AWSEC2/latest/UserGuide/EBSEncryption.html[Criptografia do Amazon EBS].

[#_request_spot_instance]
=== Solicitar Instância Spot

Ativar a opção de https://docs.aws.amazon.com/AWSEC2/latest/UserGuide/spot-requests.html[solicitar instâncias spot] e especificar o preço máximo por hora que você está disposto a pagar.

[#_use_only_private_address]
=== Usar apenas endereço privado

Ativar a opção de usar apenas https://docs.aws.amazon.com/AWSEC2/latest/UserGuide/using-instance-addressing.html[endereços privados].

[#_ebs_optimized_instance]
=== Instância Otimizada para EBS

Usar uma https://docs.aws.amazon.com/AWSEC2/latest/UserGuide/ebs-optimized.html[instância otimizada para EBS].

[#_allow_access_to_ec2_metadata]
=== Permitir acesso aos metadados do EC2

Ativar acesso a https://docs.aws.amazon.com/AWSEC2/latest/UserGuide/ec2-instance-metadata.html[metadados do EC2].

[#_use_tokens_for_metadata]
=== Usar tokens para metadados

Usar https://docs.aws.amazon.com/AWSEC2/latest/UserGuide/configuring-instance-metadata-service.html[Serviço de Metadados da Instância Versão 2 (IMDSv2)], um método baseado em token para acessar metadados.

[#_add_tag]
=== Adicionar Tag

Adicionar metadados usando https://docs.aws.amazon.com/AWSEC2/latest/UserGuide/Using_Tags.html[tags] para categorizar recursos.

[#_ipv6_address_count]
=== Contagem de Endereços IPv6

Especificar quantos endereços IPv6 atribuir à interface de rede da instância.

[#_ipv6_address_only]
=== Apenas Endereço IPv6

Ativar esta opção se a instância deve usar exclusivamente IPv6. VPCs ou sub-redes somente IPv6 requerem isso. Quando ativado, a instância tem IPv6 como seu único endereço, e a Contagem de Endereços IPv6 deve ser maior que zero.

[#_http_protocol_ipv6]
=== HTTP Protocol IPv6

Ativar ou desativar endpoints IPv6 para o serviço de metadados da instância.

[#_enable_primary_ipv6]
=== Ativar IPv6 primário

Ative esta opção para designar o primeiro endereço IPv6 atribuído como o endereço primário. Isso garante um endereço IPv6 consistente e inalterável para a instância. Isso não controla se os endereços IPv6 são atribuídos.
