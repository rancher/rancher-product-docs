[#_instance_options]
== Opções de Instância

Na seção *Instance Options*, configure o número de vCPUs, memória e tamanho do disco para as VMs criadas por este modelo.

[cols="<,^,<,<"]
|===
| Parâmetro | Obrigatória | Descrição | Default

| CPUs
|
| Número de vCPUs alocadas para a VM (núcleos)
| 2

| Memória
|
| Quantidade de RAM alocada para a VM (MB)
| 2 GB

| Imagem do Modelo
| ✓
| Nome do modelo de imagem de disco a ser clonado como o disco principal da VM (deve suportar cloud-init)
|

| Tamanho do Disco da VM
|
| Novo tamanho do disco principal da VM (em GiB)
|

| Tamanho do Disco Adicional
|
| Tamanho de um disco adicional a ser adicionado à VM (em GiB)
|

| Contêiner de Armazenamento
|
| Contêiner de armazenamento _UUID_ no qual provisionar um disco adicional
|

| Cloud Config YAML
|
| Cloud-init para fornecer à VM (será aplicado um patch com o usuário root do Rancher)
|

| Rede
| ✓
| Nome(s) da(s) rede(s) a serem anexadas à VM
|

| Categorias da VM
|
| Nome(s) de quaisquer categorias a serem aplicadas à VM
|
|===

A VM pode usar qualquer sistema operacional Linux moderno que esteja configurado com suporte para https://cloudinit.readthedocs.io/en/latest/[cloud-init] usando a https://cloudinit.readthedocs.io/en/latest/reference/datasources/configdrive.html[fonte de dados Config Drive v2].

[#_networks]
== Redes

O modelo de nó permite que uma VM seja provisionada com várias redes. No campo *Rede*, você pode clicar em *Adicionar* para adicionar quaisquer redes disponíveis para você no AOS.

[#_vm_categories]
== Categorias da VM

Uma categoria é um agrupamento de entidades em um par chave-valor. Normalmente, as VMs são atribuídas a uma categoria com base em algum critério. As políticas podem então ser vinculadas a essas entidades que são atribuídas (agrupadas por) um valor de categoria específico.

[#_cloud_init]
== cloud-init

https://cloudinit.readthedocs.io/en/latest/[Cloud-init] permite que você inicialize seus nós aplicando configuração na primeira inicialização. Isso pode envolver coisas como criar usuários ou autorizar chaves SSH.

Para fazer uso da inicialização do cloud-init, cole uma configuração de nuvem usando uma sintaxe YAML válida no campo *YAML de Configuração de Nuvem*. Consulte a https://cloudinit.readthedocs.io/en/latest/topics/examples.html[documentação do cloud-init] para um conjunto comentado de exemplos de diretivas de configuração de nuvem suportadas.

Observe que a configuração de rede baseada em cloud-init não é recomendada e é suportada apenas via dados do usuário `runcmd` em vez de NoCloud ou outras fontes de dados de configuração de rede.

O Gerenciamento de Endereços IP Nutanix (IPAM) ou outro serviço DHCP é recomendado.

[#_engine_options]
== Opções de Motor

Na seção *Opções de Motor* do modelo de nó, você pode configurar o daemon de contêiner. Você pode querer especificar a versão do contêiner ou um espelho de registro de imagem de contêiner.

[NOTE]
====
Se você estiver provisionando nós do Red Hat Enterprise Linux (RHEL) ou CentOS, deixe o campo *URL de instalação do Docker* como o valor padrão ou selecione *nenhum*. Isso ignorará uma verificação para a instalação do Docker, já que o Docker já está instalado nesses tipos de nós.

Se você definir *URL de instalação do Docker* para um valor diferente do padrão ou *nenhum*, pode ver uma mensagem de erro como a seguinte: `Error creating machine: RHEL ssh command error: command: sudo -E yum install -y curl err: exit status 1 output: Updating Subscription Management repositories.`
====
