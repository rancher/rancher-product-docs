ifeval::["{build-type}" != "srfa"]
Neste cenário, o Rancher não provisiona Kubernetes porque ele é instalado por provedores como Google Kubernetes Engine (GKE), Amazon Elastic Container Service for Kubernetes ou Azure Kubernetes Service.

Se você usar um provedor de Kubernetes como o Google GKE, o Rancher se integra com suas APIs de nuvem, permitindo que você crie e gerencie o controle de acesso com base em função para o cluster hospedado a partir da interface do Rancher.

Neste caso de uso, o Rancher envia uma solicitação a um provedor hospedado usando a API do provedor. O provedor então provisiona e hospeda o cluster para você. Quando o cluster terminar de ser construído, você pode gerenciá-lo a partir da interface do Rancher junto com clusters que você provisionou que estão hospedados localmente ou em um provedor de infraestrutura.

O Rancher suporta os seguintes provedores de Kubernetes:

* https://cloud.google.com/kubernetes-engine/[Google GKE (Google Kubernetes Engine)]
* https://aws.amazon.com/eks/[Amazon EKS (Amazon Elastic Container Service for Kubernetes)]
* https://azure.microsoft.com/en-us/services/kubernetes-service/[Microsoft AKS (Azure Kubernetes Service)]

endif::[]

== Autenticação de Provedor de Kubernetes Hospedado
