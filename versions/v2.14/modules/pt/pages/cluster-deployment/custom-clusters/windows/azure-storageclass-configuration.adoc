Se você estiver usando VMs do Azure para seus nós, pode usar https://docs.microsoft.com/en-us/azure/aks/azure-files-dynamic-pv[Arquivos do Azure] como uma StorageClass para o cluster.

Para que a plataforma Azure crie os recursos de armazenamento necessários, siga estas etapas:

. {xref-azure-cloud-provider}[Configure o provedor de nuvem Azure.]
. Configure `kubectl` para se conectar ao seu cluster.
. Copie o manifesto `ClusterRole` e `ClusterRoleBinding` para a conta de serviço:
  ```yml
  --
  apiVersion: rbac.authorization.k8s.io/v1
  kind: ClusterRole
  metadata:
    name: system:azure-cloud-provider
  rules:
 ** apiGroups: ['']
resources: ['secrets']
verbs:     ['get','create']
  --
  apiVersion: rbac.authorization.k8s.io/v1
  kind: ClusterRoleBinding
  metadata:
name: system:azure-cloud-provider
  roleRef:
kind: ClusterRole
apiGroup: rbac.authorization.k8s.io
name: system:azure-cloud-provider
  subjects:
 ** kind: ServiceAccount
name: persistent-volume-binder
namespace: kube-system
  ```
. Crie isso em seu cluster usando um dos seguintes comandos.
+
----
# kubectl create -f <MANIFEST>
----
