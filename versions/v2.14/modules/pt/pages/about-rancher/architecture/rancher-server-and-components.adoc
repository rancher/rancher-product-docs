A maioria do software Rancher 2.x é executada no Servidor Rancher. O Servidor Rancher inclui todos os componentes de software usados para gerenciar toda a implantação do Rancher.

O diagrama abaixo ilustra a arquitetura de alto nível do Rancher 2.x. A figura retrata uma instalação do Servidor Rancher que gerencia dois clusters Kubernetes downstream: um criado pelo RKE2 e outro criado pelo Amazon EKS (Elastic Kubernetes Service).

Para melhor desempenho e segurança, recomendamos um cluster Kubernetes dedicado para o servidor de gerenciamento do Rancher. Executar cargas de trabalho de usuários neste cluster não é aconselhável. Após implantar o Rancher, você pode {xref-cluster-deployment}[criar ou importar clusters] para executar suas cargas de trabalho.

O diagrama mostra como os usuários podem manipular tanto os clusters Kubernetes {xref-launch-k8s-with-rancher}[lançados pelo Rancher] quanto os clusters Kubernetes {xref-hosted-k8s-clusters}[hospedados] através do proxy de autenticação do Rancher:

image::rancher-architecture-rancher-api-server.svg[Arquitetura]

Você pode instalar o Rancher em um único nó ou em um cluster Kubernetes de alta disponibilidade.

Uma instalação Kubernetes de alta disponibilidade é recomendada para produção.

Uma instalação do Rancher via Docker é recomendada apenas para desenvolvimento e testes. A capacidade de migrar o Rancher para um cluster de alta disponibilidade depende da versão do Rancher.

Para ver os recursos implantados, execute o seguinte comando:

[,bash]
----
kubectl get all -n <namespace>
----

Se você tiver privilégios de administrador, também pode ver esses recursos listados na interface do usuário do Rancher:

. Clique *☰* e selecione um cluster.
. Selecione *Mais Recursos* no menu de navegação lateral para visualizar recursos implantados por tipo.
. Selecione *Cluster* > *Projetos/Namespaces* no menu de navegação lateral, em seguida, selecione um namespace para visualizar recursos implantados por namespace.
O operador de backup do Rancher pode ser usado para migrar o Rancher da instalação em um único contêiner Docker para uma instalação em um cluster Kubernetes de alta disponibilidade. Para mais detalhes, consulte a documentação sobre {xref-migrate-cluster}[migrando o Rancher para um novo cluster].

O servidor Rancher, independentemente do método de instalação, deve sempre ser executado em nós que são separados dos clusters de usuários downstream que ele gerencia. Se o Rancher estiver instalado em um cluster Kubernetes de alta disponibilidade, ele deve ser executado em um cluster separado dos clusters que ele gerencia.
