[#_grafana_ui]
== Interface do Grafana

https://grafana.com/grafana/[Grafana] permite que você consulte, visualize, receba alertas e entenda suas métricas, não importa onde elas estejam armazenadas. Crie, explore e compartilhe painéis com sua equipe e promova uma cultura orientada a dados.

Para ver os painéis padrão para visualização de dados de séries temporais, acesse a interface do Grafana.

[#_alertmanager_ui]
== Interface do Alertmanager

Quando `rancher-monitoring` está instalado, a interface do Alertmanager do Prometheus é implantada, permitindo que você veja seus alertas e a configuração atual do Alertmanager.

[NOTE]
====

Esta seção assume familiaridade com o funcionamento conjunto dos componentes de monitoramento. Para mais informações sobre o Alertmanager, veja xref:./how-monitoring-works.adoc#_3_how_alertmanager_works[Como o Alertmanager Funciona.]
====


[#_accessing_the_alertmanager_ui]
=== Acessando a interface do Alertmanager

A interface do Alertmanager permite que você veja os alertas disparados mais recentemente.

[NOTE]
.Pré-requisitos:
====

O aplicativo `rancher-monitoring` deve estar instalado.
====


Para ver a interface do Alertmanager,

. No canto superior esquerdo, clique em *☰ > Gerenciamento de Cluster*.
. Na página *Clusters*, vá para o cluster onde você deseja ver a interface do Alertmanager e clique em *Explorar*.
. Na barra de navegação à esquerda, clique em *Monitoramento*.
. Clique em *Alertmanager*.

*Resultado:* A interface do Alertmanager abre em uma nova aba. Para ajuda com a configuração, consulte a https://prometheus.io/docs/alerting/latest/alertmanager/[documentação oficial do Alertmanager.]

Para mais informações sobre como configurar o Alertmanager no Rancher, veja {xref-configure-alertmanager}[esta página.]

.A interface do Alertmanager
image::alertmanager-ui.png[Interface do Alertmanager]

[#_viewing_default_alerts]
=== Visualizando Alertas Padrão

Para ver os alertas que são disparados por padrão, vá para a interface do Alertmanager e clique em *Expandir todos os grupos*.

[#_prometheus_ui]
== Interface do Prometheus

Por padrão, o https://github.com/kubernetes/kube-state-metrics[serviço kube-state-metrics] fornece uma riqueza de informações sobre a utilização de CPU e memória para o aplicativo de monitoramento. Essas métricas cobrem recursos do Kubernetes em namespaces diferentes. Isso significa que, para ver as métricas de recursos de um serviço, você não precisa criar um novo ServiceMonitor para ele. Como os dados já estão no banco de dados de séries temporais, você pode ir para a interface do Prometheus e executar uma consulta PromQL para obter as informações. A mesma consulta pode ser usada para configurar um painel do Grafana para mostrar um gráfico dessas métricas ao longo do tempo.

Para ver a interface do Prometheus, instale `rancher-monitoring`. Then:

. No canto superior esquerdo, clique em *☰ > Gerenciamento de Cluster*.
. Na página *Clusters*, vá para o cluster onde você deseja ver a interface do Prometheus e clique em *Explorar*.
. Na barra de navegação à esquerda, clique em *Monitoramento*.
. Clique em *Gráfico do Prometheus*.

.Interface do Gráfico do Prometheus
image::prometheus-graph-ui.png[Interface do Gráfico do Prometheus]

[#_viewing_the_prometheus_targets]
=== Visualizando os Alvos do Prometheus

Para ver quais serviços você está monitorando, você precisará ver seus alvos. Os Targets são configurados por ServiceMonitors e PodMonitors como fontes para coletar métricas. Você não precisará editar diretamente os Targets, mas a interface do Prometheus pode ser útil para lhe dar uma visão geral de todas as fontes de métricas que estão sendo coletadas.

Para ver os Targets do Prometheus, instale `rancher-monitoring`. Then:

. No canto superior esquerdo, clique em *☰ > Gerenciamento de Cluster*.
. Na página *Clusters*, vá para o cluster onde você deseja ver os Targets do Prometheus e clique em *Explorar*.
. Na barra de navegação à esquerda, clique em *Monitoramento*.
. Clique em *Targets do Prometheus*.

.Targets na interface do Prometheus
image::prometheus-targets-ui.png[Interface de Targets do Prometheus]

[#_viewing_the_prometheusrules]
=== Visualizando as PrometheusRules

Quando você define uma Rule (que é declarada dentro de um RuleGroup em um recurso PrometheusRule), o https://github.com/prometheus-operator/prometheus-operator/blob/main/Documentation/api-reference/api.md#rule[spec da própria Rule] contém rótulos que são usados pelo Alertmanager para determinar qual Route deve receber um determinado Alert.

Para ver as PrometheusRules, instale `rancher-monitoring`. Then:

. No canto superior esquerdo, clique em *☰ > Gerenciamento de Cluster*.
. Na página *Clusters*, vá para o cluster onde você deseja ver as visualizações e clique em *Explorar*.
. Na barra de navegação à esquerda, clique em *Monitoramento*.
. Clique em *Prometheus Rules*.

Você também pode ver as regras na interface do Prometheus:

.Regras na interface do Prometheus
image::prometheus-rules-ui.png[PrometheusRules UI]

Para mais informações sobre como configurar PrometheusRules no Rancher, veja {xref-prometheusrules}[esta página.]
