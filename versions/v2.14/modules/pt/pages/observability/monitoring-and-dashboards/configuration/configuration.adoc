Esta página captura algumas das opções mais importantes para configurar o Monitoramento V2 na interface do Rancher.

Para informações sobre como configurar alvos de coleta e regras personalizadas para o Prometheus, consulte a documentação upstream para o https://github.com/prometheus-operator/prometheus-operator[Operador Prometheus.] Alguns dos recursos personalizados mais importantes são explicados na https://github.com/prometheus-operator/prometheus-operator/blob/main/Documentation/getting-started/design.md[documentação de design do Operador Prometheus.] A documentação do Operador Prometheus também pode ajudá-lo a configurar RBAC, Thanos ou configuração personalizada.

[#_setting_resource_limits_and_requests]
== Configuração de limites e solicitações de recursos

As solicitações e limites de recursos para o aplicativo de monitoramento podem ser configurados ao instalar `rancher-monitoring`. Para mais informações sobre os limites padrão, veja {xref-helm-chart-options}#_configuring_resource_limits_and_requests[esta página.]

[TIP]
====

Em um cluster ocioso, o Monitoramento pode ter alto uso de CPU. Para melhorar o desempenho, desative o adaptador Prometheus.
====


[#_prometheus_configuration]
== Configuração do Prometheus

Geralmente, não é necessário editar diretamente o recurso personalizado do Prometheus.

Em vez disso, para configurar o Prometheus para coletar métricas personalizadas, você só precisará criar um novo ServiceMonitor ou PodMonitor para configurar o Prometheus para coletar métricas adicionais.

[#_alertmanager_configuration]
== Configuração do Alertmanager

O recurso personalizado do Alertmanager geralmente não precisa ser editado diretamente. Para a maioria dos casos de uso comuns, você pode gerenciar alertas atualizando Rotas e Receptores.

Rotas e receptores fazem parte da configuração do recurso personalizado do Alertmanager. Na interface do Rancher, Rotas e Receptores não são verdadeiros recursos personalizados, mas recursos pseudo-personalizados que o Operador Prometheus usa para sincronizar sua configuração com o recurso personalizado do Alertmanager. Quando rotas e receptores são atualizados, o aplicativo de monitoramento atualizará automaticamente o Alertmanager para refletir essas mudanças.

Para alguns casos de uso avançados, você pode querer configurar o Alertmanager diretamente. Para mais informações, consulte {xref-configure-alertmanager}[esta página.]

[#_receivers]
=== Receptores

Receptores são usados para configurar notificações. Para detalhes sobre como configurar receptores, consulte {xref-receivers}[esta página.]

[#_routes]
=== Rotas

Rotas filtram notificações antes que elas cheguem aos receptores. Cada rota precisa se referir a um receptor que já foi configurado. Para detalhes sobre como configurar rotas, consulte {xref-monitoring-routes}[esta página.]
