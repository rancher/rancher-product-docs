O Rancher contém uma variedade de ferramentas que não estão incluídas no Kubernetes para auxiliar em suas operações de DevOps. O Rancher pode se integrar a serviços externos para ajudar seus clusters a funcionarem de forma mais eficiente.

[#_notifiers_and_alerts]
== Notificadores e Alertas

Notificadores e alertas são duas funcionalidades que trabalham juntas para informá-lo sobre eventos no sistema Rancher. Antes que possam ser ativados, o aplicativo de monitoramento deve ser instalado.

Notificadores são serviços que informam sobre eventos de alerta. Você pode configurar notificadores para enviar notificações de alerta para a equipe mais adequada para tomar ações corretivas. As notificações podem ser enviadas via Slack, e-mail, PagerDuty, WeChat e webhooks.

Alertas são regras que acionam essas notificações. Antes que você possa receber alertas, deve configurar um ou mais notificadores no Rancher. O escopo dos alertas pode ser definido no nível do cluster ou do projeto.

[#_logging]
== Registro

O registro é útil porque permite que você:

* Capture e analise o estado do seu cluster
* Procure por tendências em seu ambiente
* Salve seus logs em um local seguro fora do seu cluster
* Mantenha-se informado sobre eventos como um contêiner falhando, uma evacuação de pod ou um nó morrendo
* Depure e resolva problemas mais facilmente

O Rancher pode se integrar com Elasticsearch, Splunk, Kafka, Syslog e Fluentd.

Para obter detalhes, consulte a {xref-logging}[seção de registro.]

[#_monitoring]
== Monitoramento

Usando o Rancher, você pode monitorar o estado e os processos dos nós do seu cluster, componentes do Kubernetes e implantações de software através da integração com https://prometheus.io/[Prometheus], uma solução de monitoramento de código aberto líder. Para obter detalhes, consulte a {xref-monitoring-and-dashboards}[seção de monitoramento.]
