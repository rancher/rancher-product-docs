Elementalはクラウドネイティブホスト管理を可能にします。Elementalは、データセンターにあるかエッジにあるかにかかわらず、任意の場所にある任意のマシンをオンボードし、それらをKubernetesにシームレスに統合し、ワークフロー（例：OSの更新）を管理することを可能にします。

[#__elemental_product_name_with_rancher]
== {elemental-product-name} と Rancher

RancherにおけるElemental：

* ElementalはKubernetesにネイティブ対応しており、Kubernetesクラスター内でElementalを介してOSを管理できます。
* Kubernetesの運用の観点から非破壊的です。
* 宣言的であり、GitOpsに優しいです。
* 信頼性が高く、決定論的で予測可能なOCIイメージベースのフローを許可します。
* 大規模環境でも動作します。ElementalはFleet規模のOS管理を可能にします。

[#_when_should_i_use_elemental_product_name]
=== {elemental-product-name}を使用すべき時はいつですか？

* ElementalはRancherマネージャーからクラウドネイティブなOS管理を可能にします。任意のOS（例：SLE Microバニラ）で動作します。
* Elementalは、データセンターとエッジにあるマシンのためのクラウドネイティブな管理を可能にします。
* Elementalは柔軟であり、プラットフォームチームがFleet内のマシンに対してあらゆる種類のワークフローを実行できるようにします。

[#__elemental_product_name_with_rancher_prime]
== {elemental-product-name} と {rancher-prime}

* すでにRancherのGUI拡張機能として深く統合されています。
* Rancherの機能をOSまで拡張します。本日、Rancher向けにSLE Microと完璧に連携して動作しています。
