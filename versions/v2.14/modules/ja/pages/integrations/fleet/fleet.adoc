Fleetは、複数のクラスター向けに、サプライチェーンを通じてアプリケーションの継続的デリバリをオーケストレーションし、管理します。Fleetは、GitOpsを安全な運用モデルとして活用し、チームが自信と信頼をもって、タイムリーにデリバリできるよう、サプライチェーンを統括します。

[#__fleet_product_name_with_rancher]
== {fleet-product-name} と Rancher

多くのユーザーは、同時に10以上のクラスターを管理することがよくあります。クラスターの急増を考慮すると、継続的デリバリはRancherにとって重要な部分です。Fleetは、GitOpsを使用して信頼性の高い継続的デリバリ体験を保証します。これは、安全でますます一般的な運用モデルです。

[#_when_should_i_use_fleet_product_name]
=== {fleet-product-name}を使用すべき時はいつですか？

* 私は、異なる保持ポリシーを持つ地理的地域にわたって監視スタック（例：Grafana、Prometheus）をデプロイする必要があります。
* 私はプラットフォームオペレーターであり、スケーラブルで安全な運用モデル（GitOps）を活用して、すべてのコンポーネントを備えたクラスターをプロビジョニングしたいです。
* 私はアプリケーション開発者であり、最新の変更が自動的に開発環境に反映されることを望んでいます。
