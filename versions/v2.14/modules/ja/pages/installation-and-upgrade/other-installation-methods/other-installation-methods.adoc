[#_air_gapped_installations]
== エアギャップ環境でのインストール

{xref-air-gap}[これらの手順]に従って、エアギャップ環境にRancherサーバーをインストールします。

エアギャップ環境とは、Rancherサーバーがオフラインで、ファイアウォールの背後またはプロキシの背後にインストールされる場所を指します。

[#_docker_installations]
== Dockerインストール

xref:[シングルノードDockerインストール]は、Rancherを試したいユーザー向けです。Helmを使用してKubernetesクラスター上で実行する代わりに、`docker run`コマンドを使用してシングルノードにRancherサーバーコンポーネントをインストールします。

Dockerインストールは、開発およびテスト環境専用です。

ノードが1つとシングルDockerコンテナしかないため、ノードがダウンすると、他のノードにetcdデータのコピーがなくなり、Rancherサーバーのすべてのデータが失われます。

Rancherバックアップオペレーターは、単一のDockerコンテナインストールから高可用性KubernetesクラスターへのRancherの移行に使用できます。詳細については、{xref-migrate-cluster}[新しいクラスターへのRancherの移行]に関するドキュメントを参照してください。
