このページでは、MicrosoftのAzure Kubernetes Service (AKS)にRancherをインストールする方法について説明します。

このガイドでは、コマンドラインツールを使用して、イングレスを持つAKSクラスターをプロビジョニングします。Azureポータルを使用してクラスターをプロビジョニングする場合は、 https://docs.microsoft.com/en-us/azure/aks/kubernetes-walkthrough-portal[公式ドキュメント]を参照してください。

すでにAKS Kubernetesクラスターをお持ちの場合は、<<_5_install_an_ingress,イングレスのインストールに関するステップにスキップしてください。>>次に、{xref-install-rancher}#_install_the_rancher_helm_chart[このページ]の指示に従ってRancher Helmチャートをインストールします。

[#_prerequisites]
== 前提条件

[CAUTION]
====

Microsoft Azureにデプロイすると、料金が発生します。
====


* https://azure.microsoft.com/en-us/free/[Microsoft Azureアカウント]:RancherとKubernetesをデプロイするためのリソースを作成するには、Microsoft Azureアカウントが必要です。
* https://docs.microsoft.com/en-us/azure/cost-management-billing/manage/create-subscription#create-a-subscription-in-the-azure-portal[Microsoft Azureサブスクリプション]:まだMicrosoft Azureサブスクリプションをお持ちでない場合は、このリンクを使用してチュートリアルに従ってください。
* https://docs.microsoft.com/en-us/azure/active-directory/develop/quickstart-create-new-tenant[Microsoft Azureテナント]:このリンクを使用して、Microsoft Azureテナントを作成する手順に従ってください。
* サブスクリプションには、少なくとも2つのvCPUの十分なクォータがあります。Rancherサーバーのリソース要件の詳細については、{xref-install-reqs}[このセクション]を参照してください。
* AzureでHelmを使用してRancherをインストールする際は、ネットワークの問題を避けるためにL7ロードバランサーを使用してください。詳細については、 https://docs.microsoft.com/en-us/azure/load-balancer/components#_limitations[Azureロードバランサーの制限]に関するドキュメントを参照してください。

[#_1_prepare_your_workstation]
== 1.作業ステーションを準備する

作業ステーションに以下のコマンドラインツールをインストールしてください:

* Azure CLI、*az:*ヘルプが必要な場合は、これらの https://docs.microsoft.com/en-us/cli/azure/[インストール手順]を参照してください。
* *kubectl:*ヘルプが必要な場合は、これらの https://kubernetes.io/docs/tasks/tools/#kubectl[インストール手順]を参照してください。
* *helm:*ヘルプが必要な場合は、これらの https://helm.sh/docs/intro/install/[インストール手順]を参照してください。

[#_2_create_a_resource_group]
== 2.リソースグループを作成する

CLIをインストールした後、Azureアカウントでログインする必要があります。

----
az login
----

クラスターに関連するすべてのリソースを保持するために、 https://docs.microsoft.com/en-us/azure/azure-resource-manager/management/manage-resource-groups-portal[リソースグループ]を作成してください。使用ケースに適した場所を使用してください。

----
az group create --name rancher-rg --location eastus
----

[#_3_create_the_aks_cluster]
== 3.AKSクラスターを作成する

AKSクラスターを作成するには、次のコマンドを実行します。使用ケースに適したVMサイズを使用してください。利用可能なサイズとオプションについては、 https://docs.microsoft.com/en-us/azure/virtual-machines/sizes[この記事]を参照してください。Kubernetesのバージョンを選択する際は、最初に https://rancher.com/support-matrix/[サポートマトリックス]を確認して、Rancherのバージョンに対して検証された最高のKubernetesバージョンを見つけてください。

----
az aks create \
  --resource-group rancher-rg \
  --name rancher-server \
  --kubernetes-version <VERSION> \
  --node-count 3 \
  --node-vm-size Standard_D2_v3
----

クラスターのデプロイには時間がかかります。

[#_4_get_access_credentials]
== 4.アクセス資格情報を取得する

クラスターがデプロイされた後、アクセス資格情報を取得します。

----
az aks get-credentials --resource-group rancher-rg --name rancher-server
----

このコマンドは、クラスターの資格情報を既存のkubeconfigに統合し、`kubectl`がクラスターと対話できるようにします。

[#_5_install_an_ingress]
== 5.Ingressをインストールする

クラスターには、Rancherがクラスターの外部からアクセスできるようにするためにIngressが必要です。Ingressをインストールするには、パブリックIPアドレスを割り当てる必要があります。十分なクォータがあることを確認してください。そうでないと、IPアドレスの割り当てに失敗します。パブリックIPアドレスの制限は、サブスクリプションごとに地域レベルで適用されます。Azureが提供するマネージドIngressコントローラーや、TraefikのようなサードパーティのIngressコントローラーを使用できます。

[WARNING]
====
マネージドIngressコントローラーがすでに使用されている場合、TraefikのようなサードパーティのIngressコントローラーをインストールすることは推奨されません。
====

[WARNING]
====
**Ingress-NGINX EOL:**コミュニティの`ingress-nginx`コントローラーは2026年3月にサービス終了となります。このページでは、Rancher環境に推奨される移行パスであるTraefikを使用しています。
====

Traefikには、ネイティブのIngress NGINXプロバイダーが含まれています。これにより、既存のIngressオブジェクトを再記述することなくNGINXから移行でき、Traefikは`nginx.ingress.kubernetes.io`アノテーションを自動的に解釈します。すでに`ingress-nginx`を使用しているクラスターをアップグレードする場合は、詳細についてはこの https://doc.traefik.io/traefik/migrate/nginx-to-traefik/[ガイド]に従ってください。

Traefikをインストールするには、 https://apps.rancher.io/applications/traefik-proxy[Rancher Prime Application Collection]のTraefikアプリケーションを使用することを強くお勧めします。Rancher Managerと統合するための https://docs.apps.rancher.io/howto-guides/integrate-with-rancher-manager[ドキュメントを確認してください]。


[#_6_get_load_balancer_ip]
== 6.ロードバランサーIPを取得する

ロードバランサーのアドレスを取得するには、次のコマンドを実行します：

----
kubectl get service traefik --namespace=traefik
----

結果は次のようになります：

----
NAME                       TYPE           CLUSTER-IP     EXTERNAL-IP    PORT(S)
 AGE
traefik  LoadBalancer   10.0.116.18    40.31.180.83   80:31229/TCP,443:31050/TCP 67s
----

`EXTERNAL-IP`を保存します。

[#_7_set_up_dns]
== 7.DNSを設定します。

Rancherサーバーへの外部トラフィックは、作成したロードバランサーに向ける必要があります。

保存した`EXTERNAL-IP`を指すDNSを設定します。このDNSはRancherサーバーのURLとして使用されます。

DNSを設定するための有効な方法は多数あります。ヘルプが必要な場合は、 https://docs.microsoft.com/en-us/azure/dns/[Azure DNS ドキュメント]を参照してください。

[#_8_install_the_rancher_helm_chart]
== 8.Rancher Helm チャートをインストールします。

次に、{xref-install-rancher}#_install_the_rancher_helm_chart[このページ]の指示に従って Rancher Helm チャートをインストールします。Helm の指示は、任意の Kubernetes ディストリビューションに Rancher をインストールする際に同じです。

Rancher をインストールする際には、前のステップで得た DNS 名を Rancher サーバーの URL として使用します。それは Helm オプションとして渡すことができます。例えば、DNS 名が `rancher.my.org` の場合、オプション `--set hostname=rancher.my.org` を使用して Helmインストールコマンドを実行できます。

このセットアップの上に Rancher をインストールする際には、Rancher の Ingress リソースで使用する Ingress コントローラーの名前を設定するために、以下の値を Rancher Helm インストールコマンドに渡す必要があります。

----
--set ingress.ingressClassName=traefik
----

選択した証明書オプションの Helm インストールコマンドについては、{xref-install-rancher}#_5_install_rancher_with_helm_and_your_chosen_certificate_option[こちら]を参照してください。
