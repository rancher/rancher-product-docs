*アダプターをインストールした後、Rancherに「AWS Marketplace Adapter:」というバナーメッセージが表示されます。「アダプターを実行できません。アダプターログを確認してください」*

このエラーは、アダプターがクラスターにインストールされたものの、ライセンスのチェックイン/チェックアウトを適切に行うことを妨げるエラーが発生したことを示しています。

これは、IAMロールが正しく設定されていないために発生することがよくあります。{xref-aws-cloud-marketplace-adapter-reqs}[前提条件]を確認し、次のことを確認してください:

* OIDCプロバイダーが作成され、Rancherが実行されているクラスターに関連付けられています。
* IAMロールがこのOIDCプロバイダーを信頼するように構成されています。
* IAMロールには、ポリシーに記載されている権限が少なくとも含まれています。

上記のすべてが正しく構成されている場合は、サポートに連絡して支援を求めてください。

*「AWS Marketplace Adapter:」というバナーメッセージが表示されます。ライセンスされたノード数を超えています。AWSでコンプライアンスを満たすためには、少なくともx個のライセンスが必要です」*

このメッセージは、Rancherが現在管理しているノードの数に対して十分な権利がないことを示しています。

次の制限事項に注意してください:

* 各権利は特定のノード数に対して有効です。
* Rancherが現在管理しているすべてのノードは、使用量の合計にカウントされます（Rancherがインストールされているクラスター内のノードは除く）。
* 各権利は、最大で1つのRancherインスタンスによって使用できます。例えば、アカウントに2つのRancherインスタンスが実行されている場合（それぞれ別のEKSクラスターにインストールされている）、少なくとも2つの権利が必要です。

最近アダプターをアンインストール/再インストールした可能性もあります。アダプターが現在管理しているライセンスの追跡を失った場合、ライセンスの実際の状態を解決するのに最大1時間かかることがあります。
