Rancherは、コンテナを本番環境でデプロイする組織のために構築されたコンテナ管理プラットフォームです。Rancherは、Kubernetesをどこでも簡単に実行し、IT要件を満たし、DevOpsチームを支援します。

[#_run_kubernetes_everywhere]
== どこでもKubernetesを実行する

ifeval::["{build-type}" != "srfa"]
Kubernetesは、コンテナオーケストレーションの標準となりました。ほとんどのクラウドおよび仮想化ベンダーは、これを標準インフラストラクチャとして提供しています。Rancherのユーザーは、Rancher Kubernetes Engine（RKE）またはGKE、AKS、EKSなどのクラウドKubernetesサービスを使用してKubernetesクラスターを作成する選択肢があります。Rancherのユーザーは、任意のKubernetesディストリビューションまたはインストーラーを使用して作成した既存のKubernetesクラスターをインポートして管理することもできます。
endif::[]
ifeval::["{build-type}" == "srfa"]
Kubernetesは、コンテナオーケストレーションの標準となりました。ほとんどのクラウドおよび仮想化ベンダーは、これを標準インフラストラクチャとして提供しています。Rancherのユーザーは、Rancher Kubernetes Engine（RKE）またはEKSなどのクラウドKubernetesサービスを使用してKubernetesクラスターを作成する選択肢があります。Rancherのユーザーは、任意のKubernetesディストリビューションまたはインストーラーを使用して作成した既存のKubernetesクラスターをインポートして管理することもできます。

[#_meet_it_requirements]
== IT要件を満たす

Rancherは、その管理下にあるすべてのKubernetesクラスターに対して、中央認証、アクセス制御、および監視をサポートしています。たとえば、次の処理が可能です。

ifeval::["{build-type}" != "srfa"]
* Active Directoryの資格情報を使用して、GKEなどのクラウドベンダーがホストするKubernetesクラスターにアクセスします。
endif::[]
ifeval::["{build-type}" == "srfa"]
* Active Directoryの資格情報を使用して、クラウドベンダーがホストするKubernetesクラスターにアクセスします。
endif::[]
* すべてのユーザー、グループ、プロジェクト、クラスター、およびクラウドにわたってアクセス制御とセキュリティポリシーを設定し、強制します。
* 単一の画面からKubernetesクラスターの健康状態と容量を確認します。

[#_empower_devops_teams]
== DevOpsチームを支援する

Rancherは、DevOpsエンジニアがアプリケーションのワークロードを管理するための直感的なユーザーインターフェースを提供します。ユーザーは、Rancherを使用するためにKubernetesの概念について深い知識を持っている必要はありません。Rancherカタログには、便利なDevOpsツールのセットが含まれています。Rancherは、セキュリティツール、監視システム、コンテナレジストリ、ストレージおよびネットワーキングドライバーなど、幅広いクラウドネイティブエコシステム製品と認定されています。

[#_features_of_the_rancher_api_server]
== Rancher APIサーバーの機能

Rancher APIサーバーは、組み込みのKubernetes APIサーバーとetcdデータベースの上に構築されています。以下の機能を実装しています：

[#_authorization_and_role_based_access_control]
=== 承認とロールベースのアクセス制御

* *ユーザー管理：*Rancher APIサーバーは、ローカルユーザーに加えて、Active DirectoryやGitHubなどの外部認証プロバイダーに対応する{xref-authn-authz}[ユーザーID]を管理します。
* *承認：*Rancher APIサーバーは、{xref-manage-rbac}[アクセス制御]と{xref-psa-pss}[セキュリティ基準]を管理します。

[#_working_with_kubernetes]
=== Kubernetesとの連携

* *Kubernetesクラスターのプロビジョニング：*Rancher APIサーバーは、既存のノード上で{xref-cluster-deployment}[Kubernetesをプロビジョニング]したり、{xref-ds-backup-restore}[Kubernetesをアップグレードする]ことができます。
* *カタログ管理：*Rancherは、アプリケーションを繰り返しデプロイするのを容易にする{xref-helm-charts-in-rancher}[Helmチャートのカタログ]を使用する機能を提供します。
* *プロジェクトの管理：*プロジェクトは、クラスター内の複数のネームスペースとアクセス制御ポリシーのグループです。プロジェクトはRancherの概念であり、Kubernetesの概念ではなく、複数のネームスペースをグループとして管理し、その中でKubernetes操作を実行することを可能にします。Rancher UIは、{xref-project-admin}[プロジェクトの管理]および{xref-k8s-resources-setup}[プロジェクト内のアプリケーション管理]の機能を提供します。
* *Fleet 継続的デリバリ：*Rancher内では、{xref-fleet}[{fleet-product-name}]を活用して、手動操作なしでgitリポジトリからターゲットのダウンストリームKubernetesクラスターにアプリケーションをデプロイできます。
* *Istio：*私たちの{xref-istio}[Istioとの統合]は、管理者やクラスターの所有者などのRancherオペレーターが、開発者にIstioを提供できるように設計されています。その後、開発者はIstioを使用してセキュリティポリシーを強制したり、問題をトラブルシューティングしたり、グリーン/ブルーデプロイメント、カナリアデプロイメント、またはA/Bテストのためにトラフィックを管理したりできます。

[#_working_with_cloud_infrastructure]
=== クラウドインフラストラクチャとの連携

* *ノードの追跡：*Rancher APIサーバーは、すべてのクラスター内のすべての{xref-nodes-machinepools}[ノード]のIDを追跡します。
* *インフラストラクチャのセットアップ：* クラウドプロバイダーを使用するように設定されている場合、Rancherはクラウド内で{xref-infra-providers}[新しいノード]と{xref-persistent-storage}[永続ストレージ]を動的にプロビジョニングできます。

[#_cluster_visibility]
=== クラスターの可視性

* *ログ:*Rancherは、Kubernetesクラスターの外部に存在するさまざまな人気のあるログサービスやツールと統合できます。
* *監視:*Rancherを使用すると、主要なオープンソース監視ソリューションであるPrometheusとの統合を通じて、クラスターのノード、Kubernetesコンポーネント、およびソフトウェアデプロイメントの状態とプロセスを監視できます。
* *アラート：*クラスターとアプリケーションを健全に保ち、組織の生産性を向上させるためには、計画されたイベントと予期しないイベントの両方について、クラスターやプロジェクトで発生している出来事を把握しておく必要があります。
