Google Cloud Platform (GCP) および Google Compute Engine (GCE) に関する詳細は、公式の https://cloud.google.com/docs[GCP ドキュメント] を参照してください。

[#_basic_options]
== 基本的なオプション

[#_zone]
=== ゾーン

VMがデプロイされるGCPリージョンとゾーン。たとえば、`us-east1-b`と指定します。

[#_machine_image_project]
=== マシンイメージプロジェクト

希望するイメージファミリーが属するイメージプロジェクト。

[#_machine_image_family]
=== マシンイメージファミリー

希望するマシンオペレーティングシステムが属するイメージファミリー。

[#_machine_image]
=== マシンイメージ

VMにインストールされるオペレーティングシステム。

[#_disk_type]
=== Disk Type

VMに接続されるディスクのタイプ。利用可能なタイプはリージョンによって異なる場合があります。

[#_disk_size]
=== ディスクサイズ

VMに接続されるディスクのサイズ（ギガバイト単位）。

[#_machine_type]
=== コンピュータのタイプ

デプロイされるVMのタイプ。マシンタイプは、各ノードに割り当てられるリソース（vCPU、RAMなど）の数を決定します。

[#_network]
=== ネットワーク

VMが作成されるVPCネットワーク。マシンプールがプロビジョニングされた後は、この値を変更することはできません。

[#_subnet]
=== サブネット

VMが作成されるVPCサブネットワーク。マシンプールがプロビジョニングされた後は、この値を変更することはできません。

[#_username]
=== ユーザ名

GCE VMのデフォルトユーザーとして設定されたカスタムユーザー名。

[#_external_address]
=== 外部アドレス

GCE VMのための希望する外部IPアドレス。

[#_scopes]
=== スコープ

VMが他のGCP APIにアクセスすることを許可するOAuth2スコープのリスト。

[#_allow_internal_communication]
=== 内部通信を許可する

デフォルトでは、VPC内のクラスタノード間の通信を促進するために、固定されたポートのセットを公開するVPCファイアウォールルールが自動的に作成されます。この動作は、`Show Advanced`オプションをクリックし、`Allow Internal Communication`チェックボックスを無効にすることで、マシンプールごとに無効にできます。

[#_expose_external_ports]
=== 外部ポートを公開する

広範なインターネットに対して_外部_に開くポートのリスト。開いているポートはマシンプールレベルで定義されます。このオプションを有効にすると、VPCファイアウォールルールが自動的に作成されます。このルールは、クラスタまたはマシンプールが削除されると自動的に削除されます。

[#_network_tags]
=== ネットワークタグ

タグは、マシンプール内のすべてのVMに既存のファイアウォールルールを関連付けるために使用できる_ネットワークタグ_のリストです。

[#_labels]
=== ラベル

特定のマシンプール内のすべてのVMに添付されるカスタムラベルのカンマ区切りリスト。タグとは異なり、ラベルはネットワーキングの動作に影響を与えず、クラウドリソースを整理するためだけに使用されます。

[#_advanced_options]
== 詳細オプション

Rancher UIを介してクラスタを作成する際には、一部のオプションが自動的に設定されます。ただし、マシン構成オブジェクトを手動で作成する場合は、以下のフィールドを適切に設定する必要があります。

[#_external_firewall_rule_prefix]
=== external-firewall-rule-prefix

ポートを公開するためのファイアウォールルールを作成する際に使用されるプレフィックスです。理想的には、これはマシンプール名とクラスタ名の連結であるべきです。マシンプールがポートを公開するように設定されている場合、このフィールドは設定する必要があります。そうでない場合は省略できます。

[#_internal_firewall_rule_prefix]
=== internal-firewall-rule-prefix

クラスター内のノード間の通信を許可する内部ファイアウォールルールを作成する際に使用されるプレフィックスです。このフィールドが省略された場合、内部ファイアウォールルールは作成されません。
