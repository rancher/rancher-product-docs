[#_node_roles]
=== ノードの役割

RKE2 CLIは、Kubernetesノードの役割`etcd` + `controlplane`および`worker`をそれぞれ表す`server`と`agent`の2つの役割を公開します。Rancher v2.6におけるRKE2統合により、RKE2ノードプールは、`etcd`および`controlplane`の役割を表す、より細かい役割の割り当てが可能になります。

`etcd`、`controlplane`、および`worker`ノードを使用する同様の機能は、フラグとノードのテイントを使用して、ワークロードとKubernetesマスターのスケジュールを制御することにより、RKE2 CLIでも可能です。これらの役割がRKE2 CLIでファーストクラスの役割として実装されなかった理由は、RKE2がRancherのようなオーケストレーションシステムを通じて最も効果的に活用される生のビルディングブロックのセットとして概念化されているからです。

私たちの{xref-recommended-architecture}[推奨されるクラスターアーキテクチャ]では、各役割のクラスターが持つべきノードの数を概説しています：

* 1つのノードを失っても生き残るために、役割etcdを持つノードを少なくとも3つ持つこと。
* マスターコンポーネントの高可用性のために、役割controlplaneを持つノードを少なくとも2つ持つこと。
* ノードにエラーが発生した場合にワークロードの再スケジューリングを行うためには、役割workerを持つノードを少なくとも2つ用意する必要があります。
