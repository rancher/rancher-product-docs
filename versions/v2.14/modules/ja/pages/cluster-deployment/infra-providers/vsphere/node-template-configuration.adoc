[#_account_access]
== アカウントアクセス

[cols="<,^,<"]
|===
| パラメータ | 必須 | 説明

| クラウド資格情報
| *
| vSphereアカウントのアクセス情報は、{xref-cloud-credentials}[クラウド認証情報]に保存されています。
|===

クラウド認証情報には、以下のフィールドがあります：

|===
| 認証情報フィールド | 説明

| vCenterまたはESXiサーバー
| vCenterまたはESXiのホスト名/IPを入力してください。ESXiは、仮想マシンや仮想アプライアンスを作成および実行するための仮想化プラットフォームです。vCenter Serverは、ネットワークに接続された複数のホストを管理し、ホストリソースをプールするためのサービスです。

| ポート
| vCenterまたはESXiサーバーのポートを設定してください。

| ユーザ名とパスワード
| vSphereのログインユーザー名とパスワードを入力してください。
|===

[#_scheduling]
== スケジューリング

仮想マシンがスケジュールされるハイパーバイザーを選択します。

*スケジューリング*セクションのフィールドは、vSphereで利用可能なデータセンターやその他のスケジューリングオプションで自動的に入力されるべきです。

|===
| フィールド | 必須 | 説明

| データセンター
| *
| VMがスケジュールされるデータセンターの名前/パスを選択してください。

| Resource Pool
|
| VMをスケジュールするリソースプールの名前。リソースプールは、スタンドアロンホストまたはクラスターの利用可能なCPUおよびメモリリソースを分割するために使用でき、ネストすることも可能です。スタンドアロンESXiの場合は空白のままにしてください。指定されていない場合、デフォルトのリソースプールが使用されます。

| データストア
| *
| データストアクラスターがある場合は、*データストア*フィールドを切り替えることができます。これにより、VMがスケジュールされるデータストアクラスターを選択できます。フィールドが切り替えられていない場合は、個別のディスクを選択できます。

| フォルダ
|
| 仮想マシンを作成するためのデータセンター内のフォルダー名。すでに存在している必要があります。このドロップダウンメニューの仮想マシンフォルダーは、vSphere内の仮想マシンフォルダーに直接対応しています。フォルダー名は、vSphere設定ファイル内で`vm/`を前置きする必要があります。

| ホスト
|
| 仮想マシンをスケジュールするホストシステムのIP。スタンドアロンのESXiまたはDRS（分散リソーススケジューラー）を持つクラスターの場合、このフィールドは空白のままにしてください。指定された場合、ホストシステムのプールが使用され、*リソースプール*パラメーターは無視されます。

| 優雅なシャットダウンのタイムアウト
|
| Rancherがクラスター上の仮想マシンを削除する前に待機する時間（秒単位）。`0`に設定されている場合、優雅なシャットダウンは無効になります。整数値のみを受け付けます。
|===

[#_instance_options]
== インスタンスオプション

*インスタンスオプション*セクションで、このテンプレートによって作成されるVMのvCPU、メモリ、およびディスクサイズを設定します。

[cols="<,^,<"]
|===
| パラメータ | 必須 | 説明

| CPU
| *
| 仮想マシンに割り当てるvCPUの数。

| メモリ
| *
| 仮想マシンに割り当てるメモリの量。

| ディスク
| *
| 仮想マシンに接続するディスクのサイズ（MB単位）。

| 作成方法
| *
| ノードにオペレーティングシステムを設定する方法。オペレーティングシステムは、ISOまたはVMテンプレートからインストールできます。作成方法に応じて、VMテンプレート、コンテンツライブラリ、既存のVM、またはISOを指定する必要があります。作成方法の詳細については、<<_about_vm_creation_methods,仮想マシン作成方法について。>>を参照してください。

| Cloud Init
|
| `cloud-config.yml`ファイルのURLまたはVMをプロビジョニングするためのURL。このファイルは、ネットワーク設定、DNSサーバー、またはシステムデーモンなど、オペレーティングシステムのさらなるカスタマイズを可能にします。オペレーティングシステムは、`cloud-init`をサポートしている必要があります。

| ネットワーク
|
| VMに接続するネットワークの名前。

| guestinfoに使用される設定パラメータ
|
| VMの追加の設定パラメータ。これらは、vSphereコンソールの https://kb.vmware.com/s/article/1016098[高度な設定]に対応しています。使用例には、RancherOSの https://rancher.com/docs/os/v1.x/en/installation/cloud/vmware-esxi/#vmware-guestinfo[guestinfo]パラメータを提供したり、VMのディスクUUIDを有効にしたりすることが含まれます（`disk.EnableUUID=TRUE`）。
|===

[#_about_vm_creation_methods]
=== VM作成方法について

*作成方法*フィールドで、vSphereでVMをプロビジョニングするために使用される方法を設定します。利用可能なオプションには、RancherOS ISOから起動するVMを作成することや、既存の仮想マシンまたは https://docs.vmware.com/en/VMware-vSphere/6.5/com.vmware.vsphere.vm_admin.doc/GUID-F7BF0E6B-7C4F-4E46-8BBF-76229AEA7220.html[VMテンプレート]からクローンを作成することが含まれます。

既存のVMまたはテンプレートは、 https://cloudinit.readthedocs.io/en/latest/[cloud-init]をサポートするように構成された最新のLinuxオペレーティングシステムを使用できます。 https://canonical-cloud-init.readthedocs-hosted.com/en/latest/reference/datasources/nocloud.html[NoCloudデータソース]を使用します。

VMが作成される方法を選択してください：

* *テンプレートからデプロイ：データセンター：*選択したデータセンターに存在するVMテンプレートを選択してください。
* *テンプレートからデプロイ：コンテンツライブラリ：*まず、テンプレートを含む https://docs.vmware.com/en/VMware-vSphere/6.5/com.vmware.vsphere.vm_admin.doc/GUID-254B2CE8-20A8-43F0-90E8-3F6776C2C896.html[コンテンツライブラリ]を選択し、次にポピュレートされたリストからテンプレートを選択します*ライブラリテンプレート*。
* *既存の仮想マシンをクローンする：**仮想マシン*フィールドで、新しいVMがクローンされる既存のVMを選択します。
* *boot2docker ISOからインストールする：**OS ISO URL*フィールドには、RancherOSのVMware ISOリリースのURLが含まれていることを確認してください。このURLは、Rancherサーバーのインストールを実行しているノードからアクセス可能である必要があります。

[#_networks]
== ネットワーク

ノードテンプレートは、複数のネットワークでVMをプロビジョニングできるようになりました。*ネットワーク*フィールドでは、*ネットワークを追加*をクリックして、vSphereで利用可能なネットワークを追加できます。

[#_node_tags_and_custom_attributes]
== ノードタグとカスタム属性

タグを使用すると、vSphereインベントリ内のオブジェクトにメタデータを添付し、これらのオブジェクトをより簡単にソートおよび検索できるようになります。

タグについては、すべてのvSphereタグがノードテンプレートで選択可能なオプションとして表示されます。

カスタム属性では、RancherはvSphereで既に設定したすべてのカスタム属性を選択できるようにします。カスタム属性はキーであり、それぞれに値を入力できます。

[NOTE]
====

カスタム属性はレガシー機能であり、最終的にはvSphereから削除される予定です。
====


[#_cloud_init]
== cloud-init

https://cloudinit.readthedocs.io/en/latest/[Cloud-init]は、最初の起動時に構成を適用することでノードを初期化することを可能にします。これには、ユーザーの作成、SSHキーの承認、またはネットワークの設定などが含まれる場合があります。

cloud-init初期化を利用するには、有効なYAML構文を使用してcloud configファイルを作成し、そのファイルの内容を*Cloud Init*フィールドに貼り付けてください。サポートされているクラウドコンフィグディレクティブのコメント付きの例については、 https://cloudinit.readthedocs.io/en/latest/topics/examples.html[cloud-initドキュメント]を参照してください。

.iso作成方法を使用する場合、cloud-initはサポートされていないことに注意してください。

[#_engine_options]
== エンジンオプション

ノードテンプレートの *エンジンオプション* セクションで、コンテナデーモンを設定できます。コンテナのバージョンやコンテナイメージレジストリミラーを指定することをお勧めします。

[NOTE]
====
Red Hat Enterprise Linux (RHEL)またはCentOSノードをプロビジョニングする場合は、*DockerインストールURL*フィールドをデフォルト値のままにするか、*なし*を選択してください。これにより、これらのノードタイプにDockerがすでにインストールされているため、Dockerインストールのチェックがスキップされます。

*DockerインストールURL*をデフォルト値または*なし*以外の値に設定すると、次のようなエラーメッセージが表示されることがあります: `Error creating machine: RHEL ssh command error: command: sudo -E yum install -y curl err: exit status 1 output: Updating Subscription Management repositories.`
====
