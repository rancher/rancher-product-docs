Rancherダッシュボードは、最新の更新、ライフサイクルイベント、およびデプロイメントのステータスについて、常に情報を得られるように動的通知とシステムアラートを提供します。

[#_dynamic_content]
== 動的コンテンツ

Rancherは、新しいリリース、メンテナンス終了（EOM）またはサービス終了（EOL）ステータス、その他の重要な発表に関する通知を含む動的コンテンツを取得します。

動的コンテンツは、Rancherダッシュボードの3つのエリアを通じてユーザーに提供されます。

* *ホームページ通知バナー*:高優先度の発表のために、ホームページの上部に表示される動的バナーです。
* *リンクの下のホームページセクション*:*リンク*セクションの下に位置する、関連する更新とリソースを表示するためのホームページの専用エリアです。
* *<<_what_is_the_notification_center,通知センター>>*:システムアラート、リソース、および発表を含むすべての通知のための集中ハブです。

これらの通知と対話するには、リストされたリソースの*詳細はこちら*をクリックしてください。通知を閉じることもできます。

[#_what_is_the_notification_center]
== 通知センターとは何ですか？

通知センターは、システム生成のアラートと動的発表のための中央リポジトリとして機能します。Rancherダッシュボードの右上隅にあるベルアイコンをクリックすることでアクセスできます。

通知は、重大度とタイプによって分類されます：

* *エラー*は、重大度が高い問題を示します。
* *警告*は、重大度が中程度の懸念を示します。
* *情報*は、重大度が低い一般的な更新を提供します。
* *タスク*は、プロセスまたはアクションが現在進行中であることを示します。
* *成功*は、プロセスまたはアクションが成功裏に完了したことを確認します。

例えば、新しいバージョンのRancherが利用可能になったときに通知を受け取ったり、Rancherリリースノートへの直接リンクを受け取ったりすることがあります。

上下の矢印キーを使用して、通知を簡単に閲覧できます。特定の通知には関連するアクションが含まれている場合もあり、即座に応答することができます。
