このセクションには、バックアップと復元のカスタムリソースの例が含まれています。

デフォルトのバックアップストレージ場所は、`rancher-backup`オペレーターがインストールまたはアップグレードされたときに構成されます。

暗号化されたバックアップは、復元カスタムリソースがバックアップを作成するために使用されたのと同じ暗号化設定シークレットを使用している場合にのみ復元できます。

[#_backup]
== バックアップ

このセクションには、バックアップカスタムリソースの例が含まれています。

NOTE: 以下のオプションを設定するための詳細については、{xref-rancher-backup-config}[バックアップ設定リファレンスページ]を参照してください。

[#_backup_in_the_default_location_with_encryption]
=== 暗号化を使用したデフォルトの場所でのバックアップ

[,yaml]
----
apiVersion: resources.cattle.io/v1
kind: Backup
metadata:
  name: default-location-encrypted-backup
spec:
  resourceSetName: rancher-resource-set-full
  encryptionConfigSecretName: encryptionconfig
----

[#_recurring_backup_in_the_default_location]
=== デフォルトの場所での定期的なバックアップ

[,yaml]
----
apiVersion: resources.cattle.io/v1
kind: Backup
metadata:
  name: default-location-recurring-backup
spec:
  resourceSetName: rancher-resource-set-basic
  schedule: "@every 1h"
  retentionCount: 10
----

kind: Backup
metadata:
  name: default-enc-recurring-backup
spec:
  resourceSetName: rancher-resource-set-full
  encryptionConfigSecretName: encryptionconfig

[#_encrypted_backup_in_minio]
=== Minioでの暗号化されたバックアップ

[,yaml]
----
apiVersion: resources.cattle.io/v1
kind: Backup
metadata:
  name: minio-backup
spec:
  storageLocation:
    s3:
      credentialSecretName: minio-creds
      credentialSecretNamespace: default
      bucketName: rancherbackups
      endpoint: minio.xip.io
      endpointCA: <base64-encoded-cert>
  resourceSetName: rancher-resource-set-full
  encryptionConfigSecretName: encryptionconfig
----

[#_backup_from_ec2_nodes_with_iam_permission_to_access_s3]
=== S3にアクセスするためのIAM権限を持つEC2ノードからのバックアップ

この例は、`rancher-backup`を実行しているノードがS3にアクセスするための{xref-rancher-backup-config}#_iam_permissions_for_ec2_nodes_to_access_s3[これらの権限を持っている場合、バックアップを作成するためにAWS認証シークレットを提供する必要はないことを示しています。]

[,yaml]
----
apiVersion: resources.cattle.io/v1

  name: s3-iam-backup

  storageLocation:
    s3:

      folder: ecm1

      endpoint: s3.us-west-2.amazonaws.com
  resourceSetName: rancher-resource-set-full
  encryptionConfigSecretName: encryptionconfig
----

NOTE: 以下のオプションを設定するための詳細については、{xref-rancher-restore-config}[復元設定リファレンスページ]を参照してください。

[#_restore_using_the_default_backup_file_location]
=== デフォルトのバックアップファイルの場所を使用して復元

[,yaml]
----
apiVersion: resources.cattle.io/v1
kind: Restore
metadata:
  name: restore-default
spec:
  backupFilename: default-location-recurring-backup-752ecd87-d958-4d20-8350-072f8d090045-2020-09-26T12-29-54-07-00.tar.gz
#  encryptionConfigSecretName: test-encryptionconfig
----

=== Rancher移行のための復元

      credentialSecretNamespace: default

      folder: ecm1
      region: us-west-2
      endpoint: s3.us-west-2.amazonaws.com

[#_example_encryptionconfiguration]
== 例: EncryptionConfiguration

以下のスニペットは、バックアップとカスタムリソースの復元に関する2種類のシークレットとその関連性を示しています。シークレットの作成は、次のコマンドで行うことができます:

[,console]
----
kubectl create secret generic example-encryptionconfig \
  --from-file=./encryption-provider-config.yaml \
  -n cattle-resources-system
----

最初の例は、バックアップファイルを暗号化するために使用されるシークレットの例です。バックアップオペレーターは、Base64でエンコードされたEncryptionConfigurationリソースの定義を含む**encryption-provider-config.yaml**キーの内容を読み取ります。

2番目の例は、KubernetesのEncryptionConfigurationリソース自体であり、最初の例からのBase64エンコードされた内容のプレーンテキスト形式です。このリソースは、etcdに保存されるシークレットを暗号化するためや、Kubernetesにおける静止データの暗号化の一般的なケースでも使用されます。その詳細については、 https://kubernetes.io/docs/tasks/administer-cluster/encrypt-data/[アップストリームドキュメント]に記載されています。このリソースで定義されたキーは、特に復元プロセス中に暗号化されたバックアップの内容にアクセスするために不可欠です。そのため、これらは安全に保管し、機密性を保ち、容易に取得できるようにする必要があります。
