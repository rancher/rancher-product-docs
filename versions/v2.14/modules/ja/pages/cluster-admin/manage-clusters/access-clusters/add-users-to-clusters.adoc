特定のクラスターリソースを表示し、新しいプロジェクトを作成するためにユーザーにアクセスを提供するには、そのユーザーに**クラスターのメンバーシップ**を割り当てます。クラスターのメンバーは、プロジェクトを作成し、そのプロジェクト内のリソースを管理できます。ただし、クラスター内のすべてのリソース、ネームスペース、ワークロードにクラスターのメンバーがアクセスできるわけではありません。

[TIP]
====
クラスター内の_特定の_プロジェクトにユーザーにアクセスを提供したいですか？代わりに{xref-add-users-to-projects}[プロジェクトメンバーの追加]をご覧ください。
====

クラスターのメンバーを追加できるコンテキストは2つあります：

* 新しいクラスターへのメンバー追加
+
クラスターを作成する際にメンバーを追加できます（可能であれば推奨）。

* <<_editing_cluster_membership,既存のクラスターへのメンバー追加>>
+
クラスターがプロビジョニングされた後でも、クラスターにメンバーを追加することは常に可能です。

[#_editing_cluster_membership]
== クラスターのメンバーシップの編集

クラスターの管理者は、クラスターのメンバーシップを編集でき、どのRancherユーザーがクラスターにアクセスできるか、どの機能を使用できるかを制御します。

. *☰ > クラスター管理*をクリックします。
. メンバーを追加したいクラスターに移動し、*⋮ > 設定編集*をクリックします。
. *メンバーの役割*タブで、*メンバーを追加*をクリックします。
. クラスターに追加したいユーザーまたはグループを検索します。
+
[NOTE]
.備考
====
* 結果が表示されるには、検索ボックスに少なくとも2文字を入力する必要があります。
* ユーザーは、ユーザー名または表示名に基づいて検索できます。
* 検索は接頭辞ベースです（例：ユーザー名が`Stan Dard`のユーザーは`Sta`を検索すると表示されますが、`Dar`を検索すると表示されません）および大文字と小文字を区別します。
====
+
外部認証が設定されている場合：


 ** Rancherは、入力する際に{xref-authn-authz}[外部認証]ソースからユーザーを返します。
+

[NOTE]
.ADを使用していますが、ユーザーが見つかりませんか？
====
検索属性の設定に問題がある可能性があります。{xref-configure-ad}[Active Directory認証の設定：ステップ5]
====


 ** ドロップダウンを使用すると、個々のユーザーの代わりにグループを追加できます。ドロップダウンには、ログインしているユーザーが所属しているグループのみが表示されます。
+

[NOTE]
====
ローカルユーザーとしてログインしている場合、外部ユーザーは検索結果に表示されません。詳細については、{xref-authn-authz}#_external_authentication_configuration_and_principal_users[外部認証設定と主要ユーザー]を参照してください。
====

. ユーザーまたはグループに*クラスター*の役割を割り当てます。
+
{xref-cluster-project-roles}[クラスターの役割とは？]
+

[TIP]
====
カスタム役割の場合、割り当て可能な個々の役割のリストを変更できます。

* リストに役割を追加するには、{xref-custom-roles}[カスタム役割を追加]します。
* リストから役割を削除するには、{xref-locked-roles}[Lock/Unlock Roles]をクリックします。
====

*結果:*選択したユーザーがクラスターに追加されます。

* クラスターのメンバーシップを取り消すには、ユーザーを選択し、*削除する*をクリックします。この操作はメンバーシップを削除しますが、ユーザー自体は削除しません。
* クラスター内のユーザーの役割を変更するには、ユーザーをクラスターから削除し、変更された役割で再追加します。
