*ネームスペースデフォルト制限*は、作成時にプロジェクトから各ネームスペースに伝播しますが、特定のネームスペースのクォータを増やす（または減らす）必要がある場合があります。この場合、ネームスペースを編集することでデフォルト制限を上書きできます。

以下の図では、Rancher管理者がプロジェクトに対して有効なリソースクォータを持っています。しかし、管理者は`Namespace 3`のCPU制限を上書きして、より多くのリソースを利用できるようにしたいと考えています。したがって、管理者は{xref-projects-and-namespaces}[CPU制限を引き上げます]`Namespace 3`、ネームスペースがより多くのリソースにアクセスできるようにします。

[#_editing_namespace_resource_quotas]
== ネームスペースリソースクォータの編集

プロジェクトにリソースクォータが設定されている場合、特定のネームスペースに対して、より多く（またはより少ない）プロジェクトリソースを利用できるよう、ネームスペースのデフォルト制限を上書きできます。

. 左上隅で、*☰ > クラスター管理*をクリックします。
. *クラスター*ページで、ネームスペースリソースクォータを編集したいクラスターに移動し、*探索*をクリックします。
. menu:クラスター[プロジェクト/名前空間]をクリックします。
. リソースクォータを編集したいネームスペースを見つけます。*⋮ > 設定を編集*をクリックしてください。
. リソース制限を編集します。 これらの制限は、ネームスペースに利用可能なリソースを決定します。制限は、設定されたプロジェクトの制限内に設定する必要があります。
+
各*リソースタイプ*に関する詳細は、{xref-resource-quota-types}[タイプリファレンス]を参照してください。
+

[NOTE]
====

 ** プロジェクトにリソースクォータが設定されていない場合、これらのオプションは利用できません。
 ** 設定されたプロジェクトの制限を超える制限を入力した場合、Rancherは編集内容を保存させません。

+
====


*結果:*あなたのオーバーライドは、ネームスペースのリソースクォータに適用されます。
