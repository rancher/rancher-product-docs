Fleet 协调并管理通过供应链实现的应用程序持续交付，以便面向多个集群。Fleet 组织供应链，通过采用 GitOps 这一安全的操作模型，帮助团队充满信心、及时地交付。

[#__fleet_product_name_with_rancher]
== {fleet-product-name}与 Rancher 一起使用

许多用户通常同时管理超过 10 个集群。考虑到集群的激增，持续交付是 Rancher 的一个重要部分。Fleet 通过 GitOps 确保了可靠的持续交付体验，这是一种安全且日益普及的操作模型。

[#_when_should_i_use_fleet_product_name]
=== 我应该在什么时候使用 {fleet-product-name}？

* 我需要在不同的地理区域部署我的监控栈（例如，Grafana、Prometheus），每个区域都有不同的保留策略。
* 我是一名平台操作员，想要利用可扩展且安全的操作模型（GitOps）配置包含所有组件的集群。
* 我是一名应用程序开发人员，希望我的最新更改自动进入我的开发环境。

[#__fleet_product_name_with_rancher_prime]
== {fleet-product-name} 与 {rancher-prime}

Fleet 已在 Rancher 中深度整合，作为持续交付工具和 GitOps 引擎。
