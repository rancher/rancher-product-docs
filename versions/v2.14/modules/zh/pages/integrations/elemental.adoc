Elemental 使云原生主机管理成为可能。Elemental 允许您在任何位置（无论是在数据中心还是边缘）引导任何机器，并将其无缝集成到 Kubernetes 中，同时管理您的工作流（例如，操作系统更新）。

[#__elemental_product_name_with_rancher]
== {elemental-product-name} 与 Rancher 一起使用

Rancher 中的 Elemental：

* 它是为 Kubernetes 原生设计的，允许您在 Kubernetes 集群中通过 Elemental 管理操作系统。
* 从 Kubernetes 运维的角度来看，它不会造成中断。
* 它采用声明式设计，并且支持 GitOps。
* 允许基于 OCI 镜像的流程，这些流程是可信的、确定性的和可预测的。
* 能够大规模运行。它使大规模的操作系统管理成为可能。

[#_when_should_i_use_elemental_product_name]
=== 我应该在什么时候使用 {elemental-product-name}？

* Elemental 使得通过 Rancher 管理器实现云原生操作系统管理成为可能。它与任何操作系统（例如，SLE Micro 原版）兼容。
* Elemental 允许对数据中心和边缘的机器进行云原生管理。
* Elemental 灵活，并允许平台团队在其所有机器上执行各种工作流。

[#__elemental_product_name_with_rancher_prime]
== {elemental-product-name} 与 {rancher-prime}

* 已经以 GUI 扩展形式深度集成到 Rancher 中。
* 将 Rancher 的应用延伸至操作系统。目前与 SLE Micro for Rancher 完美协同工作。
