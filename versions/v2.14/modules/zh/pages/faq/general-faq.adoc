[#_is_it_possible_to_manage_azure_kubernetes_services_with_rancher_v2_x]
== 可以使用Rancher v2.x管理Azure Kubernetes服务吗？

可以。请参阅我们的{xref-manage-clusters}[集群管理]指南，了解Rancher在AKS上可用的功能，以及我们的{xref-rancher-on-aks}[AKS文档]。

[#_does_rancher_support_windows]
== Rancher支持Windows吗？

可以。Rancher支持Windows Server 1809容器。有关如何设置带有Windows工作节点的集群的详细信息，请参阅{xref-use-windows-clusters}[为Windows配置自定义集群]部分。
endif::[]

[#_does_rancher_support_istio]
== Rancher支持Istio吗？

可以。Rancher支持{xref-istio}[Istio]。

[#_will_rancher_v2_x_support_hashicorps_vault_for_storing_secrets]
== Rancher v2.x会支持Hashicorp的Vault来存储秘密吗？

秘密管理在我们的路线图上，但我们尚未将其分配给特定版本。

[#_does_rancher_v2_x_support_rkt_containers_as_well]
== Rancher v2.x也支持RKT容器吗？

目前，我们仅支持Docker。

[#_does_rancher_v2_x_support_calico_contiv_contrail_flannel_weave_net_etc_for_embedded_and_registered_kubernetes]
== Rancher v2.x支持Calico、Contiv、Contrail、Flannel、Weave net等用于嵌入式和注册的Kubernetes吗？

开箱即用，Rancher为Kubernetes集群提供以下CNI网络提供者：Canal、Flannel、Calico和Weave。 请始终参考 https://rancher.com/support-maintenance-terms/[Rancher支持矩阵]以获取有关官方支持的详细信息。

[#_are_you_planning_on_supporting_traefik_for_existing_setups]
== 您计划支持现有设置的Traefik吗？

我们目前不计划提供嵌入式Traefik支持，但我们仍在探索负载平衡的方法。

[#_can_i_import_openshift_kubernetes_clusters_into_v2_x]
== 我可以将 OpenShift Kubernetes 集群导入 v2.x 吗？

我们的目标是运行任何 Kubernetes 集群。因此，Rancher v2.x 应该可以与 OpenShift 一起使用，但我们还没有进行测试。

[#_is_longhorn_product_name_integrated_with_rancher]
== {longhorn-product-name} 是否与 Rancher 集成？

可以。Longhorn 与 Rancher v2.5 及更高版本集成。
