Rancher 支持管理由于网络中断、控制平面可用性或所有集群节点故障而可能无法在线的集群。目前在最新发布的 Rancher 版本中，没有已知的断开集群问题。

当受管集群与 Rancher 断开连接时，管理操作将不可用，Rancher 用户界面将无法导航到该集群。然而，一旦重新建立连接，功能将完全恢复。

[#_best_practices_for_managing_disconnected_clusters]
== 管理断开集群的最佳实践

* *Rancher 升级期间的集群可用性*:建议在 Rancher 升级期间，所有或至少大多数受管集群保持在线。原因是升级 Rancher 会自动升级在受管集群上运行的 Rancher 代理软件。保持代理和 Rancher 版本一致可确保功能的一致性。在升级期间断开的任何集群将在重新连接后立即更新其代理。

* *清理断开集群*:定期去除将不再与 Rancher 重新连接的集群（例如，已退役或已销毁的集群）。将此类集群保留在 Rancher 管理系统中会消耗不必要的资源，可能会随着时间的推移影响 Rancher 的性能。

* *证书轮换注意事项*:在设计涉及定期关闭集群的流程时，无论是否连接到 Rancher，都要考虑证书轮换政策。例如，RKE2/K3s 集群在启动时可能会轮换证书，如果它们超过了其生命周期。
