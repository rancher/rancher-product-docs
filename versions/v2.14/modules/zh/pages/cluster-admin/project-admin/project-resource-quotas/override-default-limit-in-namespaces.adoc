尽管*名称空间默认限制*在创建时从项目传播到每个名称空间，但在某些情况下，您可能需要增加（或减少）特定名称空间的配额。在这种情况下，您可以通过编辑名称空间来覆盖默认限制。

在下面的图示中，Rancher 管理员对其项目实施了资源配额。然而，管理员希望覆盖`Namespace 3`的处理器限制，以便它可以使用更多的资源。因此，管理员{xref-projects-and-namespaces}[提高了处理器限制]以便`Namespace 3`可以访问更多资源。

[#_editing_namespace_resource_quotas]
== 编辑名称空间资源配额

如果为项目配置了资源配额，您可以覆盖名称空间默认限制，以便为特定名称空间提供更多（或更少）项目资源的访问。

. 在左上角，单击*☰ > 集群管理*。
. 在*集群*页面上，转到您想要编辑名称空间资源配额的集群，然后点击*浏览*。
. 单击menu:集群[项目/名称空间]。
. 找到您想要编辑资源配额的名称空间。点击*⋮ > 编辑配置*。
. 编辑资源限制。 这些限制决定了可用于名称空间的资源。限制必须在配置的项目限制范围内设置。
+
有关每种*资源类型*的更多信息，请参见{xref-resource-quota-types}[类型参考]。
+

[NOTE]
====

 ** 如果项目没有配置资源配额，这些选项将不可用。
 ** 如果您输入的限制超过了配置的项目限制，Rancher 将不允许您保存编辑。

+
====


*结果：*您的覆盖已应用于名称空间的资源配额。
