_项目_是在Rancher中引入的对象，帮助组织Kubernetes集群中的名称空间。您可以使用项目创建多租户集群，这允许一组用户共享相同的基础资源，而不干扰彼此的应用程序。

在层级结构方面：

* 集群包含项目
* 项目包含名称空间

在Rancher中，项目允许您将多个名称空间作为一个实体进行管理。在不包含项目的原生Kubernetes中，基于角色的访问权限或集群资源等功能是分配给单个名称空间的。在多个名称空间需要相同访问权限的集群中，将这些权限分配给每个单独的名称空间可能会变得繁琐。即使所有名称空间都需要相同的访问权限，也没有办法在单个操作中将这些权限应用于所有名称空间。您必须重复地将这些权限分配给每个名称空间！

Rancher项目通过允许您在项目级别应用资源和访问权限来解决此问题。项目中的每个名称空间继承这些资源和策略，因此您只需将它们分配给项目一次，而不是分配给每个单独的名称空间。

您可以使用项目执行以下操作：

* {xref-add-users-to-projects}[][将用户访问权限分配给一组名称空间]
* 为用户分配{xref-cluster-project-roles}#_project_roles[项目中的特定角色]。角色可以是所有者、成员、只读或{xref-custom-roles}[自定义]。
* {xref-project-resource-quotas}[设置资源配额]
* {xref-namespaces}[管理名称空间]
* {xref-rancher-project-tools}[配置工具]

[#_authorization]
== 委任

非管理员用户只有在{xref-global-permissions}[管理员]、{xref-cluster-project-roles}#_cluster_roles[集群所有者或成员]，或{xref-cluster-project-roles}#_project_roles[项目所有者]将他们添加到项目的*成员*选项卡后，才被授权访问项目。

创建项目的用户自动成为{xref-cluster-project-roles}#_project_roles[项目所有者]。

[#_switching_between_projects]
== 在项目之间切换

要在项目之间切换，请使用导航栏中的下拉菜单。或者，您可以直接在导航栏中切换项目。

. 在左上角，单击*☰ > 集群管理*。
. 在*集群*页面上，转到您想切换项目的集群，然后单击*探索*。
. 在顶部导航栏中，选择您想要打开的项目。
