[#_basic_options]
== 基本选项

[#_environment]
=== 环境

微软提供多个 https://docs.microsoft.com/en-us/cli/azure/cloud?view=azure-cli-latest[云]以遵守地区法律，供您使用：

* AzurePublicCloud
* AzureGermanCloud
* AzureChinaCloud
* AzureUSGovernmentCloud

[#_location]
=== 位置

配置集群和节点 https://docs.microsoft.com/en-us/azure/virtual-machines/regions[位置]。

[#_resource_group]
=== 资源组

资源组是一个容器，用于保存 Azure 解决方案的相关资源。资源组可以包含解决方案的所有资源，或仅包含您希望作为一个组管理的那些资源。您可根据您组织的实际需求决定如何将资源分配给资源组。通常，将共享相同生命周期的资源添加到同一资源组，以便您可以轻松地作为一个组进行部署、更新和删除。

使用现有资源组或输入资源组名称，将为您创建一个。

有关管理资源组的信息，请参见 https://docs.microsoft.com/en-us/azure/azure-resource-manager/management/manage-resource-groups-portal[Azure 文档。]

[#_availability_set_unmanaged]
=== 可用性集（未管理）

要将虚拟机添加到的现有 https://docs.microsoft.com/en-us/azure/virtual-machines/availability-set-overview[可用性集] 的名称或 ID。

[#_image]
=== 映像

作为 ARM 资源标识符提供的操作系统映像的名称。需要使用托管磁盘。

[#_vm_size]
=== 虚拟机大小

为节点池中的每个虚拟机选择一个大小。有关每个虚拟机大小的详细信息，请参见 https://azure.microsoft.com/en-us/pricing/details/virtual-machines/linux/[此页面。]

[#_advanced_options]
== 高级选项

[#_fault_domain_count]
=== 故障域计数

故障域定义共享公共电源和网络交换机的虚拟机组。如果可用性集已经创建，则故障域计数将被忽略。

有关故障域的更多信息，请参见 https://docs.microsoft.com/en-us/azure/virtual-machines/availability-set-overview#how-do-availability-sets-work[此处参考]。

[#_update_domain_count]
=== 更新域计数

更新域表示可以同时重启的虚拟机组和底层物理硬件。如果可用性集已经创建，则更新域计数将被忽略。

有关更新域的更多信息，请参见 https://docs.microsoft.com/en-us/azure/virtual-machines/availability-set-overview#how-do-availability-sets-work[此处参考]。

[#_purchase_plan]
=== 购买计划

Azure 市场中的某些虚拟机映像需要一个计划。如适用，选择一个格式为`publisher:product:plan`的购买计划，以便与您选择的映像一起使用。

[#_subnet]
=== 子网

创建新虚拟网络或引用现有虚拟网络时的子网名称。

默认值为: `docker-machine`

[#_subnet_prefix]
=== 子网前缀

在 CIDR 格式中创建新虚拟网络时使用的子网 IP 地址前缀。

默认值为: `192.168.0.0/16`

[#_virtual_network]
=== 虚拟网络

要使用或创建的 https://docs.microsoft.com/en-us/azure/virtual-network/virtual-networks-overview[虚拟网络]（如果不存在）。格式为`[resourcegroup:]name`。

[#_public_ip_options]
=== 公用 IP 选项

[#_no_public_ip]
==== 无公用 IP

不分配公用 IP 地址。

[#_static_public_ip]
==== 静态公用 IP

分配一个静态公用 IP 地址。

[#_use_private_ip]
=== 使用私有 IP

使用静态私有 IP 地址。

[#_private_ip_address]
=== 私有 IP 地址

配置要使用的静态私有 IP 地址。

[#_network_security_group]
=== 网络安全组

要使用的 https://docs.microsoft.com/en-us/azure/virtual-network/network-security-groups-overview[网络安全组]。使用此模板的所有节点将使用提供的网络安全组。如果未提供网络安全组，将为每个节点创建一个新的网络安全组。

[#_dns_label]
=== DNS 标签

公共 IP 地址的唯一 DNS 名称标签。

[#_storage_type]
=== 存储类型

与您的虚拟机一起使用的 https://docs.microsoft.com/en-us/azure/storage/common/storage-account-overview[存储帐户] 类型。选项包括标准 LRS、标准 ZRS、标准 GRS、标准 RAGRS 和高级 LRS。

[#_use_managed_disks]
=== 使用托管磁盘

https://docs.microsoft.com/en-us/azure/virtual-machines/managed-disks-overview[Azure 托管磁盘] 是由 Azure 管理并与 Azure 虚拟机一起使用的块级存储卷。托管磁盘设计为 99.999% 的可用性。托管磁盘通过为您提供数据的三个副本来实现这一点，从而确保高耐久性。

[#_managed_disk_size]
=== 托管磁盘大小

每个节点磁盘的大小（以 GB 为单位）。

[#_ssh_username]
=== SSH 用户名

用于创建与节点的 SSH 连接的用户名。

[#_open_port]
=== 打开端口

在指定端口上打开入站流量。使用现有的网络安全组时，开放端口将被忽略。

默认值：`2379/tcp, 2380/tcp, 6443/tcp, 9796/tcp, 10250/tcp, 10251/tcp, 10252/tcp, 10256/tcp` 和 `8472/udp, 4789/udp`
