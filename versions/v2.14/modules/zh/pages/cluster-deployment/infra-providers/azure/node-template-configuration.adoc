有关 Azure 的更多信息，请参阅官方 https://docs.microsoft.com/en-us/azure/?product=featured[Azure 文档。]

账户访问信息存储为云凭证。云凭证存储为 Kubernetes 秘密。多个节点模板可以使用相同的云凭证。您可以使用现有的云凭证或创建新的云凭证。

* *位置* 设置您的集群托管的地理区域和其他位置信息。
* *网络* 配置您集群中使用的网络。
* *实例* 自定义您的虚拟机配置。
+

[NOTE]
====
如果使用与虚拟机不同资源组中的虚拟网络，则虚拟网络名称应以资源组名称为前缀。例如，`<resource group>:<vnet>`。
====


如果您使用 Docker， https://docs.docker.com/engine/docker-overview/#the-docker-daemon[Docker 守护程序] 的配置选项包括：

* *标签：*有关标签的信息，请参阅 https://docs.docker.com/config/labels-custom-metadata/[Docker对象标签文档。]。
* *Docker 引擎安装 URL：*确定将在实例上安装哪个 Docker 版本。
+

[NOTE]
====
如果您正在配置红帽企业Linux（RHEL）或CentOS节点，请将 *Docker安装URL* 字段保留为默认值，或选择 *无*。这将跳过对Docker安装的检查，因为这些节点类型上已经安装了Docker。

如果您将 *Docker安装URL* 设置为与默认值或 *无* 不同的值，您可能会看到如下错误消息： `Error creating machine: RHEL ssh command error: command: sudo -E yum install -y curl err: exit status 1 output: Updating Subscription Management repositories.`
====


* *注册表镜像：*Docker守护程序将使用的Docker注册表镜像。
* *其他高级选项：*请参阅 https://docs.docker.com/engine/reference/commandline/dockerd/[Docker 守护程序选项参考]。
