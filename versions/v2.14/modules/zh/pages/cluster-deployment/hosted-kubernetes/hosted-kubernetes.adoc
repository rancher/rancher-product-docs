ifeval::["{build-type}" != "srfa"]
在这种情况下，Rancher不会提供Kubernetes，因为它是由Google Kubernetes Engine (GKE)、Amazon Elastic Container Service for Kubernetes或Azure Kubernetes Service等提供商安装的。

如果您使用Google GKE等Kubernetes提供商，Rancher会与其云API集成，允许您从Rancher UI创建和管理托管集群的基于角色的访问控制。

在此用例中，Rancher使用提供商的API向托管提供商发送请求。然后，提供商为您提供和托管集群。当集群构建完成后，您可以通过 Rancher UI 管理它，以及管理那些您已部署并托管在本地或基础设施提供商上的集群。

Rancher支持以下Kubernetes提供商：

* https://cloud.google.com/kubernetes-engine/[Google GKE (Google Kubernetes Engine)]
* https://aws.amazon.com/eks/[Amazon EKS (Amazon Elastic Container Service for Kubernetes)]
* https://azure.microsoft.com/en-us/services/kubernetes-service/[Microsoft AKS (Azure Kubernetes Service)]

在这种情况下，Rancher不会提供Kubernetes，因为它是由Amazon Elastic Container Service for Kubernetes安装的。

在此用例中，Rancher使用提供商的API向托管提供商发送请求。然后，提供商为您提供和托管集群。当集群构建完成后，您可以通过 Rancher UI 管理它，以及管理那些您已部署并托管在本地或基础设施提供商上的集群。

Rancher支持Amazon EKS提供商：

* https://aws.amazon.com/eks/[Amazon EKS (Amazon Elastic Container Service for Kubernetes)]
endif::[]

== 托管Kubernetes提供商身份验证

ifeval::["{build-type}" != "srfa"]
当使用Rancher创建由提供商托管的集群时，系统会提示您输入身份验证信息。此信息是访问提供商API所必需的。有关如何获取此信息的更多信息，请参见以下程序：

* {xref-deploy-gke}[创建 GKE 集群]
* {xref-deploy-eks}[创建 EKS 集群]
* {xref-deploy-aks}[创建 AKS 集群]
endif::[]
