*安装适配器后，Rancher 中会出现一条横幅消息，内容为：“AWS Marketplace Adapter：无法运行适配器，请检查适配器日志”*

此错误表示虽然适配器已安装到集群中，但发生了错误，导致其无法正确登记/注销许可证。

这通常是因为 IAM 角色未正确设置。请查看 {xref-aws-cloud-marketplace-adapter-reqs}[先决条件] 并验证：

* 已为 Rancher 运行的集群创建/关联了 OIDC 提供者。
* IAM 角色已配置为信任此 OIDC 提供者。
* IAM 角色至少具有策略中列出的权限。

如果以上所有配置正确，请联系支持以获取帮助。

*我看到一条横幅消息，内容为：“AWS Marketplace Adapter：您已超出许可节点数量。在 AWS 中需要至少 x 个额外的许可证才能合规”*

此消息表示您没有足够的授权来管理 Rancher 当前管理的节点数量。

请牢记以下限制：

* 每个授权仅适用于一定数量的节点。
* Rancher 当前管理的每个节点都计入您的使用总量（安装 Rancher 的集群中的节点除外）。
* 每个授权最多可由一个 Rancher 实例使用。例如，如果您在账户中有两个运行的 Rancher 实例（每个实例安装在不同的 EKS 集群上），那么您至少需要两个授权。

您可能最近卸载/重新安装了适配器。如果适配器失去对其当前管理的许可证的跟踪，可能需要长达一个小时才能解决许可证的实际状态。
