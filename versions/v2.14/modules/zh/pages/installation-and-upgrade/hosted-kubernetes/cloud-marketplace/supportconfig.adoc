安装CSP适配器后，您将能够生成支持配置包。此包是一个tar文件，可用于快速提供支持信息。

这些包可以通过Rancher创建，或通过直接访问安装了Rancher的集群来创建。请注意，优先通过Rancher访问。

NOTE: 无论使用何种方法，只有管理员用户可以生成/下载支持配置包。

[#_accessing_through_rancher]
== 通过Rancher访问

首先，单击汉堡菜单。然后单击 `Get Support` 按钮。

image::support-help.png[获得支持]

在下一页中，单击 `Generate Support Config` 按钮。

NOTE: 如果未安装适配器，则将没有生成支持配置包的选项。您必须安装CSP适配器才能生成支持配置包。

image::generate-support-config.png[获得支持]

[#_accessing_without_rancher]
== 不通过Rancher访问

首先，为安装了Rancher的集群生成kubeconfig。

NOTE: 如果Rancher宕机，您将无法使用Rancher生成的kubeconfig令牌访问集群。

配置您的外壳环境以使用此kubeconfig令牌：

[,bash]
----
export KUBECONFIG=$MY_KUBECONFIG_PATH
----

建议在运行此命令时创建一个临时工作目录，如下所示：

[,bash]
----
mkdir temp && cd temp
----

然后，检索支持配置包：

[,bash]
----
mkdir rancher && kubectl get configmap csp-config -n cattle-csp-adapter-system -o=jsonpath='{.data.data}' >> rancher/config.json && tar -c -f supportconfig_rancher.tar rancher && rm -rf rancher
----

这将在您当前的目录中创建一个 `supportconfig_rancher.tar` 文件。

在Mac上运行这些命令的用户可能会遇到由于gnu-tar和bsd-tar之间的不兼容而导致的问题。如果支持团队在读取您生成的supportconfig时遇到问题，请在将gnu-tar设置为在您的路径上可访问的`gtar`后，尝试使用以下命令：

[,bash]
----
mkdir rancher && kubectl get configmap csp-config -n cattle-csp-adapter-system -o=jsonpath='{.data.data}' >> rancher/config.json && gtar -c -f supportconfig_rancher.tar rancher && rm -rf rancher
----
