我们建议使用 Helm，一个 Kubernetes 包管理器，在专用的 Kubernetes 集群上安装 Rancher。这被称为高可用性 Kubernetes 安装，因为通过在多个节点上运行 Rancher 来实现更高的可用性。

在标准安装中，Kubernetes 首先在三个节点上安装，这些节点托管在基础设施提供商如亚马逊的 EC2 或谷歌的计算引擎上。

然后使用 Helm 在 Kubernetes 集群上安装 Rancher。Helm 使用 Rancher 的 Helm 图表在 Kubernetes 集群中的每个节点上安装 Rancher 的副本。我们建议使用负载均衡器将流量引导到集群中每个 Rancher 的副本，以提高 Rancher 的可用性。

Rancher 服务器的数据存储在 etcd 上。这个 etcd 数据库也在所有三个节点上运行，并且需要奇数个节点，以确保在 etcd 集群中始终能够选举出获得多数票的领导者。如果 etcd 数据库无法选举出领导者，etcd 可能会失败，要求从备份中恢复集群。

有关 Rancher 工作原理的详细信息，无论安装方式如何，请参阅 {xref-rancher-architecture}[架构部分]。

[#_recommended_architecture]
== 推荐架构

* Rancher 的 DNS 应该解析到一个层 4 负载均衡器。
* 负载均衡器应该将 TCP/80 和 TCP/443 端口转发到 Kubernetes 集群中的所有 3 个节点。
* Ingress 控制器将 HTTP 重定向到 HTTPS，并在 TCP/443 端口终止 SSL/TLS。
* Ingress 控制器将流量转发到 Rancher 部署中的 pod 的 TCP/80 端口。

.Kubernetes Rancher 安装配合层 4 负载均衡器，展示了在 Ingress 控制器处的 SSL 终止。
