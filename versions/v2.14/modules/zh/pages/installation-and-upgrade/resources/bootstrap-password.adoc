安装Rancher时，可以为第一个管理员账户设置启动口令。

如果选择不设置启动口令，Rancher会随机生成一个启动口令用于第一个管理员账户。

有关如何设置启动口令的详细信息，请参见下文。

[#_password_requirements]
== 口令要求

启动口令可以是任意长度。

在首次登录后重置第一个管理员账户的口令时，新口令必须至少为12个字符。

您可以{xref-users-and-groups}#_minimum_password_length[自定义用户账户的最小口令长度]，但需遵循限制。

最小口令长度可以是2到256之间的任何正整数值。不允许使用小数值和前导零。

[#_specifying_the_bootstrap_password]
== 指定启动口令

在{xref-install-rancher}[Rancher安装]期间，设置`bootstrapPassword`与Rancher Helm图表的其他标志一起使用。例如：

[,bash]
----
helm install rancher rancher-<chart-repo>/rancher \
  --set bootstrapPassword=<password>
----

## 检索启动口令

启动口令存储在Docker容器日志中。安装Rancher后，用户界面会显示根据您的安装方法检索口令的说明。

[,bash]
----
kubectl get secret --namespace cattle-system bootstrap-secret -o go-template='{{ .data.bootstrapPassword|base64decode}}{{ "\n" }}'
----
