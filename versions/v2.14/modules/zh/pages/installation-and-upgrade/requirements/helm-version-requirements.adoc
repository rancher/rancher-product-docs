本节包含 Helm 的要求，Helm 是用于在高可用性 Kubernetes 集群上安装 Rancher 的工具。

[#_identifying_the_proper_helm_v3_version]
== 识别正确的 Helm v3 版本

从我们的 https://www.suse.com/suse-rancher/support-matrix/all-supported-versions[Rancher 支持矩阵] 中选择与您使用的 Kubernetes 版本范围正式兼容的任何 Helm v3 版本。

要应用此规则，您可能需要参考两个外部资源：

* *Helm 版本兼容性：*请参考 https://helm.sh/docs/topics/version_skew/[Helm 版本支持策略]，并选择与您的 Rancher 次要版本规则匹配的版本。
* *Rancher 的 Kubernetes 支持范围：*使用 https://www.suse.com/suse-rancher/support-matrix/all-supported-versions[Rancher 支持矩阵] 来识别您的目标 Rancher 次要版本所支持的 Kubernetes 版本。

[#_example]
=== 示例

* *方案：*您正在针对 Rancher v2.11.4，它支持 Kubernetes 版本 1.30 到 1.32。
* *应用程序：*我们的规则要求支持此范围的 Helm 版本。您可以通过检查 Helm 版本与范围内最高版本 Kubernetes v1.32 的兼容性来验证这一点。
* *结果：*您发现 Helm v3.17 和 Helm v3.18 都支持 Kubernetes v1.30-v1.32 范围。
* 虽然两者都可以工作，但我们推荐 Helm v3.18，因为它是与支持的 Kubernetes 范围重叠的最新 Helm 次要版本。

[#_additional_notes]
== 补充说明

* 安装或升级 Rancher v2.5 需要 Helm v3.2.x 或更高版本。
* 当使用为您运行 Helm 命令的工具（如 Terraform）时，您必须确保它们配置为使用正确的 Helm 版本。
