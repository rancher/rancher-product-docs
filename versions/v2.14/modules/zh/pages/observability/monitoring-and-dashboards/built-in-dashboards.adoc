[#_grafana_ui]
== Grafana UI

https://grafana.com/grafana/[Grafana] 允许您查询、可视化、设置警报以及了解您的指标，无论它们存储在哪里。创建、探索和与您的团队共享仪表板，并培养数据驱动的文化。

要查看时间序列数据可视化的默认仪表板，请访问 Grafana UI。

[#_customizing_grafana]
=== 自定义 Grafana

要查看和自定义支持 Grafana 仪表板的 PromQL 查询，请参见 {xref-customize-grafana-dashboard}[此页面。]

[#_persistent_grafana_dashboards]
=== 持久化 Grafana 仪表板

要创建持久的 Grafana 仪表板，请参见 {xref-create-persistent-grafana-dashboard}[此页面。]

[#_access_to_grafana]
=== 访问 Grafana

有关 Grafana 的基于角色的访问控制的信息，请参见 xref:./rbac-for-monitoring.adoc#_role_based_access_control_for_grafana[本节。]

[#_alertmanager_ui]
== Alertmanager UI

当 `rancher-monitoring` 安装时，Prometheus Alertmanager UI 被部署，允许您查看您的警报和当前的 Alertmanager 配置。

[NOTE]
====

本节假设您对监控组件如何协同工作有一定了解。有关 Alertmanager 的更多信息，请参见 xref:./how-monitoring-works.adoc#_3_how_alertmanager_works[Alertmanager 的工作原理。]
====


[#_accessing_the_alertmanager_ui]
=== 访问 Alertmanager UI

Alertmanager UI 让您查看最近触发的警报。

[NOTE]
.前提条件：
====

必须安装 `rancher-monitoring` 应用程序。
====


要查看 Alertmanager UI，

. 在左上角，单击*☰ > 集群管理*。
. 在 *集群* 页面上，转到您想查看 Alertmanager UI 的集群，单击 *探索*。
. 在左侧导航栏中，单击*监控*。
. 单击 *Alertmanager*。

*结果：*Alertmanager UI 将在新标签页中打开。有关配置的帮助，请参阅 https://prometheus.io/docs/alerting/latest/alertmanager/[官方 Alertmanager 文档。]

有关在Rancher中配置Alertmanager的更多信息，请参见{xref-configure-alertmanager}[此页面。]

image::alertmanager-ui.png[Alertmanager UI]

[#_viewing_default_alerts]
=== 查看默认警报

要查看默认触发的警报，请转到Alertmanager UI并单击*展开所有组*。

[#_prometheus_ui]
== Prometheus UI

默认情况下， https://github.com/kubernetes/kube-state-metrics[kube-state-metrics 服务]为监控应用程序提供了大量关于处理器和内存利用率的信息。这些指标涵盖了跨名称空间的 Kubernetes 资源。这意味着要查看服务的资源指标，您无需为其创建新的ServiceMonitor。因为数据已经在时间序列数据库中，您可以转到 Prometheus UI 并运行 PromQL 查询以获取信息。相同的查询可以用于配置 Grafana 仪表板，以显示这些指标随时间变化的图表。

要查看 Prometheus UI，请安装 `rancher-monitoring`。然后：

. 在左上角，单击*☰ > 集群管理*。
. 在 *集群*页面上，转到您想查看 Prometheus UI 的集群，然后单击 *探索*。
. 在左侧导航栏中，单击*监控*。
. 单击*Prometheus图表*。

.Prometheus 图表 UI
image::prometheus-graph-ui.png[Prometheus 图表 UI]

[#_viewing_the_prometheus_targets]
=== 查看 Prometheus 目标

要查看您正在监控的服务，您需要查看您的目标。目标由 ServiceMonitors 和 PodMonitors 设置，作为抓取指标的来源。您无需直接编辑目标，但 Prometheus UI 可以帮助您概览所有被抓取的指标来源。

要查看 Prometheus 目标，请安装 `rancher-monitoring`。然后：

. 在左上角，单击*☰ > 集群管理*。
. 在*集群*页面上，转到您想查看 Prometheus 目标的集群，然后单击*探索*。
. 在左侧导航栏中，单击*监控*。
. 单击*Prometheus 目标*。

.Prometheus UI 中的目标
image::prometheus-targets-ui.png[Prometheus 目标 UI]

[#_viewing_the_prometheusrules]
=== 查看 PrometheusRules

当您定义一个规则（在 PrometheusRule 资源中的 RuleGroup 内声明）时， https://github.com/prometheus-operator/prometheus-operator/blob/main/Documentation/api-reference/api.md#rule[规则本身的 spec] 包含 Alertmanager 用来确定哪个路由应该接收某个警报的标签。

要查看 PrometheusRules，请安装`rancher-monitoring`。然后：

. 在左上角，单击*☰ > 集群管理*。
. 在*集群*页面上，转到您想查看可视化的集群，然后单击*探索*。
. 在左侧导航栏中，单击*监控*。
. 单击*Prometheus 规则*。

您还可以在 Prometheus UI 中查看规则：

.Prometheus UI 中的规则
image::prometheus-rules-ui.png[PrometheusRules UI]

有关在 Rancher 中配置 PrometheusRules 的更多信息，请参见 {xref-prometheusrules}[此页面。]。
