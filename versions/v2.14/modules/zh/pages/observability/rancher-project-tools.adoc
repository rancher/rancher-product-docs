Rancher包含多种不包含在Kubernetes中的工具，以协助您的DevOps操作。Rancher可以与外部服务集成，以帮助您的集群更高效地运行。

[#_notifiers_and_alerts]
== 通知器和警报

通知器和警报是两个协同工作的功能，用于通知您 Rancher 系统中的事件。在启用它们之前，必须安装监控应用程序。

通知器是通知您警报事件的服务。您可以配置通知器，将警报通知发送给最适合采取纠正措施的工作人员。通知可以通过Slack、电子邮件、PagerDuty、微信和Webhooks发送。

警报是触发这些通知的规则。在您能够接收警报之前，必须在Rancher中配置一个或多个通知器。警报的范围可以设置在集群级别或项目级别。

[#_logging]
== 日志记录

日志记录很有帮助，因为它允许您：

* 捕获和分析集群的状态
* 寻找您环境中的趋势
* 将日志保存到集群外的安全位置
* 及时了解事件，例如容器崩溃、Pod驱逐或节点故障
* 更轻松地调试和排除问题

Rancher可以与Elasticsearch、Splunk、Kafka、Syslog和Fluentd集成。

有关细节，请参阅{xref-logging}[日志记录部分。]

[#_monitoring]
== 监控

使用Rancher，您可以通过与 https://prometheus.io/[Prometheus]的集成，监控集群节点、Kubernetes组件和软件部署的状态和进程。Prometheus是一个领先的开源监控解决方案。有关细节，请参阅{xref-monitoring-and-dashboards}[监控部分。]
