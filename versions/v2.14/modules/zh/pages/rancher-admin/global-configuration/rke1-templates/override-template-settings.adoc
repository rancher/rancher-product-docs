当用户创建 RKE 模板时，模板中的每个设置在 Rancher UI 中都有一个开关，指示用户是否可以覆盖该设置。该开关将这些设置标记为 *允许用户覆盖*。

在使用模板创建集群后，最终用户无法更新模板中定义的任何设置，除非模板所有者将其标记为 *允许用户覆盖*。然而，如果模板被 {xref-manage-rke1-templates}[更新到新版本]，更改了设置或允许最终用户更改设置，则可以将集群升级到模板的新版本，并将新版本中的更改应用于集群。

当 RKE 模板上的任何参数设置为 *允许用户覆盖* 时，这意味着最终用户在创建集群时必须填写这些字段，并且他们可以在任何时候编辑这些设置。

RKE 模板的 *允许用户覆盖* 模式在以下情况中非常有用：

* 管理员知道某些设置需要灵活性，以便随着时间的推移进行频繁更新。
* 最终用户需要输入自己的访问密钥或秘密密钥，例如云凭证或备份快照凭证。
