Rancher 通过使用 https://kubernetes.io/docs/concepts/extend-kubernetes/api-extension/apiserver-aggregation/[Kubernetes API 聚合层]注册扩展 API 服务器来扩展 Kubernetes。

[#_aggregation_layer_is_required]
== 需要聚合层

API 聚合层必须在本地 Kubernetes 集群上配置，以便 `v1.ext.cattle.io` `APIService` 正常工作。如果 `APIService` 在 Rancher 服务器启动后未收到注册请求，Pod 将崩溃，并记录错误日志条目。如果您的 Pod 一直无法检测到注册，尽管集群配置正确，您可以通过在 Helm 中设置 `.Values.aggregationRegistrationTimeout` 来增加超时时间。

在此版本的 Rancher 中，RKE2/K3s 支持的 Kubernetes 版本默认配置并启用聚合层。但是，如果您怀疑集群配置不正确，请参考 https://kubernetes.io/docs/tasks/extend-kubernetes/configure-aggregation-layer/[Kubernetes 聚合层文档]以获取有关配置聚合层的信息。

[NOTE]
====
如果底层 Kubernetes 发行版不支持聚合层，您必须在升级之前迁移到支持聚合层的 Kubernetes 发行版。
====
